## Analysis

### Technical Summary

SKMaskFilter operates on alpha masks produced during path/text rendering and has no effect when drawing bitmaps via DrawBitmap. To apply gamma correction to bitmap pixel data the reporter should use SKColorFilter.CreateTable() with a power-function lookup table instead.

### Rationale

The reporter filed this as a bug but provided no expected or actual behavior. The real question is 'how do I apply gamma to a bitmap?' — SKMaskFilter is not designed for bitmap pixel processing; it targets alpha masks generated by path/shape/text rendering. The correct tool is SKColorFilter.CreateTable() or SKColorFilter.CreateSrgbToLinearGamma/CreateLinearToSrgbGamma. This is by-design behavior, not a defect.

### Resolution Proposals

**Hypothesis:** SKMaskFilter is not the right tool for bitmaps; SKColorFilter is the correct SkiaSharp API for per-pixel gamma correction.

1. **Apply gamma via SKColorFilter.CreateTable** — workaround, confidence 0.92 (92%), cost/xs, validated=yes
   - Build a 256-entry byte lookup table using a power function (value/255)^gamma and assign it as ColorFilter on the SKPaint used in DrawBitmap.

```csharp
float gamma = 2.2f;
byte[] gammaTable = new byte[256];
for (int i = 0; i < 256; i++)
    gammaTable[i] = (byte)(Math.Pow(i / 255.0, gamma) * 255.0 + 0.5);

byte[] identityTable = new byte[256];
for (int i = 0; i < 256; i++)
    identityTable[i] = (byte)i;

using var colorFilter = SKColorFilter.CreateTable(identityTable, gammaTable, gammaTable, gammaTable);
using var paint = new SKPaint { ColorFilter = colorFilter };
using var canvas = new SKCanvas(destBitmap);
canvas.DrawBitmap(srcBitmap, new SKRect(0, 0, srcBitmap.Width, srcBitmap.Height), paint);
```
2. **Use built-in sRGB gamma filters** — alternative, confidence 0.88 (88%), cost/xs, validated=yes
   - For standard sRGB ↔ linear gamma, use the built-in SKColorFilter.CreateSrgbToLinearGamma() or CreateLinearToSrgbGamma() without building a custom table.

```csharp
using var paint = new SKPaint { ColorFilter = SKColorFilter.CreateLinearToSrgbGamma() };
using var canvas = new SKCanvas(destBitmap);
canvas.DrawBitmap(srcBitmap, new SKRect(0, 0, srcBitmap.Width, srcBitmap.Height), paint);
```

**Recommended proposal:** Apply gamma via SKColorFilter.CreateTable

**Why:** Supports any custom gamma value; the LUT approach exactly mirrors what SKMaskFilter.CreateGamma does but at the pixel level rather than the mask level.

**Comment draft for `add-comment`:**

```markdown
Thanks for the report! `SKMaskFilter` (including `CreateGamma` and `CreateBlur`) only operates on the **alpha mask** that Skia generates when rendering paths, shapes, and text — it has no effect on `DrawBitmap` calls because bitmaps bypass the mask step entirely.

For **gamma correction on bitmaps**, use `SKColorFilter` instead. You can build a power-function lookup table and attach it as `ColorFilter` on your `SKPaint`:

```csharp
float gamma = 2.2f; // adjust to your desired gamma value

byte[] gammaTable = new byte[256];
for (int i = 0; i < 256; i++)
    gammaTable[i] = (byte)(Math.Pow(i / 255.0, gamma) * 255.0 + 0.5);

byte[] identityTable = new byte[256];
for (int i = 0; i < 256; i++)
    identityTable[i] = (byte)i;

// Apply gamma to R, G, B channels; leave alpha unchanged
using var colorFilter = SKColorFilter.CreateTable(identityTable, gammaTable, gammaTable, gammaTable);
using var paint = new SKPaint { ColorFilter = colorFilter };
using var canvas = new SKCanvas(destBitmap);
canvas.DrawBitmap(srcBitmap, new SKRect(0, 0, srcBitmap.Width, srcBitmap.Height), paint);
```

If you specifically want standard **sRGB ↔ linear gamma**, SkiaSharp also has built-in helpers:

```csharp
using var paint = new SKPaint { ColorFilter = SKColorFilter.CreateLinearToSrgbGamma() };
```

Closing as by-design — `SKMaskFilter` is not intended for bitmap pixel processing. `SKColorFilter` is the right tool here.
```
