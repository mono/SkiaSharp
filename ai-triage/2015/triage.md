## Analysis

### Technical Summary

The reporter wants SkiaSharp to support embedding X.509 digital signatures (PKCS#7/CMS) into PDF documents at generation time. SkiaSharp's current PDF API (SKDocument) wraps Skia's SkPDF backend, which does not expose a digital signing hook in its C API surface. Upstream Skia does not natively support PDF digital signing; this feature would require either a post-processing step using a dedicated PDF library or upstream Skia changes.

### Rationale

This is a clearly stated feature request for new functionality that does not exist in SkiaSharp or upstream Skia. No broken behavior is described. The request is well-understood (PDF digital signatures via X.509/PKCS#7), but implementation requires either changes upstream or a post-processing approach external to SkiaSharp's PDF generation pipeline.

- "I believe that using a new package just for this couldn't be good." — **issue body** (Reporter prefers integrated solution but is open to alternatives.)

### Workarounds

- Use iTextSharp (LGPL) or PDFsharp to post-process the PDF output from SkiaSharp and embed X.509 digital signatures
- Use PdfPig or Docnet.Core libraries which support digital signing as a post-processing step on existing PDFs
- Use the PdfSharpCore NuGet package to sign PDFs generated by SkiaSharp after saving them

**Why:** The workaround is immediately actionable without waiting for SkiaSharp changes. PDF digital signing is a complex standard (PKCS#7/CMS) better handled by dedicated PDF libraries. SkiaSharp's PDF backend (Skia's SkPDF) does not natively support signing.

**Comment draft for `add-comment`:**

```markdown
Thanks for the feature request! PDF digital signing (X.509/PKCS#7) is not currently supported by SkiaSharp or the underlying Skia PDF backend (`SkPDF`), so this would require significant upstream work.

**Workaround:** You can generate the PDF with SkiaSharp and then use a dedicated .NET PDF library to digitally sign it:

- **iTextSharp** (LGPL) – well-established, supports X.509 digital signatures via `PdfStamper`
- **PDFsharp** – open-source, supports PKCS#7 signing via `PdfDocumentSigner`
- **PdfPig** – modern .NET library with signing support

These libraries can open the PDF output stream from `SKDocument.CreatePdf()` and embed a digital signature using your `X509Certificate2` without needing to rerender the document.

We'll keep this open to track interest in native support.
```
