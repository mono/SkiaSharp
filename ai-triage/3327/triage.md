**Issue Summary:** SkiaSharp.Views.WinUI 3.119.0 produces IL2026 trim analysis warnings when building a WinUI3 app with NativeAOT/trimming enabled, due to the bundled WinRT projection DLL using an outdated CsWinRT version that is not trim-safe.

**Analysis:** The trim warnings originate from SkiaSharp.Views.WinUI.Native.Projection.dll, which is a CsWinRT-generated WinRT projection bundled with Microsoft.WindowsAppSDK 1.4.230913002. CsWinRT 2.0.4 (used in WA SDK 1.4) contains WinRT.WinrtModule.GetActivationFactory and WinRT.DllModule.GetActivationFactory that call WinRT.ComWrappersSupport.GetObjectReferenceForInterface, which is marked RequiresUnreferencedCode. Later versions of CsWinRT (2.1+, shipping with WA SDK 1.5+) added trim-safety. Updating the Windows App SDK reference would resolve these warnings.

**Recommendations:** **needs-investigation** — Root cause is clear (outdated CsWinRT in WA SDK 1.4), but updating the WA SDK dependency may have breaking consumer implications requiring investigation before a PR.

## Analysis

### Technical Summary

The trim warnings originate from SkiaSharp.Views.WinUI.Native.Projection.dll, which is a CsWinRT-generated WinRT projection bundled with Microsoft.WindowsAppSDK 1.4.230913002. CsWinRT 2.0.4 (used in WA SDK 1.4) contains WinRT.WinrtModule.GetActivationFactory and WinRT.DllModule.GetActivationFactory that call WinRT.ComWrappersSupport.GetObjectReferenceForInterface, which is marked RequiresUnreferencedCode. Later versions of CsWinRT (2.1+, shipping with WA SDK 1.5+) added trim-safety. Updating the Windows App SDK reference would resolve these warnings.

### Rationale

The trim warnings are real and caused by a known outdated dependency (CsWinRT 2.0.4) bundled in Microsoft.WindowsAppSDK 1.4. The fix path is clear: update the WA SDK dependency. The crashes reported by the user correlate with the IL2026 warnings — trimming can strip code that CsWinRT reflection-discovers at runtime. This is a legitimate bug in the dependency management, not a usage error.

### Key Signals

- "warning IL2104: Assembly 'SkiaSharp.Views.WinUI.Native.Projection' produced trim warnings" — **issue body** (The native WinRT projection DLL shipped by SkiaSharp is not trim-safe.)
- "WinRT.WinrtModule.GetActivationFactory: Using member WinRT.ComWrappersSupport.GetObjectReferenceForInterface which has RequiresUnreferencedCodeAttribute" — **issue body** (This is a known CsWinRT 2.0.x limitation; fixed in CsWinRT 2.1+ / WA SDK 1.5+.)
- "This is because SkiaSharp is relying on an outdated version of CsWinRT (specifically, 2.0.4)." — **comment by dongle-the-gadget** (Community confirmation that the root cause is the CsWinRT version pinned in WA SDK 1.4.)

### Resolution Proposals

**Hypothesis:** Updating Microsoft.WindowsAppSDK from 1.4.230913002 to 1.5+ in SkiaSharp.Views.WinUI.csproj would bring in a trim-safe CsWinRT (2.1+), eliminating the IL2026/IL2104 warnings.

1. **Update Microsoft.WindowsAppSDK to 1.5+** — fix, confidence 0.85 (85%), cost/s, validated=untested
   - Bump the Microsoft.WindowsAppSDK PackageReference in SkiaSharp.Views.WinUI.csproj from 1.4.230913002 to 1.5+ (e.g., 1.5.250201001 or later). WA SDK 1.5 ships CsWinRT 2.1 which is trim-safe, resolving all IL2026/IL2104 warnings.
2. **Suppress trim warnings as workaround** — workaround, confidence 0.70 (70%), cost/xs, validated=untested
   - Consumers can suppress the IL2104 assembly-level warning by adding <TrimmerRootDescriptor> or by setting <SuppressTrimAnalysisWarnings>true</SuppressTrimAnalysisWarnings> as a stopgap. This does not fix the underlying risk of runtime failures under AOT.

**Recommended proposal:** Update Microsoft.WindowsAppSDK to 1.5+

**Why:** This is the correct long-term fix. CsWinRT 2.1 shipped in WA SDK 1.5 explicitly added trim and AOT safety. Suppressing warnings leaves the runtime risk unresolved.

Thanks for the detailed report. This is caused by `SkiaSharp.Views.WinUI` shipping a WinRT projection DLL (`SkiaSharp.Views.WinUI.Native.Projection.dll`) that was generated with **CsWinRT 2.0.4** (bundled in `Microsoft.WindowsAppSDK 1.4`). CsWinRT 2.0.x uses `WinRT.ComWrappersSupport.GetObjectReferenceForInterface` internally, which is annotated `[RequiresUnreferencedCode]` and triggers the IL2026/IL2104 trim warnings.

As a temporary workaround you can suppress the assembly-level warning, but this does not address the underlying AOT risk.
