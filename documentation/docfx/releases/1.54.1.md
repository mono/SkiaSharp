## Highlights

Servicing release on top of 1.54.0. No user-facing PR work landed on the branch — the release ships to align the binaries and continues the fields→properties migration for `SKMatrix`, and rationalizes disposal on a few iterator types.

## Breaking Changes

- **SKMatrix** — the nine public fields (`ScaleX`, `SkewX`, `TransX`, `SkewY`, `ScaleY`, `TransY`, `Persp0`, `Persp1`, `Persp2`) were converted to properties. Field access by name still compiles; `ref`/`out` on a field does not.
- **SKObject / SKPath.Iterator / SKPath.RawIterator** — the `virtual void Dispose()` and finalizer overrides were removed; the classes still implement `IDisposable` but delegate disposal to the base implementation.
