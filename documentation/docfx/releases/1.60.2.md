# Version 1.60.2

> **New drawing primitives and platform fixes** · Released June 28, 2018 · [NuGet](https://www.nuget.org/packages/SkiaSharp/1.60.2) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v1.60.2)

## Highlights

This release introduces `SKNWayCanvas` for drawing to multiple canvases simultaneously and `SKRoundRect` for more complex rounded rectangles. It also adds flexible encoding overloads to `SKPixmap` and resolves several platform-specific linking and rendering issues.

## New Features

### API Surface

- **`SKNWayCanvas`** — Draw to multiple canvases simultaneously, enabling broadcast-style rendering to several targets at once.
- **`SKRoundRect`** — Draw more complex rounded rectangles with per-corner radius control.
- **`SKPixmap.Encode()` overloads** — Additional `Encode()` overloads for finer control when encoding to PNG, JPEG, and WEBP formats.

## Bug Fixes

- Fixed UWP library being linked to the wrong libraries ([#566](https://github.com/mono/SkiaSharp/issues/566), [#536](https://github.com/mono/SkiaSharp/issues/536)).
- Fixed iOS not being able to AOT compile ([#547](https://github.com/mono/SkiaSharp/issues/547)).
- Fixed slow rendering in the GTK# `SKWidget` ([#534](https://github.com/mono/SkiaSharp/issues/534)).
- Fixed a memory leak in `SKXamlCanvas` on UWP and Xamarin.Forms ([#429](https://github.com/mono/SkiaSharp/issues/429)).
- Upgraded projects to the new SDK-style format; removed shared projects and enabled Android builds on Windows.
