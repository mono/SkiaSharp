# Version 4.151.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

<!-- Generated: 2026-07-06T23:24:50Z by claude-opus-4.7 -->

In-development window for the next SkiaSharp release. Adds a memory-leak-fixer skill for coordinated leak diagnosis, sharpens the `SKColor` hex parser with `ReadOnlySpan<char>` overloads, and lands a cross-backend golden-image regression harness. HarfBuzzSharp 14.2.1 co-ships — see the [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) page for its binding details. Not yet available on NuGet.

## Highlights

- New **memory-leak-fixer** skill and self-testing agentic workflow for coordinated leak diagnosis and regression capture ([#4330](https://github.com/mono/SkiaSharp/pull/4330)).
- `SKColor` hex parsing is faster and now accepts `ReadOnlySpan<char>` inputs so callers can parse straight out of tokenizers/spans without materializing a `string` ([#4345](https://github.com/mono/SkiaSharp/pull/4345)).
- Cross-backend **golden-image (visual regression) test harness** for validating rendering against reference images across raster/GPU backends, contributed by [@ramezgerges](https://github.com/ramezgerges) ([#4340](https://github.com/mono/SkiaSharp/pull/4340)).
- Upstream Skia chrome/m151 bug fixes merged in ([#4312](https://github.com/mono/SkiaSharp/pull/4312)).
- HarfBuzzSharp 14.2.1 ships alongside this release — see the linked page for binding details.

## API Surface

- New `SKColor.Parse`/`TryParse` overloads take `ReadOnlySpan<char>` and share a hot-path parser to remove allocations on the common `#RRGGBB`/`#AARRGGBB` shapes ([#4345](https://github.com/mono/SkiaSharp/pull/4345)).
- `GRContextOptions` no longer requires callers to be in an `unsafe` context to configure it, contributed by [@Copilot](https://github.com/apps/copilot-swe-agent) ([#3991](https://github.com/mono/SkiaSharp/pull/3991)).

## Engine

- Upstream chrome/m151 bug-fix merge picks up the latest Skia stability patches for the current milestone ([#4312](https://github.com/mono/SkiaSharp/pull/4312)).

## Bug Fixes

- The obsolete `skiasharp-docker` registry authentication step no longer runs on infra pipelines that don't need it, contributed by [@Copilot](https://github.com/apps/copilot-swe-agent) ([#4310](https://github.com/mono/SkiaSharp/pull/4310)).

## Breaking Changes

None in this release.

## Lifecycle & Internals

- New memory-leak-fixer skill and the workflow that self-tests it ([#4330](https://github.com/mono/SkiaSharp/pull/4330)).

- Apple certificate/provisioning-profile steps removed from tests — the harness no longer relies on maintainer-owned iOS signing ([#4320](https://github.com/mono/SkiaSharp/pull/4320)).
