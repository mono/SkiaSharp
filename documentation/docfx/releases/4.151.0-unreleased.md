# Version 4.151.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Development toward the next 4.x release is under way. So far the branch has picked
up an upstream Skia `chrome/m151` bug-fix sync, a hex-parsing optimization for
`SKColor` with new span-based overloads, and a small internal tidy on
`GRContextOptions`. HarfBuzzSharp 14.2.1 ships alongside — see its
[release notes](harfbuzzsharp/14.2.1.md) for the details.

## Breaking Changes

None in this release.

## Engine

- **Skia `chrome/m151` bug-fix sync** — upstream fixes from the m151 release branch
  merged in ahead of the SkiaSharp 4.151 stable, keeping the engine current with
  Chromium's own stability work.
  ([#4312](https://github.com/mono/SkiaSharp/pull/4312), [mono/skia#278](https://github.com/mono/skia/pull/278))

## API Surface

- **Faster `SKColor` hex parsing with span-based overloads** — the `#RRGGBB` /
  `#AARRGGBB` parser was rewritten to avoid the intermediate string allocation
  and now offers `ReadOnlySpan<char>` overloads for zero-copy parsing on the hot
  paths (color palettes, theme lookups).
  ([#4345](https://github.com/mono/SkiaSharp/pull/4345))
- **`GRContextOptions` no longer requires an `unsafe` context** — the redundant
  `unsafe` modifier was removed so consumers can construct `GRContextOptions`
  from safe code.
  ([#3991](https://github.com/mono/SkiaSharp/pull/3991))
