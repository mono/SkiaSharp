# Version 4.151.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

<!-- Generated: 2026-07-08T08:15:00Z by claude-opus-4.7 -->

## Highlights

**4.151.0** is under active development on `main`. So far it brings a faster `SKColor` hex parser with new `ReadOnlySpan<char>` overloads, an upstream `chrome/m151` bug-fix sync, and a small `GRContextOptions` cleanup that drops an unneeded `unsafe` modifier. HarfBuzzSharp bumps to **14.2.1** alongside this release; see the [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) page for that binding's details.

## Performance

- **`SKColor` hex parsing** — Optimize the `#RRGGBB` / `#AARRGGBB` parser and add `ReadOnlySpan<char>` overloads so callers can parse without allocating a string ([#4345](https://github.com/mono/SkiaSharp/pull/4345)).

## Fixes

- **Skia upstream sync** — Merge upstream `chrome/m151` bug fixes so 4.151.0 tracks the latest stable Skia branch ([#4312](https://github.com/mono/SkiaSharp/pull/4312), [mono/skia#278](https://github.com/mono/skia/pull/278)).
- **`GRContextOptions`** — Drop an unnecessary `unsafe` modifier that leaked through the wrapper, keeping the type callable from safe code ([#3991](https://github.com/mono/SkiaSharp/pull/3991)).

## Other changes

Plus various CI, documentation, and internal tooling improvements — including a new performance-fixer skill and reworked release-notes polish pipeline (❤️ [@ramezgerges](https://github.com/ramezgerges) added the cross-backend golden-image test harness in [#4340](https://github.com/mono/SkiaSharp/pull/4340)).
