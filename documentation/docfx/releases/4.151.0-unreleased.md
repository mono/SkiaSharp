# Version 4.151.0

> **In-flight main** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

In-flight work on `main` for the next 4.x line. Product-facing changes are small and targeted: `SKColor` hex parsing gains a `ReadOnlySpan<char>` overload and a faster parser, an upstream `chrome/m151` bug-fix roll from Skia, and a tidy-up removing an unnecessary `unsafe` modifier from `GRContextOptions`. HarfBuzzSharp 14.2.1 ships alongside — see its [linked page](harfbuzzsharp/14.2.1.md) for binding details.

## Breaking Changes

None in this release.

## API Surface

- **`SKColor` hex parsing gets a Span overload and is measurably faster** — new `ReadOnlySpan<char>` overloads on `SKColor.Parse` / `TryParse` avoid the intermediate string, and the hex path itself was rewritten to skip allocations on the hot path. ([#4345](https://github.com/mono/SkiaSharp/pull/4345))
- **`GRContextOptions` no longer requires `unsafe`** — a leftover pointer field that no longer needed it was removed, so callers can drop `unsafe` from `GRContextOptions` construction. ([#3991](https://github.com/mono/SkiaSharp/pull/3991))

## Engine

- **Skia `chrome/m151` bug-fix roll** — upstream stability fixes merged into the m151 branch. ([#4312](https://github.com/mono/SkiaSharp/pull/4312), [skia#278](https://github.com/mono/skia/pull/278))

## Lifecycle & Internals

- Plus various CI, documentation, and internal tooling improvements — including a new cross-backend visual regression test harness contributed by [@ramezgerges](https://github.com/ramezgerges).
