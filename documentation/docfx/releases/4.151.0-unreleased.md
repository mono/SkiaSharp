# Version 4.151.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

In-development notes for the upcoming 4.151.0 line on `main`. So far this line brings a Skia upstream sync for m151 bug fixes, an `SKColor` hex-parsing optimization with `ReadOnlySpan<char>` overloads, and a cross-backend golden-image (visual regression) test harness contributed by [@ramezgerges](https://github.com/ramezgerges). Ships alongside **[HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)** — see that page for its binding details. Additional community contributions from [Copilot](https://github.com/apps/copilot) (removing an unnecessary `unsafe` modifier and cleaning up obsolete docker registry auth).

## Breaking Changes

None in this release.

## Engine

- **Merge upstream chrome/m151 bug fixes** — Rolls up upstream Skia bug fixes from Chrome milestone m151. ([#4312](https://github.com/mono/SkiaSharp/pull/4312))

## API Surface

- **`SKColor` hex parsing + `ReadOnlySpan<char>` overloads** — Faster hex-string parsing and new `ReadOnlySpan<char>` overloads that avoid the allocation cost of intermediate substrings. ([#4345](https://github.com/mono/SkiaSharp/pull/4345))

## Fixes

- **Removed unnecessary `unsafe` modifier from `GRContextOptions`** — Trims an incorrect `unsafe` marker that was preventing use from safe contexts. ❤️ [Copilot](https://github.com/apps/copilot) ([#3991](https://github.com/mono/SkiaSharp/pull/3991))

## Testing

- **Cross-backend golden-image test harness** — Adds a visual-regression harness that compares rendered output across the CPU, GPU, and other supported backends against golden images. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4340](https://github.com/mono/SkiaSharp/pull/4340))

## Infrastructure & Tooling

- **New `memory-leak-fixer` skill + self-testing agentic workflow** — Adds a new agent skill that scans for and fixes SkiaSharp memory-ownership/disposal leaks, with a self-testing agentic workflow driving it. ([#4330](https://github.com/mono/SkiaSharp/pull/4330))
- **Auto-apply milestone labels in the Skia sync workflow and on `mono/skia` PRs** ([#4327](https://github.com/mono/SkiaSharp/pull/4327), [#4329](https://github.com/mono/SkiaSharp/pull/4329))
- **Release-notes skill improvements** — Manual notes sidecar; AI-summarized breaking changes; per-page breaking-change itemization; always-emit `Breaking Changes` heading; template relocation into the skill.
