# Version 4.151.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

SkiaSharp 4.151.0 advances the Skia engine to Chrome milestone m151 and co-ships HarfBuzzSharp 14.2.1. Community contributor [@ramezgerges](https://github.com/ramezgerges) fixed a WASM build regression by removing a redundant `SK_BUILD_FOR_WASM` symbol redefinition from the Skia submodule. The SkiaFiddle playground gains embedded image and font selectors for a richer development experience.

## Breaking Changes

*None in this release.*

## New Features

### Engine

- **Skia m151 update** — Advances the bundled Skia graphics engine to Chrome milestone m151, incorporating the latest upstream rendering improvements and fixes. ([#4294](https://github.com/mono/SkiaSharp/pull/4294))
- **WASM build fix** — Removes a redundant `SK_BUILD_FOR_WASM` redefinition in the Skia submodule that caused compilation failures in WebAssembly targets. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3788](https://github.com/mono/SkiaSharp/pull/3788))

### Tooling

- **SkiaFiddle image and font selectors** — Adds embedded image and font selectors to the SkiaFiddle playground, making it easier to experiment with different assets when prototyping SkiaSharp code. ([#4284](https://github.com/mono/SkiaSharp/pull/4284))

Plus several CI and documentation improvements.
