# Version 1.68.1

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.1)

> **API changes** · [SkiaSharp API diff](1.68.1/index.md) · [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md)

The first big servicing wave over `1.68.0`. Sixty-six PRs across six release candidates plus a small unreleased tail, focused on stabilising the `m68` engine, growing the managed surface with real user-facing bindings, and moving the CI onto Azure Pipelines. HarfBuzz `2.6.1` ships alongside — see [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md) for the standalone binding.

## Highlights

- Re-work the managed⇄native object model ([#900](https://github.com/mono/SkiaSharp/pull/900), (skia: [mono/skia#61](https://github.com/mono/skia/pull/61))) — the foundation the later `SKObject` disposal and `HandleDictionary` improvements build on.
- Split desktop projects and packages ([#899](https://github.com/mono/SkiaSharp/pull/899)) plus a new WPF backend for Xamarin.Forms ([#917](https://github.com/mono/SkiaSharp/pull/917)) and refreshed Xamarin.Forms baseline ([#916](https://github.com/mono/SkiaSharp/pull/916)).
- Introduce a P/Invoke generator ([#992](https://github.com/mono/SkiaSharp/pull/992)) — the seed for what becomes today's `*.generated.cs` pipeline — and a first pass at task-based API doc generation ([#997](https://github.com/mono/SkiaSharp/pull/997), (skia: [mono/skia#68](https://github.com/mono/skia/pull/68))).
- Bring `Span<T>` overloads to data, bitmap and text-blob APIs ([#865](https://github.com/mono/SkiaSharp/pull/865)) and add helpers to reduce delegate-proxy verbosity ([#882](https://github.com/mono/SkiaSharp/pull/882), [#878](https://github.com/mono/SkiaSharp/pull/878)).
- Bind `SkPictureShader` ([#973](https://github.com/mono/SkiaSharp/pull/973)) and add `int` unicode overloads for consistency ([#972](https://github.com/mono/SkiaSharp/pull/972)).
- Add `SKTypeface.IsFixedPitch` ([#1019](https://github.com/mono/SkiaSharp/pull/1019)).

## HarfBuzz

- HarfBuzzSharp `2.6.1` — the standalone binding ships alongside SkiaSharp ([#929](https://github.com/mono/SkiaSharp/pull/929)) with earlier bumps to `2.5.2` and `2.5.3` along the way ([#872](https://github.com/mono/SkiaSharp/pull/872), [#904](https://github.com/mono/SkiaSharp/pull/904)) and a first pass at font funcs ([#881](https://github.com/mono/SkiaSharp/pull/881)) and loading a face by table data ([#862](https://github.com/mono/SkiaSharp/pull/862)).

## Fixes

- Free memory when the view is detached, fixing [#1004](https://github.com/mono/SkiaSharp/issues/1004) ([#1005](https://github.com/mono/SkiaSharp/pull/1005)) and fix the iOS canvas implementation ([#879](https://github.com/mono/SkiaSharp/pull/879)).
- Fix a memory leak in `SKRoundRect` ([#995](https://github.com/mono/SkiaSharp/pull/995)) and stop 0-byte allocations from raster views ([#761](https://github.com/mono/SkiaSharp/pull/761), fixes [#759](https://github.com/mono/SkiaSharp/issues/759)).
- Stop treating some derived color spaces as owned instances so their disposal doesn't nuke a shared native pointer ([#922](https://github.com/mono/SkiaSharp/pull/922)).
- Fix `ToSKPixmap` producing display artifacts with transparent bitmaps ([#863](https://github.com/mono/SkiaSharp/pull/863)) and rescue bad colour output ([#1023](https://github.com/mono/SkiaSharp/pull/1023)).
- Improve UWP GPU-view stability ([#1024](https://github.com/mono/SkiaSharp/pull/1024)) and correct [#879](https://github.com/mono/SkiaSharp/issues/879)'s follow-up ([#1022](https://github.com/mono/SkiaSharp/pull/1022)).
- Land a Linux build version script to hide dependency exports ([#984](https://github.com/mono/SkiaSharp/pull/984)) and enable bitcode for watchOS and tvOS ([#971](https://github.com/mono/SkiaSharp/pull/971), (skia: [mono/skia#63](https://github.com/mono/skia/pull/63))).
- Build `arm64_32` for watchOS ([#967](https://github.com/mono/SkiaSharp/pull/967), (skia: [mono/skia#62](https://github.com/mono/skia/pull/62))) and support building SkiaSharp without fontconfig ([#821](https://github.com/mono/SkiaSharp/pull/821)).

## Engine

- Refresh the Skia submodule ([#985](https://github.com/mono/SkiaSharp/pull/985), (skia: [mono/skia#66](https://github.com/mono/skia/pull/66))) — the last engine bump captured by this range.
- Update Skia to fix a typo in the earlier snapshot ([#868](https://github.com/mono/SkiaSharp/pull/868)).

## Infrastructure

- Move to Azure DevOps Pipelines ([#776](https://github.com/mono/SkiaSharp/pull/776)) and refactor the pipeline for maximum reuse ([#785](https://github.com/mono/SkiaSharp/pull/785)) with a switchable preview label ([#857](https://github.com/mono/SkiaSharp/pull/857)).
- Set up NuGet signing ([#786](https://github.com/mono/SkiaSharp/pull/786)) — skipped on public builds ([#790](https://github.com/mono/SkiaSharp/pull/790)) — plus iOS/macOS certificate handling ([#777](https://github.com/mono/SkiaSharp/pull/777)) and publicly published native artifacts ([#889](https://github.com/mono/SkiaSharp/pull/889)).
- Prep the final release: tweaks in [#1013](https://github.com/mono/SkiaSharp/pull/1013) and making `netstandard SkiaSharp.Views.Forms.dll` ref-only ([#1012](https://github.com/mono/SkiaSharp/pull/1012)).
- Move documentation over to `SkiaSharp-API-docs` ([#843](https://github.com/mono/SkiaSharp/pull/843)) and refresh the docs pass ([#979](https://github.com/mono/SkiaSharp/pull/979)).
- Sample and PR-builder polish: [#875](https://github.com/mono/SkiaSharp/pull/875), [#869](https://github.com/mono/SkiaSharp/pull/869), [#1021](https://github.com/mono/SkiaSharp/pull/1021), [#1025](https://github.com/mono/SkiaSharp/pull/1025), [#903](https://github.com/mono/SkiaSharp/pull/903), [#891](https://github.com/mono/SkiaSharp/pull/891), [#886](https://github.com/mono/SkiaSharp/pull/886), [#885](https://github.com/mono/SkiaSharp/pull/885), [#884](https://github.com/mono/SkiaSharp/pull/884), [#850](https://github.com/mono/SkiaSharp/pull/850), [#817](https://github.com/mono/SkiaSharp/pull/817), [#779](https://github.com/mono/SkiaSharp/pull/779), [#737](https://github.com/mono/SkiaSharp/pull/737), [#724](https://github.com/mono/SkiaSharp/pull/724).
- Bring `.NET`-side sanity: `Env.Is64BitOperatingSystem` isn't available under `dotnet build`, worked around in [#1020](https://github.com/mono/SkiaSharp/pull/1020).
- Add support for Xamarin.Forms on `netcoreapp3.1` (already staged for a later cross-runtime story) — see the downstream `2.88` notes.
- Change the native-artifact name to match the Azure Build ([#892](https://github.com/mono/SkiaSharp/pull/892)).
- Rescue `master` after the split ([#962](https://github.com/mono/SkiaSharp/pull/962), [#966](https://github.com/mono/SkiaSharp/pull/966)) and add the `*.dylib` bits to the app bundle when using the full framework ([#919](https://github.com/mono/SkiaSharp/pull/919)).

## Breaking Changes

The managed-native re-work ([#900](https://github.com/mono/SkiaSharp/pull/900)) and the desktop package split ([#899](https://github.com/mono/SkiaSharp/pull/899)) together move a meaningful slice of the public surface. Walk the [SkiaSharp API diff](1.68.1/index.md) — the breaking sections cover both `SkiaSharp` and `SkiaSharp.Views.Desktop`.
