# Version 1.68.1

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.1)

> **API changes** · [SkiaSharp API diff](1.68.1/index.md) · [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md)

## Highlights

Major structural release that reshapes SkiaSharp's desktop and interop story: the SkiaSharp.Views.Desktop package is split into per-framework packages (WPF, WinForms, GTK), the managed↔native ownership layer is reworked to plug a category of memory leaks and disposal bugs, and a P/Invoke generator replaces most hand-written interop. Xamarin.Forms picks up a WPF backend, watchOS/tvOS get bitcode and `arm64_32`, HarfBuzzSharp is refreshed with new font-funcs bindings, and `Span<T>` is threaded through data/bitmap/text-blob APIs. Ships alongside **[HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md)** — see that page for its binding details.

Big community push: [@Gillibald](https://github.com/Gillibald) landed the HarfBuzzSharp 2.6.1 bump, font-funcs bindings, and Linux export-hiding; [@alexandrvslv](https://github.com/alexandrvslv) bound `SkPictureShader`; and contributions from [@newky2k](https://github.com/newky2k), [@danien](https://github.com/danien), [@Odirb](https://github.com/Odirb).

## Breaking Changes

- **Desktop packages split** — The single `SkiaSharp.Views.Desktop` package is broken into per-framework packages (WPF, WinForms, GTK), each shipping its own assembly. Consumers must reference the specific package(s) they need. ([#899](https://github.com/mono/SkiaSharp/pull/899))
- **`netstandard SkiaSharp.Views.Forms.dll` is now ref-only** — The netstandard flavor of the Xamarin.Forms views assembly is now a reference assembly; the runtime implementation lives in the platform-specific packages. ([#1012](https://github.com/mono/SkiaSharp/pull/1012))
- **Reworked managed/native ownership layer** — The internal object-lifetime plumbing was rewritten; downstream code that reached into `SKObject` internals may need updating. Public API-level usage is unaffected. ([#900](https://github.com/mono/SkiaSharp/pull/900))
- **Span-based overloads for data/bitmap/text-blob** — Previous array-based paths remain, but new `Span<T>` overloads are the preferred entry points. ([#865](https://github.com/mono/SkiaSharp/pull/865))

See the [full API diff](1.68.1/index.md) for the complete list.

## Engine

- **Skia sync + build defines** — Updates the Skia submodule and disables `fontconfig` when unavailable. ([#985](https://github.com/mono/SkiaSharp/pull/985), [#821](https://github.com/mono/SkiaSharp/pull/821))
- **Reworked managed/native types** — Rewritten object-lifetime layer that unifies disposal, cleanup, and finalization. ([#900](https://github.com/mono/SkiaSharp/pull/900))
- **P/Invoke generator** — Adds a code generator that produces the interop layer, replacing hand-written signatures. ([#992](https://github.com/mono/SkiaSharp/pull/992))
- **Improved delegate proxies + reduced verbosity for multi-delegates** ([#878](https://github.com/mono/SkiaSharp/pull/878), [#882](https://github.com/mono/SkiaSharp/pull/882))
