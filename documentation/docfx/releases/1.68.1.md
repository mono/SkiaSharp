# Version 1.68.1

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.1)

> **API changes** · [SkiaSharp API diff](1.68.1/index.md) · [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md)

<!-- Generated: 2026-07-06T23:24:50Z by claude-opus-4.7 -->

Big infrastructure-and-API refresh of the 1.68 line. Managed–native interop is re-worked to use `Span<T>` across pixels/bitmaps/text-blobs, the desktop packages split apart, a WPF backend for Xamarin.Forms lands, HarfBuzz jumps to 2.6.1 (via 2.5.2/2.5.3), and the pipeline moves onto Azure DevOps with NuGet signing. HarfBuzzSharp 2.6.1 co-ships — see the [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md) page for its binding details.

## Highlights

- Re-worked managed–native types ([#900](https://github.com/mono/SkiaSharp/pull/900)) sit under a new `Span<T>`-based surface for pixel data, bitmap data, and text blobs ([#865](https://github.com/mono/SkiaSharp/pull/865)) — allocation-free interop is now the default across the API.
- **WPF backend for Xamarin.Forms** ([#917](https://github.com/mono/SkiaSharp/pull/917)) rounds out the Forms platform matrix.
- Desktop projects/packages split apart so consumers pull in only the presentation packages they need ([#899](https://github.com/mono/SkiaSharp/pull/899)); the netstandard `SkiaSharp.Views.Forms.dll` is now ref-only ([#1012](https://github.com/mono/SkiaSharp/pull/1012)).
- watchOS bitcode plus the `arm64_32` architecture ([#967](https://github.com/mono/SkiaSharp/pull/967), [#971](https://github.com/mono/SkiaSharp/pull/971)) — SkiaSharp now ships for Apple Watch Series 4+.
- HarfBuzzSharp 2.6.1 (via 2.5.2 → 2.5.3 → 2.6.1) ships alongside this release ([#872](https://github.com/mono/SkiaSharp/pull/872), [#904](https://github.com/mono/SkiaSharp/pull/904), [#929](https://github.com/mono/SkiaSharp/pull/929)) — see the linked page for binding details.
- Azure DevOps pipeline with NuGet signing lands ([#776](https://github.com/mono/SkiaSharp/pull/776), [#785](https://github.com/mono/SkiaSharp/pull/785), [#786](https://github.com/mono/SkiaSharp/pull/786)); on Windows the native build now uses Clang for the m68 Skia bump ([#1007](https://github.com/mono/SkiaSharp/pull/1007)).

## API Surface

- `Span<T>` overloads for pixel/bitmap data and text blobs ([#865](https://github.com/mono/SkiaSharp/pull/865)) alongside the managed–native type re-work ([#900](https://github.com/mono/SkiaSharp/pull/900)).
- `SkPictureShader` bindings ([#973](https://github.com/mono/SkiaSharp/pull/973)) and int unicode overloads for consistency across the shaping APIs ([#972](https://github.com/mono/SkiaSharp/pull/972)) — both from [@Gillibald](https://github.com/Gillibald) and [@alexandrvslv](https://github.com/alexandrvslv).
- `SKTypeface.IsFixedPitch` binding for detecting monospace fonts ([#1019](https://github.com/mono/SkiaSharp/pull/1019)).
- HarfBuzzSharp gains font funcs (custom `hb_font_funcs_t` callbacks) contributed by [@Gillibald](https://github.com/Gillibald) ([#881](https://github.com/mono/SkiaSharp/pull/881)); HarfBuzz can load a face from raw table data ([#862](https://github.com/mono/SkiaSharp/pull/862)).
- Delegate proxies simplified ([#878](https://github.com/mono/SkiaSharp/pull/878)); helper method reduces boilerplate for multi-delegate registration ([#882](https://github.com/mono/SkiaSharp/pull/882)).

## Engine

- HarfBuzz native engine bumped through 2.5.2 → 2.5.3 → 2.6.1 by [@Gillibald](https://github.com/Gillibald) ([#872](https://github.com/mono/SkiaSharp/pull/872), [#904](https://github.com/mono/SkiaSharp/pull/904), [#929](https://github.com/mono/SkiaSharp/pull/929)); a few tweaks to align the HarfBuzz surface ([#915](https://github.com/mono/SkiaSharp/pull/915)).
- Skia submodule refreshed ([#985](https://github.com/mono/SkiaSharp/pull/985)) with a typo fix ([#868](https://github.com/mono/SkiaSharp/pull/868)) and a P/Invoke generator ([#992](https://github.com/mono/SkiaSharp/pull/992)) so future updates rebase cleanly.

## Text & Fonts

- HarfBuzzSharp font funcs contributed by [@Gillibald](https://github.com/Gillibald) ([#881](https://github.com/mono/SkiaSharp/pull/881)).
- HarfBuzz can now load a face directly from table data instead of a full font file ([#862](https://github.com/mono/SkiaSharp/pull/862)).

## Platform

- **WPF backend for Xamarin.Forms** ([#917](https://github.com/mono/SkiaSharp/pull/917)) so all Forms targets have native SkiaSharp presentation.
- Desktop projects/packages split apart ([#899](https://github.com/mono/SkiaSharp/pull/899)); netstandard `SkiaSharp.Views.Forms.dll` becomes a reference assembly ([#1012](https://github.com/mono/SkiaSharp/pull/1012)); the full-framework Windows Forms build stays in step ([#886](https://github.com/mono/SkiaSharp/pull/886)).
- watchOS bitcode plus `arm64_32` for Apple Watch Series 4+ ([#967](https://github.com/mono/SkiaSharp/pull/967), [#971](https://github.com/mono/SkiaSharp/pull/971)).
- Linux linker version script hides dependency symbols to prevent accidental symbol capture, contributed by [@Gillibald](https://github.com/Gillibald) ([#984](https://github.com/mono/SkiaSharp/pull/984)); SkiaSharp can now build without fontconfig ([#821](https://github.com/mono/SkiaSharp/pull/821)).
- Tizen builds pin the correct OpenJDK combination for reliable native builds ([#817](https://github.com/mono/SkiaSharp/pull/817)).
- Bundled `*.dylib` files land in the app bundle for full-framework Mac apps ([#919](https://github.com/mono/SkiaSharp/pull/919)).
- Xamarin.Forms package refresh ([#916](https://github.com/mono/SkiaSharp/pull/916)).

## Bug Fixes

- Fix iOS canvas implementation regression ([#879](https://github.com/mono/SkiaSharp/pull/879)) and its correction ([#1022](https://github.com/mono/SkiaSharp/pull/1022)).
- `ToSKPixmap` no longer creates display artifacts on transparent bitmaps, contributed by [@Odirb](https://github.com/Odirb) ([#863](https://github.com/mono/SkiaSharp/pull/863)).
- Raster views stop trying to allocate zero-byte buffers ([#761](https://github.com/mono/SkiaSharp/pull/761)).
- Memory leak in `SKRoundRect` fixed ([#995](https://github.com/mono/SkiaSharp/pull/995)); UWP GPU view stability ([#1024](https://github.com/mono/SkiaSharp/pull/1024)); freeing memory when a view is detached ([#1005](https://github.com/mono/SkiaSharp/pull/1005)); color-space objects that were meant to be references stop being disposed ([#922](https://github.com/mono/SkiaSharp/pull/922)).
- `Environment.Is64BitOperatingSystem` isn't available on all TFMs — replaced by a runtime probe ([#1020](https://github.com/mono/SkiaSharp/pull/1020)); bad colors in a test path corrected ([#1023](https://github.com/mono/SkiaSharp/pull/1023)); patch for #994 landed via [#1000](https://github.com/mono/SkiaSharp/pull/1000).
