# Version 1.68.1

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.1)

> **API changes** · [SkiaSharp API diff](1.68.1/index.md) · [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md)

<!-- Generated: 2026-07-03T21:22:41Z by claude-opus-4.7 -->

A large point release that consolidated the desktop and Xamarin.Forms story, adopted `Span<T>` on the hot paths, and modernised the CI/build infrastructure. [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md) ships alongside for text-shaping consumers; refer to its page for binding details.

## Highlights

- Split the desktop projects and packages so WPF, WinForms, and GTK consumers each get a focused package instead of one bundle ([#899](https://github.com/mono/SkiaSharp/pull/899), [#917](https://github.com/mono/SkiaSharp/pull/917)).
- Rewrote the managed-native layer for stability and performance, including new `Span<T>` overloads for data, bitmap data, and text blobs ([#900](https://github.com/mono/SkiaSharp/pull/900), [#865](https://github.com/mono/SkiaSharp/pull/865)).
- Generated the P/Invoke layer from a code generator so signatures stay consistent across the binding ([#992](https://github.com/mono/SkiaSharp/pull/992)).
- Added support for building SkiaSharp without fontconfig for minimal Linux containers ([#821](https://github.com/mono/SkiaSharp/pull/821)).
- Enabled bitcode + `arm64_32` for watchOS/tvOS so the assemblies pass App Store submission ([#971](https://github.com/mono/SkiaSharp/pull/971), [#967](https://github.com/mono/SkiaSharp/pull/967)).

## New features

- **`SKPictureShader`** — new binding to reuse pre-recorded `SKPicture` content as a shader ([#973](https://github.com/mono/SkiaSharp/pull/973) ❤️).
- **`SKTypeface.IsFixedPitch`** — new property to detect monospaced typefaces ([#1019](https://github.com/mono/SkiaSharp/pull/1019)).
- **HarfBuzzSharp `font_funcs`** — new callback-based font-metrics extension surface ([#881](https://github.com/mono/SkiaSharp/pull/881) ❤️).
- **HarfBuzz `hb_face_create_from_table`** — load HarfBuzz faces from raw table data ([#862](https://github.com/mono/SkiaSharp/pull/862)).
- **Integer unicode overloads** — added `int`-taking overloads for the code-point APIs so callers don't have to cast to `char` first ([#972](https://github.com/mono/SkiaSharp/pull/972) ❤️).
- **Custom preview labels** — build infrastructure supports naming preview channels other than the default ([#857](https://github.com/mono/SkiaSharp/pull/857)).
- **WPF backend for Xamarin.Forms** — parity with the other desktop backends ([#917](https://github.com/mono/SkiaSharp/pull/917)).

## Bug fixes

- Fixed transparent-bitmap display artifacts when converting `SKBitmap` to `SKPixmap` ([#863](https://github.com/mono/SkiaSharp/pull/863) ❤️).
- Fixed the iOS canvas implementation so drawing lands in the correct pixel bounds ([#879](https://github.com/mono/SkiaSharp/pull/879), and follow-up correction in [#1022](https://github.com/mono/SkiaSharp/pull/1022)).
- Fixed a memory leak in `SKRoundRect` ([#995](https://github.com/mono/SkiaSharp/pull/995)).
- Raster views no longer allocate a zero-byte backing buffer ([#761](https://github.com/mono/SkiaSharp/pull/761)).
- Freed the native GPU state when a view is detached, resolving several memory-pressure reports ([#1005](https://github.com/mono/SkiaSharp/pull/1005)).
- Stopped disposing color spaces that are shared static references ([#922](https://github.com/mono/SkiaSharp/pull/922)).
- Improved the UWP GPU view's stability under repeated attach/detach ([#1024](https://github.com/mono/SkiaSharp/pull/1024)).
- Fixed bad colors when drawing with certain painting configurations ([#1023](https://github.com/mono/SkiaSharp/pull/1023)).

## Platform support

- **watchOS / tvOS** — Bitcode enabled and `arm64_32` added to satisfy Apple's submission requirements.
- **Linux** — Native builds now switched to Docker containers, and consumers can opt out of `fontconfig` for minimal images.
- **Windows** — Clang toolchain adopted for the m68 Skia build.
- **Signing** — NuGet signing added; iOS/macOS certificates rotated.
