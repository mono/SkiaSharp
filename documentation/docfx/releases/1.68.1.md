# Version 1.68.1

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.1)

> **API changes** · [SkiaSharp API diff](1.68.1/index.md) · [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md)

## Highlights

Large maintenance and infrastructure release for the 1.x line — 66 PRs spanning a P/Invoke generator, `Span<T>` support for data/bitmap/text-blob buffers, an early WPF backend for Xamarin.Forms, split desktop packages, and a fresh Azure DevOps pipeline with NuGet code signing. Community contributors also delivered `SKPictureShader` bindings, `int` unicode overloads, HarfBuzz `font-funcs`, a raft of HarfBuzz version bumps, and Linux linker-script fixes. HarfBuzzSharp 2.6.1 ships alongside this release — see [HarfBuzzSharp 2.6.1](harfbuzzsharp/2.6.1.md) for its binding details.

## Breaking Changes

- **`SKColorSpace.CreateRgb` defaults dropped** — The optional `SKColorSpaceFlags flags = 0` argument on all four `CreateRgb` overloads is now required; pass `SKColorSpaceFlags.None` explicitly to preserve the previous behaviour.
- **`SKPath` iterators moved to `SKObject`** — `SKPath.Iterator`, `SKPath.RawIterator`, and `SKPath.OpBuilder` now derive from `SKObject` instead of `SKNativeObject`, aligning them with the standard managed-native lifecycle machinery.
- **`SKObject.OwnsHandle` removed** — The protected `OwnsHandle` property is gone; ownership is now expressed through the constructor and disposal path.

## Engine

- **Bind `SKPictureShader`** ([#973](https://github.com/mono/SkiaSharp/pull/973) by @alexandrvslv) — Exposes the picture-backed shader factory so recorded pictures can be reused as fill sources.
- **`SKShader` improved Perlin noise binding, `SKColorF`/`SKColorSpace` overloads, more `SKImage` and `SKRegion` members, and `SKCanvas` additions** — Several waves of API completeness work fill in previously missing overloads.

## API Surface

- **`Span<T>` for data, bitmap data, and text blobs** ([#865](https://github.com/mono/SkiaSharp/pull/865) by @mattleibow) — Replaces byte-array parameters with span-based overloads across the pixel/data surface, unlocking allocation-free callers.
- **`int` unicode overloads on HarfBuzz** ([#972](https://github.com/mono/SkiaSharp/pull/972) by @Gillibald) — Adds `int` versions of the code-point APIs so callers don't need to widen manually.
- **P/Invoke generator** ([#992](https://github.com/mono/SkiaSharp/pull/992) by @mattleibow) — Introduces the internal generator that keeps the P/Invoke layer synchronised with the C API.

## Text & Fonts

- **HarfBuzz refresh** — Multiple community bumps land alongside this release, ending at HarfBuzz 2.6.1 ([#872](https://github.com/mono/SkiaSharp/pull/872), [#904](https://github.com/mono/SkiaSharp/pull/904), [#929](https://github.com/mono/SkiaSharp/pull/929) by @Gillibald). The `font-funcs` binding ([#881](https://github.com/mono/SkiaSharp/pull/881) by @Gillibald) exposes custom font-function tables, and [#862](https://github.com/mono/SkiaSharp/pull/862) allows loading a face directly from table data.
- **Bind `SKTypeface.IsFixedPitch`** ([#1019](https://github.com/mono/SkiaSharp/pull/1019) by @mattleibow) — Surfaces the monospace check on typefaces.

## Bug Fixes

- **Free memory when the view is detached** ([#1005](https://github.com/mono/SkiaSharp/pull/1005) by @mattleibow) — Fixes a native-memory leak on view teardown ([#1004](https://github.com/mono/SkiaSharp/issues/1004)).
- **`SKRoundRect` memory leak** ([#995](https://github.com/mono/SkiaSharp/pull/995) by @mattleibow) — Plugs a leak in the round-rect constructor path.
- **Colour spaces created as references** ([#922](https://github.com/mono/SkiaSharp/pull/922) by @mattleibow) — Fixes ownership on some `SKColorSpace` factory results so they are not incorrectly destroyed.
- **iOS canvas implementation fix** ([#879](https://github.com/mono/SkiaSharp/pull/879) by @mattleibow) — Straightens out the iOS canvas rendering path.
- **`ToSKPixmap` transparent artefacts** ([#863](https://github.com/mono/SkiaSharp/pull/863) by @Odirb) — Fixes display artefacts when converting transparent bitmaps ([#849](https://github.com/mono/SkiaSharp/issues/849)).
- **Raster views no longer allocate zero bytes** ([#761](https://github.com/mono/SkiaSharp/pull/761) by @mattleibow) — Avoids the zero-byte allocation crash ([#759](https://github.com/mono/SkiaSharp/issues/759)).

## Platform

- **Improve stability of UWP GPU View** ([#1024](https://github.com/mono/SkiaSharp/pull/1024) by @mattleibow) — Tightens the UWP swap-chain / GPU view lifecycle.
- **WPF backend for Xamarin.Forms** ([#917](https://github.com/mono/SkiaSharp/pull/917) by @mattleibow) — Adds the WPF renderer for Forms so shared code can target WPF apps.
- **Split desktop projects and packages** ([#899](https://github.com/mono/SkiaSharp/pull/899) by @mattleibow) — Reorganises the desktop deliverables into per-technology packages instead of one aggregate.
- **watchOS bitcode + `arm64_32`** ([#967](https://github.com/mono/SkiaSharp/pull/967), [#971](https://github.com/mono/SkiaSharp/pull/971) by @mattleibow) — Enables bitcode for watchOS/tvOS and adds the `arm64_32` architecture required by Apple Watch (skia: mono/skia#62, mono/skia#63).
- **Build without fontconfig on Linux** ([#821](https://github.com/mono/SkiaSharp/pull/821) by @mattleibow) — Makes fontconfig optional for containerised Linux scenarios.
- **Linux linker script hides dependency exports** ([#984](https://github.com/mono/SkiaSharp/pull/984) by @Gillibald) — Prevents accidental symbol collisions on Linux by hiding third-party symbols.

## Lifecycle & Internals

- **Azure DevOps pipeline, NuGet signing, and platform certificates** ([#776](https://github.com/mono/SkiaSharp/pull/776), [#777](https://github.com/mono/SkiaSharp/pull/777), [#786](https://github.com/mono/SkiaSharp/pull/786), [#790](https://github.com/mono/SkiaSharp/pull/790) by @mattleibow) — Introduces the Azure DevOps pipeline, iOS/macOS signing certificates, NuGet signing, and public-build handling.
- **Re-work managed-native types** ([#900](https://github.com/mono/SkiaSharp/pull/900) by @mattleibow) — Overhauls the managed-native lifecycle plumbing that the iterator base-type shift depends on (skia: mono/skia#61).
- **Move documentation to SkiaSharp-API-docs** ([#843](https://github.com/mono/SkiaSharp/pull/843) by @mattleibow) — Splits the docs into their own submodule.
