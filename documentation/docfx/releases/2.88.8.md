## Highlights

Servicing release for the 2.88 line focused on API-compat overloads, trimming/AOT compatibility, and build/CI cleanup. Ships alongside **[HarfBuzzSharp 7.3.0.2](harfbuzzsharp/7.3.0.2.md)** — see that page for its binding details. Backports from the [Copilot bot](https://github.com/apps/copilot) roll in fixes originally landed on the 3.x mainline: an Apple-side script fix, a `PlatformConfiguration` trimming fix, an `async void` fix in `SKXamlCanvas`, and an old-Alpine `gn` workaround.

## Breaking Changes

- **`SKImageFilter.Create*` — implicit defaults removed** — A large batch of `SKImageFilter.Create*` overloads (`CreateAlphaThreshold`, `CreateArithmetic`, `CreateBlendMode`, `CreateBlur`, `CreateColorFilter`, `CreateDilate`, `CreateDisplacementMapEffect`, `CreateDistantLitDiffuse/Specular`, `CreateDropShadow`, `CreateDropShadowOnly`, `CreateErode`, `CreateMagnifier`, `CreateMatrixConvolution`, `CreateMerge`, `CreateOffset`, `CreatePaint`, `CreatePointLitDiffuse/Specular`, `CreateSpotLitDiffuse/Specular`) had their trailing `SKImageFilter input = null` and/or `SKImageFilter.CropRect cropRect = null` default values removed. Callers using positional arguments stay source-compatible; anyone omitting the trailing arguments must add explicit `null`s and recompile.

See the [full API diff](2.88.8/index.md) for the complete list.

- **`SKXamlCanvas` no longer uses `async void`** — Replaces the fire-and-forget pattern with a proper async flow so exceptions are observable. ❤️ Copilot bot backport of community work ([#2731](https://github.com/mono/SkiaSharp/pull/2731))
