# Version 2.88.8

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.8)

> **API changes** · [SkiaSharp API diff](2.88.8/index.md) · [HarfBuzzSharp 7.3.0.2](harfbuzzsharp/7.3.0.2.md)

<!-- Generated: 2026-07-03T21:22:41Z by claude-opus-4.7 -->

The final patch release on the 2.x line, folding in servicing fixes back-ported from the 3.x branch along with a few new overloads that make cross-version code easier to write. [HarfBuzzSharp 7.3.0.2](harfbuzzsharp/7.3.0.2.md) ships alongside; refer to its page for the shaping-side binding details.

## Highlights

- Added new overloads so callers written against the 3.x branch can compile back down to 2.88.8 with fewer changes ([#2810](https://github.com/mono/SkiaSharp/pull/2810)).
- Made `PlatformConfiguration` properties trim-friendly so trimmed WASM and mobile apps no longer lose the values at publish time ([#2734](https://github.com/mono/SkiaSharp/pull/2734)).
- Removed `async void` from `SKXamlCanvas` so unhandled exceptions no longer bring down the host process ([#2731](https://github.com/mono/SkiaSharp/pull/2731)).
- Fixed the macOS build script so contributors can rebuild the native library locally on Apple Silicon and Intel ([#2783](https://github.com/mono/SkiaSharp/pull/2783)).
- Pinned the `gn` version so the native build no longer breaks on older Alpine base images ([#2712](https://github.com/mono/SkiaSharp/pull/2712)).

Plus several CI, APIScan, and compliance cleanups.

## Community contributors ❤️

Thanks to the community members who back-ported these fixes to the 2.x servicing branch — every one of these changes shipped as an automatic back-port via `github-actions[bot]`, but they were originated by the community: [Ivan Tomilov](https://github.com/mono/SkiaSharp/pull/2717), [Cameron White](https://github.com/mono/SkiaSharp/pull/2720), [dellis1972](https://github.com/mono/SkiaSharp/pull/2749) and [chylik](https://github.com/mono/SkiaSharp/pull/2711).
