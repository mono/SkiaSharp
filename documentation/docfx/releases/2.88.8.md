# Version 2.88.8

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.8)

> **API changes** · [SkiaSharp API diff](2.88.8/index.md) · [HarfBuzzSharp 7.3.0.2](harfbuzzsharp/7.3.0.2.md)

Servicing release for the `2.88` line. Nine PRs — compatibility overloads, a few compliance/YAML cleanups and a handful of trimming-safety fixes cherry-picked from the `main` branch by the merge bot. HarfBuzz `7.3.0.2` ships alongside — see [HarfBuzzSharp 7.3.0.2](harfbuzzsharp/7.3.0.2.md).

## Highlights

- Add compatibility overloads to the core surface for consumers upgrading from earlier `2.88.x` releases ([#2810](https://github.com/mono/SkiaSharp/pull/2810)).
- Make `PlatformConfiguration` properties trimmable so `SkiaSharp.Views` assemblies play nicely with published trimmed builds (`#2717`, cherry-picked via [#2734](https://github.com/mono/SkiaSharp/pull/2734)).
- Avoid `async void` inside `SKXamlCanvas` to make the Uno/UWP path safer against unobserved exceptions (`#2720`, cherry-picked via [#2731](https://github.com/mono/SkiaSharp/pull/2731)).
- Stop emitting invalid `extern` declarations that produced link-time warnings on some toolchains ([#2710](https://github.com/mono/SkiaSharp/pull/2710)).

## Fixes

- Pin the `gn` used on old Alpine to the version that actually works there (`#2711`, cherry-picked via [#2712](https://github.com/mono/SkiaSharp/pull/2712)).
- Fix the release script for macOS (`#2749`, cherry-picked via [#2783](https://github.com/mono/SkiaSharp/pull/2783)).

## Infrastructure

- Clean up APIScan and the surrounding compliance YAML ([#2805](https://github.com/mono/SkiaSharp/pull/2805)), plus a couple of YAML-only bumps ([#2791](https://github.com/mono/SkiaSharp/pull/2791), [#2792](https://github.com/mono/SkiaSharp/pull/2792)).

## Breaking Changes

`2.88.8` intentionally minimises surface change to unblock consumers on the `2.x` line. A small number of signatures moved during the compat-overload cleanup; the exact list is in the [SkiaSharp API diff](2.88.8/index.md).
