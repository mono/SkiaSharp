## Highlights

Servicing release on the 2.88.x line. New compat overloads restore a handful of shapes that had drifted, several third-party bug-fix PRs land via the `[release/2.x]` cherry-pick queue, and CI/build scripts are tightened. Ships alongside [HarfBuzzSharp 7.3.0.2](harfbuzzsharp/7.3.0.2.md).

- 🎨 **Core** — new overloads improve source-compat with 2.88.7 callers; several `SKImageFilter` factories no longer default their region/paint parameters to `null` (see Breaking Changes).
- 🪟 **Windows / Uno** — `SKXamlCanvas` no longer uses `async void`, avoiding intermittent crashes.
- 🏗️ **Build / CI** — trim-safety fixes for `PlatformConfiguration`, cleaner Alpine-compatible `gn` selection, and APIScan/YAML tidy-ups.

## Breaking Changes

The `SKImageFilter` cleanup dropped default `null` values from several factory parameters — positional calls compile unchanged, but callers relying on the defaults must now pass them explicitly:

- **SKImageFilter.CreateBlur / CreateColorFilter / CreateCompose / CreateDisplacementMapEffect / CreateDropShadow / CreateDropShadowOnly / CreateImage / CreateMagnifier / CreateMatrixConvolution / CreateMerge / CreateOffset / CreatePaint / CreatePicture / CreateTile / CreateXfermode** — the trailing `SKImageFilter input = null, SKImageFilter.CropRect cropRect = null` parameters lost their default values in several overloads. Pass `null` (or an actual filter/crop) explicitly.
