# Version 2.88.8

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.8)

> **API changes** · [SkiaSharp API diff](2.88.8/index.md) · [HarfBuzzSharp 7.3.0.2](harfbuzzsharp/7.3.0.2.md)

<!-- Generated: 2026-07-06T23:24:50Z by claude-opus-4.7 -->

Servicing update for the 2.88 line. Adds a handful of overloads to broaden ABI compatibility, forwards two behavioral fixes from the 3.x line, and tidies compliance/CI plumbing. HarfBuzzSharp 7.3.0.2 co-ships — see the [HarfBuzzSharp 7.3.0.2](harfbuzzsharp/7.3.0.2.md) page for its binding details.

## API Surface

- Backport of new overloads that keep the 2.88 line binary-compatible with consumers that were built against later patches ([#2810](https://github.com/mono/SkiaSharp/pull/2810)).

## Bug Fixes

- `SKXamlCanvas` no longer uses `async void`, avoiding lost exceptions and re-entrancy hazards during redraws (backport of [#2720](https://github.com/mono/SkiaSharp/pull/2720) via [#2731](https://github.com/mono/SkiaSharp/pull/2731)).
- `PlatformConfiguration` properties are marked trimmable so linker-enabled apps drop the platform detection tables cleanly (backport of [#2717](https://github.com/mono/SkiaSharp/pull/2717) via [#2734](https://github.com/mono/SkiaSharp/pull/2734)).
- Removed the invalid extern declarations that could confuse P/Invoke resolution on some hosts ([#2710](https://github.com/mono/SkiaSharp/pull/2710)).
- Pinned the `gn` version that works on the older Alpine base image so 2.88 native builds keep succeeding (backport of [#2711](https://github.com/mono/SkiaSharp/pull/2711) via [#2712](https://github.com/mono/SkiaSharp/pull/2712)).

## Lifecycle & Internals

- APIScan and other compliance YAML on the 2.x branch cleaned up ([#2805](https://github.com/mono/SkiaSharp/pull/2805)).
- Plus several CI and documentation improvements.

## Breaking Changes

None in this release.
