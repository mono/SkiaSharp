# Version 4.151.0

> **New rendering APIs, Skia m151 engine, and v4 modernization** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.151.0-preview.1.1) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v4.151.0-preview.1.1)

> **API changes** · [SkiaSharp API diff](4.151.0/index.md) · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Version 4.151.0 rolls up the full 4.150.0 preview cycle and delivers the Skia m151 engine milestone. This first preview introduces four new rendering API methods — `SKPaint.GetFastBounds()`, `SKColorFilter.CreateOverdraw()`, `SKImageFilter.CreateCrop()`, and `SKImageFilter.CreateEmpty()` — plus a `SKSurface.Canvas` performance boost and a raster blur bug fix. Two breaking changes complete the v4 modernization: pre-v4 obsoletes are now errors, and the deprecated .NET Interactive (Polyglot Notebooks) support has been removed. HarfBuzz 14.2.1 ships alongside this release — see the API changes link above for binding details. ❤️ [@ramezgerges](https://github.com/ramezgerges) was the standout community contributor, delivering the performance improvement, raster blur fix, v4 API cleanup, and a WASM build fix.

## Breaking Changes

- **Pre-v4 obsoletes promoted to errors** — Remaining v3-era APIs that were `[Obsolete]` are now compile-time errors, and stale obsolete enums have been trimmed from the reference assembly. Update any call sites to their v4 equivalents before upgrading. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))
- **Remove .NET Interactive / Polyglot Notebooks support** — The long-deprecated .NET Interactive integration has been removed. Projects that relied on the Polyglot Notebooks NuGet package should migrate to the standard SkiaSharp NuGet packages. ([#4186](https://github.com/mono/SkiaSharp/pull/4186))

## Engine

- **Skia milestone m150** — Updates the engine baseline to Chrome/m150, pulling in all upstream improvements and fixes from that milestone. ([#4146](https://github.com/mono/SkiaSharp/pull/4146))
- **Skia m150 upstream bug fixes** — Merges Chrome/m150 upstream patch fixes into the release branch. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))
- **Skia milestone m151** — Advances the engine to Chrome/m151, the latest Skia milestone. ([#4294](https://github.com/mono/SkiaSharp/pull/4294))
- **Fix WASM `SK_BUILD_FOR_WASM` redefinition** — Bumps the Skia submodule to eliminate a build-breaking `SK_BUILD_FOR_WASM` redefinition that affected WebAssembly targets. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3788](https://github.com/mono/SkiaSharp/pull/3788))

## API Surface

- **`SKPaint.GetFastBounds()`** — New method for quick-reject culling: returns a conservative bounding box of what the paint would draw, enabling fast visibility tests before committing to a full render pass. ([#4271](https://github.com/mono/SkiaSharp/pull/4271))
- **`SKColorFilter.CreateOverdraw()`** — New factory that creates an overdraw visualization color filter, useful for debugging rendering hot-spots. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — New factory that crops the result of a filter graph to a specified rectangle, with optional chaining to a child filter. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New factory that produces a transparent-black output regardless of input, useful as a no-op or placeholder node in filter graphs. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

## Performance

- **`SKSurface.Canvas` wrapper caching** — The `Canvas` property now returns the same `SKCanvas` wrapper instance across calls rather than allocating a new wrapper each time, reducing allocation pressure in tight render loops. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

## Bug Fixes

- **Fix small-sigma raster blur regression** — Restores correct fast-path raster blur behavior for small sigma values by defining `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS`, fixing a visible rendering quality regression. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4269](https://github.com/mono/SkiaSharp/pull/4269))

## Platform

- **Tizen native dependencies in Docker** — The Tizen native build now runs inside a Docker container, providing a reproducible and isolated build environment. ([#4154](https://github.com/mono/SkiaSharp/pull/4154))
