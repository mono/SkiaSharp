# Version 4.151.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Version 4.151.0 advances the Skia engine from m148 through m151, adds four new API methods, and ships HarfBuzz 14.2.1 alongside (see the **API changes** link above). Community contributors delivered a notable `SKSurface.Canvas` caching performance improvement, fixed a small-sigma raster blur regression, and promoted pre-v4 legacy APIs to compile errors.

## Breaking Changes

- **Pre-v4 obsolete APIs promoted to errors** — APIs marked obsolete in v4 now produce compile errors instead of warnings; some obsolete enum values have been trimmed from the reference assembly. Projects with v3-era code will need to update before targeting this release. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))

- **Remove .NET Interactive (Polyglot Notebooks) support** — The deprecated Polyglot Notebooks integration has been removed. ([#4186](https://github.com/mono/SkiaSharp/pull/4186))

## New Features

### Engine

- **Skia milestone m151** — Updates the bundled Skia engine to Chrome/m151. ([#4294](https://github.com/mono/SkiaSharp/pull/4294))
- **Skia milestone m150** — Baseline engine update to Chrome/m150. ([#4146](https://github.com/mono/SkiaSharp/pull/4146))
- **Chrome/m150 upstream bug fixes** — Merges Google's m150 fix commits into the build. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))
- **WASM `SK_BUILD_FOR_WASM` redefinition fix** — Removes a preprocessor redefinition that broke WebAssembly builds. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3788](https://github.com/mono/SkiaSharp/pull/3788))

### API Surface

- **`SKPaint.GetFastBounds()`** — Returns a conservative bounding box for a paint, enabling fast geometry culling before drawing. ([#4271](https://github.com/mono/SkiaSharp/pull/4271))
- **`SKColorFilter.CreateOverdraw()`** — Creates a color filter that visualizes overdraw — a debugging aid that highlights pixels drawn multiple times per frame. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — Creates an image filter that crops its input to a rectangle. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — Creates an image filter that outputs a transparent image, useful as a no-op placeholder in filter graphs. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Platform

- **Tizen native builds in Docker** — Tizen native dependencies are now built inside a Docker container for improved build reproducibility. ([#4154](https://github.com/mono/SkiaSharp/pull/4154))

## Performance

- **`SKSurface.Canvas` wrapper caching** — The `SKCanvas` wrapper returned by `SKSurface.Canvas` is now cached across calls instead of being reallocated on each access, reducing per-frame allocations in rendering loops. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

## Bug Fixes

- **Small-sigma raster blur regression** — Fixes a regression where small-sigma Gaussian blurs took a slow raster pipeline path. The `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS` flag now restores fast-path blur rendering. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4269](https://github.com/mono/SkiaSharp/pull/4269))
