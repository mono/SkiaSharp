# Version 4.151.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.151.0-preview)

> **API changes** · [SkiaSharp API diff](4.151.0/index.md) · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

First preview on the 4.151 line. Skia moves to Chrome's m151 branch and the
WASM build drops a stale `SK_BUILD_FOR_WASM` redefinition. Ships with the
existing [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md).

## Breaking Changes

None so far. See the [4.151.0 API diff](4.151.0/index.md) as previews land.

## Native Engine

- **Skia bumped to milestone 151** — the shipped Skia engine moves to Chrome's
  m151 branch. ([#4294](https://github.com/mono/SkiaSharp/pull/4294),
  [mono/skia#274](https://github.com/mono/skia/pull/274))
- **`SK_BUILD_FOR_WASM` redefinition removed** — a stale WASM build define was
  redundant with what Skia sets itself; dropping it silences the redefinition
  warning on WASM. ❤️ [@ramezgerges](https://github.com/ramezgerges)
  ([#3788](https://github.com/mono/SkiaSharp/pull/3788),
  [mono/skia#204](https://github.com/mono/skia/pull/204))
