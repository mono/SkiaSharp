# Version 4.151.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Version 4.151.0 advances the Skia engine from m148 to m151, expands the API surface with new image-filter, color-filter, and paint-bounds APIs, and ships HarfBuzzSharp 14.2.1. ⚠️ Pre-v4 deprecation warnings are now **compile errors** — migration is required before upgrading. Community contributor [@ramezgerges](https://github.com/ramezgerges) resolved the long-standing WASM build conflict, fixed a raster blur regression, and improved `SKSurface` canvas caching performance.

## ⚠️ Breaking Changes

- **Pre-v4 API obsoletes promoted to compile errors** — All remaining `[Obsolete]` markers from the v4 migration guide are now hard compile errors, and trimmed obsolete enum values have been removed from the reference assembly. Migrate any remaining v3-era API usages before upgrading. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))
- **Removed .NET Interactive / Polyglot Notebooks support** — The deprecated integration with .NET Interactive (Polyglot Notebooks) has been removed. ([#4186](https://github.com/mono/SkiaSharp/pull/4186))

## New Features

### Engine

- **Skia milestone m151** — Advances the bundled Skia engine to Chrome milestone m151, bringing the latest upstream graphics improvements and fixes. ([#4294](https://github.com/mono/SkiaSharp/pull/4294))
- **Skia milestone m150 with upstream bug fixes** — Includes the initial upgrade to Skia m150 and a follow-up merge of all upstream chrome/m150 bug fixes. ([#4146](https://github.com/mono/SkiaSharp/pull/4146), [#4190](https://github.com/mono/SkiaSharp/pull/4190))
- **WASM `SK_BUILD_FOR_WASM` build conflict resolved** — Removes a duplicate `SK_BUILD_FOR_WASM` preprocessor definition that was preventing WASM builds from compiling. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3788](https://github.com/mono/SkiaSharp/pull/3788))

### API Surface

- **`SKPaint.GetFastBounds()`** — Returns a conservative bounding box for a paint's effect area, enabling efficient quick-reject culling before expensive draw calls. ([#4271](https://github.com/mono/SkiaSharp/pull/4271))
- **`SKColorFilter.CreateOverdraw()`** — New overdraw visualization filter: maps successive pixel passes to distinct colors, making over-rendered areas immediately visible during profiling. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — Crops filter chain output to a specified rectangle, useful for constraining effects to a region of interest. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — A no-op image filter producing transparent black output, useful as a placeholder or to suppress rendering unconditionally in a filter graph. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

## Performance

- **`SKSurface.Canvas` wrapper is now cached** — `SKSurface.Canvas` returns the same `SKCanvas` instance on every call instead of allocating a new wrapper each time, reducing allocations in tight render loops. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

## Bug Fixes

- **Small-sigma raster blur regression fixed** — Restores correct raster-pipeline blur output for small sigma values by enabling `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS`. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4269](https://github.com/mono/SkiaSharp/pull/4269))

Plus several CI, tooling, and documentation infrastructure improvements throughout this release cycle.
