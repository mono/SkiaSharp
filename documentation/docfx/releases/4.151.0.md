# Version 4.151.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.151.0-preview)

> **API changes** · [SkiaSharp API diff](4.151.0/index.md) · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

First preview on the m151 line after the 4.150.0 stable. The two shipping-behavior changes are the Skia bump to milestone 151, which rolls in a stack of upstream fixes, and a build-time drop of the `SK_BUILD_FOR_WASM` macro redefinition that was warning on WASM. HarfBuzzSharp 14.2.1 ships alongside — see its [linked page](harfbuzzsharp/14.2.1.md).

## Breaking Changes

None in this preview. The [SkiaSharp API diff](4.151.0/index.md) shows no signature changes since 4.150.0.

## Engine

- **Skia milestone 151** — moved SkiaSharp onto the upstream m151 branch, picking up the associated stability and rendering fixes. ([#4294](https://github.com/mono/SkiaSharp/pull/4294), [skia#274](https://github.com/mono/skia/pull/274))

## Platform

- **`SK_BUILD_FOR_WASM` redefinition warning dropped** — the WASM native build no longer redefines a macro Skia itself now sets, silencing the warning and simplifying the WASM toolchain path. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3788](https://github.com/mono/SkiaSharp/pull/3788), [skia#204](https://github.com/mono/skia/pull/204))
