# Version 4.151.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.151.0-preview)

> **API changes** · [SkiaSharp API diff](4.151.0/index.md) · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

SkiaSharp 4.151.0 upgrades the Skia engine to Chrome's milestone m151 and rolls up all work from the 4.150.0 preview and release-candidate cycle — four new rendering filter and paint APIs, an `SKSurface` performance improvement, and a raster blur bug fix. HarfBuzz 14.2.1 ships alongside this release; see the API changes link above for binding details. ❤️ [@ramezgerges](https://github.com/ramezgerges) drove several key improvements across the 4.150.0 cycle, from engine fixes to performance work and API cleanup.

## Breaking Changes

- **Pre-v4 obsolete APIs now raise compile errors** — APIs deprecated in the v4 transition that previously emitted warnings now produce compile errors. Remove any remaining uses of pre-v4 patterns before upgrading. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))
- **Polyglot Notebooks / .NET Interactive removed** — The deprecated .NET Interactive integration has been removed. ([#4186](https://github.com/mono/SkiaSharp/pull/4186))

## New Features

### Engine

- **Skia upgraded to milestone m151** — Updates the upstream Skia engine to Chrome/m151. ([#4294](https://github.com/mono/SkiaSharp/pull/4294))
- **Skia upgraded to milestone m150** — Includes the Chrome/m150 milestone update and subsequent m150 upstream bug-fix merge from the 4.150.0 development cycle. ([#4146](https://github.com/mono/SkiaSharp/pull/4146), [#4190](https://github.com/mono/SkiaSharp/pull/4190))
- **HarfBuzz updated to 14.2.1** — Updates the HarfBuzz text-shaping library. ([#4167](https://github.com/mono/SkiaSharp/pull/4167))
- **WebAssembly: drop stale `SK_BUILD_FOR_WASM` redefinition** — Removes a redundant `SK_BUILD_FOR_WASM` define that caused build failures in WASM targets. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3788](https://github.com/mono/SkiaSharp/pull/3788))

### API Surface

- **`SKPaint.GetFastBounds()`** — New method for quick-reject culling; lets renderers skip painting when a bounds check shows the primitive is entirely outside the clip. ([#4271](https://github.com/mono/SkiaSharp/pull/4271))
- **`SKColorFilter.CreateOverdraw()`** — New factory for overdraw visualization, useful for identifying rendering hot-spots during development. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — New factory for clipping an image filter's output to a specified rect. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New factory for a no-op image filter that produces fully transparent output. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Performance

- **`SKSurface.Canvas` wrapper cached across calls** — The `Canvas` property now reuses its wrapper object between calls instead of allocating a new one each time. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

## Bug Fixes

- **Small-sigma raster blur regression fixed** — Restores the fast raster blur path for small sigma values (`SK_AVOID_SLOW_RASTER_PIPELINE_BLURS`), fixing visual regressions on raster backends that were inadvertently introduced. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4269](https://github.com/mono/SkiaSharp/pull/4269))
