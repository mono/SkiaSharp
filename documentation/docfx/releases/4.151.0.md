# Version 4.151.0

## Highlights

Preview cut on the m151 line. Rolls up two previews and an RC of the (skipped) 4.150.0 line, and lands several new image-filter and color-filter APIs, a raster-blur regression fix, and a Surface/Canvas caching perf win. Ships alongside [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) — the shaper is on its new baseline.

- 🎨 **Core** — `SKPaint.GetFastBounds()` for quick-reject culling; `SKImageFilter.CreateCrop()` and `CreateEmpty()`; `SKColorFilter.CreateOverdraw()` for overdraw visualization.
- 🎨 **Core** — small-sigma raster blur regression fixed by `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS`.
- 🎨 **Core** — `SKSurface` now caches its `Canvas` wrapper across calls, avoiding per-call allocation.
- 🎨 **Core** — Skia updated to milestone 151 (see [mono/skia#274](https://github.com/mono/skia/pull/274)), with subsequent m151 bug-fix backports; HarfBuzz updated to 14.2.1 ([mono/skia#257](https://github.com/mono/skia/pull/257)).
- 🌐 **WebAssembly** — Skia submodule bump removes the `SK_BUILD_FOR_WASM` redefinition warning (❤️ @ramezgerges).
- 🏗️ **Build / CI** — Skia upstream-sync workflow gains version-driven rotation, RC/main modes, honest run signals, and Tizen native builds now run inside Docker.
- 🌐 **Docs site** — release cadence surfaced on the releases index, "AI in SkiaSharp" site page, and doc generation is now Linux-only and reproducible.

## Breaking Changes

- **Obsoleted enums and members from before v4** were promoted to `[Obsolete(...IsError = true)]` in the ref assembly and trimmed. Callers still building against those long-deprecated members will now see a compile error — see the API diff for the full list. (❤️ @ramezgerges in [#4195](https://github.com/mono/SkiaSharp/pull/4195).)
