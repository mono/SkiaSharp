# Version 4.151.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

SkiaSharp 4.151.0 upgrades the native engine to **Skia milestone m151** and ships four new compositing APIs — `SKPaint.GetFastBounds()`, `SKColorFilter.CreateOverdraw()`, `SKImageFilter.CreateCrop()`, and `SKImageFilter.CreateEmpty()`. `SKSurface.Canvas` now caches the wrapper object, reducing allocation pressure in tight render loops. HarfBuzz 14.2.1 ships alongside — see [the HarfBuzzSharp 14.2.1 page](harfbuzzsharp/14.2.1.md) for details. Community contributor ❤️ [@ramezgerges](https://github.com/ramezgerges) landed the raster blur fix, the `SKSurface` performance improvement, the WASM build fix, and the final pre-v4 API cleanup.

## Breaking Changes

- **Polyglot Notebooks / .NET Interactive support removed** — The long-deprecated `.NET Interactive` integration has been removed. Notebook environments should render via `SKBitmap.ToArray()` or equivalent instead. ([#4186](https://github.com/mono/SkiaSharp/pull/4186))
- **Pre-v4 obsolete APIs are now compile errors** — APIs deprecated in v3 that were carried as `[Obsolete(error: true)]` in v4 have been removed from the reference assembly. Upgrade call sites to v4 equivalents before updating. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))

## Engine

- **Skia m151** — Updates the native Skia engine to Chrome milestone m151. ([#4294](https://github.com/mono/SkiaSharp/pull/4294))
- **Skia m150 upstream bug fixes** — Backports stability and correctness fixes from the Chrome m150 Skia upstream. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))
- **Skia m150** — Initial engine update to Chrome milestone m150. ([#4146](https://github.com/mono/SkiaSharp/pull/4146))
- **HarfBuzz 14.2.1** — Updates the bundled HarfBuzz text-shaping library to version 14.2.1. ([#4167](https://github.com/mono/SkiaSharp/pull/4167))
- **WASM submodule: drop `SK_BUILD_FOR_WASM` redefinition** — Removes a conflicting preprocessor redefinition that caused build failures in WebAssembly targets. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3788](https://github.com/mono/SkiaSharp/pull/3788))

## API Surface

- **`SKPaint.GetFastBounds()`** — Returns the conservative bounding box of what the paint would draw, enabling efficient quick-reject culling before issuing draw calls. ([#4271](https://github.com/mono/SkiaSharp/pull/4271))
- **`SKColorFilter.CreateOverdraw()`** — Creates an overdraw-visualization color filter, useful for diagnosing over-rendering in complex scenes. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — Adds a crop image filter that restricts the output of a filter pipeline to a given rectangle. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — Adds a no-op image filter suitable as a placeholder or sentinel in filter chains. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

## GPU & Rendering

- **Fix small-sigma raster blur regression** — Defines `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS` to restore the fast-path blur code path for small sigma values that regressed during the m150 engine update. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4269](https://github.com/mono/SkiaSharp/pull/4269))

## Performance

- **`SKSurface` Canvas wrapper caching** — Caches the `SKCanvas` instance returned by `SKSurface.Canvas` across repeated accesses, eliminating repeated native lookups and wrapper allocations in tight render loops. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

## Platform

- **Tizen native dependencies in Docker** — Tizen native library compilation now runs inside a reproducible Docker container, improving build reliability and portability across CI environments. ([#4154](https://github.com/mono/SkiaSharp/pull/4154))

## Developer Tools

- **SkiaFiddle: embedded image and font selectors** — The SkiaFiddle online playground gains inline asset selectors for images and fonts, making it easier to test rendering in the browser without external uploads. ([#4284](https://github.com/mono/SkiaSharp/pull/4284))
