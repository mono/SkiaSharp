# Version 4.151.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.151.0-preview)

> **API changes** · [SkiaSharp API diff](4.151.0/index.md) · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Cadence-aligned bump to Chrome milestone m151. Adds new low-level API surface — `SKPaint.GetFastBounds()` for quick-reject culling, `SKColorFilter.CreateOverdraw()` for overdraw visualization, and `SKImageFilter.CreateCrop()` / `SKImageFilter.CreateEmpty()` — plus infra work: migrated the test suite to xUnit v3, removed deprecated .NET Interactive support, made docs generation reproducible and Linux-only, and picked up Skia m150 and m151 bug-fix sweeps. Ships alongside the HarfBuzz 14.2.1 bump — see [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md).

## Breaking Changes

None in this release.

## Engine

- **Update Skia to milestone m151** ([#4294](https://github.com/mono/SkiaSharp/pull/4294) by @mattleibow) — Cadence-aligned bump to Chrome m151 (skia: mono/skia#274).
- **Update Skia to milestone m150** ([#4146](https://github.com/mono/SkiaSharp/pull/4146) by @mattleibow) — Cadence-aligned bump to Chrome m150 (skia: mono/skia#253).
- **Merge upstream `chrome/m150` bug fixes** ([#4190](https://github.com/mono/SkiaSharp/pull/4190) by @mattleibow) — Picks up upstream `chrome/m150` bug fixes (skia: mono/skia#259).
- **Bump skia submodule to drop `SK_BUILD_FOR_WASM` redef** ([#3788](https://github.com/mono/SkiaSharp/pull/3788) by @ramezgerges) — Drops the redundant `SK_BUILD_FOR_WASM` definition after an upstream cleanup (skia: mono/skia#204).
- **Update HarfBuzz to 14.2.1** ([#4167](https://github.com/mono/SkiaSharp/pull/4167) by @mattleibow) — Refreshes the HarfBuzz native, shipped as [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) (skia: mono/skia#257).

## API Surface

- **`SKPaint.GetFastBounds()` for quick-reject culling** ([#4271](https://github.com/mono/SkiaSharp/pull/4271) by @mattleibow) — Adds a fast, conservative bounds query on `SKPaint`, useful for hoisting draw calls out of the rendering path when their expanded bounds are off-screen (skia: mono/skia#271).
- **`SKColorFilter.CreateOverdraw()`** ([#4227](https://github.com/mono/SkiaSharp/pull/4227) by @mattleibow) — Adds the overdraw visualization color filter for GPU diagnostic tooling (skia: mono/skia#262).
- **`SKImageFilter.CreateCrop()`** ([#4229](https://github.com/mono/SkiaSharp/pull/4229) by @mattleibow) — Exposes Skia's `SkImageFilters::Crop` factory (skia: mono/skia#261).
- **`SKImageFilter.CreateEmpty()`** ([#4228](https://github.com/mono/SkiaSharp/pull/4228) by @mattleibow) — Exposes the empty image-filter factory (skia: mono/skia#260).

## GPU & Rendering

- **Fix small-sigma raster blur regression** ([#4269](https://github.com/mono/SkiaSharp/pull/4269) by @ramezgerges) — Defines `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS` so small-sigma raster blurs use the fast path again.
- **Cache the `Canvas` wrapper across `SKSurface` calls** ([#4247](https://github.com/mono/SkiaSharp/pull/4247) by @ramezgerges) — Avoids reallocating the managed `SKCanvas` wrapper on every call.

## Platform

- **Promote remaining pre-v4 obsoletes to error and trim obsolete enums from the ref assembly** ([#4195](https://github.com/mono/SkiaSharp/pull/4195) by @ramezgerges) — The last of the pre-v4 obsolete surface is escalated to error and the obsolete enum members are removed from the reference assembly. Downstream code still compiling with warnings will now need to migrate off those APIs.
- **Build Tizen native dependencies in a Docker container** ([#4154](https://github.com/mono/SkiaSharp/pull/4154) by Matthew Leibowitz) — Moves the Tizen native build into a reproducible container image.
- **Remove deprecated .NET Interactive (Polyglot Notebooks) support** ([#4186](https://github.com/mono/SkiaSharp/pull/4186) by @mattleibow) — The .NET Interactive integration is no longer maintained upstream; the SkiaSharp-side support is dropped.
