# Version 4.151.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.151.0-preview)

> **API changes** · [SkiaSharp API diff](4.151.0/index.md) · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

**4.151.0** ships the Skia m151 milestone bump on top of everything that landed in 4.150.0. The rolled-up delta over 4.148.0 therefore includes the four new drawing APIs (`SKPaint.GetFastBounds()`, `SKColorFilter.CreateOverdraw()`, `SKImageFilter.CreateCrop()` and `CreateEmpty()`), the `SKSurface.Canvas` cache, the small-sigma blur fix, and the pre-v4 obsolete cleanup. New in 4.151.0 preview 1 itself: the m151 sync and a follow-up submodule bump that drops a `SK_BUILD_FOR_WASM` redefinition warning. HarfBuzz 14.2.1 ships alongside — see [its release notes](harfbuzzsharp/14.2.1.md) for the binding details.

## New APIs

- **`SKPaint.GetFastBounds()`** — Conservative paint bounds for quick-reject culling before the raster pipeline ([#4271](https://github.com/mono/SkiaSharp/pull/4271), [mono/skia#271](https://github.com/mono/skia/pull/271)).
- **`SKColorFilter.CreateOverdraw()`** — Standard overdraw-visualization color filter for perf tooling ([#4227](https://github.com/mono/SkiaSharp/pull/4227), [mono/skia#262](https://github.com/mono/skia/pull/262)).
- **`SKImageFilter.CreateCrop()`** — Crop an image filter graph to an explicit rectangle ([#4229](https://github.com/mono/SkiaSharp/pull/4229), [mono/skia#261](https://github.com/mono/skia/pull/261)).
- **`SKImageFilter.CreateEmpty()`** — Explicit no-op image filter for graph plumbing ([#4228](https://github.com/mono/SkiaSharp/pull/4228), [mono/skia#260](https://github.com/mono/skia/pull/260)).

## Performance

- **`SKSurface.Canvas` caches its wrapper** — Repeated access no longer allocates or crosses P/Invoke on every call (❤️ [@ramezgerges](https://github.com/ramezgerges) in [#4247](https://github.com/mono/SkiaSharp/pull/4247)).

## Fixes

- **Small-sigma raster blur** — Restore the fast small-sigma raster blur path by defining `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS`, fixing a regression that surfaced with m150 (❤️ [@ramezgerges](https://github.com/ramezgerges) in [#4269](https://github.com/mono/SkiaSharp/pull/4269)).
- **WASM redefinition warning** — Bump the skia submodule to drop a duplicate `SK_BUILD_FOR_WASM` definition (❤️ [@ramezgerges](https://github.com/ramezgerges) in [#3788](https://github.com/mono/SkiaSharp/pull/3788), [mono/skia#204](https://github.com/mono/skia/pull/204)).

## Breaking Changes

- **Pre-v4 obsoletes now error** — Remaining pre-v4 `[Obsolete]` members are promoted to error and obsolete enums are trimmed from the ref assembly. Rebuild against the current APIs; see the [SkiaSharp API diff](4.151.0/index.md) for the full list (❤️ [@ramezgerges](https://github.com/ramezgerges) in [#4195](https://github.com/mono/SkiaSharp/pull/4195)).
- **.NET Interactive support removed** — Drops the deprecated .NET Interactive (Polyglot Notebooks) surface ([#4186](https://github.com/mono/SkiaSharp/pull/4186)).

## Platform support

- **Skia m151** — Bump the Skia submodule to milestone 151 ([#4294](https://github.com/mono/SkiaSharp/pull/4294), [mono/skia#274](https://github.com/mono/skia/pull/274)).
- **Skia m150 rollup** — Includes the m150 milestone bump and chrome/m150 fixes from 4.150.0 ([#4146](https://github.com/mono/SkiaSharp/pull/4146), [#4190](https://github.com/mono/SkiaSharp/pull/4190)).
- **HarfBuzz 14.2.1** — Ships alongside; see [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) ([#4167](https://github.com/mono/SkiaSharp/pull/4167), [mono/skia#257](https://github.com/mono/skia/pull/257)).

## Other changes

Plus various CI, documentation, and internal tooling improvements — xUnit v3 migration, Tizen native builds in a Docker container, doc generation reproducibility, and release-notes/API-diff pipeline consolidation.
