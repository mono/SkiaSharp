# Version 1.56.1

> **.NET Standard, .NET Core, and major API expansion** ┬╖ Released February 8, 2017 ┬╖ [NuGet](https://www.nuget.org/packages/SkiaSharp/1.56.1) ┬╖ [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v1.56.1)

## Highlights

A landmark release that adds .NET Standard 1.3 and .NET Core support, bringing SkiaSharp to modern .NET platforms. Four new types arrive тАФ `SKFontManager`, `SKSvgCanvas`, `SKMask`, and `SK3dView` тАФ along with extensive additions to `SKPath`, `SKBitmap`, and `SKCodec`. Unicode, UTF-32, and emoji support is significantly improved. The Xamarin.Forms view renderers are now public and extensible, and the SVG viewbox handling is fixed.

## Breaking Changes

### Renames

- **`SKStrokeJoin.Mitter` тЖТ `SKStrokeJoin.Miter`** тАФ Corrected the typo in the enum value name. ([#227](https://github.com/mono/SkiaSharp/issues/227))

## New Features

### API Surface

- **`SKFontManager`** тАФ Find typefaces for a specific character, enabling better font fallback. ([#232](https://github.com/mono/SkiaSharp/issues/232))
- **`SKSvgCanvas`** _(preview)_ тАФ Create SVG files programmatically via a canvas API. ([#107](https://github.com/mono/SkiaSharp/issues/107))
- **`SKMask`** тАФ New type for working with mask data, plus `SKBitmap` can now be created from an `SKMask`. ([#220](https://github.com/mono/SkiaSharp/issues/220))
- **`SK3dView`** тАФ Create 3D `SKMatrix` transformations. ([#214](https://github.com/mono/SkiaSharp/issues/214))
- **`SKPath` expansions** тАФ Many new members including `AddPoly` and `SegmentMasks`.
- **`SKBitmap` expansions** тАФ Many new members including `ExtractAlpha` and `ExtractSubset`.
- **Unicode and emoji support** тАФ Improved support for Unicode, UTF-32, and emoji rendering. ([#232](https://github.com/mono/SkiaSharp/issues/232))
- **`StringUtilities`** тАФ New string helper methods for text processing.
- **High quality `SKMaskFilter` blurring** тАФ Support for high quality blurring effects.
- **`SKCodec` scanline decoding** тАФ Codecs now support scanline-by-scanline decoding.
- **PDF metadata and annotations** тАФ `SKDocument` now supports PDF metadata and link annotations.
- **`SKImageInfo.PlatformColorType` fix** тАФ Now correctly obtains the platform color type.

### Platform

- **.NET Standard 1.3 support** тАФ SkiaSharp now targets .NET Standard 1.3 project types. ([#111](https://github.com/mono/SkiaSharp/issues/111))
- **.NET Core support** тАФ Native libraries for Windows (win7, win10) and macOS. ([#111](https://github.com/mono/SkiaSharp/issues/111))
- **Reduced native library sizes** тАФ Native binaries reduced to ~5 MB per platform architecture. ([#174](https://github.com/mono/SkiaSharp/issues/174))
- **Class library native copy fix** тАФ .NET 4.5 class library builds (including tests) now correctly copy the native assembly to output. ([#44](https://github.com/mono/SkiaSharp/issues/44), [#190](https://github.com/mono/SkiaSharp/issues/190))
- **Public Xamarin.Forms renderers** тАФ `SKCanvasViewRenderer` and `SKGLViewRenderer` are now public and can be extended by apps and libraries. ([#218](https://github.com/mono/SkiaSharp/issues/218))
- **SVG viewbox fix** тАФ Fixed handling of the SVG viewbox in SkiaSharp.Svg. ([#230](https://github.com/mono/SkiaSharp/issues/230))

## Bug Fixes

- Fixed `SKImageInfo.PlatformColorType` to correctly return the platform-specific color type.
- Fixed SVG viewbox handling. ([#230](https://github.com/mono/SkiaSharp/issues/230))
