# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Three new image-processing APIs have landed since the last preview: `SKColorFilter.CreateOverdraw()` for overdraw visualization, `SKImageFilter.CreateCrop()`, and `SKImageFilter.CreateEmpty()`. Upstream Skia m150 bug fixes have been merged into the engine, and [@ramezgerges](https://github.com/ramezgerges) contributed a performance improvement that caches the `SKCanvas` wrapper across `SKSurface` calls, reducing allocations on repeated access. HarfBuzzSharp 14.2.1 ships alongside this release — see its [release page](harfbuzzsharp/14.2.1.md) for details.

## New Features

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — New color filter that visualizes overdraw by rendering areas in different colors based on how many times they have been drawn, useful for GPU performance diagnostics. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — New image filter that crops the filter output to a specified rectangle, enabling precise control over which region of a filter chain contributes to the final result. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New image filter that produces no output, useful as a null-filter placeholder or to suppress rendering in a filter chain. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Engine

- **Upstream Skia m150 bug fixes** — Merged the latest bug fixes from the upstream Chrome m150 branch of the Skia engine. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))

## Performance

- **`SKSurface` canvas wrapper caching** — The `SKCanvas` wrapper is now cached across `SKSurface` calls, reducing object allocations when the canvas is accessed repeatedly. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))
