# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Three new filter APIs round out the 4.150.0 API surface since preview.2: `SKImageFilter.CreateCrop()`, `SKImageFilter.CreateEmpty()`, and `SKColorFilter.CreateOverdraw()` for GPU overdraw visualization. Upstream chrome/m150 Skia bug fixes are merged in alongside build system improvements. ❤️ [@ramezgerges](https://github.com/ramezgerges) contributes a notable performance win that caches the `SKCanvas` wrapper inside `SKSurface`, eliminating a redundant wrapper allocation on every property access.

## New Features

### Engine

- **Upstream Skia m150 bug fixes** — Cherry-picked bug fixes from the upstream chrome/m150 Skia branch. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — New color filter that visualizes GPU overdraw by mapping draw-call depth to distinct colors, useful for rendering performance profiling. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — New image filter that clips filter output to a specified rectangle, enabling tighter filter pipeline composition without manual clipping layers. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New image filter that produces transparent-black output regardless of source, useful as a null or no-op node in filter graphs. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Performance

- **`SKSurface` canvas wrapper cache** — `SKSurface.Canvas` now caches the `SKCanvas` wrapper across calls, eliminating repeated wrapper allocations for long-lived surfaces. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))
