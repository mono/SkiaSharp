# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Two new `SKImageFilter` factory methods — `CreateCrop()` and `CreateEmpty()` — have landed on the upcoming 4.150.0 branch alongside a Skia engine sync that incorporates the latest Chrome m150 upstream bug fixes. HarfBuzzSharp 14.2.1 ships alongside this release; see the [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) page for details.

## New Features

### Engine

- **Skia m150 upstream bug fixes** — Merges Chrome/m150 bug fixes from upstream Skia, bringing the latest engine stability and correctness improvements. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))

### API Surface

- **`SKImageFilter.CreateCrop()`** — New factory method that creates a crop image filter, restricting a filter graph's output to a specified rectangular region. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New factory method that creates a no-op empty image filter, useful as a neutral placeholder or default in filter pipelines. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))
