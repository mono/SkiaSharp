# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md)

## Highlights

Version 4.150.0 continues the v4 API finalization push with a key community contribution from [@ramezgerges](https://github.com/ramezgerges), who promoted all remaining pre-v4 obsolete members to errors and trimmed stale enum entries from the reference assembly — making the SkiaSharp 4.x API contract explicit and final. The release also removes the long-deprecated .NET Interactive (Polyglot Notebooks) integration, moves Tizen native dependency builds into Docker containers for improved reproducibility, and upgrades the test suite to xUnit v3. HarfBuzzSharp 14.2.0 ships alongside — see [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md) for details.

## Breaking Changes

- **Pre-v4 obsoletes promoted to errors** — All members previously marked `[Obsolete(error: false)]` as part of the v3→v4 migration are now `[Obsolete(error: true)]`. Code referencing these APIs will no longer compile; migrate to the v4 replacement APIs before upgrading. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))
- **Removed .NET Interactive / Polyglot Notebooks support** — The deprecated SkiaSharp integration with .NET Interactive notebooks has been removed. ([#4186](https://github.com/mono/SkiaSharp/pull/4186))

## New Features

### API Surface

- **Obsolete enum member cleanup** — Stale obsolete enum entries trimmed from the reference assembly, reducing surface ambiguity in the v4 API contract. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))

### Platform

- **Tizen Docker builds** — Tizen native dependencies are now built inside a Docker container, improving reproducibility and isolation across CI and developer environments. ([#4154](https://github.com/mono/SkiaSharp/pull/4154))
