# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

This in-development snapshot (beyond Preview 2) adds three new filter APIs —
`SKColorFilter.CreateOverdraw()`, `SKImageFilter.CreateCrop()`, and
`SKImageFilter.CreateEmpty()` — along with a `SKSurface.Canvas` performance
improvement from community contributor
[@ramezgerges](https://github.com/ramezgerges). The Skia engine has also
received upstream m150 bug fixes. HarfBuzzSharp 14.2.1 ships alongside this
release; see its [dedicated page](harfbuzzsharp/14.2.1.md) for binding details.

## New Features

### Engine

- **Skia m150 upstream bug fixes** — Merges upstream Chrome/m150 bug fixes
  into the native Skia engine. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — New color filter for overdraw
  visualization, useful for diagnosing scenes where pixels are painted multiple
  times. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — New image filter that clips the output to
  a specified rectangle, composable with other filter effects.
  ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New image filter that produces a
  transparent-black result, useful as a sentinel or no-op node in filter graphs.
  ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Performance

- **`SKSurface.Canvas` wrapper caching** — The `Canvas` property now returns
  the same C# wrapper instance across calls instead of allocating a new one each
  time, reducing GC pressure in tight rendering loops. ❤️
  [@ramezgerges](https://github.com/ramezgerges)
  ([#4247](https://github.com/mono/SkiaSharp/pull/4247))
