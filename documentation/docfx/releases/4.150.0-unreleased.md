# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Three new filter APIs land in this development window: `SKColorFilter.CreateOverdraw()` for overdraw visualization, `SKImageFilter.CreateCrop()` for bounded filter output, and `SKImageFilter.CreateEmpty()` as a transparent-output sentinel. Upstream Skia m150 bug fixes are merged in as well. HarfBuzz 14.2.1 ships alongside this release — see the [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) page for binding details.

## Breaking Changes

*None in this development window.*

## New Features

### Engine

- **Skia m150 upstream bug fixes** — Merges chrome/m150 stability and correctness fixes from the upstream Skia repository. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — New factory method for overdraw visualization. Maps each overdraw depth level to a user-defined color, making it easy to identify overdraw hotspots in rendering pipelines. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — Crops the output of an image filter to a specified rectangle, allowing composited effects to be precisely bounded without a separate clip step. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — Returns a filter that always outputs fully transparent pixels. Useful as a no-op sentinel in conditional or branching filter pipelines. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))
