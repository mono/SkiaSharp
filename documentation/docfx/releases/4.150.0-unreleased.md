# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Three new filter APIs — `SKColorFilter.CreateOverdraw()`, `SKImageFilter.CreateCrop()`, and `SKImageFilter.CreateEmpty()` — expand SkiaSharp's compositing toolkit. ❤️ [@ramezgerges](https://github.com/ramezgerges) contributed a performance improvement that caches the `SKCanvas` wrapper on `SKSurface`, reducing allocations in tight rendering loops. This branch also merges upstream Skia chrome/m150 bug fixes. HarfBuzz 14.2.1 ships alongside this release — see its [dedicated page](harfbuzzsharp/14.2.1.md) for details.

## New Features

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — New color filter for overdraw visualization. Maps pixel overdraw counts to configurable colors, making it easy to identify over-drawn regions during performance debugging. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — New image filter that clips the output of another filter to a specified rectangle, without any other transformation. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New image filter that produces a fully transparent (empty) output, useful as a terminal or placeholder node in filter graphs. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Performance

- **`SKSurface` canvas wrapper caching** — `SKSurface` now caches its `SKCanvas` wrapper across calls rather than recreating it on every access, cutting unnecessary allocations in hot rendering paths. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

### Engine

- **Upstream Skia m150 bug fixes** — Merges chrome/m150 upstream bug fixes from the Skia graphics engine into SkiaSharp's fork. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))
