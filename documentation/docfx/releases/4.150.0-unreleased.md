# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Version 4.150.0 expands the API surface with new image filter constructors (`CreateEmpty`, `CreateCrop`), a color filter for overdraw visualization, and fast-bounds culling on `SKPaint`. Two community contributions sharpen rendering: [@ramezgerges](https://github.com/ramezgerges) fixed a small-sigma raster blur regression and improved `SKSurface` canvas access performance. HarfBuzzSharp 14.2.1 ships alongside this release — see its [linked page](harfbuzzsharp/14.2.1.md) for binding details.

## Breaking Changes

*None in this release.*

## New Features

### Engine

- **Skia m150 upstream bug fixes** — Merges upstream Chrome m150 Skia bug fixes. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))

### API Surface

- **`SKImageFilter.CreateEmpty()`** — New factory method that creates a pass-through (no-op) image filter. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))
- **`SKImageFilter.CreateCrop()`** — New factory method that crops the input to a specified rectangle before passing it to downstream filters. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKColorFilter.CreateOverdraw()`** — New factory method for an overdraw visualization color filter, useful for identifying rendering hot spots. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKPaint.GetFastBounds()`** — New method that returns the quick-reject culling bounds of a paint, enabling efficient skip-draw decisions without rendering. ([#4271](https://github.com/mono/SkiaSharp/pull/4271))

## Performance

- **`SKSurface` canvas wrapper caching** — The `Canvas` property on `SKSurface` now caches the managed wrapper across calls, eliminating repeated allocations in tight render loops. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

## Bug Fixes

- **Fixed small-sigma raster blur regression** — Defines `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS` to restore the fast code path for small-sigma Gaussian blurs that was inadvertently regressed. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4269](https://github.com/mono/SkiaSharp/pull/4269))
