# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

This in-development snapshot adds three new rendering filter APIs, merges upstream Chrome m150 Skia bug fixes, and includes a community-contributed performance improvement to `SKSurface`. `SKImageFilter.CreateCrop()`, `SKImageFilter.CreateEmpty()`, and `SKColorFilter.CreateOverdraw()` are now available. [@ramezgerges](https://github.com/ramezgerges) contributed caching of the `SKSurface` Canvas wrapper to eliminate repeated allocation on every access. Plus several CI and workflow improvements.

## Breaking Changes

*None in this release.*

## New Features

### Engine

- **Upstream m150 bug fixes** — Merges Chrome/m150 Skia bug fixes into the native libraries. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — Adds an overdraw visualization color filter that maps per-pixel draw counts to configurable colors, useful for debugging rendering coverage. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — Adds a crop image filter that restricts output to a specified rectangle. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — Adds a no-op empty image filter that discards all pixels, useful as a terminal sink in filter graphs. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Performance

- **`SKSurface` Canvas caching** — Caches the `SKCanvas` wrapper instance across calls to `SKSurface.Canvas`, avoiding repeated object allocation on every access. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))
