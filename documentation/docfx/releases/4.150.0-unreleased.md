# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

This in-development update adds three new image filter APIs — `SKColorFilter.CreateOverdraw()`, `SKImageFilter.CreateCrop()`, and `SKImageFilter.CreateEmpty()` — backed by upstream Skia engine bug fixes merged from Chrome/m150.

## New Features

### Engine

- **Merge upstream Chrome/m150 bug fixes** — Incorporates the latest stability and correctness fixes from the upstream Skia library at Chrome milestone 150. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — New color filter for overdraw visualization, enabling developers to identify regions where pixels are drawn multiple times in a frame. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — New image filter that crops the output of any filter pipeline to a given rectangle, simplifying compositing workflows. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New no-op image filter that produces fully transparent output, useful as a neutral element or default in filter chains. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))
