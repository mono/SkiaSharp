# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

This in-development snapshot adds three new image filter and color filter APIs, merges upstream Skia m150 bug fixes, and brings a performance improvement from the community. New `SKImageFilter.CreateCrop()`, `SKImageFilter.CreateEmpty()`, and `SKColorFilter.CreateOverdraw()` expand the compositing and diagnostic toolkit. A canvas wrapper caching optimization from [@ramezgerges](https://github.com/ramezgerges) reduces allocations in render loops. HarfBuzzSharp 14.2.1 ships alongside this release — see the [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) page for details.

## New Features

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — New color filter for overdraw visualization, useful for diagnosing rendering overdraw in complex scenes. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — Crops a filter's output to a specified rectangle; useful in compositing pipelines. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — A no-op image filter that produces fully transparent output regardless of its input; serves as a placeholder in filter graphs. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Performance

- **`SKSurface` canvas wrapper caching** — The `Canvas` property now caches its wrapper object across calls instead of re-creating it each time, reducing GC pressure in tight render loops. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

### Engine

- **Upstream Skia m150 bug fixes** — Merges the latest Chrome m150 bug-fix commits from the upstream Skia repository. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))
