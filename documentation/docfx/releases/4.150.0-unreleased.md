# Version 4.150.0

> **Upcoming release** · In development · Not yet available on NuGet

> **API changes** · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

Three new image-pipeline APIs arrive for 4.150.0: `SKColorFilter.CreateOverdraw()` for overdraw visualization, `SKImageFilter.CreateCrop()` for cropping filter graphs, and `SKImageFilter.CreateEmpty()` for a transparent no-op placeholder. The upstream Skia m150 engine also received cherry-picked bug fixes from Chrome. HarfBuzzSharp 14.2.1 ships alongside this release — see its [release page](harfbuzzsharp/14.2.1.md) for details.

## New Features

### API Surface

- **`SKColorFilter.CreateOverdraw()`** — New color filter that maps successive draw counts to distinct colors, making overdraw patterns immediately visible. Useful for diagnosing areas where too many layers are being composited. ([#4227](https://github.com/mono/SkiaSharp/pull/4227))
- **`SKImageFilter.CreateCrop()`** — New image filter that clips the output of a filter graph to a specified rectangle before passing it downstream. ([#4229](https://github.com/mono/SkiaSharp/pull/4229))
- **`SKImageFilter.CreateEmpty()`** — New image filter that produces a transparent black output regardless of input — a no-op placeholder useful when constructing or debugging filter graphs. ([#4228](https://github.com/mono/SkiaSharp/pull/4228))

### Engine

- **Skia m150 upstream bug fixes** — Merges cherry-picked fixes from the Chrome m150 branch, bringing the latest stability improvements from the upstream Skia engine. ([#4190](https://github.com/mono/SkiaSharp/pull/4190))
