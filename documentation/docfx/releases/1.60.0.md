## Highlights

Servicing release with no user-facing PR work. Skia's own type cleanup rippled through the binding refresh — several helper enums and one-off classes are gone, and the pixel-config enum was re-numbered again. Ships alongside [HarfBuzzSharp 1.4.6](harfbuzzsharp/1.4.6.md) — see that page for shaping-side details.

## Breaking Changes

- **Removed types** — `GRContextOptions.GpuPathRenderers`, `SKEncodedInfo`, `SKEncodedInfoAlpha`, `SKEncodedInfoColor`, `SKPMColor`, `SKColorTable`, and the obsolete `GRContextOptions` batching helpers were removed. Callers that persisted these enums as integers need to switch to the current equivalents.
- **SKColorType** — the `Index8` value was removed.
- **SKPathEffect** — `CreateArcTo(float radius)` was removed. Use `CreateCorner` or path construction directly.
- **SKImageFilter** — `CreateMerge(SKImageFilter[] filters, SKXferMode[] modes)` overload was removed; use `CreateMerge(SKImageFilter[] filters)`.
- **SKLatticeFlags** — the enum was removed. Use the current `SKLattice` struct fields instead.
- **GRContext.Flush(GRContextFlushBits)** — the previously obsolete overload was removed. Call `Flush()` (no args).
- **GRPixelConfig** — several values were removed and remaining members re-numbered.
