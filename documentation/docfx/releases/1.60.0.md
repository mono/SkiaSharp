# Version 1.60.0

## Highlights

Historical release for the 1.x line. This tag predates SkiaSharp's modern PR-tracked release cadence, so the pull-request history for the branch is empty; the signature changes captured from the previous tag are the interesting part and are surfaced through the API diff linked above. HarfBuzzSharp 1.4.6 ships alongside this release — see [HarfBuzzSharp 1.4.6](harfbuzzsharp/1.4.6.md) for its binding details.

## Breaking Changes

- **`GRContextOptions` op-combine tuning removed** — `MaxOpCombineLookahead` and `MaxOpCombineLookback` were dropped; the Skia backend now manages op combining internally.
- **`GRContextOptionsGpuPathRenderers` cleanup** — `DistanceField` and `Pls` renderer flags were removed, and the remaining values (including `All`, `Default`, and `Tessellating`) were renumbered. If you were composing a custom `GpuPathRenderers` bitmask, rebuild it against the new values.
- **Obsolete overloads removed** — `GRContext.Flush(GRContextFlushBits)` and `GRGlInterface.CreateNativeInterface()`, both previously marked `[Obsolete]`, are gone. Use the parameterless `GRContext.Flush()` and `GRGlInterface.CreateNativeGlInterface()`/`CreateDefaultInterface()` respectively.
- **`GRSurfaceOrigin` renumbered** — `TopLeft` moved from `0` to `1` and `BottomLeft` from `1` to `2`. Code that serialised or persisted the raw integer values needs remapping; name-based use is unaffected.
