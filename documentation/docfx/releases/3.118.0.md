# Version 3.118.0

> **Preview only** · Superseded by [3.119.0](3.119.0.md) · Never released as stable — these changes rolled up into 3.119.0 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.118.0-preview)

> **API changes** · [SkiaSharp API diff](3.118.0/index.md) · [HarfBuzzSharp 8.3.1](harfbuzzsharp/8.3.1.md)

## Highlights

3.118.0 was the first preview after 3.116.1, moving SkiaSharp to Skia milestone 118 and picking up the associated dependency refreshes (freetype, libexpat, libjpeg-turbo, harfbuzz). It also broadens platform coverage — Metal is exposed uniformly for `GRBackendRenderTarget` across every Apple platform, and Blazor/WASM get several stability fixes for .NET 9. **This preview line was never released as stable; everything below rolled up into [3.119.0](3.119.0.md).**

## Breaking Changes

None in this preview — see the [SkiaSharp API diff](3.118.0/index.md) for the full signature-level view.

## Engine

- **Skia milestone 118** — moved SkiaSharp onto the upstream m118 branch. ([#3048](https://github.com/mono/SkiaSharp/pull/3048), [skia#139](https://github.com/mono/skia/pull/139))
- **freetype2 2.13.3+** — refresh of the font engine tracked by Skia. ([#3061](https://github.com/mono/SkiaSharp/pull/3061), [skia#143](https://github.com/mono/skia/pull/143))
- **libexpat 2.6.4** — pulls in upstream security fixes. ([#3056](https://github.com/mono/SkiaSharp/pull/3056), [skia#142](https://github.com/mono/skia/pull/142))
- **libjpeg-turbo 2.1.5.1+** — advances to the latest commit on the 2.1.5.1 branch. ([#3060](https://github.com/mono/SkiaSharp/pull/3060), [skia#141](https://github.com/mono/skia/pull/141))

## GPU & Rendering

- **Metal `GRBackendRenderTarget` API on every Apple platform** — the Metal-backed render-target APIs, previously partial, are now uniformly available on iOS, tvOS, macOS and Mac Catalyst. ❤️ [@spouliot](https://github.com/spouliot) ([#3070](https://github.com/mono/SkiaSharp/pull/3070))

## Platform

- **.NET 9 ASP.NET Blazor fixes** — resolves a set of issues that broke Blazor apps on .NET 9. ([#3064](https://github.com/mono/SkiaSharp/pull/3064))
- **WASM/Blazor workaround moved into the WASM package** — the Blazor-specific initialization tweak now lives with the WASM native assets, so consumers no longer wire it up themselves. ([#3082](https://github.com/mono/SkiaSharp/pull/3082), [#3088](https://github.com/mono/SkiaSharp/pull/3088))
- **Explicit `WindowsSdkPackageVersion` pin** — pins the exact Windows SDK NuGet used to build the Windows targets so restores are reproducible. ([#3085](https://github.com/mono/SkiaSharp/pull/3085))
- **Correct import path for apps with a base path** — the module `<script>` in the Blazor host now resolves relative to the base path, fixing sub-app hosting. ([#3092](https://github.com/mono/SkiaSharp/pull/3092))
