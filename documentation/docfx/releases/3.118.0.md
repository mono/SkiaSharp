# Version 3.118.0

> **Preview only** · Superseded by [3.119.0](3.119.0.md) · Never released as stable — these changes rolled up into 3.119.0 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.118.0-preview)

> **API changes** · [SkiaSharp API diff](3.118.0/index.md) · [HarfBuzzSharp 8.3.1](harfbuzzsharp/8.3.1.md)

## Highlights

**3.118.0** brought the Skia m117/m118 milestone bumps, added the Metal API for `GRBackendRenderTarget` on all platforms, and rounded out .NET 9 support for Blazor/WebAssembly hosts. It was never released as stable — every change rolled up into [3.119.0](3.119.0.md). HarfBuzz 8.3.1 ships alongside — see [its release notes](harfbuzzsharp/8.3.1.md) for the binding details.

## New APIs

- **Metal `GRBackendRenderTarget`** — Expose the Metal-backed constructor for `GRBackendRenderTarget` on every platform where SkiaSharp is built, unifying the Metal integration surface (❤️ [@spouliot](https://github.com/spouliot) in [#3070](https://github.com/mono/SkiaSharp/pull/3070)).

## Fixes

- **.NET 9 Blazor** — Fix ASP.NET Blazor host issues that surfaced on .NET 9, including the WASM initialization path and the Blazor base-path import ([#3064](https://github.com/mono/SkiaSharp/pull/3064), [#3092](https://github.com/mono/SkiaSharp/pull/3092)).
- **Blazor/WASM decoupling** — Move the Blazor and WASM workarounds into SkiaSharp so consumers no longer have to copy them into their own projects ([#3088](https://github.com/mono/SkiaSharp/pull/3088), [#3082](https://github.com/mono/SkiaSharp/pull/3082)).
- **Windows SDK pinning** — Use `<WindowsSdkPackageVersion>` to pin the Windows SDK so mismatched host SDKs no longer break the build ([#3085](https://github.com/mono/SkiaSharp/pull/3085)).

## Platform support

- **Skia m117 + m118** — Sync the Skia submodule through milestones 117 and 118 ([#3047](https://github.com/mono/SkiaSharp/pull/3047), [#3048](https://github.com/mono/SkiaSharp/pull/3048), [mono/skia#138](https://github.com/mono/skia/pull/138), [mono/skia#139](https://github.com/mono/skia/pull/139)).
- **Native dependency bumps** — Update the bundled freetype2 to 2.13.3+ ([#3061](https://github.com/mono/SkiaSharp/pull/3061), [mono/skia#143](https://github.com/mono/skia/pull/143)), libexpat to 2.6.4 ([#3056](https://github.com/mono/SkiaSharp/pull/3056), [mono/skia#142](https://github.com/mono/skia/pull/142)), and libjpeg-turbo to 2.1.5.1+commits ([#3060](https://github.com/mono/SkiaSharp/pull/3060), [mono/skia#141](https://github.com/mono/skia/pull/141)).
- **HarfBuzz 8.3.1** — Bump the HarfBuzz native to 8.3.1; see [HarfBuzzSharp 8.3.1](harfbuzzsharp/8.3.1.md) ([#3052](https://github.com/mono/SkiaSharp/pull/3052), [mono/skia#140](https://github.com/mono/skia/pull/140)).

## Other changes

Plus various CI, documentation, and internal tooling improvements — 16 KB page-size alignment for Android NDK r27c and the correct NuGet `PackagePath` for runtime assets.
