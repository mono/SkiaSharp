## Highlights

**3.116.1** is a small servicing update to the 3.116.0 stable line. Two automated back-ports arrived on the branch: the Android NDK is bumped to r27c with 16 KB page-size alignment, and the NuGet `PackagePath` attribute is corrected so runtime assets land in the expected folders. There are no HarfBuzz binding API changes — HarfBuzz 8.3.0.1 is a packaging refresh; see [its release notes](harfbuzzsharp/8.3.0.1.md).

## Platform support

- **Android NDK r27c + 16 KB alignment** — Update the Android native toolchain to NDK r27c and align the shared libraries to 16 KB pages so they load on modern Android devices ([#3098](https://github.com/mono/SkiaSharp/pull/3098)).

## Other changes

Plus various CI, documentation, and internal tooling improvements — the NuGet `PackagePath` attribute is corrected so runtime assets are placed in the expected folders ([#3102](https://github.com/mono/SkiaSharp/pull/3102)).
