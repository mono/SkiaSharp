> **API changes** · [SkiaSharp API diff](3.116.1/index.md) · [HarfBuzzSharp 8.3.0.1](harfbuzzsharp/8.3.0.1.md)

## Highlights

A tiny follow-up to 3.116.0 shipped three days after the .0 release. It fixes a packaging attribute that broke a specific NuGet consumption path and picks up an Android NDK bump to r27c with 16 KB page alignment so SkiaSharp keeps loading cleanly on newer Android devices. No public API changes — see the [SkiaSharp API diff](3.116.1/index.md).

## Breaking Changes

None in this release. The [SkiaSharp API diff](3.116.1/index.md) reports no signature changes for this servicing hotfix.

## Platform

- **Android NDK r27c with 16 KB page alignment** — rebuilt the native library against NDK r27c so it aligns to the 16 KB page size Android 15 devices are moving to. ([#3098](https://github.com/mono/SkiaSharp/pull/3098))

## Lifecycle & Internals

- Plus various CI, documentation, and internal tooling improvements — including the NuGet `PackagePath` attribute fix that motivated the hotfix ([#3102](https://github.com/mono/SkiaSharp/pull/3102)).
