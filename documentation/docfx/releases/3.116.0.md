# Version 3.116.0

> **First stable v3 release** · Released December 2, 2024 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.116.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.116.0)

> **Supersedes [3.0.0](3.0.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](3.116.0/index.md) · [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md)

## Highlights

The **first stable `3.x` release** — the culmination of six previews and
the biggest cross-cutting version bump since 2.80. `3.116.0` adopts Skia's
`chrome/mNNN` milestone versioning (so `3.116.x` ships against Skia m116),
removes every `[Obsolete]` member accumulated across 2.x, moves interop to
function pointers + `LibraryImport` on .NET 8/9, refreshes Skia to m116
along with HarfBuzz 8.3.0, webp 1.3.2, and libjpeg-turbo, and adds
first-class WinUI accelerated views, `SKGLElement` on WPF, Metal on Mac
Catalyst, Blazor DPI, multi-thread + SIMD on WebAssembly, `SKBlender`,
extra `SKPicture` APIs, Span overloads across the geometry surface, and
Skottie's `Animation.Builder`. **Thirteen** community contributors landed
work. Ships with [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md).

## Breaking Changes

`3.116.0` is the first stable release of the `3.x` line and rolls up the
full 3.0.0 preview series. The assembly version jumps from `2.88.0.0` to
`3.116.0.0` and the API surface is significantly reshaped. See the
[3.116.0 API diff](3.116.0/index.md#breaking-changes) for the complete,
per-assembly signature list. The biggest themes:

- **Every `[Obsolete]` member from 2.x is removed** — anything you were
  ignoring an obsolete warning on is now gone. Follow the migration hint
  in the original obsolete attribute (the diffs have them per-member).
  ([#2539](https://github.com/mono/SkiaSharp/pull/2539))
- **`GRContext.Create(GRBackend, …)` legacy factories removed** — use the
  explicit `CreateGl(...)` / `CreateVulkan(...)` / `CreateMetal(...)`
  factories. `GetResourceCacheLimits` / `SetResourceCacheLimits` are
  replaced by the singular `GetResourceCacheLimit` / `SetResourceCacheLimit(long)`.
- **`GRGlInterface.AssembleXxxInterface(...)` variants removed** — use the
  new `Create*(GRGlGetProcedureAddressDelegate)` factories (`CreateAngle`,
  `CreateOpenGl`, `CreateGles`, `Create`).
- **`SKPaint.FilterQuality` removed from the sampling contract** — pass an
  `SKSamplingOptions` explicitly to draw calls. Compatibility overloads that
  accept the old `SKFilterQuality` enum are reintroduced where they help
  migration. ❤️ [@Youssef1313](https://github.com/Youssef1313)
  ([#2963](https://github.com/mono/SkiaSharp/pull/2963))
- **Text APIs consolidated on `SKFont`** — measure/break/glyph paths that
  were on `SKPaint` in 2.x are consolidated into `SKFont`, and text-based
  `SKTextBlob` builders come back.
  ([#2775](https://github.com/mono/SkiaSharp/pull/2775),
  [#2545](https://github.com/mono/SkiaSharp/pull/2545))
- **`GRMtlTextureInfo` getters are read-only** — the Metal texture-info
  struct's properties are now `{ get; }`-only.
  ([#2833](https://github.com/mono/SkiaSharp/pull/2833))
- **Views assemblies rebuilt** — the platform Views assemblies
