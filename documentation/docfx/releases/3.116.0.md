# Version 3.116.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/3.116.0)

> **Supersedes [3.0.0](3.0.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](3.116.0/index.md) · [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md)

The first stable release of the `3.x` line. `3.116.0` **supersedes** the never-released [`3.0.0`](3.0.0.md) preview series and rolls up all of its work — Skia through m137, HarfBuzz `8.3.0`, .NET 8/9 support, WinUI accelerated views, `SKGLView` for MAUI, Mac Catalyst Metal, `SKBlender`, `SKRuntimeEffect` cleanups — plus a small "Preview 6" that adds multi-thread + SIMD WebAssembly and a Blazor stability wave for .NET 9. The version scheme now aligns with the underlying Skia milestone. HarfBuzz `8.3.0` ships alongside — see [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md).

## Highlights

- Rolls up the entire [`3.0.0`](3.0.0.md) preview series: modernise the repository ([#2505](https://github.com/mono/SkiaSharp/pull/2505)); remove all `[Obsolete]` members and types ([#2539](https://github.com/mono/SkiaSharp/pull/2539)); Skia milestones from m112 → m137 ([#2582](https://github.com/mono/SkiaSharp/pull/2582), [#2829](https://github.com/mono/SkiaSharp/pull/2829), [#2885](https://github.com/mono/SkiaSharp/pull/2885), [#3026](https://github.com/mono/SkiaSharp/pull/3026)); HarfBuzz to `8.3.0` ([#2624](https://github.com/mono/SkiaSharp/pull/2624)).
- Add support for multi-thread and/or SIMD WebAssembly ([#2620](https://github.com/mono/SkiaSharp/pull/2620)) — the new "Preview 6" landmark that finally opens threaded rendering in the browser.
- Add `SKMatrix.MapPoints`/`MapVectors` and `SKPath.AddPoly` overloads with `Span<SKPoint>` ([#3030](https://github.com/mono/SkiaSharp/pull/3030)) — filling out the `Span<>` surface.
- Blazor stabilisation for .NET 9: fix `.NET 9` ASP.NET Blazor issues ([#3081](https://github.com/mono/SkiaSharp/pull/3081)), move the Blazor workaround into WASM ([#3087](https://github.com/mono/SkiaSharp/pull/3087)), move the WASM fix to SkiaSharp ([#3089](https://github.com/mono/SkiaSharp/pull/3089)), fix the import for apps with a base path ([#3093](https://github.com/mono/SkiaSharp/pull/3093)) and use `<WindowsSdkPackageVersion>` to set the SDK version ([#3086](https://github.com/mono/SkiaSharp/pull/3086)).
- Rolled up from `3.0.0`: WinUI accelerated views ([#2733](https://github.com/mono/SkiaSharp/pull/2733)), `SKGLView` for MAUI ([#2598](https://github.com/mono/SkiaSharp/pull/2598)), `SKGLElement` for WPF ([#2317](https://github.com/mono/SkiaSharp/pull/2317)), Metal on Mac Catalyst ([#2747](https://github.com/mono/SkiaSharp/pull/2747)), Metal APIs on a common .NET TFM ([#2788](https://github.com/mono/SkiaSharp/pull/2788)), `SKBlender` ([#2830](https://github.com/mono/SkiaSharp/pull/2830)), `SKImage.ToRawShader` ([#2748](https://github.com/mono/SkiaSharp/pull/2748)), R-Tree `beginRecording` overload ([#2889](https://github.com/mono/SkiaSharp/pull/2889)), more `SKPicture` APIs ([#2883](https://github.com/mono/SkiaSharp/pull/2883)), Skottie `Animation.Builder` ([#2630](https://github.com/mono/SkiaSharp/pull/2630)), Blazor components DPI ([#1832](https://github.com/mono/SkiaSharp/pull/1832)), .NET 9 support ([#3010](https://github.com/mono/SkiaSharp/pull/3010)), .NET 8 baseline ([#2927](https://github.com/mono/SkiaSharp/pull/2927)), function pointers + `LibraryImport` ([#2917](https://github.com/mono/SkiaSharp/pull/2917)), `Unsafe.As` perf ([#2780](https://github.com/mono/SkiaSharp/pull/2780)).

## Breaking Changes

`3.116.0` is the major-version cutover — it deletes every `[Obsolete]` member ([#2539](https://github.com/mono/SkiaSharp/pull/2539)), reshapes Metal APIs, and rewrites view types across `SkiaSharp.Views.Android`, `.iOS`, `.tvOS`, `.Mac`, `.Tizen`, `.Desktop.Common`, `.Maui.Controls`, `.WPF`, `.WinUI`, `.Uno.WinUI`, `.WindowsForms` and `.Blazor`. Consult the [SkiaSharp API diff](3.116.0/index.md) — the per-assembly "breaking" pages linked at the top of the diff are the authoritative migration guide.
