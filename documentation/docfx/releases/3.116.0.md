# Version 3.116.0

> **First 3.x stable release** · Released December 2, 2024 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.116.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.116.0)

## Highlights

3.116.0 is the first stable release of the v3 series — it rolls up the entire 3.0 preview cycle cumulatively so upgraders don't need to hop through preview builds. Headlines: a full v3 API cleanup that removed every `[Obsolete]` member (see per-package breaking-changes notes), Skia m116, Metal on Mac Catalyst, first-class WinUI accelerated views, `SKGLElement` for WPF, .NET 8/9 support, and multi-threaded / SIMD WebAssembly. HarfBuzzSharp [8.3.0](harfbuzzsharp/8.3.0.md) ships alongside — see that page for its binding details.

## API and Runtime

- **v3 cleanup** — removed all `[Obsolete]` members and types, then re-added carefully-scoped compatibility overloads (including `SKFilterQuality` for smooth 2.x-to-3.x migration) ([#2539](https://github.com/mono/SkiaSharp/pull/2539), [#2789](https://github.com/mono/SkiaSharp/pull/2789), [#2963](https://github.com/mono/SkiaSharp/pull/2963)).
- **Function pointers and `LibraryImport`** — modernized interop for AoT scenarios, contributed by @maxkatz6 ❤️ ([#2917](https://github.com/mono/SkiaSharp/pull/2917)).
- **Metal on Mac Catalyst** — Skia now uses Metal instead of GL on Mac Catalyst ([#2747](https://github.com/mono/SkiaSharp/pull/2747)); common .NET TFM for Metal APIs contributed by @maxkatz6 ❤️ ([#2788](https://github.com/mono/SkiaSharp/pull/2788)).
- **.NET 9 support** — added .NET 9 targets and ASP.NET Blazor fixes ([#3010](https://github.com/mono/SkiaSharp/pull/3010), [#3081](https://github.com/mono/SkiaSharp/pull/3081), [#3086](https://github.com/mono/SkiaSharp/pull/3086), [#3087](https://github.com/mono/SkiaSharp/pull/3087), [#3089](https://github.com/mono/SkiaSharp/pull/3089), [#3093](https://github.com/mono/SkiaSharp/pull/3093)).
- **APIs added** — `SKImage.ToRawShader` ([#2748](https://github.com/mono/SkiaSharp/pull/2748)), R-Tree overload of `BeginRecording` ([#2889](https://github.com/mono/SkiaSharp/pull/2889)), more `SKPicture` APIs ([#2883](https://github.com/mono/SkiaSharp/pull/2883)), `SKBlender` ([#2830](https://github.com/mono/SkiaSharp/pull/2830)), Skottie `Animation.Builder` ([#2630](https://github.com/mono/SkiaSharp/pull/2630)), Span overload for `SetRectRadii` contributed by @Youssef1313 ❤️ ([#2949](https://github.com/mono/SkiaSharp/pull/2949)), Span overloads for `SKMatrix.MapPoints`/`MapVectors` and `SKPath.AddPoly` contributed by @alexandrvslv ❤️ ([#3030](https://github.com/mono/SkiaSharp/pull/3030)), `GetPixelSpan()` with offsets ([#2609](https://github.com/mono/SkiaSharp/pull/2609)), Span overloads for color filters ([#2879](https://github.com/mono/SkiaSharp/pull/2879)), and text-based blobs re-added ([#2545](https://github.com/mono/SkiaSharp/pull/2545)).
- **Fixes** — `SKPaint` clone-and-dispose no longer AVEs, contributed by @jeremy-visionaid ❤️ ([#2904](https://github.com/mono/SkiaSharp/pull/2904)); `SKRectI` floors outward for correct rounding ([#2568](https://github.com/mono/SkiaSharp/pull/2568)); MAUI-view memory leaks fixed ([#2955](https://github.com/mono/SkiaSharp/pull/2955)); `GetKerningPairAdjustments` fixed, contributed by @pdjonov ❤️ ([#2858](https://github.com/mono/SkiaSharp/pull/2858)); `GRMtlTextureInfo` now has readonly getters ([#2833](https://github.com/mono/SkiaSharp/pull/2833)); Metal `Snapshot` no longer returns magenta on Apple, contributed by @taublast ❤️ ([#2804](https://github.com/mono/SkiaSharp/pull/2804)).

## Breaking Changes

3.116.0 is a source-breaking major release. Per-assembly signature diffs are linked from the [3.116.0 API index](3.116.0/index.md) — every affected package has a `SkiaSharp*.breaking.md` companion covering SkiaSharp core, `SkiaSharp.Views.Blazor`, and every `SkiaSharp.Views.*` package for Android, iOS, Mac, tvOS, Tizen, WinUI, Uno.WinUI, WindowsForms, and `Desktop.Common`, plus `SkiaSharp.Views.Maui.Controls`. If you're consuming any of these packages you should read the corresponding page before upgrading.
