# Version 3.116.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/3.116.0)

The first stable release on the 3.x line. This release rolls up **all** of the preview-only 3.0.0 work (which never shipped as a tagged stable) alongside the additional fixes and features that landed after that preview series. Consumers on 2.88.x can migrate directly here without stepping through the 3.0.0 preview NuGets. [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md) ships alongside; refer to its page for shaping-side binding details.

## Highlights

- **Skia m116** — the native library is upgraded to Chrome milestone 116, bringing years of upstream rendering, text-shaping, and GPU improvements ([#2829](https://github.com/mono/SkiaSharp/pull/2829)). A subsequent community-driven roll-forward from mono/skia [#137](https://github.com/mono/skia/pull/137) picked up additional upstream fixes ([#3026](https://github.com/mono/SkiaSharp/pull/3026) ❤️).
- **.NET 8 and .NET 9** — the primary managed target moved to .NET 8 across every binding and sample ([#2927](https://github.com/mono/SkiaSharp/pull/2927), [#2971](https://github.com/mono/SkiaSharp/pull/2971)), with .NET 9 support added shortly after ([#3010](https://github.com/mono/SkiaSharp/pull/3010) ❤️).
- **Function pointers via `LibraryImport`** — the P/Invoke layer uses .NET 8 function pointers where possible, removing runtime marshalling shims ([#2917](https://github.com/mono/SkiaSharp/pull/2917) ❤️).
- **`[Obsolete]` sweep** — every `[Obsolete]` member and type accumulated during 1.x/2.x has been removed; the 3.x surface is the supported one ([#2539](https://github.com/mono/SkiaSharp/pull/2539)).
- **Skottie animation builder** — build Skottie animations programmatically from managed code ([#2630](https://github.com/mono/SkiaSharp/pull/2630)).
- **Metal on Mac Catalyst** — the Catalyst target now uses Metal as its default backend ([#2747](https://github.com/mono/SkiaSharp/pull/2747)), and the Metal APIs share a common .NET TFM across iOS/macOS/Catalyst ([#2788](https://github.com/mono/SkiaSharp/pull/2788) ❤️).
- **WinUI accelerated views** — `SKSwapChainPanel`-based accelerated rendering for WinUI 3 ([#2733](https://github.com/mono/SkiaSharp/pull/2733)).
- **`SKGLElement` for WPF** — the community-contributed WPF GL surface is back ([#2317](https://github.com/mono/SkiaSharp/pull/2317) ❤️).
- **Native dependency roll-forward** — HarfBuzz 8.3.0 ([#2624](https://github.com/mono/SkiaSharp/pull/2624)), libjpeg-turbo 3.0.0 ([#2583](https://github.com/mono/SkiaSharp/pull/2583)), libwebp 1.3.2 ([#2622](https://github.com/mono/SkiaSharp/pull/2622)), plus a targeted libjpeg-turbo roll-back to 2.1.5.1 to work around a regression ([#2699](https://github.com/mono/SkiaSharp/pull/2699)).
- **WebAssembly multi-thread and SIMD** — WASM native asset can be built with SIMD/threads enabled for Blazor and Uno consumers that opt in ([#2620](https://github.com/mono/SkiaSharp/pull/2620)).

## New features

- **`SKBlender`** — bind the Skia blender construct so callers can compose custom blending pipelines ([#2830](https://github.com/mono/SkiaSharp/pull/2830)).
- **`SKImage.ToRawShader`** — construct a shader from raw image data without going through `ToShader` ([#2748](https://github.com/mono/SkiaSharp/pull/2748)).
- **Skottie `Animation.Builder`** — build Lottie/Skottie animations without the JSON round-trip ([#2630](https://github.com/mono/SkiaSharp/pull/2630)).
- **R-Tree `BeginRecording` overload** — build spatially-indexed pictures for faster hit-testing ([#2889](https://github.com/mono/SkiaSharp/pull/2889)).
- **Additional `SKPicture` APIs** — parity fill-ins across the picture surface ([#2883](https://github.com/mono/SkiaSharp/pull/2883)).
- **Color-filter span overloads** — pass `Span<float>` into color filters instead of arrays ([#2879](https://github.com/mono/SkiaSharp/pull/2879)).
- **`SetRectRadii` span overload** — set a rounded rect's radii from a `Span<SKPoint>` ([#2949](https://github.com/mono/SkiaSharp/pull/2949) ❤️).
- **`SKMatrix.MapPoints/MapVectors`, `SKPath.AddPoly` span overloads** — additional `Span<SKPoint>` overloads throughout ([#3030](https://github.com/mono/SkiaSharp/pull/3030) ❤️).
- **`GetPixelSpan(offsets)`** — access a pixel span at a specific offset within the bitmap ([#2609](https://github.com/mono/SkiaSharp/pull/2609)).
- **`SKFilterQuality` compatibility overloads** — smoother migration path from `SamplingOptions` back to `SKFilterQuality` where callers need it ([#2963](https://github.com/mono/SkiaSharp/pull/2963) ❤️).
- **Blazor DPI exposure** — the Blazor components now surface the current device DPI ([#1832](https://github.com/mono/SkiaSharp/pull/1832) ❤️).
- **`SKGLView` re-implementation** — restored the GL-backed view after the m116 upgrade ([#2598](https://github.com/mono/SkiaSharp/pull/2598)).
- **`SKGLElement` for WPF** — accelerated WPF drawing surface ([#2317](https://github.com/mono/SkiaSharp/pull/2317) ❤️).
- **Trimmable `PlatformConfiguration`** — properties survive assembly trimming so mobile and WASM apps keep the values at publish time ([#2717](https://github.com/mono/SkiaSharp/pull/2717) ❤️).
- **Text-based `SKTextBlob` builders** — restored the higher-level text APIs ([#2545](https://github.com/mono/SkiaSharp/pull/2545)).
- **Vulcanized compiler options** — expose the SkSL vulcanized-mode compiler options ([#2753](https://github.com/mono/SkiaSharp/pull/2753)).

## Bug fixes

- **`SKBitmap.Encode(SKPngEncoderOptions)`** — restored the 2.88.8 behaviour after a preview-only regression ([#3014](https://github.com/mono/SkiaSharp/pull/3014) ❤️).
- **MAUI view memory leaks** — closed the leak in the MAUI handler-based views ([#2955](https://github.com/mono/SkiaSharp/pull/2955)).
- **Cloned `SKPaint` `AccessViolationException`** — cloned paints could crash on disposal ([#2904](https://github.com/mono/SkiaSharp/pull/2904) ❤️).
- **`SKXamlCanvas` unloaded `XamlRoot`** — no longer NREs when the canvas is unloaded ([#2854](https://github.com/mono/SkiaSharp/pull/2854) ❤️).
- **`SKXamlCanvas` on Uno Skia** — switched to `Bgra8888` so pixels render with the correct color order ([#2918](https://github.com/mono/SkiaSharp/pull/2918) ❤️).
- **`SKXamlCanvas` `async void`** — replaced with `async Task` so unhandled exceptions don't tear down the process ([#2720](https://github.com/mono/SkiaSharp/pull/2720) ❤️).
- **Uno WASM canvas not found** — no longer throws when the DOM element hasn't been mounted yet ([#2563](https://github.com/mono/SkiaSharp/pull/2563) ❤️).
- **Uno context inactive during render** — ensure the canvas' context is active before drawing ([#2559](https://github.com/mono/SkiaSharp/pull/2559) ❤️).
- **`GetKerningPairAdjustments`** — API returned incorrect values, now fixed ([#2858](https://github.com/mono/SkiaSharp/pull/2858) ❤️).
- **`SKBitmap.ToSKRectI` rounding** — floor outwards so bounds enclose the source rect ([#2568](https://github.com/mono/SkiaSharp/pull/2568)).
- **Apple Metal snapshot magenta** — fixed the tinted output when snapshotting a Metal surface ([#2804](https://github.com/mono/SkiaSharp/pull/2804) ❤️).
- **WASM nosimd builds** — restored for Edge Enhanced Security Mode and Safari compatibility ([#2612](https://github.com/mono/SkiaSharp/pull/2612) ❤️).
- **.NET 9 ASP.NET Blazor** — fixed loading in .NET 9 Blazor apps ([#3081](https://github.com/mono/SkiaSharp/pull/3081) ❤️, [#3087](https://github.com/mono/SkiaSharp/pull/3087) ❤️, [#3089](https://github.com/mono/SkiaSharp/pull/3089) ❤️, [#3093](https://github.com/mono/SkiaSharp/pull/3093) ❤️).
- **WinUI SDK version pinning** — use `<WindowsSdkPackageVersion>` so consumers can pin the SDK explicitly ([#3086](https://github.com/mono/SkiaSharp/pull/3086) ❤️).

## Platform support

- **Skia** — m116 baseline, with additional upstream fixes pulled forward through m119 territory via mono/skia [#137](https://github.com/mono/skia/pull/137).
- **Managed** — .NET 8 baseline with first-class .NET 9 support.
- **Mac Catalyst** — Metal-backed as the default.
- **WinUI 3 (WinAppSDK / Uno WinUI)** — accelerated views + SDK version pinning + `XamlRoot` null-safety.
- **WPF** — new community-contributed `SKGLElement`.
- **WinForms** — layered on the new `GLControl` packages.
- **WebAssembly** — nosimd builds for Edge/Safari, plus opt-in multi-thread/SIMD builds; Blazor .NET 9 compatibility.
- **Native deps** — HarfBuzz 8.3.0, libwebp 1.3.2, libjpeg-turbo 2.1.5.1 (rolled back from 3.0.0 pending regression fixes).
