# Version 3.116.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/3.116.0)

> **Supersedes [3.0.0](3.0.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](3.116.0/index.md) · [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md)

<!-- AI polish -->

## Highlights

SkiaSharp 3.116.0 is the first stable 3.x release. It **rolls up the entire [3.0.0](3.0.0.md) preview line cumulatively** — the platform pivot to `net8.0`, Mac Catalyst Metal, WinUI accelerated views, the `SKImage.ToRawShader`/`SKBlender`/Skottie Animation Builder bindings, function-pointer P/Invoke, `SKSamplingOptions` — and adds Preview 6's SIMD-enabled WASM builds plus a batch of unreleased backports. HarfBuzzSharp ships in lock-step at 8.3.0 — see the [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md) page for the binding details, and the [SkiaSharp API diff](3.116.0/index.md) for the full surface.

## Breaking Changes

Because 3.116.0 supersedes the 3.0.0 preview line, all of the 3.0.0 breaking changes flow through here:

- **`[Obsolete]` members removed** — every pre-3.0 API that was already marked obsolete has been deleted. Consult the [SkiaSharp API diff](3.116.0/index.md) for the exact list before upgrading.
- **Legacy Xamarin targets removed** — `MonoAndroid`, `Xamarin.iOS`, `Xamarin.Mac`, and `Xamarin.TVOS` no longer ship. Use `net8.0-android`, `net8.0-ios`, `net8.0-maccatalyst`, and `net8.0-tvos` instead.
- **`SKFilterQuality` → `SKSamplingOptions`** — added source-compatible overloads across the drawing surface so most callers can migrate by passing `SKSamplingOptions.Default`. ❤️ [@Youssef1313](https://github.com/Youssef1313)

## New Features

### Engine

- **Skia milestone 116** — Modern Skia primitives replace the older raster pipeline.
- **HarfBuzzSharp 8.3.0** — HarfBuzz shaper ships in lock-step; see [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md).

### GPU & Rendering

- **Multi-threaded / SIMD WASM builds** — Preview 6 adds `.wasm` variants that light up SIMD and multithreading in browsers that support them, dramatically improving raster-pipeline performance. ([#2620](https://github.com/mono/SkiaSharp/pull/2620))
- **Mac Catalyst Metal backend** — SkiaSharp on Mac Catalyst gains the same Metal-backed `SKSurface`/`GRContext` support already present on iOS.
- **WinUI accelerated views** — `SKSwapChainPanel` and `SKXamlCanvas` land for WinUI 3 with D3D-backed rendering.
- **Metal snapshot fix (magenta artifacts)** — Corrects a Metal-backed `SKSurface.Snapshot()` regression. ❤️ [@taublast](https://github.com/taublast)
- **WPF `SKGLElement`** — GL-backed WPF control brought back to parity. ❤️ [@gmurray81](https://github.com/gmurray81)

### API Surface

- **`SKImage.ToRawShader`** — Bind Skia's raw-shader creation path without going through a `SKPaint`.
- **`SKBlender`** — Bind the `SkBlender` interface for custom color blending operations.
- **Skottie Animation Builder** — Bindings for the fluent animation-builder API in the Skottie module.
- **`SKGLView` on Blazor** — First-class Blazor WASM GL control (see `SkiaSharp.Views.Blazor`).
- **Function-pointer interop + `LibraryImport`** — P/Invoke layer rewritten to use `[LibraryImport]` and function pointers, cutting startup allocations. ❤️ [@maxkatz6](https://github.com/maxkatz6)
- **`GetKerningPairAdjustments`** — Bind the kerning-pair lookup on `SKTypeface`. ❤️ [@pdjonov](https://github.com/pdjonov)
- **Span overloads for `SKMatrix`/`SKPath`** — Zero-alloc overloads that accept `ReadOnlySpan<T>` on the hot matrix and path APIs. ❤️ [@alexandrvslv](https://github.com/alexandrvslv) ([#3030](https://github.com/mono/SkiaSharp/pull/3030))
- **`SetRectRadii` Span overload** — Zero-alloc overload for setting rounded-rect corner radii. ❤️ [@Youssef1313](https://github.com/Youssef1313)

### Platform

- **.NET 8 baseline** — Every managed package now targets `net8.0` (and derivative TFMs).
- **.NET 9 previews** — Follow-up work adds early .NET 9 support for Views packages. ❤️ [@jeromelaban](https://github.com/jeromelaban)
- **Metal TFM + trimmable `PlatformConfiguration`** — Views packages annotate their platform-configuration types for trim-safety. ❤️ [@maxkatz6](https://github.com/maxkatz6)
- **Uno Bgra8888** — Uno WinUI branch swaps its pixel format to match the shared Skia surface layout. ❤️ [@Youssef1313](https://github.com/Youssef1313)
- **Late Skia update** — Preview 6 pulls in an additional upstream Skia refresh. ❤️ [@Redth](https://github.com/Redth) ([#3026](https://github.com/mono/SkiaSharp/pull/3026))
- **Blazor DPI** — `SKCanvasView` honours the effective DPI when rendering. ❤️ [@beto-rodriguez](https://github.com/beto-rodriguez)

## Bug Fixes

- **`SKPaint` AVE** — Fix an `AccessViolationException` when disposing a `SKPaint` after its owner. ❤️ [@jeremy-visionaid](https://github.com/jeremy-visionaid)
- **`SKBitmap.Encode` revert** — Restore the previous behaviour of `SKBitmap.Encode(SKEncodedImageFormat, int)` after a regression. ❤️ [@sungaila](https://github.com/sungaila)
- **`SKXamlCanvas` async void** — Replace an `async void` handler that swallowed exceptions. ❤️ [@lindexi](https://github.com/lindexi)
- **XamlRoot fix** — Correct WinUI XamlRoot lookup so Uno pages render on secondary windows. ❤️ [@jeromelaban](https://github.com/jeromelaban)
- **Post-release fixes** — A stream of `unreleased` backports covers WindowsSdkPackageVersion pins, Blazor/WASM ancillary fixes, and CI hardening.
