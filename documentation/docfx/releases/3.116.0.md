## Highlights

The **first stable SkiaSharp 3.x release**. It **supersedes [3.0.0](3.0.0.md)** (which stayed preview-only), so all of that work is included here cumulatively — the SkiaSharp 2.x→3.x modernization, removal of every `[Obsolete]` member, Metal-on-Mac-Catalyst, WinUI accelerated views, `SKBlender`, `SKImage.ToRawShader`, `SKPicture` expansion, Skottie `Animation.Builder`, and the harfbuzz 8.3.0 baseline. On top of that this preview adds **multi-thread and SIMD WebAssembly** natives and a couple of managed-side conveniences. Ships alongside [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md).

- 🎨 **Core** — `SKMatrix.MapPoints` / `SKMatrix.MapVectors` and `SKPath.AddPoly` gained `Span<SKPoint>` overloads (❤️ @alexandrvslv); Skia bumped ([mono/skia#137](https://github.com/mono/skia/pull/137), ❤️ @Redth); the full 3.0 preview surface is included: Skia [m116](https://github.com/mono/skia/pull/122), harfbuzz 8.3.0 ([mono/skia#118](https://github.com/mono/skia/pull/118)), [`SKBlender`](https://github.com/mono/skia/pull/123), [`SKImage.ToRawShader`](https://github.com/mono/skia/pull/119), [R-Tree `beginRecording`](https://github.com/mono/skia/pull/129), more [`SKPicture` APIs](https://github.com/mono/skia/pull/127), Skottie [`Animation.Builder`](https://github.com/mono/skia/pull/114), color-filter and rect-radii spans (❤️ @Youssef1313), `Unsafe.As` micro-perf paths ([mono/skia#120](https://github.com/mono/skia/pull/120)), text-based `SKTextBlobBuilder` overloads restored, `GetKerningPairAdjustments` fix (❤️ @pdjonov), `SKPngEncoderOptions` 2.88.8-behavior restore (❤️ @sungaila), `GetPixelSpan(offset, length)`.
- ⚠️ **Major version bump** — `[Obsolete]` members were **all** removed. `SKFilterQuality` came back as compatibility overloads (❤️ @Youssef1313), and `SKPaint.FilterQuality` is honored by the obsolete `SamplingOptions` helpers.
- 🍏 **Apple** — Metal is the backend on Mac Catalyst; Metal snapshot magenta-color fix (❤️ @taublast); Metal APIs unified onto a common .NET TFM (❤️ @maxkatz6); `GRMtlTextureInfo` getters are read-only; `arm64e` removed.
- 🌐 **WebAssembly** — [multi-thread and SIMD WASM natives](https://github.com/mono/SkiaSharp/pull/2620); .NET 9 Blazor fixes; base-path import fix; `SKXamlCanvas` WASM null-canvas fix (❤️ @jeromelaban); nosimd builds restored (❤️ @jeromelaban); WASM SDK-version pinning via `<WindowsSdkPackageVersion>`.
- 🪟 **Windows / WinUI / UWP / WPF** — WinUI accelerated views; WinForms `SKGLControl` on the new `GLControl` packages; `SKXamlCanvas` fixes (❤️ @jeromelaban, ❤️ @Youssef1313, ❤️ @lindexi); new WPF `SKGLElement` (❤️ @gmurray81); `IBufferByteAccess` updated for AOT.
- 🖼️ **MAUI** — memory-leak fixes.
- 🏗️ **Build / CI** — full repository modernization; migrated to 1ES PT; MSVC build added; function-pointer / `LibraryImport` codegen (❤️ @maxkatz6); .NET 8 baseline + .NET 9 support (❤️ @jeromelaban); libjpeg-turbo 3.0 then partial roll-back to 2.1.5.1 ([mono/skia#115](https://github.com/mono/skia/pull/115)); libwebp 1.3.2 ([mono/skia#112](https://github.com/mono/skia/pull/112)); harfbuzz 7.3.0 ([mono/skia#104](https://github.com/mono/skia/pull/104)); OpenJDK 17; Tizen workload pinning; trimmable `PlatformConfiguration` (❤️ @maxkatz6); dev containers; Vulcanized compiler options; APIScan added; nightly-only full compliance.
- 📦 **General** — new YAML issue templates, README refresh, human-readable diff added, docs regenerated, similar-issues automation.

## Breaking Changes

This is the SkiaSharp **major-version bump from 2.x to 3.x**. Beyond the assembly-version change, the following surface changes affect callers:

- **SkiaSharp** — deep API cleanup (~35 changed types + 9 removed types across the base assembly).
  - **Removed types**: `SK3dView`, `SKAutoMaskFreeImage`, `SKCropRectFlags`, `SKMask`, `SKMaskFormat`, `SKMatrix44TypeMask`, `SKXmlStreamWriter`, `SKXmlWriter`, and the nested `SKImageFilter.CropRect`. These were long-deprecated `[Obsolete]` helpers or thin wrappers around Skia APIs that were removed upstream.
  - **Renames & signature changes**: many `SKImageFilter.Create*`, `SKColorFilter.Create*`, and matrix helpers had their parameter shapes/orders normalized as part of the `[Obsolete]` sweep. If you were relying on the deprecated overloads, migrate to the ones the compiler was already warning about.
- **SkiaSharp.Views.Android / .iOS / .tvOS / .Mac / .Tizen** — API renames and cleanup as part of the SkiaSharp 3.x sweep; consult the linked per-assembly breaking pages if you subclass or interop with the view types directly.
- **SkiaSharp.Views.WindowsForms** — surface cleanup as part of the migration to the modern `GLControl` packages.
- **SkiaSharp.Views.WinUI / .Uno.WinUI** — renames coming from the shared `SkiaSharp.Views.Windows` surface refactor.
- **SkiaSharp.Views.Desktop.Common** — misc cleanup mirroring the desktop reorganization.
- **SkiaSharp.Views.Maui.Controls** — the initial MAUI-controls package churned during preview.
