# Version 4.148.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/4.148.0)

> **Supersedes [4.147.0](4.147.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](4.148.0/index.md) · [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md)

## Highlights

4.148.0 is the **first stable release on the v4 line** — the point where SkiaSharp's major version aligns with the underlying Skia milestone. It rolls up every 4.147.x preview (which was never released as stable) into a single cumulative shipping release, so consumers upgrading from 3.119.x land here directly. Skia is fast-forwarded from m119 through m132, m133, m147, and finally to **m148**, and HarfBuzz jumps to **14.2.0** (see [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md)). Highlights include **variable font support** across SkiaSharp and HarfBuzzSharp (contributed by @ramezgerges), **color font palette support**, **animated WebP encoding** via the new `SKWebpEncoder`, **`SKStream.GetData()`** for zero-copy stream-to-data conversion, a new **`HBColor`** struct for correct `hb_color_t` marshalling, **`SKSamplingOptions`** support on `SKSurface.Draw` and `SKCanvas.DrawSurface` (contributed by @Copilot), a **singleton-lifecycle rework** (contributed by @ramezgerges), and correctness fixes for `SKPixmap`/`SKBitmap.GetPixelSpan` (stride and offset bugs). Legacy obsolete APIs from the 2.x/3.x lines that had been marked `[Obsolete]` are now `[Obsolete(error: true)]` and produce compile errors, and the device / WASM test infrastructure has moved from XHarness to `dotnet test` DeviceRunners.

**⚠️ Migration note:** The v4 line has behavioral changes around `SKFont`, `SKPaint`, and `SKTypeface` default handling that compile unchanged but can cause text to measure `0` or draw nothing if you were relying on lazy null-then-resolve semantics. Read the **Breaking Changes** section carefully before upgrading.

## Breaking Changes

4.148.0 lands the bulk of the v4 breaking changes from the Skia m132/m147 upgrade. Some are signature changes visible in the API diff; others are behavioral (same signature, different runtime) or apply to interop structs outside the diff tool's public-API surface.

### Signature breaks

- **`GRVkImageInfo.YcbcrConversionInfo` — type name casing retype** — Renamed from mixed-case `GrVkYcbcrConversionInfo` to the SkiaSharp-standard upper-case `GRVkYcbcrConversionInfo`.

  ```diff
  -public GrVkYcbcrConversionInfo YcbcrConversionInfo { get; set; }
  +public GRVkYcbcrConversionInfo YcbcrConversionInfo { get; set; }
  ```

  Update locals and fields that stored this property under the old-cased type name.

- **`GrVkYcbcrConversionInfo` (old-cased) lost `IEquatable<T>` and equality operators** — The `Equals(GrVkYcbcrConversionInfo)` override, `op_Equality`, and `op_Inequality` are gone from the old-cased type. Value comparisons should move to the new upper-cased `GRVkYcbcrConversionInfo` type (which is what `GRVkImageInfo.YcbcrConversionInfo` now returns), or compare fields explicitly.

- **`SKNativeObject.IgnorePublicDispose` — setter removed** — The protected property remains readable but is no longer settable from derived classes. This is part of the singleton-lifecycle rework — the value is now decided by construction rather than mutated post-hoc.

  ```diff
   protected bool IgnorePublicDispose { get; ---set;--- }
  ```

- **Assembly version bump to `4.148.0.0`** — All SkiaSharp assemblies now version as `4.148.0.0`. Binding redirects and package references pinned to `3.x` or `4.147.x` must be updated.

- **Legacy `[Obsolete]` members promoted to compile errors** — [#4068](https://github.com/mono/SkiaSharp/pull/4068) by @ramezgerges, [#4114](https://github.com/mono/SkiaSharp/pull/4114) by @mattleibow, and the follow-up [#4205](https://github.com/mono/SkiaSharp/pull/4205) promote the remaining pre-v4 obsoletes to error and trim obsolete enums from the ref assembly. Any code still calling `SKPaint`-based text/font APIs, or other pre-v4 obsoletes, will now fail to compile. Follow the migration advice in the individual `[Obsolete]` messages (visible in the 3.119.x source), typically pointing at `SKFont`, `SKTextBlob`, and modern-shaping equivalents.

### Behavioral breaks (same signature, different runtime)

These compile unchanged but behave differently at runtime — most notably around text.

- **`new SKFont()` now has an empty typeface** ([#3735](https://github.com/mono/SkiaSharp/pull/3735)) — Upstream m132 removed the lazy "resolve default at draw time" pattern. In 3.x, `new SKFont().Typeface` was `null` and resolved to the platform default at draw time; in 4.x, `new SKFont().Typeface` is `SKTypeface.Empty` (0 glyphs), so `MeasureText("Hello")` returns `0` and nothing is drawn.

  Migrate by passing an explicit typeface and detecting "unset" with `IsEmpty`:

  ```csharp
  // After (4.x) — pass the typeface you want
  var font = new SKFont(SKTypeface.Default);
  var width = font.MeasureText("Hello"); // > 0

  // Detecting "no typeface set"
  bool unset = font.Typeface.IsEmpty; // was: font.Typeface == null
  ```

- **`new SKPaint().Typeface` returns Default instead of null** ([#3736](https://github.com/mono/SkiaSharp/pull/3736)) — SkiaSharp mitigates the behavior above by seeding the paint's default font, so text still works out of the box. The only observable break is that `paint.Typeface == null` no longer fires as a "no typeface set" check — use `paint.Typeface.IsEmpty` (or drop the check entirely; a usable default is always present).

- **`SKTypeface.FromFamilyName("missing")` returns Default, not null, on Android** ([#3737](https://github.com/mono/SkiaSharp/pull/3737)) — All platforms now behave consistently: an unknown family returns `SKTypeface.Default` rather than `null` on Android. For a strict "was the font actually found?" check, drop down to the font manager:

  ```csharp
  var tf = SKFontManager.Default.MatchFamily("MissingFont");
  if (tf is null) { /* font not found — apply your own fallback */ }
  ```

- **`SKTypeface.Default` now resolves through the font manager** ([#3738](https://github.com/mono/SkiaSharp/pull/3738)) — `SKTypeface.Default` is now **guaranteed non-null** on every platform (macOS, Windows, Linux, iOS, Android). On Android API 36+ (where the manager could previously return `null`, causing #3693), it falls back to `SKTypeface.Empty` instead of crashing.

- **`SKStream.Move(long)` now throws and is `[Obsolete]`** ([#3740](https://github.com/mono/SkiaSharp/pull/3740)) — The native `sk_stream_move` offset is a 32-bit `int`. The `Move(long)` overload previously did an unchecked `(int)` cast, silently truncating large offsets (a latent data-corruption bug). It now does `checked((int)offset)` and throws `OverflowException` on out-of-range values, and the overload is `[Obsolete]` (compiler warning **CS0618**). Most callers already pass int-range values and only need to switch to `Move(int)` to silence the warning.

### Interop / native-struct breaks (not in any API diff)

- **`GRVkBackendContextNative` — five fields removed** ([#3741](https://github.com/mono/SkiaSharp/pull/3741)) — Upstream simplified `GrVkBackendContext`; the redundant legacy fields were dropped. Removed: `fMinAPIVersion`, `fInstanceVersion`, `fExtensions`, `fFeatures`, `fOwnsInstanceAndDevice`. Delete those assignments — use the typed `fVkExtensions` instead of `fExtensions`, and `fDeviceFeatures` / `fDeviceFeatures2` instead of `fFeatures`. Skia no longer manages instance/device ownership through the backend context, so `fOwnsInstanceAndDevice` has no replacement:

  ```csharp
  var ctx = new GRVkBackendContextNative
  {
      fInstance = instance,
      fPhysicalDevice = physicalDevice,
      fDevice = device,
      fQueue = queue,
      fGraphicsQueueIndex = queueIndex,
      fMaxAPIVersion = maxVersion,
      fVkExtensions = extensions,      // instead of fExtensions
      fDeviceFeatures = features,      // instead of fFeatures
      fGetProc = getProc,
      // fMinAPIVersion / fInstanceVersion / fExtensions /
      // fFeatures / fOwnsInstanceAndDevice — removed, delete these
  };
  ```

- **`GRVkYcbcrConversionInfo` layout change: new `Components` field** ([#3741](https://github.com/mono/SkiaSharp/pull/3741)) — A new `Components` field (backed by a new `GRVkYcbcrComponents` struct) was added, changing the struct layout. Code that constructs the info struct manually or relies on its layout needs to account for the new field.

See the [full API diff](4.148.0/index.md) for the exact removed / renamed signatures across every assembly.
