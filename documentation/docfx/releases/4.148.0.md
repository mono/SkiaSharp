## Highlights

SkiaSharp 4.148.0 is the first stable release on the new **4.x** major line — it rolls up every preview cut from the 4.147.0 line (which was preview-only and never shipped as its own stable NuGet) and adds Skia milestone m148 on top, so the full changelog spans the 3.119.4 → 4.148.0 window. This release lands the Skia m132 / m147 / m148 uplift, variable-font and color-font palette support, animated WebP encoding, `SKStream.GetData()`, `SKPaint.GetFastBounds()`, `SKSamplingOptions` overloads on `SKSurface.Draw`/`SKCanvas.DrawSurface`, an `HBColor` struct for `hb_color_t`, and the completed migration from XHarness to `dotnet test`/DeviceRunners. It also brings a full slate of behavioral and interop breaking changes from the v3→v4 boundary — see the section below. HarfBuzzSharp 14.2.0 ships alongside this release; see the [linked page](harfbuzzsharp/14.2.0.md) for its binding details.

## Breaking Changes

4.148.0 is the first stable **v4** and lands the bulk of the 3→4 breaking changes. Signature breaks are captured in the [breaking-changes diff](4.148.0/SkiaSharp/SkiaSharp.breaking.md); the behavioral and interop breaks below don't appear in any signature diff and must be tested for after upgrading.

- **`new SKFont()` now has an empty typeface** ([#3735](https://github.com/mono/SkiaSharp/pull/3735)) — Upstream m132 removed the lazy "resolve default at draw time" pattern. `new SKFont().Typeface` is now `SKTypeface.Empty` (0 glyphs), so `MeasureText("Hello")` returns `0` and nothing draws. Construct with an explicit typeface (e.g. `new SKFont(SKTypeface.Default)`) and detect "unset" with `font.Typeface.IsEmpty` instead of `== null`.
- **`new SKPaint().Typeface` returns `Default` instead of `null`** ([#3736](https://github.com/mono/SkiaSharp/pull/3736)) — SkiaSharp seeds `SKPaint` with `SKTypeface.Default` so text still works out of the box, but `paint.Typeface == null` checks no longer fire. Use `paint.Typeface.IsEmpty` or drop the check.
- **`SKTypeface.FromFamilyName("missing")` no longer returns `null` on Android** ([#3737](https://github.com/mono/SkiaSharp/pull/3737)) — All platforms now return `SKTypeface.Default` for an unknown family. For a strict "found?" check, use `SKFontManager.Default.MatchFamily(name)`, which still returns `null` on no match.
- **`SKTypeface.Default` now resolves through the font manager** ([#3738](https://github.com/mono/SkiaSharp/pull/3738)) — Guaranteed non-null on every platform, with an `SKTypeface.Empty` fallback on Android API 36+ where the manager could previously return `null` (fixing #3693).
- **`SKStream.Move(long)` now throws and is `[Obsolete]`** ([#3740](https://github.com/mono/SkiaSharp/pull/3740)) — The 32-bit native offset used to be silently truncated by an unchecked cast (latent data-corruption bug); it now uses `checked((int)offset)` and throws `OverflowException` on overflow. Call `Move(int)` with an in-range offset to silence the CS0618 warning.
- **Vulkan `GRVkBackendContextNative` — five fields removed** ([#3741](https://github.com/mono/SkiaSharp/pull/3741)) — This is an interop struct outside the API-diff surface. Removed: `fMinAPIVersion`, `fInstanceVersion`, `fExtensions`, `fFeatures`, `fOwnsInstanceAndDevice`. Use `fVkExtensions` (typed) in place of `fExtensions` and `fDeviceFeatures`/`fDeviceFeatures2` in place of `fFeatures`; delete the rest — Skia no longer manages instance/device ownership through the context.
- **Vulkan `GrVkYcbcrConversionInfo` layout change** ([#3741](https://github.com/mono/SkiaSharp/pull/3741)) — A new `Components` field (and `GRVkYcbcrComponents` struct) was added, changing the struct layout. The related `GRVkImageInfo.YcbcrConversionInfo` retype from `GrVkYcbcrConversionInfo` to `GRVkYcbcrConversionInfo` (and the loss of value-equality on that type) is captured by the signature diff.
- **Pre-v4 obsoletes promoted to error** ([#4068](https://github.com/mono/SkiaSharp/pull/4068), [#4195](https://github.com/mono/SkiaSharp/pull/4195), [#4205](https://github.com/mono/SkiaSharp/pull/4205), [#4114](https://github.com/mono/SkiaSharp/pull/4114)) — All `SKPaint` text/font state-reading obsoletes, remaining pre-v4 obsoletes, and obsolete enums have been promoted to errors and trimmed from the reference assembly. Migrate to `SKFont` for text/font APIs and to the current enum members.
- **Singleton lifecycle rework** ([#4080](https://github.com/mono/SkiaSharp/pull/4080)) — Singleton instances (font managers, defaults, etc.) had their disposal / re-creation semantics reworked. Consumers that stored these across app-domain reloads should re-fetch them.
