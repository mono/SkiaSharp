# Version 4.148.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.148.0-preview)

## Highlights

4.148.0 is the first release candidate of SkiaSharp v4, rolling up all [4.147.0](4.147.0.md) preview work cumulatively — variable fonts, animated WebP, Linux Bionic support, the major version upgrade, and more — and adding the Skia milestone 148 engine update, critical `SKPixmap` memory-layout bug fixes, legacy API retirement to errors, and the co-shipped [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md) with a major upstream HarfBuzz version jump. Community contributors [@ramezgerges](https://github.com/ramezgerges) and [@4Darmygeometry](https://github.com/4Darmygeometry) drove much of the work across both release cycles.

## Engine

## Bug Fixes

- **Fix `SKPixmap`/`SKBitmap` `GetPixelSpan` stride** — `GetPixelSpan` was using `Width * BytesPerPixel` instead of `RowBytes`, causing incorrect bounds on padded bitmaps. ([#4148](https://github.com/mono/SkiaSharp/pull/4148))
- **Fix `SKPixmap.GetPixelSpan<T>(x, y)` offset** — The per-pixel offset calculation was using `Height` instead of `Width`, producing wrong results for non-square images. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **Rework singleton instance lifecycle** — Fixes object lifetime issues with singleton managed wrapper instances. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
