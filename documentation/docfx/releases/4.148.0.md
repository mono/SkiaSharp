# Version 4.148.0

> **SkiaSharp v4 — Skia m148, variable fonts, animated WebP** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.148.0-preview)

> **Supersedes 4.147.0** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md)

## Highlights

SkiaSharp 4.148.0 is the first release of the new v4 major version, powered by Skia milestone 148 and co-shipping [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md). This release rolls up all of 4.147.0's preview work cumulatively — variable font support, animated WebP encoding, zero-copy stream access, color font palette APIs, Linux Bionic and Tizen x64/arm64 native targets, and the long-awaited major version bump from 3.x to 4.x. Community contributors drove a large portion of the new features across the entire 4.x preview cycle.

## ⚠️ Breaking Changes

The v4 major version promotes previously deprecated APIs to compile errors. Callers using legacy `SKPaint` text/font members or pre-v4 obsolete enum values must migrate to current equivalents before upgrading.

- **Legacy `SKPaint` text/font members are now errors** — `SKPaint` text-state and font-state setters deprecated in 3.x are now compile errors. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))

- **Variable font support** — Full variable-font axis control via `SKTypeface` and `HarfBuzzSharp`, enabling dynamic weight, width, and custom axis values at runtime. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))

- **`SKSamplingOptions` on `DrawSurface`** — `SKSurface.Draw` and `SKCanvas.DrawSurface` now accept a `SKSamplingOptions` parameter for higher-quality compositing. ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))

## Bug Fixes

- **`SKPixmap.GetPixelSpan` stride** — Fixed two bugs: `RowBytes` is now used for stride, and the `(x, y)` overload no longer uses `Height` instead of `Width` for the row offset. ([#4148](https://github.com/mono/SkiaSharp/pull/4148), [#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **`SKPath` finalizer crash** — Fixed a native crash when `SKPathBuilder` is garbage-collected before the `SKPath` it produced. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3796](https://github.com/mono/SkiaSharp/pull/3796))
- **Default typeface resolution** — Moved default-typeface resolution from native to the managed layer, fixing edge cases on platforms without a system font manager. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))
- **Android `SKGLView` tab switch** — Fixed `SKGLView` not rendering after switching tabs in a MAUI `TabBar`. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **WinUI Projection DLL on .NET 9** — Fixed `DllNotFoundException` for the WinUI projection assembly when targeting .NET 9. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
- **Native-compat gate** — Moved the native-compatibility check from `[ModuleInitializer]` to the `SkiaApi` static constructor for more reliable startup-time sequencing. ([#4133](https://github.com/mono/SkiaSharp/pull/4133))
- **Singleton lifecycle** — Reworked internal singleton lifecycle management to eliminate potential ordering and disposal issues at startup. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
