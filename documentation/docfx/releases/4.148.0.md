# Version 4.148.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.148.0-preview)

> **Supersedes [4.147.0](4.147.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

## Highlights

SkiaSharp 4.148.0 marks the **major version jump from 3.x to 4.x**, bundling four Skia engine milestones (m132 through m148), full **variable font support**, **animated WebP encoding**, a new `SKStream.GetData()` zero-copy API, and many other API additions and bug fixes. This release also rolls up all 4.147.0 preview-only work cumulatively. Community contributors [@ramezgerges](https://github.com/ramezgerges), [@4Darmygeometry](https://github.com/4Darmygeometry), [@SimonvBez](https://github.com/SimonvBez), [@ebariche](https://github.com/ebariche), [@sasakrsmanovic](https://github.com/sasakrsmanovic), and [@Copilot](https://github.com/Copilot) drove major portions of this release.

### ⚠️ Breaking Changes

- **Major version bump (3.x → 4.x)** — SkiaSharp now uses a 4.x versioning scheme aligned with Skia milestones. Review any version constraints in your project files. ([#3640](https://github.com/mono/SkiaSharp/pull/3640))
- **`SKPaint` text/font APIs are now compile errors** — Previously deprecated paint text members are now hard errors. Migrate to `SKFont`-based APIs. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))
- **Legacy paint state-reading APIs obsoleted** — Additional `SKPaint` read-state APIs marked `[Obsolete]`. ([#4114](https://github.com/mono/SkiaSharp/pull/4114))
- **WASM pre-.NET 8 builds removed** — Native WASM assets for Emscripten toolchains predating .NET 8 are no longer produced. ([#4022](https://github.com/mono/SkiaSharp/pull/4022))

## New Features

### Engine

- **Skia milestone m148** — Updates the engine to Skia m148, incorporating all upstream improvements and fixes. ([#4125](https://github.com/mono/SkiaSharp/pull/4125))
- **Skia milestone m147** — ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **Skia milestone m133** — ([#3660](https://github.com/mono/SkiaSharp/pull/3660))
- **Skia milestone m132** — ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3560](https://github.com/mono/SkiaSharp/pull/3560))
- **Upstream m147 bug-fix cherry-picks** — Two rounds of chrome/m147 upstream fixes merged into the SkiaSharp fork. ([#4081](https://github.com/mono/SkiaSharp/pull/4081), [#4044](https://github.com/mono/SkiaSharp/pull/4044))

### Text & Fonts

- **Variable font support** — Full variable font axis enumeration and value control via `SKFont` and `HarfBuzzSharp`, supporting OpenType `fvar`/`gvar` axes. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette support** — Palette enumeration and selection for color (COLR) fonts, with improved variable font robustness. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct** — Proper managed representation of `hb_color_t` for HarfBuzz color operations. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Default typeface resolution moved to managed layer** — Fixes font fallback edge cases on platforms without a system font manager. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))

### API Surface

- **Animated WebP encoding (`SKWebpEncoder`)** — Encode animated WebP images with full per-frame control. ([#3771](https://github.com/mono/SkiaSharp/pull/3771))
- **`SKStream.GetData()`** — Zero-copy conversion from `SKStream` to `SKData`, avoiding unnecessary buffer duplication. ([#3772](https://github.com/mono/SkiaSharp/pull/3772))
- **`SKSamplingOptions` on surface draw** — `SKSurface.Draw` and `SKCanvas.DrawSurface` now accept explicit sampling filter options. ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))
- **`SKPixmap.GetPixelSpan<T>` fixes** — Fixed stride calculation (was using `Height` instead of `Width`) and corrected row-bytes offset. ([#4128](https://github.com/mono/SkiaSharp/pull/4128), [#4148](https://github.com/mono/SkiaSharp/pull/4148))
- **Missing API wrappers** — Several previously un-wrapped Skia C++ APIs are now accessible, with naming convention violations corrected. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))
- **Singleton lifecycle rework** — Native singleton instances are now managed more safely to prevent use-after-free. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
- **Native compatibility gate improved** — The native ABI check moved from `[ModuleInitializer]` to the `SkiaApi` static constructor for earlier, more reliable detection. ([#4133](https://github.com/mono/SkiaSharp/pull/4133))

### Platform

- **Linux Bionic native assets** — New native binaries for environments using the Android Bionic libc (e.g. WSL2 with Android NDK sysroot). ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 native builds** — 64-bit Tizen architectures now ship native binaries. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **Android MAUI TabBar rendering fix** — `SKGLView` no longer fails to render after a tab switch inside a MAUI `TabBar`. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **Apple TFM alignment** — Libraries use TFM `26.0`; app projects use unversioned TFMs, fixing downstream compatibility issues. ([#3798](https://github.com/mono/SkiaSharp/pull/3798))
- **WinUI .NET 9 DLL fix** — Resolves "Projection DLL not found" errors for .NET 9 WinUI consumers. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
- **C# 13 on legacy TFMs** — PolySharp enables C# 13 language features in the binding project on .NET Framework and older targets. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))
- **Linux Docker cross-compilation** — Linux native builds migrated to .NET official cross-compilation Docker images for more reliable CI. ([#4062](https://github.com/mono/SkiaSharp/pull/4062))
