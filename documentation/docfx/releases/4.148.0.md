# Version 4.148.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.148.0-preview)

> **Supersedes [4.147.0](4.147.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](4.148.0/index.md) · [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md)

## Highlights

Version 4.148.0 builds on the entire [4.147.0](4.147.0.md) preview series — the SkiaSharp 4.x major version upgrade, Skia m132 through m147 engine updates, variable and color font support, new API surface additions, and platform expansions — which shipped as previews only and was never released as stable. On top of that rolled-up base, 4.148.0 updates the Skia engine to **milestone m148**, fixes two `SKPixmap`/`SKBitmap` pixel-span stride and offset bugs, reworks the singleton object lifecycle for correctness and predictability ([@ramezgerges](https://github.com/ramezgerges)), advances the v4 API cleanup by obsoleting legacy paint state-reading APIs and promoting SKPaint text/font obsoletes to errors, and fixes WinUI Projection DLL loading on .NET 9. HarfBuzzSharp 14.2.0 ships alongside — see [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md) for details.

## Breaking Changes

- **Legacy paint state-reading APIs obsoleted** — Several `SKPaint` members used to read text and font state are now marked obsolete with errors. Migrate to the corresponding `SKFont` APIs. ([#4114](https://github.com/mono/SkiaSharp/pull/4114))
- **SKPaint text/font obsolete members → errors** — `SKPaint` text and font properties previously marked `[Obsolete(error: false)]` are now `[Obsolete(error: true)]`. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))
- **Major version 4.x** — (rolled up from 4.147.0) SkiaSharp is now versioned 4.x; verify NuGet and assembly references before upgrading. ([#3640](https://github.com/mono/SkiaSharp/pull/3640))
- **WebAssembly pre-.NET 8 native builds dropped** — (rolled up from 4.147.0) Native WASM builds for pre-.NET 8 Emscripten targets are no longer produced. ([#4022](https://github.com/mono/SkiaSharp/pull/4022))

## New Features

### Engine

- **Skia milestone m148** — Updated the underlying Skia engine to m148, the version this release tracks. ([#4125](https://github.com/mono/SkiaSharp/pull/4125))
- **Skia m132 → m133 → m147** — (rolled up from 4.147.0) Three consecutive engine updates culminating in m147. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3560](https://github.com/mono/SkiaSharp/pull/3560), [#3660](https://github.com/mono/SkiaSharp/pull/3660), [#3702](https://github.com/mono/SkiaSharp/pull/3702))

### Text & Fonts

- **Variable font support** — (rolled up from 4.147.0) Full variable font axis control in SkiaSharp and HarfBuzzSharp. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette support** — (rolled up from 4.147.0) Enumerate and select color font palettes. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct** — (rolled up from 4.147.0) Wraps `hb_color_t` for colour font palette work. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))

### API Surface

- **Reworked singleton instance lifecycle** — Internal singleton objects (`SKColorSpace`, `SKFontManager`, etc.) now use a deterministic lifecycle that prevents use-after-free races during finalization. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
- **`SKSamplingOptions` on `SKSurface.Draw` / `SKCanvas.DrawSurface`** — (rolled up from 4.147.0) ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))
- **`SKStream.GetData()`** — (rolled up from 4.147.0) Zero-copy stream-to-data conversion. ([#3772](https://github.com/mono/SkiaSharp/pull/3772))
- **Animated WebP encoding (`SKWebpEncoder`)** — (rolled up from 4.147.0) ([#3771](https://github.com/mono/SkiaSharp/pull/3771))

## Bug Fixes

- **`SKPixmap`/`SKBitmap` `GetPixelSpan` stride** — Fixed the pixel-span calculation to use `RowBytes` instead of width, correcting the stride for images with row padding. ([#4148](https://github.com/mono/SkiaSharp/pull/4148))
- **`SKPixmap.GetPixelSpan<T>(x, y)` offset** — Fixed an off-by-one bug where the row offset used `Height` instead of `Width`, producing incorrect pixel addresses. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **WinUI Projection DLL not found on .NET 9** — Fixed a missing DLL reference that caused `FileNotFoundException` for WinUI consumers on .NET 9. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
- **`SKPath` finalizer crash** — (rolled up from 4.147.0) ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3796](https://github.com/mono/SkiaSharp/pull/3796))
