## Highlights

The **first stable SkiaSharp 4.x release**. It **supersedes [4.147.0](4.147.0.md)** (which stayed preview-only), so all of that work is included here cumulatively — the 3.x→4.x major bump, Skia m132/m133/m147, variable font support, animated WebP encoding, color font palettes, `SKSamplingOptions` on more surface/canvas members, plus the `GrVkYcbcrConversionInfo` rename. On top of that, this release lands **Skia m148**, a full singleton-lifecycle rework, and the behavioral changes to `SKFont` / `SKPaint` / `SKTypeface` defaults from the m132 upgrade. Ships alongside [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md).

- 🎨 **Core** — [Skia m148](https://github.com/mono/skia/pull/250) merged, plus two upstream `chrome/m147` bug-fix pulls ([mono/skia#242](https://github.com/mono/skia/pull/242), pulled into RC); [harfbuzz 14.2.0](https://github.com/mono/skia/pull/241); the full m132/m133/m147 stack from the 4.147.x line: [m132](https://github.com/mono/skia/pull/171), [m133](https://github.com/mono/skia/pull/179), [m147](https://github.com/mono/skia/pull/184); singleton lifecycle rework (❤️ @ramezgerges); [`SKStream.GetData()`](https://github.com/mono/skia/pull/200); [animated WebP encoding](https://github.com/mono/skia/pull/199); [`SKSamplingOptions` on `SKSurface.Draw` / `SKCanvas.DrawSurface`](https://github.com/mono/SkiaSharp/pull/3491) (❤️ @Copilot); [variable font support](https://github.com/mono/skia/pull/185) (❤️ @ramezgerges); [color font palettes](https://github.com/mono/skia/pull/197); [`sk_fontmgr_legacy_create_typeface`](https://github.com/mono/skia/pull/198); [managed-side default typeface resolution](https://github.com/mono/skia/pull/195) (❤️ @ramezgerges); [`SKPath` finalizer crash fix](https://github.com/mono/SkiaSharp/pull/3796) when `SKPathBuilder` is GC'd first (❤️ @ramezgerges); missing API wrappers filled in; naming-convention violations cleaned; `HBColor` struct added; native-compat gate moved from `[ModuleInitializer]` to the `SkiaApi` cctor; `SKPixmap.GetPixelSpan<T>(x, y)` and `SKPixmap`/`SKBitmap.GetPixelSpan()` bugs fixed (stride now honors `RowBytes`; offset now uses `Width` instead of `Height`).
- ⚠️ **Major version bump** — 3.x → 4.x. Remaining pre-v4 obsoletes were **promoted to error** and the obsolete enums were trimmed from the ref assembly, and `SKPaint` text/font obsolete members were also promoted to error (❤️ @ramezgerges). See "Breaking Changes" below for the runtime-behavior changes that don't show up in the signature diff.
- 🍏 **Apple** — Apple TFMs normalized: `26.0` for libraries, unversioned for apps.
- 🪟 **Windows / WinUI** — WinUI Projection DLL "not found" error fixed for .NET 9 consumers.
- 🌐 **WebAssembly** — pre-.NET 8 Emscripten native builds dropped from CI.
- 🖼️ **Samples / Gallery** — Blazor Gallery "API Atlas" redesign with tag-cloud filtering; five new interactive samples; SkiaFiddle Uno WebAssembly sample (❤️ @ramezgerges), then trimmed ~60% (❤️ @ebariche); Uno Basic sample bumped to `Uno.Sdk/6.6.0-dev.208` (❤️ @ramezgerges); FillPath / TextToPath / Hero-Image samples added; Uno gallery restructured as a multi-page app (❤️ @ramezgerges); v4 icon; Uno README link freshened (❤️ @sasakrsmanovic).
- 🏗️ **Build / CI** — Linux Docker builds migrated to .NET official cross-compilation images; .NET SDK moved to a common step across all builds; content-based build caching with `Cache@2`; expat [2.8.1](https://github.com/mono/skia/pull/245); libjpeg-turbo [3.1.4.1](https://github.com/mono/skia/pull/237); freetype [2.14.3](https://github.com/mono/skia/pull/194); zlib [1.3.2.1-motley](https://github.com/mono/skia/pull/189); libpng [1.6.58](https://github.com/mono/skia/pull/188); libexpat [2.7.5](https://github.com/mono/skia/pull/187); `SkiaSharpGenerator` on `CppAst 0.21.4` (❤️ @ramezgerges); Skia dep-manifest cleanup ([mono/skia#243](https://github.com/mono/skia/pull/243)); `TreatWarningsAsErrors` enabled across `binding/` with all warnings cleaned; Tizen SDK installer bumped to 10.0; automated Skia upstream-sync and API-diff workflows; automated API-docs pipeline; agentic-workflow data persisted to an `aw-data` branch; device and WASM tests migrated from XHarness to DeviceRunners `dotnet test`; new `ci-status` skill for daily CI health dashboard; Chrome Releases blog and Component Governance integration for security-audit skill; docs `docs-live` branch force-push to prevent repo bloat.

## Breaking Changes

This is the **first stable SkiaSharp 4.x** release, so downstream projects must retarget package references. The changes below come from four sources: the signature diff, the deep `SkiaSharp` breaking companion, obsolete promotions, and behavioral / interop changes that never surface in an API diff.

### Managed API breaks (signature diff)

- **SkiaSharp**
  - **`GrVkYcbcrConversionInfo` was renamed to `GRVkYcbcrConversionInfo`** to match the `GR*` capitalization convention. As part of the rename the struct no longer implements `IEquatable<GrVkYcbcrConversionInfo>` and its `Equals(GrVkYcbcrConversionInfo)`, `op_Equality`, and `op_Inequality` overloads were removed. `SKGRVkImageInfo.YcbcrConversionInfo` now returns the new-cased type. Update any code that referenced the old name or those equality operators.
  - **`SKNativeObject`** — the setter on the `Handle` property was removed. Only code that reached into that internal-facing plumbing needs to change.

### Obsolete promotions (compile-time break)

- The remaining pre-v4 `[Obsolete]` members were **promoted to error** and the obsolete enums were trimmed from the ref assembly. If your build still emitted deprecation warnings against 3.x, it will now fail against 4.x — migrate to the replacements the compiler already pointed at.
- `SKPaint` text/font obsolete members (the pre-`SKFont` text-drawing convenience API) are now errors as well. Move text drawing to `SKFont` + `SKCanvas.DrawText(SKTextBlob, ...)` or the `SKCanvas.DrawText(string, ...)` overloads that accept an `SKFont`.

### Behavioral breaks (same signature, different runtime behavior)

These compile unchanged but behave differently at runtime — test text layout and stream seeking after upgrading from 3.x.

- **`new SKFont()` now has an empty typeface.** Upstream m132 removed the "resolve default at draw time" pattern, so `new SKFont().Typeface` is now `SKTypeface.Empty` (0 glyphs) instead of `null`, and `MeasureText("Hello")` returns `0`. Detect "unset" with `font.Typeface.IsEmpty`, and construct with an explicit typeface:

  ```csharp
  // Before (3.x) — worked via lazy default
  var font = new SKFont();
  var width = font.MeasureText("Hello"); // > 0

  // After (4.x) — pass the typeface you want
  var font = new SKFont(SKTypeface.Default);
  var width = font.MeasureText("Hello"); // > 0

  bool unset = font.Typeface.IsEmpty; // was: font.Typeface == null
  ```

- **`new SKPaint().Typeface` returns `SKTypeface.Default` instead of `null`.** SkiaSharp seeds a default font so drawing still works out of the box (and `SKPaint.Reset()` restores that default). The only observable break: code that did `paint.Typeface == null` to detect "no typeface set" no longer fires. Use `paint.Typeface.IsEmpty` or drop the check.

- **`SKTypeface.FromFamilyName("missing")` returns `Default`, not `null`, on Android.** All platforms now behave consistently. For a strict "was the font actually found?" check, use the font manager, which still returns `null` on no match:

  ```csharp
  var tf = SKFontManager.Default.MatchFamily("MissingFont");
  if (tf is null) { /* font not found — apply your own fallback */ }
  ```

- **`SKTypeface.Default` is now guaranteed non-null on every platform.** Default resolution moved to `matchFamilyStyle(null, Normal)` with an empty-typeface fallback (Android API 36+, where the manager could return `null`, no longer crashes). No migration needed.

- **`SKStream.Move(long)` now throws and is obsolete.** The native offset is 32-bit; the old `long` overload silently truncated large offsets. The `long` overload is now `[Obsolete]` (CS0618) and its cast is `checked`, so out-of-range offsets throw `OverflowException`. Migrate to `Move(int)` with an in-range offset.

### Interop / native-struct breaks (not in any API diff)

- **`GRVkBackendContextNative` — five fields removed.** Upstream simplified `GrVkBackendContext`. Remove any assignments to `fMinAPIVersion`, `fInstanceVersion`, `fExtensions`, `fFeatures`, `fOwnsInstanceAndDevice`. Use the typed `fVkExtensions` in place of `fExtensions`, and `fDeviceFeatures` / `fDeviceFeatures2` in place of `fFeatures`. Skia no longer tracks instance/device ownership through the backend context.

  ```csharp
  var ctx = new GRVkBackendContextNative
  {
      fInstance = instance,
      fPhysicalDevice = physicalDevice,
      fDevice = device,
      fQueue = queue,
      fGraphicsQueueIndex = queueIndex,
      fMaxAPIVersion = maxVersion,
      fVkExtensions = extensions,      // instead of fExtensions
      fDeviceFeatures = features,      // instead of fFeatures
      fGetProc = getProc,
      // fMinAPIVersion / fInstanceVersion / fExtensions /
      // fFeatures / fOwnsInstanceAndDevice — removed, delete these
  };
  ```

- **`GrVkYcbcrConversionInfo` layout change.** A new `Components` field (backed by a new `GRVkYcbcrComponents` struct) was added to the Ycbcr conversion info, changing its layout. Code that manually constructs the struct or relies on its layout must account for the new field.
