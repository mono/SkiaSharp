# Version 4.148.0

> **The SkiaSharp 4.0 generation — variable fonts, Skia m148, and a cleaner API surface** · Released June 23, 2026 · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.148.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v4.148.0)

> **Supersedes 4.147.0** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md)

## Highlights

SkiaSharp 4.148.0 is the first stable release of the v4 major line, rolling up an extensive development cycle that spans three Skia engine milestones (m132 through m148). The headline additions are **variable font support** in both SkiaSharp and HarfBuzzSharp, **animated WebP encoding**, **color font palette APIs**, and a co-shipped **HarfBuzz 14.2.0**. Because 4.147.0 shipped only as previews, all of that work is included cumulatively here. Eight community contributors drove significant user-facing changes across the release cycle.

## ⚠️ Breaking Changes

- **Pre-v4 obsolete APIs are now errors** — Members deprecated in the SkiaSharp 3.x cycle (and obsolete enum members removed from the reference assembly) are now compile errors. Migrate to the replacements called out on each member. ([#4205](https://github.com/mono/SkiaSharp/pull/4205), [#4195](https://github.com/mono/SkiaSharp/pull/4195))
- **`SKPaint` text and font APIs are errors** — The legacy text/font state on `SKPaint` (superseded by the `SKFont` API) is now a compile error; use `SKFont` and the `SKPaint`-plus-`SKFont` overloads instead. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))

## New Features

### Engine

- **Skia milestone 148** — Updates the underlying Skia engine through milestones m132 → m133 → m147 → m148, bringing three generations of upstream rendering improvements, bug fixes, and security hardening. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3560](https://github.com/mono/SkiaSharp/pull/3560), [#3702](https://github.com/mono/SkiaSharp/pull/3702), [#4125](https://github.com/mono/SkiaSharp/pull/4125))
- **HarfBuzz 14.2.0** — Co-ships the latest HarfBuzz release, including improved variable-font hinting and OpenType feature coverage. See the linked [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md) page for binding details. ([#4035](https://github.com/mono/SkiaSharp/pull/4035))
- **Upstream m147 bug fixes merged** — Cherry-picked upstream Chrome/m147 Skia bug fixes in two rounds, ahead of the full m148 bump. ([#4081](https://github.com/mono/SkiaSharp/pull/4081), [#4044](https://github.com/mono/SkiaSharp/pull/4044))

### Text & Fonts

- **Variable font support** — Full variable-font binding in both SkiaSharp and HarfBuzzSharp: axis discovery, named instances, and coordinate setting on `SKTypeface` and the HarfBuzz font APIs. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette APIs** — Exposes palette selection and per-glyph color overrides, enabling rich color/emoji font rendering. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct** — Adds the `HBColor` value type to correctly represent `hb_color_t` in managed code. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Default typeface resolution in managed layer** — Moves default-typeface lookup to the managed layer for more predictable cross-platform behavior. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))

### API Surface

- **Animated WebP encoding** — New `SKWebpEncoder` supporting multi-frame animated WebP output. ([#3771](https://github.com/mono/SkiaSharp/pull/3771))
- **`SKStream.GetData()`** — Zero-copy stream-to-`SKData` conversion, eliminating an extra allocation for in-memory streams. ([#3772](https://github.com/mono/SkiaSharp/pull/3772))
- **`SKSamplingOptions` overloads for `SKSurface` / `SKCanvas`** — Adds quality-controlled `SKSamplingOptions` parameter to `SKSurface.Draw` and `SKCanvas.DrawSurface`. ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))
- **Missing API wrappers and naming fixes** — Fills in missing C-API bindings and corrects several naming-convention violations discovered in the surface audit. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))
- **Singleton instance lifecycle** — Reworks the lifecycle of long-lived singleton instances for more deterministic disposal and to avoid premature collection. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))

### Platform

- **Apple TFM alignment** — Corrects Apple platform TFMs so libraries target `26.0` and apps use the unversioned TFM, aligning macOS Catalyst, tvOS, and watchOS with the rest of the Apple family. ([#3798](https://github.com/mono/SkiaSharp/pull/3798))
- **WASM: drop pre-.NET 8 Emscripten builds** — Removes legacy Emscripten native builds for .NET 6/7, reducing WASM package size. ([#4022](https://github.com/mono/SkiaSharp/pull/4022))
- **SkiaFiddle ~60% smaller** — The interactive online demo is dramatically smaller thanks to aggressive size optimizations to the WASM bundle. ❤️ [@ebariche](https://github.com/ebariche) ([#3849](https://github.com/mono/SkiaSharp/pull/3849))
- **C# 13 on legacy TFMs** — Uses PolySharp to enable C# 13 language features across all supported target frameworks, including .NET Framework. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))
