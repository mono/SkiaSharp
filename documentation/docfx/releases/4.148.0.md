# Version 4.148.0

> **First stable v4 release** · Released June 22, 2026 · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.148.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v4.148.0)

> **Supersedes [4.147.0](4.147.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](4.148.0/index.md) · [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md)

## Highlights

**The first stable `4.x` release.** SkiaSharp jumps from Skia m119 to m148,
lands variable fonts, color-font palettes, animated WebP encoding,
`SKStream.GetData()`, and `SKSamplingOptions` on `SKSurface.Draw`. Text
handling is reworked — the legacy "lazy default typeface" pattern is gone
and text-adjacent `SKPaint` state-readers are marked `[Obsolete]` in favor
of `SKFont`. `libpng`, `libexpat`, `libjpeg-turbo`, `freetype`, `zlib`, and
HarfBuzz are all refreshed. Five community contributors landed at least one
change. Ships with [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md).

## Breaking Changes

This is the first stable v4 release and lands the bulk of the v4 breaking
changes. The full signature list is on the
[4.148.0 API diff](4.148.0/index.md#breaking-changes); the biggest ones
are called out below.

**Signature breaks (in the API diff)**

- **Vulkan `GRVkImageInfo.YcbcrConversionInfo` retyped** —
  `GrVkYcbcrConversionInfo` → `GRVkYcbcrConversionInfo` (uppercase `GR`).
  The old struct also loses `IEquatable<GrVkYcbcrConversionInfo>` and its
  `Equals`/`==`/`!=` operators. Only Vulkan interop code that touches the
  Ycbcr conversion struct is affected.
- **`SKNativeObject.IgnorePublicDispose` is now read-only** — the internal
  base type no longer exposes a `set` on this property. It was always
  intended to be a subclass hook, not a runtime toggle; subclasses that
  override the getter continue to work unchanged.

**Behavioral breaks (compile unchanged, run differently)** — test text
layout and stream seeking after upgrading from 3.x:

- **`new SKFont()` now has an empty typeface, not a lazy default
  ([#3735](https://github.com/mono/SkiaSharp/issues/3735)).** In 3.x,
  `new SKFont().MeasureText("Hello")` returned a real width via a lazy
  platform-default typeface; in 4.x, the typeface is `SKTypeface.Empty`
  (zero glyphs) and `MeasureText` returns `0`. Detect "unset" with
  `font.Typeface.IsEmpty`, and construct with an explicit typeface
  (e.g. `new SKFont(SKTypeface.Default)`).
- **`new SKPaint().Typeface` returns `SKTypeface.Default` instead of `null`
  ([#3736](https://github.com/mono/SkiaSharp/issues/3736)).** Text drawing
  still works out of the box (SkiaSharp seeds a default); only code that
  used `paint.Typeface == null` as a sentinel needs to switch to
  `paint.Typeface.IsEmpty`.
- **`SKTypeface.FromFamilyName("missing")` never returns `null` (including
  on Android, [#3737](https://github.com/mono/SkiaSharp/issues/3737)).** All
  platforms now return `SKTypeface.Default` on no match. For a strict
  "was the font found?" check, use `SKFontManager.Default.MatchFamily(...)`
  which still returns `null`.
- **`SKTypeface.Default` now resolves through the font manager
  ([#3738](https://github.com/mono/SkiaSharp/issues/3738)).** It is now
  guaranteed non-null on every platform, and Android API 36+ (where the
  manager itself could return `null`) falls back to `SKTypeface.Empty`
  instead of crashing.
- **`SKStream.Move(long)` throws on overflow and is `[Obsolete]`
  ([#3740](https://github.com/mono/SkiaSharp/issues/3740)).** The native
  offset is a 32-bit `int`; the old `Move(long)` silently truncated.
  It is now `checked((int)offset)` and throws `OverflowException` on
  out-of-range values. Call `Move(int)` with an in-range offset.

**Interop struct changes (not in any API diff)** —
`GRVkBackendContextNative` lives outside the diff surface:

- **Vulkan `GRVkBackendContextNative` — five fields removed
  ([#3741](https://github.com/mono/SkiaSharp/issues/3741)):**
  `fMinAPIVersion`, `fInstanceVersion`, `fExtensions`, `fFeatures`, and
  `fOwnsInstanceAndDevice`. Use `fVkExtensions` in place of `fExtensions`
  and `fDeviceFeatures` / `fDeviceFeatures2` in place of `fFeatures`; Skia
  no longer manages instance/device ownership through the backend context.
- **Vulkan `GrVkYcbcrConversionInfo` layout change
  ([#3741](https://github.com/mono/SkiaSharp/issues/3741)):** a new
  `Components` field (with a new `GRVkYcbcrComponents` struct) is added.
  Code that constructs the struct manually or relies on its layout must
  account for the new field.

**Obsolete promotions**

- **Legacy `SKPaint` text/font state-readers are now `[Obsolete]` errors**
  — properties and methods that read font state off `SKPaint` (measure,
  break, glyph, path, etc.) are promoted to error, pushing you to
  `SKFont`. ❤️ [@ramezgerges](https://github.com/ramezgerges)
  ([#4068](https://github.com/mono/SkiaSharp/pull/4068),
  [#4114](https://github.com/mono/SkiaSharp/pull/4114))
