# Version 4.148.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.148.0-preview)

> **Supersedes [4.147.0](4.147.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](4.148.0/index.md) · [HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md)

## Highlights

Version 4.148.0 is a release candidate that **rolls up all of 4.147.0's preview work** cumulatively — the v4 major version, Skia milestones 132 through 147, variable fonts, color palettes, animated WebP encoding, Linux Bionic native assets, and more — then adds Skia m148, completes the SKPaint API migration, and fixes pixel-stride bugs in `SKPixmap`. ❤️ [@ramezgerges](https://github.com/ramezgerges) reworked the lifecycle of singleton instances for more deterministic resource management and promoted all remaining SKPaint font/text obsoletes to compiler errors, helping finalise the v4 API surface. HarfBuzzSharp [14.2.0](harfbuzzsharp/14.2.0.md) ships alongside this release.

## Breaking Changes

- **SKPaint text/font API errors** — Members on `SKPaint` that relate to text and font selection, deprecated in favour of `SKFont` in earlier releases, are now compiler errors. Applications still using the legacy paint-based text API must migrate to `SKFont`. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))

- **`SKPixmap.GetPixelSpan<T>(x, y)` offset** — Fixes the `(x, y)` overload using `Height` instead of `Width` when computing the pixel byte offset. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
