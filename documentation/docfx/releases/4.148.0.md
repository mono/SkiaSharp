[HarfBuzzSharp 14.2.0](harfbuzzsharp/14.2.0.md) ships alongside — refer to its page for shaping-side binding details.

## Highlights

- **Skia milestone m148** — the native library is upgraded through m132, m133, m147 and finally to m148 in this cycle, bringing years of upstream rendering and text improvements ([#4125](https://github.com/mono/SkiaSharp/pull/4125), [#3660](https://github.com/mono/SkiaSharp/pull/3660), [#3702](https://github.com/mono/SkiaSharp/pull/3702) ❤️, [#3560](https://github.com/mono/SkiaSharp/pull/3560) ❤️).
- **HarfBuzzSharp 14.2.0** — the shaping engine is rebased on HarfBuzz 14.2.0 ([#4035](https://github.com/mono/SkiaSharp/pull/4035)); see the linked HarfBuzz release page for the full binding delta.
- **Variable-font & color-palette support** — both `SkiaSharp` and `HarfBuzzSharp` now expose the axes/variations API for variable fonts and the color-font palette selectors ([#3703](https://github.com/mono/SkiaSharp/pull/3703) ❤️, [#3742](https://github.com/mono/SkiaSharp/pull/3742)).
- **Animated WebP encoding** — `SKWebpEncoder` supports frame-by-frame animated WebP output ([#3771](https://github.com/mono/SkiaSharp/pull/3771)).
- **`SKSamplingOptions` overloads** — `SKSurface.Draw` and `SKCanvas.DrawSurface` now accept sampling options so surface composition matches image composition ([#3491](https://github.com/mono/SkiaSharp/pull/3491) ❤️).
- **`SKStream.GetData()` zero-copy** — construct an `SKData` snapshot from a stream without an intermediate copy ([#3772](https://github.com/mono/SkiaSharp/pull/3772)).
- **`SkiaSharp.Views.Gtk4`** — a modern GTK 4 view built on GirCore.Gtk-4.0 bindings ([#3527](https://github.com/mono/SkiaSharp/pull/3527) ❤️).
- **Automated release-notes pipeline** — an AI-assisted release-notes generator now regenerates the site pages from git history and API diffs ([#3763](https://github.com/mono/SkiaSharp/pull/3763), [#3787](https://github.com/mono/SkiaSharp/pull/3787), [#3815](https://github.com/mono/SkiaSharp/pull/3815), [#4149](https://github.com/mono/SkiaSharp/pull/4149), [#4152](https://github.com/mono/SkiaSharp/pull/4152)).
- **v4 obsolete cleanup** — the legacy paint state-reading APIs and the `SKPaint` text/font members are promoted from obsolete to error, so the v4 surface is the supported one ([#4068](https://github.com/mono/SkiaSharp/pull/4068) ❤️, [#4114](https://github.com/mono/SkiaSharp/pull/4114)).
- **Singleton lifecycle rework** — the shared singletons (default typeface, empty typeface, etc.) now have a robust lifecycle that survives finalizer ordering ([#4080](https://github.com/mono/SkiaSharp/pull/4080) ❤️).
- **Native-compat gate moved to `SkiaApi` cctor** — the runtime version check that ensures managed and native ABI match no longer relies on `[ModuleInitializer]`, which improved startup ordering under Native AOT ([#4133](https://github.com/mono/SkiaSharp/pull/4133)).
- **WinUI Projection DLL** — fixed the "Microsoft.Web.WebView2.Core.dll not found" style error for .NET 9 consumers of the WinUI package ([#4084](https://github.com/mono/SkiaSharp/pull/4084)).
- **`SKPixmap`/`SKBitmap.GetPixelSpan` stride and offset fixes** — pixel-span calls that specified an `(x, y)` origin were computing the offset with `Height` instead of `Width`, and the base overloads were mis-using `Width * bpp` instead of `RowBytes` for stride ([#4128](https://github.com/mono/SkiaSharp/pull/4128), [#4148](https://github.com/mono/SkiaSharp/pull/4148)).
- **Native security roll-ups** — `libpng` 1.6.58 ([#3718](https://github.com/mono/SkiaSharp/pull/3718)), `libexpat` 2.7.5 then 2.8.1 ([#3717](https://github.com/mono/SkiaSharp/pull/3717), [#4079](https://github.com/mono/SkiaSharp/pull/4079)), `zlib` 1.3.2.1-motley ([#3720](https://github.com/mono/SkiaSharp/pull/3720)), `freetype` 2.14.3 ([#3726](https://github.com/mono/SkiaSharp/pull/3726)), `libjpeg-turbo` 3.1.4.1 ([#4012](https://github.com/mono/SkiaSharp/pull/4012)).

## Breaking changes

4.148.0 lands the bulk of the v4 breaking changes from the Skia m132/m147/m148 upgrade. The full signature-level diff is in the linked [SkiaSharp API diff](4.148.0/index.md); the summary below groups the changes by category and highlights the ones that a signature diff **cannot** show (behaviour changes and interop-struct changes).

### Signature changes (from the API diff)

- **`SKNativeObject.IgnorePublicDispose`** — the property's `set` accessor is removed; it is now read-only.
- **`GRVkImageInfo.YcbcrConversionInfo`** — retyped from `GrVkYcbcrConversionInfo` (the removed value-equality struct) to the new `GRVkYcbcrConversionInfo`.
- **`GrVkYcbcrConversionInfo`** — no longer implements `IEquatable<GrVkYcbcrConversionInfo>`, and the `Equals(GrVkYcbcrConversionInfo)`, `op_Equality`, and `op_Inequality` members are removed. Migrate consumers to the new `GRVkYcbcrConversionInfo` type on the retyped `GRVkImageInfo.YcbcrConversionInfo` property above.

### Behavioural changes (same signature, different runtime behaviour)

These compile unchanged but behave differently at runtime, so they never appear in a signature diff. Test text layout and stream seeking after upgrading from 3.x.

- **`new SKFont()` now has an empty typeface** ([#3735](https://github.com/mono/SkiaSharp/pull/3735)) — upstream m132 removed the lazy "resolve default at draw time" pattern. The `SkFont` constructor now eagerly stores an empty typeface when given none, so `new SKFont().MeasureText("Hello")` returns `0` and nothing is drawn. Setting `font.Typeface = null` likewise yields `SKTypeface.Empty`. **Migration:** construct with an explicit typeface (`new SKFont(SKTypeface.Default)`) and detect "unset" with `font.Typeface.IsEmpty` instead of `font.Typeface == null`.
- **`new SKPaint().Typeface` returns `SKTypeface.Default` instead of `null`** ([#3736](https://github.com/mono/SkiaSharp/pull/3736)) — SkiaSharp seeds a default typeface so text still works out of the box, but the null semantics changed. Code that used `paint.Typeface == null` to detect "no typeface set" no longer fires. Use `paint.Typeface.IsEmpty` or drop the check (a usable default is always present). `SKPaint.Reset()` also restores the default.
- **`SKTypeface.FromFamilyName("missing")` returns `Default`, not `null`, on Android** ([#3737](https://github.com/mono/SkiaSharp/pull/3737)) — Android was the outlier and now matches every other platform: `FromFamilyName` never returns `null`. For a strict "was the font actually found?" check, call `SKFontManager.Default.MatchFamily("MissingFont")`, which still returns `null` on no match.
- **`SKTypeface.Default` now resolves through the font manager** ([#3738](https://github.com/mono/SkiaSharp/pull/3738)) — `SkTypeface::RefDefault()`/`MakeDefault()` were removed upstream in m132. Default resolution moved to `matchFamilyStyle(null, Normal)` with an empty-typeface fallback. `SKTypeface.Default` is now guaranteed non-null on every platform; Android API 36+ (where the manager could return null) falls back to `SKTypeface.Empty` instead of crashing.
- **`SKStream.Move(long)` now throws and is obsolete** ([#3740](https://github.com/mono/SkiaSharp/pull/3740)) — the native `sk_stream_move` offset is 32-bit. The `long` overload previously did an unchecked `(int)` cast, silently truncating large offsets. It now uses `checked((int)offset)` so out-of-range values throw `OverflowException`, and the overload is `[Obsolete]` (**CS0618**). **Migration:** call `Move(int)` with an in-range offset; validate large values yourself first.

### Interop / native-struct changes (not in any API diff)

`GRVkBackendContextNative` is an interop struct outside the public-API surface the diff tool compares, so its field removals appear in **no** `*.breaking.md`.

- **Vulkan `GRVkBackendContextNative` — five fields removed** ([#3741](https://github.com/mono/SkiaSharp/pull/3741)) — `fMinAPIVersion`, `fInstanceVersion`, `fExtensions`, `fFeatures`, and `fOwnsInstanceAndDevice` are gone. Delete those assignments in your Vulkan initialisation code. Use the typed `fVkExtensions` in place of `fExtensions`, and `fDeviceFeatures` / `fDeviceFeatures2` in place of `fFeatures`. Skia no longer manages instance/device ownership through the backend context, so `fOwnsInstanceAndDevice` has no replacement.
- **Vulkan `GrVkYcbcrConversionInfo` layout change** ([#3741](https://github.com/mono/SkiaSharp/pull/3741)) — a new `Components` field (and a new `GRVkYcbcrComponents` struct) was added to the Ycbcr conversion info, changing its layout. Code that manually constructs the struct or relies on its layout must account for the new field. See also the retype/value-equality break on `GRVkImageInfo.YcbcrConversionInfo` in the signature-change list above.
