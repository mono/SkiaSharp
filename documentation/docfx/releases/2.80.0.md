# Version 2.80.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.0)

> **API changes** · [SkiaSharp API diff](2.80.0/index.md) · [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md)

## Highlights

The first 2.x release — a major refresh that lands the m80 Skia bump, Vulkan for Win32 (plus the platform-agnostic Vulkan APIs), an initial WebAssembly build, Alpine native binaries, ARM32/ARM64 Linux support, and a rewritten `SKFont` public API. It also includes the first `SKCanvas.DrawTextOnPath` community binding. HarfBuzzSharp 2.6.1.5 ships alongside this release — see [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md) for its binding details.

## Breaking Changes

- **Assembly version bumped to 2.80.0.0** — Consuming projects need to reference the new major/minor. Binding-redirect entries pinned to `1.68.x` must be updated.
- **Default parameter values dropped on `SKPaint.GetFontMetrics`** — The optional `scale = 0` argument on the `out`-based overload is now required; pass `0f` explicitly to preserve the previous behaviour.
- **`SKPath.Iterator.Next` defaults dropped** — The optional `doConsumeDegenerates = true` and `exact = false` arguments are now required.

## Engine

- **Update to Skia m80** ([#986](https://github.com/mono/SkiaSharp/pull/986) by @mattleibow) — Ports SkiaSharp forward to a much later Skia snapshot, unlocking the m80 rendering features (skia: mono/skia#64).
- **Rewrite `SKFont` as glyph-first** ([#1299](https://github.com/mono/SkiaSharp/pull/1299) by @mattleibow) — Makes the public `SKFont` API glyph-oriented; the other typography types now consume `SKFont` internally.
- **Improve the GPU APIs** ([#1294](https://github.com/mono/SkiaSharp/pull/1294) by @mattleibow) — Reshapes the GPU surface for the new backend model.
- **Snapshot with bounds** ([#1357](https://github.com/mono/SkiaSharp/pull/1357) by @mattleibow) — Adds an overload for capturing surface snapshots inside a rectangle.

## GPU & Rendering

- **Vulkan Support for Win32 (and the shared APIs)** ([#1010](https://github.com/mono/SkiaSharp/pull/1010) by @mattleibow) — Adds the Vulkan backend to Win32 alongside the platform-agnostic Vulkan APIs that every backend can now consume.
- **Build for Vulkan on all supported platforms** ([#1287](https://github.com/mono/SkiaSharp/pull/1287) by @mattleibow) — Enables Vulkan in the native build for every platform that supports it.

## API Surface

- **Implement `SKCanvas.DrawTextOnPath`** ([#1198](https://github.com/mono/SkiaSharp/pull/1198) by @ziriax) — Long-requested text-on-path drawing surfaces at last.
- **Expose `SKPath.ToWinding`** ([#1326](https://github.com/mono/SkiaSharp/pull/1326) by @mattleibow) — Converts a path to its winding fill representation.
- **Expose `SKPaint.ColorF`** ([#1325](https://github.com/mono/SkiaSharp/pull/1325) by @mattleibow) — Adds a wide-gamut `SKColorF` accessor on `SKPaint`.
- **Add back removed enum members** ([#1285](https://github.com/mono/SkiaSharp/pull/1285) by @mattleibow) — Restores enum values that were dropped during the m80 rebase for source-compat.

## Bug Fixes

- **Correctly dispose managed/native object relationships** ([#1344](https://github.com/mono/SkiaSharp/pull/1344) by @mattleibow) — Reworks disposal so the managed/native pair is torn down in the right order.
- **Keep the text-blob builder alive** ([#1365](https://github.com/mono/SkiaSharp/pull/1365) by @mattleibow) — Pins the builder for the duration of the native call.
- **Keep the font set alive** ([#1362](https://github.com/mono/SkiaSharp/pull/1362) by @mattleibow) — Same pattern for font-set lifetime.
- **Fix a crash when there is no GL context** ([#1328](https://github.com/mono/SkiaSharp/pull/1328) by @mattleibow) — Gracefully handles the missing-context case.

## Platform

- **Alpine native libraries** ([#1339](https://github.com/mono/SkiaSharp/pull/1339) by @mattleibow) — Ships musl-based `libSkiaSharp` for Alpine containers.
- **Linux ARM32 (armhf) and ARM64 (aarch64)** ([#1382](https://github.com/mono/SkiaSharp/pull/1382) by @mattleibow) — Adds native binaries for both Linux ARM flavours.
- **Win32 ARM64** ([#1358](https://github.com/mono/SkiaSharp/pull/1358) by @mattleibow) — Builds SkiaSharp for Windows on ARM64.
- **WebAssembly build** ([#1359](https://github.com/mono/SkiaSharp/pull/1359), [#1361](https://github.com/mono/SkiaSharp/pull/1361), [#1387](https://github.com/mono/SkiaSharp/pull/1387), [#1389](https://github.com/mono/SkiaSharp/pull/1389) by @mattleibow) — Builds `libSkiaSharp` for WASM, runs WASM unit tests, pins the WASM branch, and packages the WASM deliverables.
- **Request GL ES 3.x explicitly** ([#1388](https://github.com/mono/SkiaSharp/pull/1388) by @mattleibow) — Picks the 3.x profile when it's available on the device.
- **Don't embed `libSkiaSharp` in the assemblies** ([#1296](https://github.com/mono/SkiaSharp/pull/1296) by @mattleibow) — Moves the native binaries out of the managed assemblies to align with the new packaging model.
- **Load specific native libraries on desktop/netfx** ([#1342](https://github.com/mono/SkiaSharp/pull/1342) by @mattleibow) — Improves discovery of the platform-appropriate binary on desktop and .NET Framework hosts.

## Lifecycle & Internals

- **Use Clang 10 and updated Tizen Clang** ([#1286](https://github.com/mono/SkiaSharp/pull/1286), [#1386](https://github.com/mono/SkiaSharp/pull/1386) by @mattleibow) — Refreshes the toolchain.
- **Allow the path to `gn` and `ninja` to be specified** ([#1300](https://github.com/mono/SkiaSharp/pull/1300) by @mattleibow) — Makes local builds easier by not requiring PATH-installed tools.
- **Reduce usage of deprecated native functions** ([#1356](https://github.com/mono/SkiaSharp/pull/1356) by @mattleibow) — Migrates internal calls off deprecated Skia entry points.
- **Fix packaging** ([#1366](https://github.com/mono/SkiaSharp/pull/1366) by @mattleibow) — Fixes the packaging pipeline for the new binary set (skia: mono/skia#81).
