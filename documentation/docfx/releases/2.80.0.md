# Version 2.80.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.0)

> **API changes** · [SkiaSharp API diff](2.80.0/index.md) · [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md)

<!-- Generated: 2026-07-06T23:24:50Z by claude-opus-4.7 -->

First 2.x release. Jumps upstream Skia to milestone m80, adds a **Vulkan backend for Windows** (plus Vulkan APIs across all platforms), lands a **WASM native library** with a full test loop, splits the `SKFont` public API from `SKPaint`, and adds Alpine, ARM64 Win32, and Linux ARM32/ARM64 to the native matrix. HarfBuzzSharp 2.6.1.5 co-ships — see the [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md) page for its binding details.

## Highlights

- **Skia m80 upgrade** ([#986](https://github.com/mono/SkiaSharp/pull/986)) is the flagship engine bump — many years of Skia rendering, GPU, and text improvements arrive together.
- **Vulkan backend on Windows** (and Vulkan APIs surfaced on every supported platform) ([#1010](https://github.com/mono/SkiaSharp/pull/1010)); the native build now compiles Vulkan for every platform we support ([#1287](https://github.com/mono/SkiaSharp/pull/1287)).
- **WebAssembly native library** with unit tests running on it ([#1359](https://github.com/mono/SkiaSharp/pull/1359), [#1361](https://github.com/mono/SkiaSharp/pull/1361)).
- **`SKFont` is now the shape-aware API**; `SKPaint`'s public surface goes glyph-only, with other text types routed through `SKFont` under the hood ([#1299](https://github.com/mono/SkiaSharp/pull/1299)).
- New native RIDs: Win32 ARM64 ([#1358](https://github.com/mono/SkiaSharp/pull/1358)), Alpine ([#1339](https://github.com/mono/SkiaSharp/pull/1339)), Linux ARM32/ARM64 ([#1382](https://github.com/mono/SkiaSharp/pull/1382)).
- HarfBuzzSharp 2.6.1.5 ships alongside — see the linked page for binding details.

## API Surface

- Public `SKFont` split from `SKPaint`; text APIs now route through `SKFont` under the hood ([#1299](https://github.com/mono/SkiaSharp/pull/1299)).
- `SKPath.ToWinding` ([#1326](https://github.com/mono/SkiaSharp/pull/1326)) and `SKPaint.ColorF` ([#1325](https://github.com/mono/SkiaSharp/pull/1325)) exposed.
- `SKCanvas.DrawTextOnPath` implemented, contributed by [@ziriax](https://github.com/ziriax) ([#1198](https://github.com/mono/SkiaSharp/pull/1198)).
- Snapshots with bounds ([#1357](https://github.com/mono/SkiaSharp/pull/1357)); enum members that were removed have been added back ([#1285](https://github.com/mono/SkiaSharp/pull/1285)); reduce the usage of deprecated native functions ([#1356](https://github.com/mono/SkiaSharp/pull/1356)); improve the GPU APIs ([#1294](https://github.com/mono/SkiaSharp/pull/1294)).

## Engine

- Upstream Skia bumped to milestone m80 ([#986](https://github.com/mono/SkiaSharp/pull/986)); use the correct shaders ([#1323](https://github.com/mono/SkiaSharp/pull/1323)); Clang 10 used to build ([#1286](https://github.com/mono/SkiaSharp/pull/1286)); Tizen's Clang updated ([#1386](https://github.com/mono/SkiaSharp/pull/1386)); prefer GL ES 3.x when available ([#1388](https://github.com/mono/SkiaSharp/pull/1388)).

## GPU & Rendering

- **Vulkan backend for Windows** ([#1010](https://github.com/mono/SkiaSharp/pull/1010)); Vulkan native library built on every supported platform ([#1287](https://github.com/mono/SkiaSharp/pull/1287)).
- Fix a crash when there is no GL context ([#1328](https://github.com/mono/SkiaSharp/pull/1328)).

## Platform

- **Win32 ARM64** support ([#1358](https://github.com/mono/SkiaSharp/pull/1358)); **Alpine** native libraries ([#1339](https://github.com/mono/SkiaSharp/pull/1339)); **Linux ARM32 (armhf) and ARM64 (aarch64)** ([#1382](https://github.com/mono/SkiaSharp/pull/1382)); native libraries loaded from a specific location on desktop/netfx ([#1342](https://github.com/mono/SkiaSharp/pull/1342)); `libSkiaSharp` is no longer embedded in the managed assemblies ([#1296](https://github.com/mono/SkiaSharp/pull/1296)); dependencies stop taking the `SkiaSharp` prefix ([#1331](https://github.com/mono/SkiaSharp/pull/1331)).
- **WebAssembly** native library and unit-test loop ([#1359](https://github.com/mono/SkiaSharp/pull/1359), [#1361](https://github.com/mono/SkiaSharp/pull/1361), [#1387](https://github.com/mono/SkiaSharp/pull/1387), [#1389](https://github.com/mono/SkiaSharp/pull/1389)).

## Bug Fixes

- Keep the text-blob builder alive during a shape call ([#1365](https://github.com/mono/SkiaSharp/pull/1365)); keep the font set alive ([#1362](https://github.com/mono/SkiaSharp/pull/1362)); correctly dispose the managed/native object relationship, again ([#1344](https://github.com/mono/SkiaSharp/pull/1344)); fix #1353 ([#1368](https://github.com/mono/SkiaSharp/pull/1368)).

## Lifecycle & Internals
