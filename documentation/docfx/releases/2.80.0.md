# Version 2.80.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.0)

> **API changes** · [SkiaSharp API diff](2.80.0/index.md) · [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md)

The first `2.x` release — rebased on Skia `m80`, retooled around a glyph-first `SKFont`, and gaining Vulkan support across all supported desktop platforms. Thirty-seven PRs across five previews plus an unreleased tail set up the new package layout and native architecture matrix that carries the whole `2.x` line. HarfBuzz `2.6.1.5` ships alongside — see [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md).

## Highlights

- Make `SKFont`'s public API glyph-only and route all other text types through it ([#1299](https://github.com/mono/SkiaSharp/pull/1299)) — the core shift that unlocks the entire `2.x` text pipeline.
- Add Vulkan support across every supported platform, complete with the higher-level APIs consumers see ([#1287](https://github.com/mono/SkiaSharp/pull/1287)), including full Win32 Vulkan support ([#1010](https://github.com/mono/SkiaSharp/pull/1010)).
- Improve the GPU APIs ([#1294](https://github.com/mono/SkiaSharp/pull/1294)); expose `SKPaint.ColorF` ([#1325](https://github.com/mono/SkiaSharp/pull/1325)) and `SKPath.ToWinding` ([#1326](https://github.com/mono/SkiaSharp/pull/1326)); add `SKCanvas.DrawTextOnPath` ([#1198](https://github.com/mono/SkiaSharp/pull/1198)); support snapshots-with-bounds ([#1357](https://github.com/mono/SkiaSharp/pull/1357)).
- Native surface expansion: build Win32 ARM64 ([#1358](https://github.com/mono/SkiaSharp/pull/1358)), Linux `armhf`/`aarch64` ([#1382](https://github.com/mono/SkiaSharp/pull/1382)), Alpine native libraries ([#1339](https://github.com/mono/SkiaSharp/pull/1339)) and a WASM package ([#1389](https://github.com/mono/SkiaSharp/pull/1389)).

## Fixes

- Correctly dispose (again) the managed/native object relationship ([#1344](https://github.com/mono/SkiaSharp/pull/1344)) and reduce usage of deprecated native functions ([#1356](https://github.com/mono/SkiaSharp/pull/1356)).
- Fix packaging ([#1366](https://github.com/mono/SkiaSharp/pull/1366), (skia: [mono/skia#81](https://github.com/mono/skia/pull/81))) and fix [#1353](https://github.com/mono/SkiaSharp/issues/1353) ([#1368](https://github.com/mono/SkiaSharp/pull/1368)).
- Keep the font set alive ([#1362](https://github.com/mono/SkiaSharp/pull/1362)) and keep the text-blob builder alive ([#1365](https://github.com/mono/SkiaSharp/pull/1365)).
- Fix a crash when there is no GL context ([#1328](https://github.com/mono/SkiaSharp/pull/1328)).

## Engine

- Update Skia to `m80` ([#986](https://github.com/mono/SkiaSharp/pull/986), (skia: [mono/skia#64](https://github.com/mono/skia/pull/64))) — the version bump that names the entire `2.x` line.
- Use the correct shaders ([#1323](https://github.com/mono/SkiaSharp/pull/1323)) and re-add previously removed enum members ([#1285](https://github.com/mono/SkiaSharp/pull/1285)).

## Infrastructure

- Load specific native libraries on desktop/netfx ([#1342](https://github.com/mono/SkiaSharp/pull/1342)) and don't add the `SkiaSharp` prefix to dependencies ([#1331](https://github.com/mono/SkiaSharp/pull/1331)); don't embed `libSkiaSharp` in the assemblies ([#1296](https://github.com/mono/SkiaSharp/pull/1296)).
- Add and run WASM unit tests ([#1361](https://github.com/mono/SkiaSharp/pull/1361)); build `libSkiaSharp` for WASM only ([#1359](https://github.com/mono/SkiaSharp/pull/1359)); clone a specific WASM branch ([#1387](https://github.com/mono/SkiaSharp/pull/1387)); update the Tizen Clang ([#1386](https://github.com/mono/SkiaSharp/pull/1386)); explicitly request GL ES 3.x when available ([#1388](https://github.com/mono/SkiaSharp/pull/1388)).
- Use Clang 10 for native builds ([#1286](https://github.com/mono/SkiaSharp/pull/1286)), allow overriding paths to `gn` and `ninja` ([#1300](https://github.com/mono/SkiaSharp/pull/1300)) and improve the CI build label ([#1322](https://github.com/mono/SkiaSharp/pull/1322)); retry a couple of times on flake ([#1360](https://github.com/mono/SkiaSharp/pull/1360)); update samples ([#1313](https://github.com/mono/SkiaSharp/pull/1313)); introduce new images/icons ([#1281](https://github.com/mono/SkiaSharp/pull/1281)); improve the WPF sample startup story ([#1369](https://github.com/mono/SkiaSharp/pull/1369)).

## Breaking Changes

Turning `SKFont` glyph-only ([#1299](https://github.com/mono/SkiaSharp/pull/1299)) and re-shaping the GPU/Vulkan surface changes several long-standing signatures. Consult the [SkiaSharp API diff](2.80.0/index.md) — the breaking sections cover `SkiaSharp` and `SkiaSharp.Views.iOS`/`.tvOS`.
