# Version 2.80.0

> **API changes** · [SkiaSharp API diff](2.80.0/index.md) · [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md)

## Highlights

The major 2.80 line, cutting over to a much newer Skia (m80). Highlights include first-class Vulkan support on Win32, a WASM package, Linux ARM32/ARM64/Alpine builds, and a rewritten glyph-first `SKFont`. Ships alongside [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md).

- 🎨 **Core** — Skia bumped to m80 (see [mono/skia#64](https://github.com/mono/skia/pull/64)); GPU/vulkan APIs improved; `SKFont` public surface reworked around glyphs; `SKPath.ToWinding` and `SKPaint.ColorF` exposed; `SKCanvas.DrawTextOnPath` implemented (❤️ @ziriax); snapshot-with-bounds added; better native-lib disposal semantics.
- 🪟 **Windows** — Vulkan support for Win32 lands; ARM64 native builds; per-platform native library loading on desktop/netfx; libSkiaSharp no longer embedded in managed assemblies.
- 🐧 **Linux** — ARM32 (armhf), ARM64 (aarch64), and Alpine native libraries are now published.
- 🌐 **WebAssembly** — first WASM package; WASM-only libSkiaSharp build; WASM unit-test harness.
- 🍎 **Apple** — improved GPU/GL context handling to avoid a null-context crash.
- 🏗️ **Build / CI** — Clang 10 baseline, configurable `gn`/`ninja` paths, refreshed samples/icons.

## Breaking Changes

- **SkiaSharp**
  - **SKPaint.GetFontMetrics(SKFontMetrics, float scale = 1.0f)** — the default value for `scale` was removed. Pass it explicitly.
  - **SKPath.Iterator.Next(SKPoint[] points, bool doConsumeDegenerates = true, bool exact = false)** — the default values for `doConsumeDegenerates`/`exact` were removed. Pass them explicitly.
- **SkiaSharp.Views.Forms** — the Uno WinRT XAML type-info namespace `SkiaSharp.Views.Forms.Uno.XamlMetaDataProvider` was removed. It was an infrastructure type; consumer code should never have referenced it directly.
