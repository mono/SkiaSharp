# Version 2.80.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.0)

> **API changes** · [SkiaSharp API diff](2.80.0/index.md) · [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md)

<!-- Generated: 2026-07-03T21:22:41Z by claude-opus-4.7 -->

The first major release on the 2.x line, marking the jump to Skia m80. This release added first-class Vulkan, WASM (Blazor), Alpine Linux, Win32 ARM64, and Linux ARM32/ARM64 support, alongside a redesigned `SKFont` API and improved GPU surface bindings. [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md) ships alongside; refer to its page for shaping-side binding details.

## Highlights

- Upgraded the native library to Skia m80, bringing years of upstream rendering, text, and GPU improvements to SkiaSharp consumers ([#986](https://github.com/mono/SkiaSharp/pull/986)).
- Added Vulkan support for Win32 and unified the Vulkan APIs across the Vulkan-capable platforms ([#1010](https://github.com/mono/SkiaSharp/pull/1010), [#1287](https://github.com/mono/SkiaSharp/pull/1287)).
- Introduced a dedicated WASM (Blazor / Uno WebAssembly) build alongside a WASM-specific test suite ([#1359](https://github.com/mono/SkiaSharp/pull/1359), [#1361](https://github.com/mono/SkiaSharp/pull/1361), [#1389](https://github.com/mono/SkiaSharp/pull/1389)).
- Expanded platform coverage: Alpine Linux, Win32 ARM64, Linux ARM32 (armhf), and Linux ARM64 (aarch64) native assets ([#1339](https://github.com/mono/SkiaSharp/pull/1339), [#1358](https://github.com/mono/SkiaSharp/pull/1358), [#1382](https://github.com/mono/SkiaSharp/pull/1382)).
- Redesigned `SKFont` — the public API is now glyph-focused, and the higher-level text types delegate to it under the hood ([#1299](https://github.com/mono/SkiaSharp/pull/1299)).
- Improved the GPU-context APIs so callers can inspect and configure backend state more directly ([#1294](https://github.com/mono/SkiaSharp/pull/1294)).

## New features

- **Vulkan for Win32** — full Vulkan surface support, joining Android and Linux ([#1010](https://github.com/mono/SkiaSharp/pull/1010)).
- **`SKCanvas.DrawTextOnPath`** — draw text along an arbitrary `SKPath` ([#1198](https://github.com/mono/SkiaSharp/pull/1198) ❤️).
- **`SKPath.ToWinding`** — convert a path's fill rule to winding ([#1326](https://github.com/mono/SkiaSharp/pull/1326)).
- **`SKPaint.ColorF`** — set the paint color using the wide-gamut `SKColorF` type ([#1325](https://github.com/mono/SkiaSharp/pull/1325)).
- **`SKImage.Snapshot(bounds)`** — new overloads that snapshot a sub-region ([#1357](https://github.com/mono/SkiaSharp/pull/1357)).

## Bug fixes

- Fixed a crash when there is no GL context available at draw time ([#1328](https://github.com/mono/SkiaSharp/pull/1328)).
- Reworked managed/native disposal to close the last of the finaliser races ([#1344](https://github.com/mono/SkiaSharp/pull/1344)).
- Kept the text-blob builder and font set alive across native calls to avoid mid-draw GC ([#1365](https://github.com/mono/SkiaSharp/pull/1365), [#1362](https://github.com/mono/SkiaSharp/pull/1362)).
- Reduced usage of deprecated Skia native functions to stay ahead of upstream removals ([#1356](https://github.com/mono/SkiaSharp/pull/1356)).
- Load platform-specific native libraries on desktop netfx instead of the generic one ([#1342](https://github.com/mono/SkiaSharp/pull/1342)).
- Restored enum members that were accidentally removed during the m80 upgrade ([#1285](https://github.com/mono/SkiaSharp/pull/1285)).
- Fixed the correct shader binding after the Skia upgrade ([#1323](https://github.com/mono/SkiaSharp/pull/1323)).
- Explicitly request GL ES 3.x when available ([#1388](https://github.com/mono/SkiaSharp/pull/1388)).

## Platform support

- **WebAssembly** — new build and package for Blazor / Uno WebAssembly consumers.
- **Alpine Linux, Linux ARM32/64, Win32 ARM64** — new native asset packages.
- **Vulkan** — supported on Win32 in addition to Android and Linux.
- **Skia** — upgraded to milestone m80.
