# Version 2.80.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.0)

> **API changes** · [SkiaSharp API diff](2.80.0/index.md) · [HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md)

<!-- Generated: 2026-07-07T01:08:21Z by claude-opus-4.7 -->

## Highlights

Marks the 2.x major transition — Skia jumps to milestone m80, Vulkan becomes a first-class cross-platform backend, WebAssembly gets a shipping native build, Linux ARM32/ARM64 join the platform matrix, and the `SKFont` API is reshaped around glyph-only text. Ships alongside **[HarfBuzzSharp 2.6.1.5](harfbuzzsharp/2.6.1.5.md)** — see that page for its binding details. Community contribution: [@ziriax](https://github.com/ziriax) implemented `SKCanvas.DrawTextOnPath`; [@bender2k14](https://github.com/bender2k14) fixed the WPF sample startup project.

## Breaking Changes

- **`SKFont` public API is glyph-only** — `SKFont` no longer exposes the higher-level text-drawing surface directly. Text-drawing types now use `SKFont` under the hood via the glyph API; migrate call sites that used string overloads on `SKFont` to the equivalent `SKTextBlob`/`SKCanvas` shapes.
- **GPU API reshape** — The GPU surface was tightened as part of the m80 upgrade and cross-platform Vulkan enablement. Recompile after reviewing any code that constructed or reconfigured GPU contexts.
- **Reinstated legacy enum members** — Several enum values that were dropped mid-preview were added back for compatibility. Rebuild against the final assembly.

See the [full API diff](2.80.0/index.md) for the complete list.

## Engine

- **Skia milestone m80** — Upgrades the native Skia engine to m80. ([#986](https://github.com/mono/SkiaSharp/pull/986))
- **Fixed disposal correctness** — Managed/native ownership was reworked to close a category of double-dispose issues. ([#1344](https://github.com/mono/SkiaSharp/pull/1344))
- **Text-blob and font-set lifetime** — Keeps the underlying builder and font-set alive across the call chain so intermediate objects survive the drawing operation. ([#1362](https://github.com/mono/SkiaSharp/pull/1362), [#1365](https://github.com/mono/SkiaSharp/pull/1365))

## API Surface

- **`SKCanvas.DrawTextOnPath`** — Adds the classic text-on-path drawing API. ❤️ [@ziriax](https://github.com/ziriax) ([#1198](https://github.com/mono/SkiaSharp/pull/1198))
- **Snapshots with bounds** — New overloads for `SKSurface` snapshots that take an explicit bounds rectangle. ([#1357](https://github.com/mono/SkiaSharp/pull/1357))
- **`SKPath.ToWinding`** — Exposes the winding-fill conversion. ([#1326](https://github.com/mono/SkiaSharp/pull/1326))
- **`SKPaint.ColorF`** — Exposes the float-color property that was previously internal. ([#1325](https://github.com/mono/SkiaSharp/pull/1325))
- **`SKFont` glyph-only API** — Reshapes `SKFont` so the public surface is glyph-based, with the other text types built on top. ([#1299](https://github.com/mono/SkiaSharp/pull/1299))
- **Improved GPU APIs** — Cleans up the GPU-context construction and query surface. ([#1294](https://github.com/mono/SkiaSharp/pull/1294))

## GPU & Rendering

- **Vulkan on all supported platforms** — Vulkan becomes a first-class backend across every supported OS. ([#1287](https://github.com/mono/SkiaSharp/pull/1287))
- **Win32 Vulkan support** — Adds Vulkan support for Win32 and the shared API surface everyone else uses. ([#1010](https://github.com/mono/SkiaSharp/pull/1010))
- **Correct shaders** — Fixes a shipped-shader mismatch after the m80 bump. ([#1323](https://github.com/mono/SkiaSharp/pull/1323))
- **Explicit GL ES 3.x request** — Requests GL ES 3.x when available for better GPU feature coverage. ([#1388](https://github.com/mono/SkiaSharp/pull/1388))
