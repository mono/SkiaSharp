# Version 2.80.3

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.3)

> **API changes** · [SkiaSharp API diff](2.80.3/index.md) · [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md)

<!-- Generated: 2026-07-03T21:22:41Z by claude-opus-4.7 -->

A large point release that expanded the platform matrix (Metal, WinUI, .NET 5, .NET Interactive), split the packages into native-asset variants, added the SkiaSharp shader (SKSL) surface for the first time, and modernised the Uno Platform integration. [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md) ships alongside; refer to its page for shaping-side binding details.

## Highlights

- Added Metal APIs for macOS and iOS, giving Apple-platform apps an alternative to OpenGL for `SKSurface` creation ([#1394](https://github.com/mono/SkiaSharp/pull/1394), [#1615](https://github.com/mono/SkiaSharp/pull/1615)).
- Introduced the `SKRuntimeEffect` binding so callers can compile and use SkSL shaders directly from managed code ([#1604](https://github.com/mono/SkiaSharp/pull/1604)).
- Split the SkiaSharp package into smaller native-asset packages (`SkiaSharp.NativeAssets.*`) so consumers only download the runtimes they need ([#1660](https://github.com/mono/SkiaSharp/pull/1660)).
- Added WinUI 3 for Desktop support and a corresponding `SKXamlCanvas` implementation on Uno Platform ([#1696](https://github.com/mono/SkiaSharp/pull/1696), [#1704](https://github.com/mono/SkiaSharp/pull/1704)).
- Adopted .NET 5 as the primary managed target ([#1697](https://github.com/mono/SkiaSharp/pull/1697)).
- Added .NET Interactive support so SkiaSharp images render inline in notebook cells ([#1710](https://github.com/mono/SkiaSharp/pull/1710)).
- Correctly detected and handled musl-based Linux systems (Alpine) at runtime ([#1657](https://github.com/mono/SkiaSharp/pull/1657)).
- Refactored the Android surface factory and added iOS device tests to the pipeline ([#1617](https://github.com/mono/SkiaSharp/pull/1617), [#1638](https://github.com/mono/SkiaSharp/pull/1638)).

## New features

- **`SKRuntimeEffect`** — first-class SkSL runtime shader support ([#1604](https://github.com/mono/SkiaSharp/pull/1604)).
- **`SKSurface.FromMetal*` / `GRContext.CreateMetal`** — Metal-backed surface creation on macOS and iOS ([#1394](https://github.com/mono/SkiaSharp/pull/1394)).
- **`SKPicture` serialization** — bindings for `Serialize` and `Deserialize` ([#1514](https://github.com/mono/SkiaSharp/pull/1514)).
- **`SKGRContextOptions`** — expose the full options struct when creating a GPU context ([#1529](https://github.com/mono/SkiaSharp/pull/1529)).
- **`SKGRContext.IsAbandoned`** — inspect whether the GPU context has been abandoned ([#1659](https://github.com/mono/SkiaSharp/pull/1659)).
- **Exif SubIFD reading** — decode nested Exif metadata blocks ([#1518](https://github.com/mono/SkiaSharp/pull/1518)).
- **ANGLE `GRGlInterface`** — construct an ANGLE-backed GL interface without going through the default resolver ([#1519](https://github.com/mono/SkiaSharp/pull/1519) ❤️).
- **`SKXamlCanvas` on Uno Platform** — bring the WinUI drawing surface to every Uno backend ([#1704](https://github.com/mono/SkiaSharp/pull/1704)).
- **HarfBuzz interop generator** — the HarfBuzz P/Invoke layer now runs through the same generator as SkiaSharp ([#1447](https://github.com/mono/SkiaSharp/pull/1447)).

## Bug fixes

- Made `SKElement.OnRender` a no-op when there is no `PresentationSource`, avoiding null-reference crashes in unloaded WPF elements ([#1606](https://github.com/mono/SkiaSharp/pull/1606) ❤️).
- Ensured `SKMatrix.IsInvertible` does not crash when the matrix has a zero determinant ([#1527](https://github.com/mono/SkiaSharp/pull/1527)).
- Fixed a GL surface creation failure that could leave the swap chain in a bad state ([#1642](https://github.com/mono/SkiaSharp/pull/1642) ❤️).
- Swap the buffers after resize so the newly-sized surface is presented immediately ([#1668](https://github.com/mono/SkiaSharp/pull/1668)).
- Read image pixels correctly through the `SKBitmap` copy path ([#1636](https://github.com/mono/SkiaSharp/pull/1636)).
- Made the display links more defensive against unexpected teardown ([#1620](https://github.com/mono/SkiaSharp/pull/1620)).
- Stopped allocating temporary arrays when copying `SKStream` data ([#1510](https://github.com/mono/SkiaSharp/pull/1510)).
- Loaded fonts correctly on iOS ([#1539](https://github.com/mono/SkiaSharp/pull/1539)).

## Platform support

- **macOS / iOS** — Metal-backed surfaces available in parallel with GL.
- **WinUI 3 for Desktop / Uno** — new backends including a WinUI 3-compatible `SKXamlCanvas`.
- **Uno Platform 3.0** — updated across the board ([#1489](https://github.com/mono/SkiaSharp/pull/1489)).
- **Alpine / musl** — automatic detection so the correct native runtime loads.
- **.NET 5** — first-class managed target.
- **VS 2013** — still supported for legacy consumers ([#1536](https://github.com/mono/SkiaSharp/pull/1536)).
