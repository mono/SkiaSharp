# Version 2.80.3

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.3)

> **API changes** · [SkiaSharp API diff](2.80.3/index.md) · [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md)

Fifty-five PRs across five previews take the `2.80` line through a comprehensive infrastructure and cross-platform expansion — Metal on Apple platforms, Mac Catalyst, Uno Platform / WinUI, .NET Interactive, plus device-based test rigs for Android and iOS. HarfBuzz `2.6.1.8` ships alongside — see [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md).

## Highlights

- Add Metal APIs for macOS and iOS ([#1394](https://github.com/mono/SkiaSharp/pull/1394)), then bring them in sync ([#1615](https://github.com/mono/SkiaSharp/pull/1615)) and merge the Metal work from `main` ([#1598](https://github.com/mono/SkiaSharp/pull/1598)).
- Add Mac Catalyst support ([#1760](https://github.com/mono/SkiaSharp/pull/1760)) with the right `.targets` file name ([#1762](https://github.com/mono/SkiaSharp/pull/1762)), split SkiaSharp into smaller `NativeAssets.*` packages ([#1758](https://github.com/mono/SkiaSharp/pull/1758)) and add `net6.0-*` targets to the binding projects ([#1707](https://github.com/mono/SkiaSharp/pull/1707)).
- Add the `SKRuntimeEffect` API ([#1604](https://github.com/mono/SkiaSharp/pull/1604)) — the first native SkSL wrapper.
- Add Uno Platform SkiaSharp backends v2 (`SKXamlCanvas` support) ([#1704](https://github.com/mono/SkiaSharp/pull/1704), fixes [#1634](https://github.com/mono/SkiaSharp/issues/1634)), WinUI-for-Desktop support ([#1696](https://github.com/mono/SkiaSharp/pull/1696)) and Uno Platform `3.0` ([#1489](https://github.com/mono/SkiaSharp/pull/1489)) and `4.0` ([#1873](https://github.com/mono/SkiaSharp/pull/1873)).
- Add .NET Interactive support ([#1710](https://github.com/mono/SkiaSharp/pull/1710)) and add bindings for `SKPicture` serialisation ([#1514](https://github.com/mono/SkiaSharp/pull/1514)) and Exif SubIFD reading ([#1518](https://github.com/mono/SkiaSharp/pull/1518)).
- Add ANGLE-based `GRGlInterface` creation ([#1519](https://github.com/mono/SkiaSharp/pull/1519)) and expose `IsAbandoned` in the C# API ([#1659](https://github.com/mono/SkiaSharp/pull/1659)).
- Expose `GRContextOptions` ([#1529](https://github.com/mono/SkiaSharp/pull/1529)) and add a few APIs that make later work easier ([#1603](https://github.com/mono/SkiaSharp/pull/1603)).

## Fixes

- Ensure `SKMatrix.IsInvertible` does not crash ([#1527](https://github.com/mono/SkiaSharp/pull/1527)) and read image pixels correctly ([#1636](https://github.com/mono/SkiaSharp/pull/1636)).
- Swap the buffers after a resize ([#1668](https://github.com/mono/SkiaSharp/pull/1668)) and avoid the failure to create a GL surface ([#1642](https://github.com/mono/SkiaSharp/pull/1642)).
- Make `SKElement.OnRender` a no-op when there is no `PresentationSource` ([#1606](https://github.com/mono/SkiaSharp/pull/1606)); load the fonts on iOS ([#1539](https://github.com/mono/SkiaSharp/pull/1539)); be a bit more defensive in the display links ([#1620](https://github.com/mono/SkiaSharp/pull/1620)).
- Don't allocate arrays when copying streams ([#1510](https://github.com/mono/SkiaSharp/pull/1510)).
- Properly detect and handle musl systems ([#1657](https://github.com/mono/SkiaSharp/pull/1657)) and support VS 2013 for downstream builds because it's easy to do so ([#1536](https://github.com/mono/SkiaSharp/pull/1536)).

## Engine

- Remove the `externals/harfbuzz` submodule ([#1599](https://github.com/mono/SkiaSharp/pull/1599)) and generate the HarfBuzz interop through the same generator as SkiaSharp ([#1447](https://github.com/mono/SkiaSharp/pull/1447)).
- Register the new HarfBuzzSharp WASM package ([#1525](https://github.com/mono/SkiaSharp/pull/1525)).

## Infrastructure

- Add tests for mobile (Android and iOS) ([#1630](https://github.com/mono/SkiaSharp/pull/1630)) and add iOS device tests ([#1638](https://github.com/mono/SkiaSharp/pull/1638)); attempt to use the local ChromeDriver first ([#1653](https://github.com/mono/SkiaSharp/pull/1653)) and update it again ([#1540](https://github.com/mono/SkiaSharp/pull/1540)); run nightly builds to stay green ([#1651](https://github.com/mono/SkiaSharp/pull/1651)); fix the Android test runner ([#1665](https://github.com/mono/SkiaSharp/pull/1665)).
- Add the `arm64` build target ([#1627](https://github.com/mono/SkiaSharp/pull/1627)); refactor the Android surface factory ([#1617](https://github.com/mono/SkiaSharp/pull/1617)); split packages up for better downloads ([#1660](https://github.com/mono/SkiaSharp/pull/1660)); Roll back to a specific Xamarin.iOS that works with Xcode 12.4 ([#1703](https://github.com/mono/SkiaSharp/pull/1703)); update to macOS 11 (`#1988`) via later notes.
- Update to .NET 5 ([#1697](https://github.com/mono/SkiaSharp/pull/1697)); update Xcode & .NET Core 3.1 ([#1594](https://github.com/mono/SkiaSharp/pull/1594)); pin the xharness version and pack correctly ([#1685](https://github.com/mono/SkiaSharp/pull/1685)); build and include every Emscripten version for WASM ([#1590](https://github.com/mono/SkiaSharp/pull/1590)) with a picture test to prove it ([#1592](https://github.com/mono/SkiaSharp/pull/1592)); update externals for the new `gn` and add a helper script ([#1591](https://github.com/mono/SkiaSharp/pull/1591)); update to a later Tizen Studio ([#1566](https://github.com/mono/SkiaSharp/pull/1566)); use v2 signing ([#1568](https://github.com/mono/SkiaSharp/pull/1568)); update MSBuild.Sdk.Extras ([#1511](https://github.com/mono/SkiaSharp/pull/1511)); update the version to `2.80.3` ([#1512](https://github.com/mono/SkiaSharp/pull/1512)); fix the bad things ([#1662](https://github.com/mono/SkiaSharp/pull/1662)); improve build props/targets ([#1698](https://github.com/mono/SkiaSharp/pull/1698)); improve the local builds twice ([#1593](https://github.com/mono/SkiaSharp/pull/1593), [#1597](https://github.com/mono/SkiaSharp/pull/1597)); make things consistent ([#1595](https://github.com/mono/SkiaSharp/pull/1595)); introduce Metal consistency ([#1615](https://github.com/mono/SkiaSharp/pull/1615)); Uno docs cross-links ([#1602](https://github.com/mono/SkiaSharp/pull/1602)); WinUI `0.8.0` update ([#1741](https://github.com/mono/SkiaSharp/pull/1741)).

## Breaking Changes

The Uno-Platform binding surface receives the biggest reshaping — `SkiaSharp.Views.UWP` in particular. Consult the [SkiaSharp API diff](2.80.3/index.md) before upgrading if you consume that binding.
