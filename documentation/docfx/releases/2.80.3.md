# Version 2.80.3

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.3)

> **API changes** · [SkiaSharp API diff](2.80.3/index.md) · [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md)

## Highlights

Version 2.80.3 introduces **`SKRuntimeEffect`**, Metal APIs for macOS and iOS, .NET 5.0 support, WinUI-for-Desktop views, Uno Platform v3 (and v2 SkiaSharp backends for `SKXamlCanvas`), the removal of the `externals/harfbuzz` submodule (in favor of the shared HarfBuzz build), and .NET Interactive support. It also lands SkPicture serialization bindings, EXIF SubIFD reading, musl detection on Linux, and the switch to the generated interop layer. Community contributions include GL-surface failure handling from [@gmurray81](https://github.com/gmurray81), a null-guard for `SKElement.OnRender` from [@HarlanHugh](https://github.com/HarlanHugh), Uno documentation from [@jeromelaban](https://github.com/jeromelaban), and ANGLE `GRGlInterface` creation from [@Mikolaytis](https://github.com/Mikolaytis). HarfBuzzSharp 2.6.1.8 co-ships — see the HarfBuzz link on the API changes line above for its per-line page.

## Breaking Changes

Some UWP-view signatures shifted with the Uno v3 update. See the linked `SkiaSharp.Views.Uno` / `SkiaSharp.Views.UWP` breaking-changes file for the exact deltas.

## New Features

- **`SKRuntimeEffect` API** — First-class runtime shader effects for the 2.x line. ([#1604](https://github.com/mono/SkiaSharp/pull/1604))
- **Metal APIs for macOS and iOS** — Native Metal context bindings. ([#1394](https://github.com/mono/SkiaSharp/pull/1394)); merged from main ([#1598](https://github.com/mono/SkiaSharp/pull/1598)); consistency pass ([#1615](https://github.com/mono/SkiaSharp/pull/1615))
- **`SKPicture` serialization bindings** — Round-trip pictures to bytes. ([#1514](https://github.com/mono/SkiaSharp/pull/1514))
- **EXIF SubIFD reading** ([#1518](https://github.com/mono/SkiaSharp/pull/1518))
- **Expose `GRContext.IsAbandoned`** ([#1659](https://github.com/mono/SkiaSharp/pull/1659))

- **Additional APIs for future-friendly compatibility** ([#1603](https://github.com/mono/SkiaSharp/pull/1603))
