# Version 2.80.3

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.3)

> **API changes** · [SkiaSharp API diff](2.80.3/index.md) · [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md)

<!-- Generated: 2026-07-06T23:24:50Z by claude-opus-4.7 -->

Second servicing release on the 2.80 line. Adds **Apple Metal APIs** (for macOS and iOS), the **`SKRuntimeEffect` API** for compiled SkSL, **WinUI for Desktop** and **Uno Platform SKXamlCanvas** support, splits ANGLE creation to accept an existing GL interface, and packs several stability/build fixes. HarfBuzzSharp 2.6.1.8 co-ships — see the [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md) page for its binding details.

## Highlights

- **Apple Metal APIs** for macOS and iOS ([#1394](https://github.com/mono/SkiaSharp/pull/1394)), and later merged with the main-branch Metal work for consistency ([#1598](https://github.com/mono/SkiaSharp/pull/1598), [#1615](https://github.com/mono/SkiaSharp/pull/1615)).
- **`SKRuntimeEffect` API** — compile and run SkSL fragment shaders from managed code ([#1604](https://github.com/mono/SkiaSharp/pull/1604)).
- **WinUI for Desktop** support ([#1696](https://github.com/mono/SkiaSharp/pull/1696)) and **`SKXamlCanvas` on Uno Platform SkiaSharp backends v2.0** ([#1704](https://github.com/mono/SkiaSharp/pull/1704), Uno v3.0 bump [#1489](https://github.com/mono/SkiaSharp/pull/1489)).
- `.NET 5.0` supported ([#1697](https://github.com/mono/SkiaSharp/pull/1697)); the netstandard `HarfBuzzSharp` interop is now auto-generated ([#1447](https://github.com/mono/SkiaSharp/pull/1447)).
- **VS 2013** is supported again for netfx consumers ([#1536](https://github.com/mono/SkiaSharp/pull/1536)); the packages have been split up for faster downloads ([#1660](https://github.com/mono/SkiaSharp/pull/1660)).
- Musl systems properly detected and handled ([#1657](https://github.com/mono/SkiaSharp/pull/1657)); ARM64 build target added on macOS ([#1627](https://github.com/mono/SkiaSharp/pull/1627)); iOS/Android device test loops land ([#1638](https://github.com/mono/SkiaSharp/pull/1638), [#1630](https://github.com/mono/SkiaSharp/pull/1630), [#1665](https://github.com/mono/SkiaSharp/pull/1665)).
- .NET Interactive support ([#1710](https://github.com/mono/SkiaSharp/pull/1710)); nightly builds ensure green ([#1651](https://github.com/mono/SkiaSharp/pull/1651)).
- HarfBuzzSharp 2.6.1.8 ships alongside — see the linked page for binding details.

## API Surface

- **Metal** APIs on Apple ([#1394](https://github.com/mono/SkiaSharp/pull/1394)); consistency pass on Metal ([#1615](https://github.com/mono/SkiaSharp/pull/1615)) and merge of parallel Metal work from main ([#1598](https://github.com/mono/SkiaSharp/pull/1598)).
- **`SKRuntimeEffect`** for SkSL shaders ([#1604](https://github.com/mono/SkiaSharp/pull/1604)).
- `GRContextOptions` exposed ([#1529](https://github.com/mono/SkiaSharp/pull/1529)); `IsAbandoned` exposed on the C# `GRContext` ([#1659](https://github.com/mono/SkiaSharp/pull/1659)); create an ANGLE-backed `GRGlInterface` from an existing GL interface, contributed by [@Mikolaytis](https://github.com/Mikolaytis) ([#1519](https://github.com/mono/SkiaSharp/pull/1519)); several small APIs added to enable later work ([#1603](https://github.com/mono/SkiaSharp/pull/1603)).
- `SKPicture` serialization bindings ([#1514](https://github.com/mono/SkiaSharp/pull/1514)); Exif SubIFD reading ([#1518](https://github.com/mono/SkiaSharp/pull/1518)); guard `SKMatrix.IsInvertible` against crashes ([#1527](https://github.com/mono/SkiaSharp/pull/1527)); WASM HarfBuzzSharp package registered ([#1525](https://github.com/mono/SkiaSharp/pull/1525)).

## Engine

- Generate the HarfBuzz interop instead of hand-writing it ([#1447](https://github.com/mono/SkiaSharp/pull/1447)); the `externals/harfbuzz` submodule is removed ([#1599](https://github.com/mono/SkiaSharp/pull/1599)); externals updated for a newer gn plus helper script ([#1591](https://github.com/mono/SkiaSharp/pull/1591)); Tizen Studio updated ([#1566](https://github.com/mono/SkiaSharp/pull/1566), [#1594](https://github.com/mono/SkiaSharp/pull/1594)); tooling consistency pass ([#1595](https://github.com/mono/SkiaSharp/pull/1595)); improved local builds ([#1593](https://github.com/mono/SkiaSharp/pull/1593), [#1597](https://github.com/mono/SkiaSharp/pull/1597)).

## GPU & Rendering

- Metal APIs (see above); avoid failure when creating a GL surface, contributed by [@gmurray81](https://github.com/gmurray81) ([#1642](https://github.com/mono/SkiaSharp/pull/1642)); swap the buffers after a resize ([#1668](https://github.com/mono/SkiaSharp/pull/1668)); display links behave more defensively ([#1620](https://github.com/mono/SkiaSharp/pull/1620)); Android surface factory refactored ([#1617](https://github.com/mono/SkiaSharp/pull/1617)); Fonts load on iOS ([#1539](https://github.com/mono/SkiaSharp/pull/1539)).
