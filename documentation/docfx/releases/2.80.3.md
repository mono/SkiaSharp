# Version 2.80.3

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.80.3)

> **API changes** · [SkiaSharp API diff](2.80.3/index.md) · [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md)

## Highlights

Third servicing drop on the 2.80 line — adds `SKRuntimeEffect`, brings in the Metal APIs for macOS/iOS, exposes `GRContext.IsAbandoned` and `GRContextOptions`, adds `SKPicture` serialization bindings, generates the HarfBuzz interop, and lights up WinUI-for-Desktop, Uno Platform (up to 3.0), and .NET Interactive support. Also ships new .NET 5.0 targets, VS 2013 support, arm64 builds, and Musl-system detection. HarfBuzzSharp 2.6.1.8 ships alongside this release — see [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md) for its binding details.

## Breaking Changes

- **`SkiaSharp.Views.UWP` cleanup** — The generated UWP views API surface received a small internal-only cleanup captured in the linked API diff. Consuming code should not be affected.

## Engine

- **Add the `SKRuntimeEffect` API** ([#1604](https://github.com/mono/SkiaSharp/pull/1604) by @mattleibow) — Exposes Skia's SkSL runtime-effect API, letting callers author custom shaders in Skia's shading language.
- **Introduce Metal consistency** ([#1615](https://github.com/mono/SkiaSharp/pull/1615) by @mattleibow) — Aligns the Metal-related APIs across macOS and iOS.

## GPU & Rendering

- **Add Metal APIs for macOS and iOS** ([#1394](https://github.com/mono/SkiaSharp/pull/1394) by @mattleibow) — First-class bindings for the Metal backend.
- **Expose `GRContextOptions`** ([#1529](https://github.com/mono/SkiaSharp/pull/1529) by @mattleibow) — Surfaces the GPU-context tuning knobs.
- **Expose `IsAbandoned` on the C# API** ([#1659](https://github.com/mono/SkiaSharp/pull/1659) by @mattleibow) — Lets callers detect an abandoned GPU context without a native round-trip.
- **Allow creating ANGLE `GRGlInterface`** ([#1519](https://github.com/mono/SkiaSharp/pull/1519) by @Mikolaytis) — Adds a factory for building an ANGLE-backed `GRGlInterface`.
- **Swap buffers after resize** ([#1668](https://github.com/mono/SkiaSharp/pull/1668) by @mattleibow) — Ensures the swap chain settles after window resizes.

## API Surface

- **Bindings for `SKPicture` serialization** ([#1514](https://github.com/mono/SkiaSharp/pull/1514) by @mattleibow) — Exposes `SKPicture.Serialize`/`Deserialize` so pictures can be persisted.
- **Read the Exif SubIFD** ([#1518](https://github.com/mono/SkiaSharp/pull/1518) by @mattleibow) — Adds Exif SubIFD reading to the codec pipeline.
- **Generate HarfBuzz interop** ([#1447](https://github.com/mono/SkiaSharp/pull/1447) by @mattleibow) — Regenerates HarfBuzz P/Invoke via the generator introduced on the 1.68 line.

## Bug Fixes

- **Avoid failure to create GL surface** ([#1642](https://github.com/mono/SkiaSharp/pull/1642) by @gmurray81) — Handles the case where the platform can't create a GL surface without crashing.
- **`SKMatrix.IsInvertible` doesn't crash** ([#1527](https://github.com/mono/SkiaSharp/pull/1527) by @mattleibow) — Fixes a native call that could crash on non-invertible matrices.
- **`SKElement.OnRender` no-ops without `PresentationSource`** ([#1606](https://github.com/mono/SkiaSharp/pull/1606) by @HarlanHugh) — Prevents an NRE when the WPF element hasn't been added to the visual tree yet.
- **Load fonts on iOS** ([#1539](https://github.com/mono/SkiaSharp/pull/1539) by @mattleibow) — Restores the iOS font-loading path.
- **Read image pixels correctly** ([#1636](https://github.com/mono/SkiaSharp/pull/1636) by @mattleibow) — Fixes a regression when reading pixels from decoded images.
- **Don't allocate arrays when copying streams** ([#1510](https://github.com/mono/SkiaSharp/pull/1510) by @mattleibow) — Reuses buffers on stream copy.
- **Musl detection on Linux** ([#1657](https://github.com/mono/SkiaSharp/pull/1657) by @mattleibow) — Correctly picks the musl runtime on Alpine and similar distros.

## Platform

- **WinUI for Desktop** ([#1696](https://github.com/mono/SkiaSharp/pull/1696) by @mattleibow) — Adds initial WinUI-for-Desktop support.
- **`SKXamlCanvas` on Uno Platform SkiaSharp backends v2.0** ([#1704](https://github.com/mono/SkiaSharp/pull/1704) by @mattleibow) — Ports `SKXamlCanvas` support to the new Uno SkiaSharp backends ([#1634](https://github.com/mono/SkiaSharp/pull/1634)).
- **Update Uno Platform** ([#1489](https://github.com/mono/SkiaSharp/pull/1489), [#1708](https://github.com/mono/SkiaSharp/pull/1708) by @mattleibow) — Moves Uno to 3.0 and later.
- **Add .NET Interactive support** ([#1710](https://github.com/mono/SkiaSharp/pull/1710) by @mattleibow) — Enables notebook rendering scenarios.
- **arm64 build target** ([#1627](https://github.com/mono/SkiaSharp/pull/1627) by @mattleibow) — Adds ARM64 to the native build matrix.
- **VS 2013 support** ([#1536](https://github.com/mono/SkiaSharp/pull/1536) by @mattleibow) — Because it's easy to do so.
- **All emscripten variants for WASM** ([#1590](https://github.com/mono/SkiaSharp/pull/1590) by @mattleibow) — Ships every supported emscripten WASM variant.
- **Register HarfBuzzSharp WASM package** ([#1525](https://github.com/mono/SkiaSharp/pull/1525) by @mattleibow) — Publishes HarfBuzzSharp for WebAssembly.

## Lifecycle & Internals

- **Update to .NET 5.0** ([#1697](https://github.com/mono/SkiaSharp/pull/1697) by @mattleibow) — Adds .NET 5 targets to the projects and pipeline.
- **Split packages for better downloads** ([#1660](https://github.com/mono/SkiaSharp/pull/1660) by @mattleibow) — Breaks the shipping packages into smaller, download-friendlier pieces.
- **Remove `externals/harfbuzz` submodule** ([#1599](https://github.com/mono/SkiaSharp/pull/1599) by @mattleibow) — Now consumed via the shared externals path.
- **Update Xcode & .NET Core 3.1** ([#1594](https://github.com/mono/SkiaSharp/pull/1594) by @mattleibow) — Toolchain refresh.
- **v2 signing** ([#1568](https://github.com/mono/SkiaSharp/pull/1568) by @mattleibow) — Moves the signing pipeline to v2.
