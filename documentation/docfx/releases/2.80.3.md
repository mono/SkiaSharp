# Version 2.80.3

## Highlights

Servicing rev on the 2.80 line that quietly absorbs a large amount of Metal-on-Apple work, WinUI-for-Desktop support, WASM-on-Uno, and the first `SKRuntimeEffect` API. Ships alongside [HarfBuzzSharp 2.6.1.8](harfbuzzsharp/2.6.1.8.md).

- 🎨 **Core** — `SKRuntimeEffect` API added (Skia's SkSL); `GRContextOptions` now exposed; `GRContext.IsAbandoned` visible in the C# API; Exif SubIFD parsing; `SKMatrix.IsInvertible` no longer crashes on singular inputs.
- 🍎 **Apple** — Metal APIs for macOS and iOS land (originally cherry-picked from main), with consistency fixes around display links and surface management.
- 🪟 **Windows / Uno** — WinUI for Desktop supported; WinUI updated to 0.8.0; `SKXamlCanvas` now backs the Uno Platform SkiaSharp v2 backend (see Breaking Changes for the base-type change).
- 🐧 **Linux** — musl detection lands.
- 🌐 **WebAssembly** — WASM published for every supported Emscripten; new HarfBuzzSharp WASM package registered.
- 🏗️ **Build / CI** — Nightly greening, packages split for smaller downloads, .NET 5.0 baseline, iOS device tests, and Android/mobile test coverage.
- 📦 **General** — .NET Interactive (Polyglot Notebooks) support added.

## Breaking Changes

- **SkiaSharp.Views.Uno.WinUI (SKXamlCanvas)** — the base type changed from `Windows.UI.Xaml.FrameworkElement` to `Windows.UI.Xaml.Controls.Canvas` so it can host child elements. Code that relied on the previous base type (e.g. a `FrameworkElement`-typed field/property) must be updated; `Canvas` derives from `Panel` → `FrameworkElement`, so most consumers see no behavior change.
