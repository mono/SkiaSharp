# Version 3.119.0

> **Skia m119 stable** · Released April 25, 2025 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.0)

> **Supersedes [3.118.0](3.118.0.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](3.119.0/index.md) · [HarfBuzzSharp 8.3.1.1](harfbuzzsharp/8.3.1.1.md)

## Highlights

**3.119.0** stabilizes the Skia m117 → m119 milestone bumps and rolls up the preview-only 3.118.0 work that never shipped standalone. This release adds Direct3D on Windows, `SkCanvas::SaveLayerRec`, `SKColorFilter`s that were missing, and Metal-backed `GRBackendRenderTarget` on every platform. It broadens the Linux native matrix (LoongArch64, RISC-V 64, ARM fixes), enables GL on Windows ARM, and picks up 10 community fixes. HarfBuzz 8.3.1.1 ships alongside — see [its release notes](harfbuzzsharp/8.3.1.1.md) for the binding details.

## New APIs

- **`SKCanvas.SaveLayerRec`** — Wire up `SkCanvas::SaveLayerRec` so callers can drive `SaveLayer` with the full parameter record instead of the trimmed overloads (❤️ [@ahmed605](https://github.com/ahmed605) in [#2962](https://github.com/mono/SkiaSharp/pull/2962), [mono/skia#130](https://github.com/mono/skia/pull/130)).
- **Missing `SKColorFilter` types** — Add the color-filter factories that were previously unbound so the color-filter surface matches Skia's shipping set ([#2882](https://github.com/mono/SkiaSharp/pull/2882)).
- **`SKMetalView` on tvOS** — Bring `SKMetalView` support to tvOS to match the existing iOS/Mac Catalyst surface (❤️ [@MartinZikmund](https://github.com/MartinZikmund) in [#3114](https://github.com/mono/SkiaSharp/pull/3114)).
- **Metal `GRBackendRenderTarget`** — Expose the Metal-backed constructor on every platform SkiaSharp is built for, unifying the Metal integration surface (❤️ [@spouliot](https://github.com/spouliot) in [#3070](https://github.com/mono/SkiaSharp/pull/3070)).

## Fixes

- **`SKImage.FromPicture`** — Fix the implementation so drawn pictures survive round-tripping into images ([#3231](https://github.com/mono/SkiaSharp/pull/3231), [mono/skia#156](https://github.com/mono/skia/pull/156)).
- **.NET Framework packages.config regression** — Restore compatibility for projects using .NET Framework/Mono with `packages.config` (❤️ [@sungaila](https://github.com/sungaila) in [#3112](https://github.com/mono/SkiaSharp/pull/3112)).
- **iOS Simulator Metal performance** — Fix the Metal path on the iOS Simulator so it no longer degrades to a slow path (❤️ [@taublast](https://github.com/taublast) in [#3156](https://github.com/mono/SkiaSharp/pull/3156)).
- **Incorrect internal call** — Fix an incorrect P/Invoke call that mis-routed a Skia entry point (❤️ [@kkwpsv](https://github.com/kkwpsv) in [#3143](https://github.com/mono/SkiaSharp/pull/3143)).
- **.NET 9 Blazor** — Fix ASP.NET Blazor host issues that surfaced on .NET 9, including the WASM initialization path and the Blazor base-path import ([#3064](https://github.com/mono/SkiaSharp/pull/3064), [#3092](https://github.com/mono/SkiaSharp/pull/3092)).
- **Blazor/WASM decoupling** — Move the Blazor and WASM workarounds into SkiaSharp so consumers no longer have to copy them into their own projects ([#3088](https://github.com/mono/SkiaSharp/pull/3088), [#3082](https://github.com/mono/SkiaSharp/pull/3082)).
- **Windows SDK pinning** — Use `<WindowsSdkPackageVersion>` to pin the Windows SDK so mismatched host SDKs no longer break the build ([#3085](https://github.com/mono/SkiaSharp/pull/3085)).

## Platform support

- **Skia m117 → m119** — Sync the Skia submodule through milestones 117, 118 and 119 ([#3047](https://github.com/mono/SkiaSharp/pull/3047), [#3048](https://github.com/mono/SkiaSharp/pull/3048), [#3062](https://github.com/mono/SkiaSharp/pull/3062), [mono/skia#138](https://github.com/mono/skia/pull/138), [mono/skia#139](https://github.com/mono/skia/pull/139), [mono/skia#148](https://github.com/mono/skia/pull/148)).
- **Direct3D on Windows** — Compile Skia with Direct3D on Windows so D3D-backed rendering becomes an option (❤️ [@Kation](https://github.com/Kation) in [#2823](https://github.com/mono/SkiaSharp/pull/2823), [mono/skia#121](https://github.com/mono/skia/pull/121)).
- **Windows ARM GL** — Enable GL on Windows ARM so ARM64 desktop hosts can drive GPU rendering ([#3189](https://github.com/mono/SkiaSharp/pull/3189), [mono/skia#151](https://github.com/mono/skia/pull/151)).
- **LoongArch64** — Add native builds for Loongson-based Linux distributions (❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) in [#3198](https://github.com/mono/SkiaSharp/pull/3198)).
- **RISC-V 64** — Add native builds for `riscv64` Linux (❤️ [@kasperk81](https://github.com/kasperk81) in [#3192](https://github.com/mono/SkiaSharp/pull/3192)).
- **Alpine cross-compile** — Extend the `clang-cross` build to support Alpine (❤️ [@kasperk81](https://github.com/kasperk81) in [#3200](https://github.com/mono/SkiaSharp/pull/3200)) and fix Linux test wiring (❤️ [@kasperk81](https://github.com/kasperk81) in [#3194](https://github.com/mono/SkiaSharp/pull/3194)).
- **ARM Linux + tvOS packaging** — Fix Linux ARM native targets ([#3190](https://github.com/mono/SkiaSharp/pull/3190)) and set the correct tvOS target ([#3199](https://github.com/mono/SkiaSharp/pull/3199), [mono/skia#153](https://github.com/mono/skia/pull/153)).
- **CVE-2024-30105** — Pull in the upstream fix for CVE-2024-30105 (❤️ [@pjcollins](https://github.com/pjcollins) in [#3166](https://github.com/mono/SkiaSharp/pull/3166)).
- **vulkanmemoryallocator 3.2.1** — Bump the bundled VMA to 3.2.1 (❤️ [@kasperk81](https://github.com/kasperk81) in [#3196](https://github.com/mono/SkiaSharp/pull/3196), [mono/skia#152](https://github.com/mono/skia/pull/152)).
- **Native dependency bumps** — Refresh libpng to 1.6.44 ([#3059](https://github.com/mono/SkiaSharp/pull/3059), [mono/skia#146](https://github.com/mono/skia/pull/146)), freetype2 to 2.13.3+ ([#3061](https://github.com/mono/SkiaSharp/pull/3061), [mono/skia#143](https://github.com/mono/skia/pull/143)), libexpat to 2.6.4 ([#3056](https://github.com/mono/SkiaSharp/pull/3056), [mono/skia#142](https://github.com/mono/skia/pull/142)), libjpeg-turbo to 2.1.5.1+commits ([#3060](https://github.com/mono/SkiaSharp/pull/3060), [mono/skia#141](https://github.com/mono/skia/pull/141)) and HarfBuzz to 8.3.1 ([#3052](https://github.com/mono/SkiaSharp/pull/3052), [mono/skia#140](https://github.com/mono/skia/pull/140)).
- **Reduced unused dependencies** — Trim unused Skia externals and disable unused/problematic tools to slim the native surface ([#3171](https://github.com/mono/SkiaSharp/pull/3171), [#3180](https://github.com/mono/SkiaSharp/pull/3180), [#3186](https://github.com/mono/SkiaSharp/pull/3186), [#3187](https://github.com/mono/SkiaSharp/pull/3187), [#3150](https://github.com/mono/SkiaSharp/pull/3150), [mono/skia#149](https://github.com/mono/skia/pull/149), [mono/skia#150](https://github.com/mono/skia/pull/150)).
- **Android NDK r27c + 16 KB alignment** — Rebase Android on NDK r27c with 16 KB page-size alignment ([#3096](https://github.com/mono/SkiaSharp/pull/3096)) and correct the NuGet `PackagePath` for runtime assets ([#3100](https://github.com/mono/SkiaSharp/pull/3100)).
