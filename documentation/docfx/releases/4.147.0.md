# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

SkiaSharp 4.147.0 marks the launch of the **4.x series**, powered by the Skia graphics engine at milestone 147 (Chrome 147) — a 14-milestone leap from the previous 3.x baseline. This release delivers complete **OpenType variable font support** (design axes, named instances, allocation-free `Span` APIs), **COLRv1 color font palettes** with per-entry override control, a critical **Android API 36 crash fix**, and surface-level sampling options for high-quality rendering. Four community contributors drove the most impactful work, led by [@ramezgerges](https://github.com/ramezgerges) and [@4Darmygeometry](https://github.com/4Darmygeometry).

## Breaking Changes

The Skia engine upgrade from m133 to m147 and the managed typeface refactor introduce the following changes:

- **`SKPath` is now immutable** — Path mutation methods have been removed. Use `SKPathBuilder` to construct paths and call `Detach()` to get an immutable `SKPath`. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **`SKPath.Rewind()` removed** — Use `SKPathBuilder.Reset()` instead. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **`SKPaint.GetFillPath` dst parameter changed** — The destination parameter is now `SKPathBuilder` instead of `SKPath`. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **`SKPathMeasure.GetSegment` dst parameter changed** — The destination parameter is now `SKPathBuilder` instead of `SKPath`. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **`SKColorType` enum values shifted** — Do not rely on numeric values; use named members. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **Encoder option ICC profile fields removed** — `SKJpegEncoderOptions` and related structs no longer accept ICC profile constructor parameters. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **7 native C API symbols removed** — `sk_typeface_create_default`, `sk_typeface_ref_default`, `sk_typeface_create_from_name`, `sk_typeface_create_from_file`, `sk_typeface_create_from_stream`, `sk_typeface_create_from_data`, and `sk_fontmgr_ref_default` have been removed from the native ABI. Native-level consumers must migrate to `sk_fontmgr_*` entry points. ([#3730](https://github.com/mono/SkiaSharp/pull/3730))

## New Features

### Engine

- **Skia milestone 147** — Upgrades the underlying Skia engine from m133 to m147, a 14-milestone jump spanning Chrome 133 through Chrome 147. Includes `SKColorType.R16Unorm` and many upstream rendering improvements. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3702](https://github.com/mono/SkiaSharp/pull/3702))

### Text & Fonts

- **Variable font support** — Full OpenType variable font API: query all design axes via `SKTypeface.VariationDesignParameters`, read the current axis positions via `VariationDesignPosition`, and clone a typeface to any position with `SKTypeface.Clone(position)` or `Clone(SKFontArguments)`. Allocation-free `Span` overloads for every query API. HarfBuzzSharp `Face` gains `VariationAxisInfos`, `NamedInstanceCount`, named instance queries, and `TryFindVariationAxis`; `Font` gains `SetVariations`, `SetVariationCoordsDesign`, and `SetVariationNamedInstance`. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palettes** — `SKTypeface.Clone(paletteIndex)` and `Clone(SKFontArguments)` let you select and override COLRv1 color font palettes at runtime via `SKFontArguments.PaletteIndex` and `PaletteOverrides`. HarfBuzzSharp `Face` exposes `HasPalettes`, `PaletteCount`, `GetPaletteColors`, `GetPaletteFlags`, `GetPaletteNameId`, `HasColorLayers`, `HasColorPng`, and `HasColorSvg`. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))

### API Surface

- **`SKSurface.Draw` sampling options** — New `SKSurface.Draw` and `SKCanvas.DrawSurface` overloads accept `SKSamplingOptions` for bilinear, bicubic, or mipmap-filtered rendering. ([#3491](https://github.com/mono/SkiaSharp/pull/3491))

### Platform

- **Tizen x64 and ARM64 support** — Native libraries now ship for `tizen-x64` and `tizen-arm64` in addition to the existing `tizen-armel` and `tizen-x86` targets. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **C# 13 language features on legacy TFMs** — PolySharp 1.15.0 is included as a build-time dependency, enabling modern C# 13 language features on `netstandard2.0`, `netstandard2.1`, and `net462`. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))

## Bug Fixes

- **Android API 36 crash at static initialization** — Fixes a `SIGSEGV` at `sk_typeface_ref_default+44` that crashed any app touching an `SKObject`-derived type on Android 16 (API 36). Default typeface and font manager resolution is now handled entirely in managed code, eliminating the crash and ensuring `SKTypeface.FromFamilyName` always returns a usable typeface instead of potentially `null`. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))
- **Windows crash on systems without DirectX 12** — Delay-loads `d3d12.dll`, `dxgi.dll`, and `D3DCOMPILER_47.dll` so those DLLs are only loaded when the GPU backend is actually used. Fixes a crash that affected approximately 0.1% of installations on Windows 7/8, Windows Server 2008, and some Windows 10 machines without DX12. ([#3633](https://github.com/mono/SkiaSharp/pull/3633))
- **`SKGLView` not rendering in Android `TabBar`** — Fixed `SKGLView` on Android not being rendered when hosted inside a `TabBar`. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))

## Security

- **libpng 1.6.58** — Updates the bundled libpng from 1.6.54 to 1.6.58. ([#3718](https://github.com/mono/SkiaSharp/pull/3718))
- **libexpat 2.7.5** — Updates the bundled libexpat from 2.7.3 to 2.7.5. ([#3717](https://github.com/mono/SkiaSharp/pull/3717))
