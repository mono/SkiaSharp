# Version 4.147.0

> **Preview only** · Superseded by 4.148.0 · Never released as stable — these changes rolled up into 4.148.0 · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.147.0-preview)

> **API changes** · [SkiaSharp API diff](4.147.0/index.md) · [HarfBuzzSharp 8.3.1.6](harfbuzzsharp/8.3.1.6.md)

## Highlights

SkiaSharp 4.0 — the library's first major version release — launched in this preview cycle, advancing the Skia engine through milestones 132, 133, and 147. The headline features are full variable font support (in both SkiaSharp and HarfBuzzSharp, shipping alongside as [HarfBuzzSharp 8.3.1.6](harfbuzzsharp/8.3.1.6.md)), color font palette queries, animated WebP encoding, and zero-copy stream-to-data access. Platform coverage expanded to Linux Bionic and Tizen x64/arm64, and a redesigned Blazor gallery introduced an API Atlas with tag-cloud filtering. This preview was never released as stable — all changes rolled up into [4.148.0](4.148.0.md). Six community contributors drove many of the user-facing changes.

## Breaking Changes

This is a **major version upgrade from 3.x to 4.x**. Breaking API changes are expected; see the [SkiaSharp API diff](4.147.0/index.md) for the complete surface. ([#3640](https://github.com/mono/SkiaSharp/pull/3640))

## Engine

- **Skia milestones 132, 133, and 147** — Advances through three Skia engine milestones, bringing the latest upstream graphics improvements. ❤️ [@ramezgerges](https://github.com/ramezgerges) contributed the m132 and m147 bumps. ([#3560](https://github.com/mono/SkiaSharp/pull/3560), [#3660](https://github.com/mono/SkiaSharp/pull/3660), [#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **Native dependency security updates** — libpng updated to 1.6.58 ([#3718](https://github.com/mono/SkiaSharp/pull/3718)), libexpat to 2.7.5 ([#3717](https://github.com/mono/SkiaSharp/pull/3717)), freetype to 2.14.3 ([#3726](https://github.com/mono/SkiaSharp/pull/3726)), zlib to 1.3.2.1-motley ([#3720](https://github.com/mono/SkiaSharp/pull/3720)), and libjpeg-turbo to 3.1.4.1 ([#4012](https://github.com/mono/SkiaSharp/pull/4012)).

## Text & Fonts

- **Variable font support** — Full variable font axis enumeration and value queries in both SkiaSharp and HarfBuzzSharp. See [HarfBuzzSharp 8.3.1.6](harfbuzzsharp/8.3.1.6.md) for HarfBuzz binding details. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette support** — Query OpenType CPAL palettes and set active palette entries for color fonts, plus improved variable font robustness. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct** — Proper managed struct for `hb_color_t`, replacing ad-hoc integer representations in HarfBuzz color queries. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Default typeface resolution** — Moved default-typeface resolution to the managed layer, fixing font fallback behaviour on platforms without a system font manager. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))

## API Surface

- **`SKStream.GetData()`** — Zero-copy conversion from `SKStream` to `SKData`, avoiding a redundant memory allocation. ([#3772](https://github.com/mono/SkiaSharp/pull/3772))
- **Animated WebP encoding** — New `SKWebpEncoder` produces animated WebP output from a sequence of frames. ([#3771](https://github.com/mono/SkiaSharp/pull/3771))
- **`SKSamplingOptions` for surface drawing** — `SKSurface.Draw` and `SKCanvas.DrawSurface` now accept `SKSamplingOptions`, enabling filtering control when blitting surfaces. ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))
- **Missing API wrappers** — Added missing wrappers and fixed several naming-convention violations across the binding layer. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))
