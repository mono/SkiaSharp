# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

SkiaSharp 4.147.0 is the first release under the new **version 4** major version, powered by the **Skia graphics engine milestone 147** — a 14-milestone jump from m133 covering upstream improvements from Chrome 133 through Chrome 147. This release brings full **variable font support** (axis queries, runtime weight/style control, allocation-free Span APIs) and **color font palette support** (COLRv1, palette selection, per-entry overrides). A critical Android API 36 crash at startup is fixed by moving default-typeface resolution to the managed layer. Community contributor [@ramezgerges](https://github.com/ramezgerges) delivered the Skia engine upgrade, variable font APIs, and the Android crash fix.

## ⚠️ Breaking Changes

- **`SKPath` is now immutable** — Construction and mutation methods have been removed from `SKPath`. Use `SKPathBuilder` to build paths, then call `Detach()` to get an immutable `SKPath`. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **`SKPath.Rewind()` removed** — Use `SKPathBuilder.Reset()` instead. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **`SKPaint.GetFillPath` destination parameter** — Now accepts `SKPathBuilder` instead of `SKPath`. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **`SKPathMeasure.GetSegment` destination parameter** — Now accepts `SKPathBuilder` instead of `SKPath`. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **`SKColorType` enum values shifted** — Use named members rather than numeric values. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **Encoder option ICC profile fields removed** — Remove ICC profile arguments from encoder option constructors. ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **Default typeface resolution moved to managed layer** — 7 native C API symbols removed (`sk_typeface_create_default`, `sk_typeface_ref_default`, `sk_typeface_create_from_name`, `sk_typeface_create_from_file`, `sk_typeface_create_from_stream`, `sk_typeface_create_from_data`, `sk_fontmgr_ref_default`). External C/C++ consumers must migrate to `sk_fontmgr_*` entry points. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))
- **`SKPaint()` default constructor** — The backing default `SkFont` now wraps `SkTypeface::MakeEmpty()`. Apps using the obsolete `SKPaint` text APIs without an explicit `SKFont` will render no glyphs until migrated to `SKFont`. ([#3730](https://github.com/mono/SkiaSharp/pull/3730))

## New Features

### Engine

- **Skia milestone 147** — Updates the Skia graphics engine from m133 to m147, a 14-milestone jump covering upstream improvements in rendering, font handling, GPU, and image codecs from Chrome 133 through Chrome 147. Adds `SKColorType.R16Unorm`. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3702](https://github.com/mono/SkiaSharp/pull/3702))

### Text & Fonts

- **Variable font support** — Full OpenType variable font APIs for both SkiaSharp and HarfBuzzSharp. Query variation axes (`VariationDesignParameters`), read/set design positions, clone typefaces with new axis values, and select named instances — all with allocation-free `Span` overloads. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette support** — New `SKTypeface.Clone(int paletteIndex)` and `Clone(SKFontArguments)` overloads enable selecting color palettes on COLRv1 fonts with optional per-entry overrides. HarfBuzzSharp gains `Face.PaletteCount`, `GetPaletteColors`, palette flags, name IDs, and color-capability detection (`HasColorLayers`, `HasColorPng`, `HasColorSvg`). ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`SKFontArguments` ref struct** — New `SKFontArguments` for composing typeface clone arguments: variation design position, collection index, palette index, and palette overrides. ([#3703](https://github.com/mono/SkiaSharp/pull/3703), [#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`SKFourByteTag` struct** — New `SKFourByteTag` wrapping Skia's four-byte font tag with `Parse`, `ToString`, and implicit `uint` conversion. ([#3703](https://github.com/mono/SkiaSharp/pull/3703))

### API Surface

- **`SKSamplingOptions` on `SKSurface.Draw` and `SKCanvas.DrawSurface`** — New overloads accepting `SKSamplingOptions` allow callers to control filtering quality when compositing surfaces. ([#3491](https://github.com/mono/SkiaSharp/pull/3491))

## Security

- **libexpat updated to 2.7.5** — Bumps the bundled libexpat from 2.7.3. ([#3717](https://github.com/mono/SkiaSharp/pull/3717))
- **libpng updated to 1.6.58** — Bumps the bundled libpng from 1.6.54. ([#3718](https://github.com/mono/SkiaSharp/pull/3718))

## Bug Fixes

- **Fixed Android API 36 startup crash** — `SIGSEGV` at `sk_typeface_ref_default+44` on Android 16 (API 36) during static initialization is resolved by resolving the default typeface entirely in managed code via `SKFontManager.Default`. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))
- **Fixed `SKTypeface.ContainsGlyphs` infinite recursion** — Pre-existing `StackOverflowException` caused by a method calling itself instead of the underlying font resolved. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **Fixed `SKTypeface.FromFamilyName` returning null** — Now always returns a usable typeface (falls back to `SKTypeface.Default`) instead of potentially returning null on Android API 36 or for typo'd family names. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))
