# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This upcoming release brings major infrastructure improvements: content-based build caching, an automated Skia upstream sync workflow, and agentic workflow data persistence. Community contributor ❤️ [@ramezgerges](https://github.com/ramezgerges) fixed SkiaSharp references in the Uno Platform Gallery sample.

## Breaking Changes

*None in this release.*

## New Features

### 🏗️ Build & CI

- **Content-based build caching** — Integrates `ADO Cache@2` to cache build artifacts based on content hashes, significantly speeding up repeat builds. ([#3826](https://github.com/mono/SkiaSharp/pull/3826))
- **Automated Skia upstream sync** — Adds a new workflow that automatically tracks and merges upstream Skia changes, reducing manual effort for milestone updates. ([#3825](https://github.com/mono/SkiaSharp/pull/3825))
- **Agentic workflow data persistence** — Persists agentic workflow outputs to a dedicated `aw-data` branch for improved traceability. ([#3907](https://github.com/mono/SkiaSharp/pull/3907))
- **Raw PR data preserved in release notes** — Release notes files now retain raw PR data as an HTML comment, enabling AI-assisted regeneration from git history. ([#3877](https://github.com/mono/SkiaSharp/pull/3877))
