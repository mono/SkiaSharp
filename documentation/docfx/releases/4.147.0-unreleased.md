# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This release syncs the Skia engine to the chrome/m147 milestone, bringing upstream bug fixes
directly from Google. HarfBuzz is updated to 14.2.0 for improved text shaping, a WinUI
projection DLL issue affecting .NET 9 consumers is resolved, and Linux builds now use .NET's
official cross-compilation Docker images for a more reliable build pipeline.

## Breaking Changes

*None in this release.*

## New Features

### Engine

- **Skia chrome/m147 upstream sync** — Merges bug fixes from Google's chrome/m147 branch into the SkiaSharp Skia fork. ([#4044](https://github.com/mono/SkiaSharp/pull/4044))

### Text & Fonts

- **HarfBuzz updated to 14.2.0** — Upgrades the text shaping engine to HarfBuzz 14.2.0, picking up bug fixes and broader script support. ([#4035](https://github.com/mono/SkiaSharp/pull/4035))

## Bug Fixes

- **WinUI Projection DLL not found on .NET 9** — Fixes a missing DLL reference that prevented WinUI apps targeting .NET 9 from loading correctly. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))

## Platform Support

| Platform | What's New |
|----------|-----------|
| 🪟 Windows | Fixed WinUI Projection DLL resolution for .NET 9 |
| 🐧 Linux | Migrated to .NET official cross-compilation Docker images |

## Infrastructure

- **Linux Docker cross-compilation images** — Linux native builds now use the official .NET cross-compilation images, improving consistency and reducing maintenance overhead. ([#4062](https://github.com/mono/SkiaSharp/pull/4062))
- **Device and WASM test migration** — Tests migrated from XHarness to DeviceRunners `dotnet test`, simplifying the test pipeline. ([#4004](https://github.com/mono/SkiaSharp/pull/4004))

*Plus several CI and documentation improvements.*
