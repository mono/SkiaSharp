# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This release expands the interactive sample gallery with five new demos and a redesigned Blazor home page, making it easier to explore SkiaSharp's capabilities. Apple platform builds now correctly use versioned TFMs for libraries and unversioned TFMs for apps. Community contributor [@ramezgerges](https://github.com/ramezgerges) delivered SkiaFiddle UX improvements and a Uno gallery fix.

## Breaking Changes

*None in this release.*

## New Features

### Gallery & Samples

- **Five new interactive samples** — Adds five new interactive demos to the Gallery, covering a range of SkiaSharp drawing features, alongside a redesigned Blazor home page for better discoverability. ([#3757](https://github.com/mono/SkiaSharp/pull/3757))
- **SkiaFiddle UX improvements** — Polishes the SkiaFiddle interactive experience with a set of UX refinements. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3790](https://github.com/mono/SkiaSharp/pull/3790))

## Bug Fixes

- **Apple platform TFM correction** — Libraries now target versioned Apple TFMs (e.g., `net9.0-ios26.0`) while app projects continue to use unversioned TFMs, resolving build warnings and ensuring correct framework resolution. ([#3798](https://github.com/mono/SkiaSharp/pull/3798))
- **Skottie package downgrade in Uno gallery** — Fixes a `NU1605` package downgrade warning for the Skottie dependency in the Uno sample gallery. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3785](https://github.com/mono/SkiaSharp/pull/3785))
