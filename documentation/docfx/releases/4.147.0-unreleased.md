# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This release focuses on developer infrastructure: an automated API docs pipeline, content-based build caching, and automated Skia upstream sync. Community contributor ❤️ [@ebariche](https://github.com/ebariche) delivered a significant **~60% reduction in SkiaFiddle WASM application size**, and ❤️ [@ramezgerges](https://github.com/ramezgerges) fixed SkiaSharp references in the Uno Platform Gallery sample.

## Breaking Changes

*None in this release.*

## New Features

### WebAssembly

- **SkiaFiddle WASM size reduced by ~60%** — Significantly reduces the download size of the SkiaFiddle interactive sample application, improving load times for browser-based demos. ❤️ [@ebariche](https://github.com/ebariche) ([#3849](https://github.com/mono/SkiaSharp/pull/3849))

### Build & Infrastructure

- **Automated API docs pipeline** — Adds a fully automated pipeline for generating and publishing API documentation, ensuring docs stay in sync with code changes. ([#3918](https://github.com/mono/SkiaSharp/pull/3918))
- **Content-based build caching** — Integrates ADO `Cache@2` for content-addressed build caching, reducing CI build times significantly. ([#3826](https://github.com/mono/SkiaSharp/pull/3826))
- **Automated Skia upstream sync workflow** — Adds a workflow to automatically track and sync changes from the upstream Skia repository. ([#3825](https://github.com/mono/SkiaSharp/pull/3825))

## Bug Fixes

- **Fixed SkiaSharp references for Uno Platform Gallery sample** — Corrects broken references in the Uno Platform Gallery sample project. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3867](https://github.com/mono/SkiaSharp/pull/3867))
