# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This release brings significant build infrastructure improvements: content-based caching with ADO Cache@2 slashes CI times, and a new automated Skia upstream sync workflow keeps SkiaSharp current with upstream changes. Community contributor [@ramezgerges](https://github.com/ramezgerges) fixed SkiaSharp references for the Uno Platform Gallery sample.

## Breaking Changes

*None in this release.*

## New Features

### Build & CI

- **Content-based build caching** — Adds ADO Cache@2 integration to cache build artifacts based on content hashes, significantly reducing CI build times. ([#3826](https://github.com/mono/SkiaSharp/pull/3826))
- **Automated Skia upstream sync workflow** — New agentic workflow that automatically detects and integrates upstream Skia changes. ([#3825](https://github.com/mono/SkiaSharp/pull/3825))
- **Agentic workflow data persistence** — Adds a workflow to persist agentic workflow data to the `aw-data` branch for traceability. ([#3907](https://github.com/mono/SkiaSharp/pull/3907))
- **Daily stale staging sweep** — Changes the stale staging cleanup job from weekly to daily to keep the environment tidy. ([#3911](https://github.com/mono/SkiaSharp/pull/3911))

### Samples

- **Uno Platform Gallery sample fix** — Corrects SkiaSharp references in the Uno Platform Gallery sample so it builds cleanly. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3867](https://github.com/mono/SkiaSharp/pull/3867))
