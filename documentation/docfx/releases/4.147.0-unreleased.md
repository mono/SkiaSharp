# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

Two community contributions from [@ramezgerges](https://github.com/ramezgerges) drive the headline changes: a reworked singleton lifecycle that improves correctness and resource management, and a cleanup of long-deprecated `SKPaint` text/font members that are now compile errors. The Skia engine received two rounds of upstream chrome/m147 bug fixes, harfbuzz was bumped to 14.2.0, and expat was updated to 2.8.1 for a security fix.

## ⚠️ Breaking Changes

- **`SKPaint` obsolete text/font members are now errors** — Members deprecated in earlier releases (`TextSize`, `Typeface`, `IsLinearText`, etc.) are now `[Obsolete(error: true)]`. Code that previously compiled with warnings will now fail to compile. Migrate to the `SKFont`-based API. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))

## New Features

### Engine

- **Skia chrome/m147 upstream sync** — Two rounds of upstream bug fixes from Chrome milestone 147 merged into the Skia fork, improving stability across rendering and GPU paths. ([#4081](https://github.com/mono/SkiaSharp/pull/4081), [#4044](https://github.com/mono/SkiaSharp/pull/4044))
- **HarfBuzz updated to 14.2.0** — Picks up the latest text shaping improvements and bug fixes. ([#4035](https://github.com/mono/SkiaSharp/pull/4035))
- **expat updated to 2.8.1** — Security maintenance update for the XML parsing library. ([#4079](https://github.com/mono/SkiaSharp/pull/4079))

### API Surface

- **Reworked singleton instance lifecycle** — Internal singleton objects (e.g., `SKColorSpace`, `SKFont`, `SKTypeface`) are now managed with a cleaner lifecycle, preventing premature disposal and improving correctness in edge cases. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))

## Bug Fixes

- **`SKPixmap.GetPixelSpan<T>(x, y)` byte offset** — Fixed an off-by-one calculation that used `Height` instead of `Width` when computing the pixel offset, causing incorrect pixel reads for non-square images. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **WinUI Projection DLL not found on .NET 9** — Fixed a packaging issue where `Microsoft.WinUI` projection DLLs were missing for .NET 9 consumers of the WinUI view. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
