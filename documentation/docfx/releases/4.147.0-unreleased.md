# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This development snapshot introduces a breaking API cleanup — obsolete `SKPaint` text and font members are now compile-time errors, nudging consumers to migrate to `SKFont`-based APIs. The release also picks up two rounds of upstream Skia m147 bug fixes, updates expat and harfbuzz for improved security, and fixes a WinUI Projection DLL resolution failure for .NET 9 consumers. Community contributor [@ramezgerges](https://github.com/ramezgerges) drove the API cleanup.

## ⚠️ Breaking Changes

- **`SKPaint` text/font obsolete members are now errors** — Members previously marked `[Obsolete]` with a warning are now `[Obsolete(error: true)]`. Projects that referenced deprecated text or font APIs on `SKPaint` will fail to compile and must migrate to the `SKFont`-based equivalents. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))

## New Features

### Engine

- **Skia m147 upstream bug fixes (round 2)** — Merges the latest chrome/m147 bug fixes from upstream Skia into the SkiaSharp fork, improving rendering correctness and stability. ([#4081](https://github.com/mono/SkiaSharp/pull/4081))
- **Skia m147 upstream bug fixes** — Earlier batch of chrome/m147 upstream fixes. ([#4044](https://github.com/mono/SkiaSharp/pull/4044))

### Security

- **expat updated to 2.8.1** — Bumps the XML parsing library used internally to the latest release, addressing security advisories. ([#4079](https://github.com/mono/SkiaSharp/pull/4079))
- **HarfBuzz updated to 14.2.0** — Upgrades the text shaping engine to the latest version. ([#4035](https://github.com/mono/SkiaSharp/pull/4035))

### Platform

- **WinUI Projection DLL resolution fixed for .NET 9** — Fixes a `DllNotFoundException` for the WinUI Projection DLL that affected .NET 9 consumers of the WinUI target. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
