# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This release brings native dependency security updates (expat 2.8.1 and HarfBuzz 14.2.0), two rounds of Skia engine bug fixes from upstream Chrome m147, and a critical fix for WinUI consumers on .NET 9. The Linux build infrastructure has been modernised to use .NET's official cross-compilation Docker images.

## Breaking Changes

*None in this release.*

## New Features

### Engine

- **Skia m147 upstream bug fixes (round 1)** — Merges upstream Chrome m147 fixes from the Skia repository into SkiaSharp. ([#4044](https://github.com/mono/SkiaSharp/pull/4044))
- **Skia m147 upstream bug fixes (round 2)** — A second batch of upstream Chrome m147 fixes applied to the Skia submodule. ([#4081](https://github.com/mono/SkiaSharp/pull/4081))
- **Skia submodule: disable unused dependency manifests** — Reduces build noise by disabling unused dependency manifests in the Skia submodule. ([#4054](https://github.com/mono/SkiaSharp/pull/4054))

## Bug Fixes

- **WinUI Projection DLL not found on .NET 9** — Fixes a missing-DLL error for WinUI consumers targeting .NET 9, where the projection DLL was not being located correctly. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))

## Security

- **expat updated to 2.8.1** — Bumps the libexpat native dependency to 2.8.1, picking up the latest security and stability fixes. ([#4079](https://github.com/mono/SkiaSharp/pull/4079))
- **HarfBuzz updated to 14.2.0** — Upgrades the HarfBuzz text-shaping library to 14.2.0, incorporating upstream security and bug fixes. ([#4035](https://github.com/mono/SkiaSharp/pull/4035))
