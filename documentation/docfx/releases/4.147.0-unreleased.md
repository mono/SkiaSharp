# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This upcoming release brings major infrastructure improvements: content-based build caching, an automated Skia upstream sync workflow, and a significant reduction in WASM application size. Community contributors drove the WASM size optimization and a sample compatibility fix for Uno Platform. The build and CI pipeline has been substantially modernized.

## Breaking Changes

*None in this release.*

## New Features

### 🌐 WebAssembly

- **SkiaFiddle WASM app size reduced by ~60%** — Optimized the SkiaFiddle sample application to dramatically reduce its WebAssembly bundle size through asset trimming and build configuration improvements. ❤️ [@ebariche](https://github.com/ebariche) ([#3849](https://github.com/mono/SkiaSharp/pull/3849))

### 🏗️ Build & CI

- **Content-based build caching** — Adds ADO `Cache@2` task integration so builds skip redundant work when inputs haven't changed, speeding up CI turnaround. ([#3826](https://github.com/mono/SkiaSharp/pull/3826))
- **Automated Skia upstream sync workflow** — Introduces a new workflow to keep the Skia submodule synchronized with upstream automatically. ([#3825](https://github.com/mono/SkiaSharp/pull/3825))
- **Agentic workflow data persistence** — Adds a workflow to persist agentic workflow data to the `aw-data` branch for improved traceability. ([#3907](https://github.com/mono/SkiaSharp/pull/3907))
- **Daily stale staging sweep** — Increases the frequency of the stale staging cleanup from weekly to daily. ([#3911](https://github.com/mono/SkiaSharp/pull/3911))

## Bug Fixes

- **Fixed SkiaSharp references for Uno Platform Gallery sample** — Corrects package references so the Uno Platform Gallery sample builds correctly against recent SkiaSharp versions. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3867](https://github.com/mono/SkiaSharp/pull/3867))
