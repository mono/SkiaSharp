# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This release brings Skia chrome/m147 upstream bug fixes, a harfbuzz update to 14.2.0, and a fix for WinUI applications failing to load the projection DLL under .NET 9. Linux builds have been migrated to .NET's official cross-compilation Docker images for more reliable and maintainable builds.

## Breaking Changes

*None in this release.*

## New Features

### Engine

- **Skia chrome/m147 upstream bug fixes** — Merges upstream bug fixes from the Skia chrome/m147 branch, keeping SkiaSharp's native engine up to date with the latest stability improvements. ([#4044](https://github.com/mono/SkiaSharp/pull/4044))
- **Disable unused Skia dependency manifests** — Updates the Skia submodule to disable unused dependency manifests, reducing unnecessary overhead in the build process. ([#4054](https://github.com/mono/SkiaSharp/pull/4054))
- **HarfBuzz 14.2.0** — Updates the bundled HarfBuzz text shaping library to version 14.2.0, picking up the latest bug fixes and improvements. ([#4035](https://github.com/mono/SkiaSharp/pull/4035))

### Platform

- **Fix WinUI Projection DLL not found on .NET 9** — Resolves a runtime failure where the WinUI projection DLL could not be located when consuming SkiaSharp in .NET 9 WinUI applications. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
