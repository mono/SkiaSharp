# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This release expands the HarfBuzz API surface with a proper `HBColor` struct, fixes naming convention violations across the bindings, and reduces SkiaFiddle WASM application size by ~60%. Native dependencies freetype and zlib have been updated for improved stability. Community contributors [@ebariche](https://github.com/ebariche) and [@ramezgerges](https://github.com/ramezgerges) drove key improvements in this release.

## Breaking Changes

*None in this release.*

## New Features

### API Surface

- **`HBColor` struct** — Adds a dedicated `HBColor` struct for proper `hb_color_t` representation in the HarfBuzz bindings, replacing raw integer usage. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Missing API wrappers** — Adds previously missing API wrappers and corrects naming convention violations throughout the binding layer, improving API completeness and consistency. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))

### WebAssembly

- **SkiaFiddle WASM size reduction (~60%)** — Significantly reduces the SkiaFiddle WebAssembly application size through build optimizations, making the interactive demo load much faster. ❤️ [@ebariche](https://github.com/ebariche) ([#3849](https://github.com/mono/SkiaSharp/pull/3849))

### Platform

- **Tizen SDK 10.0** — Updates the Tizen SDK from 6.1 to 10.0, keeping Tizen support aligned with the latest toolchain. ([#3743](https://github.com/mono/SkiaSharp/pull/3743))

### Native Dependencies

- **freetype 2.14.3** — Updates the bundled FreeType library to 2.14.3 for bug fixes and improved font rendering reliability. ([#3726](https://github.com/mono/SkiaSharp/pull/3726))
- **zlib 1.3.2.1-motley** — Updates the bundled zlib to the latest motley-patched version for stability improvements. ([#3720](https://github.com/mono/SkiaSharp/pull/3720))

## Bug Fixes

- **Uno Platform Gallery sample references** — Fixes incorrect SkiaSharp package references in the Uno Platform Gallery sample project. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3867](https://github.com/mono/SkiaSharp/pull/3867))

## Infrastructure

- **Automated API docs pipeline** — Adds a fully automated pipeline for generating and publishing API documentation, eliminating manual doc publishing steps. ([#3918](https://github.com/mono/SkiaSharp/pull/3918))
- **Automated Skia upstream sync** — Introduces an automated workflow to keep SkiaSharp's Skia fork in sync with upstream, reducing manual merge work. ([#3825](https://github.com/mono/SkiaSharp/pull/3825))
- **Content-based build caching** — Adds ADO `Cache@2`-based content-addressed caching to significantly speed up CI builds for unchanged native artifacts. ([#3826](https://github.com/mono/SkiaSharp/pull/3826))
- **TreatWarningsAsErrors in binding** — Enables `TreatWarningsAsErrors` across all projects under `binding/` and resolves all resulting warnings, improving long-term code quality. ([#3752](https://github.com/mono/SkiaSharp/pull/3752))
- **Hero Image gallery sample** — Adds a new Hero Image sample to the gallery demonstrating advanced image composition techniques. ([#3792](https://github.com/mono/SkiaSharp/pull/3792))
- **Blazor gallery redesign** — Redesigns the Blazor gallery with an API Atlas view and tag cloud filtering for easier sample discovery. ([#3970](https://github.com/mono/SkiaSharp/pull/3970))

Plus several CI and documentation improvements.
