# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This upcoming release focuses on developer experience and build infrastructure improvements: an automated API docs pipeline, content-based build caching for faster CI, and an automated Skia upstream sync workflow. Community contributor [@ebariche](https://github.com/ebariche) cut the SkiaFiddle WASM application size by ~60%, and [@ramezgerges](https://github.com/ramezgerges) fixed SkiaSharp references for the Uno Platform Gallery sample.

## Breaking Changes

*None in this release.*

## New Features

### 🏗️ Build & CI

- **Automated API docs pipeline** — Adds a new pipeline to automatically generate and publish API documentation. ([#3918](https://github.com/mono/SkiaSharp/pull/3918))
- **Content-based build caching** — Introduces ADO `Cache@2` task for content-based caching, significantly speeding up CI builds by avoiding redundant work. ([#3826](https://github.com/mono/SkiaSharp/pull/3826))
- **Automated Skia upstream sync** — Adds a workflow to automatically track and sync upstream Skia changes, reducing manual effort in milestone updates. ([#3825](https://github.com/mono/SkiaSharp/pull/3825))
- **Agentic workflow data persistence** — Adds a workflow to persist agentic workflow data to the `aw-data` branch for audit and review. ([#3907](https://github.com/mono/SkiaSharp/pull/3907))

### 🌐 WebAssembly

- **SkiaFiddle WASM size reduced by ~60%** — Significant reduction in WASM application download size for SkiaFiddle, improving load times for browser-based demos. ❤️ [@ebariche](https://github.com/ebariche) ([#3849](https://github.com/mono/SkiaSharp/pull/3849))

## Bug Fixes

- **Uno Platform Gallery sample references** — Fixed SkiaSharp references in the Uno Platform Gallery sample so it builds and runs correctly. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3867](https://github.com/mono/SkiaSharp/pull/3867))
