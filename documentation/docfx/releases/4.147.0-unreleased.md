# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

Community contributor [@ramezgerges](https://github.com/ramezgerges) delivered UX improvements to the SkiaFiddle sample app and fixed a Skottie package downgrade issue in the Uno gallery. The docs landing page received a fresh layout with an improved hero section and better card grid. This work builds on the 4.147.0 Preview 1 foundation.

## Breaking Changes

*None in this release.*

## New Features

### Samples

- **SkiaFiddle UX improvements** — Adds a Monaco editor focus guard for WebAssembly, updates the page layout, and enhances the code-behind logic for a smoother editing experience. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3790](https://github.com/mono/SkiaSharp/pull/3790))

## Bug Fixes

- **Fixed Skottie package downgrade NU1605 in Uno gallery** — Pins SkiaSharp.Skottie in the Uno sample project to prevent a version conflict caused by `Uno.WinUI.Lottie` pulling in an older 3.x dependency. Also updates native asset include targets for HarfBuzzSharp and SkiaSharp. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3785](https://github.com/mono/SkiaSharp/pull/3785))

## Documentation

- **Improved docs landing page layout** — Reorganized the documentation card grid into balanced rows, redesigned the hero section with SkiaFiddle as the primary CTA, and added spacing improvements for a cleaner landing experience. ([#3794](https://github.com/mono/SkiaSharp/pull/3794))

Plus several CI, workflow, and automation improvements including automated release notes generation ([#3787](https://github.com/mono/SkiaSharp/pull/3787)), the skia-analyst and sample-scout skills ([#3755](https://github.com/mono/SkiaSharp/pull/3755)), and various workflow reliability fixes.
