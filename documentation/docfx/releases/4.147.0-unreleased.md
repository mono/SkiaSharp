# Version 4.147.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This upcoming release brings a range of API surface improvements, new developer tooling, and significant infrastructure investments. Noteworthy additions include a proper `HBColor` struct for HarfBuzz color representation, a batch of missing API wrappers and naming-convention fixes, and a redesigned Blazor API Atlas gallery with tag cloud filtering. Community contributors [@sasakrsmanovic](https://github.com/sasakrsmanovic), [@ebariche](https://github.com/ebariche), and [@ramezgerges](https://github.com/ramezgerges) each landed meaningful improvements.

## Breaking Changes

*None in this release.*

## New Features

### API Surface

- **`HBColor` struct** — Adds a proper managed struct for `hb_color_t`, replacing raw integer usage and making HarfBuzz color APIs more ergonomic. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Missing API wrappers and naming fixes** — Fills in several gaps in the binding layer and corrects naming-convention violations for consistency across the API surface. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))

### Security

- **zlib updated to 1.3.2.1-motley** — Bumps the bundled zlib to the latest patched release, closing known security advisories. ([#3720](https://github.com/mono/SkiaSharp/pull/3720))

### Samples & Gallery

- **Blazor API Atlas with tag cloud filtering** — Redesigns the Blazor gallery into an interactive API Atlas where every sample is discoverable through a searchable tag cloud. ([#3970](https://github.com/mono/SkiaSharp/pull/3970))
- **Hero Image gallery sample** — Adds a new showcase sample demonstrating large-format hero image rendering techniques. ([#3792](https://github.com/mono/SkiaSharp/pull/3792))
- **SkiaFiddle WASM size reduced ~60%** — Dramatically shrinks the WebAssembly application bundle for the SkiaFiddle sample by stripping unused assets. ❤️ [@ebariche](https://github.com/ebariche) ([#3849](https://github.com/mono/SkiaSharp/pull/3849))
- **Uno Platform sample references fixed** — Corrects SkiaSharp package references in the Uno Platform Gallery sample so it builds cleanly. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3867](https://github.com/mono/SkiaSharp/pull/3867))

### Platform

- **Tizen SDK updated to 10.0** — Moves CI and tooling from the end-of-life Tizen SDK 6.1 to 10.0, keeping Tizen builds healthy. ([#3743](https://github.com/mono/SkiaSharp/pull/3743))

### Tooling & Build

- **Automated API docs pipeline** — Adds a fully automated pipeline that generates and publishes API reference documentation on every build. ([#3918](https://github.com/mono/SkiaSharp/pull/3918))
- **Warnings-as-errors in binding/** — Enables `TreatWarningsAsErrors` across the entire `binding/` project set and resolves all pre-existing warnings. ([#3752](https://github.com/mono/SkiaSharp/pull/3752))
- **Content-based build caching** — Integrates ADO `Cache@2` with content hashing so incremental CI builds skip unchanged work. ([#3826](https://github.com/mono/SkiaSharp/pull/3826))
- **Automated Skia upstream sync** — Adds a workflow that automatically proposes upstream Skia merges, keeping the fork in sync without manual intervention. ([#3825](https://github.com/mono/SkiaSharp/pull/3825))

Plus several CI and documentation improvements.
