## Highlights

SkiaSharp **4.150.0** rolls up the chrome/m150 Skia bump and a bundle of correctness fixes that came out of the 4.148.0 debut cycle — GPU context lifetime, WPF/WinUI paint stability, and canvas state management.

## Breaking Changes

None in this release. See the [SkiaSharp API diff](4.150.0/index.md) for the full surface.

## Engine & Skia

- Bump skia to milestone 150 ([#4200](https://github.com/mono/SkiaSharp/pull/4200)) (Skia mono/skia#210)

## Bug Fixes

- Fix GRContext disposal race on backend swap ([#4215](https://github.com/mono/SkiaSharp/pull/4215))
- Fix SKPaint stroke width caching on WPF ([#4220](https://github.com/mono/SkiaSharp/pull/4220))
- Fix canvas Save/Restore imbalance detection in debug builds ([#4225](https://github.com/mono/SkiaSharp/pull/4225))
- Fix Skottie animation seek precision ([#4230](https://github.com/mono/SkiaSharp/pull/4230))
