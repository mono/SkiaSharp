# Version 4.150.0

> **New image-filter APIs and blur fix** · Released July 7, 2026 · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.150.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v4.150.0)

> **API changes** · [SkiaSharp API diff](4.150.0/index.md) · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

4.150.0 moves onto Skia milestone m150 and adds several new pieces of API surface: `SKPaint.GetFastBounds()` for quick-reject culling, `SKColorFilter.CreateOverdraw()` for overdraw visualisation, and `SKImageFilter.CreateCrop()` / `CreateEmpty()`. A hot-path `SKSurface.Canvas` optimisation caches the wrapper across calls, and a small-sigma raster blur regression is fixed. This release also completes the v4 obsolete-cleanup by promoting the remaining pre-v4 obsoletes to errors and trimming obsolete enums from the reference assembly. HarfBuzzSharp 14.2.1 ships alongside — see its [linked page](harfbuzzsharp/14.2.1.md) for the binding changes.

## Breaking Changes

- **Remaining pre-v4 obsoletes promoted to errors + obsolete enums trimmed from the reference assembly** — members that had been `[Obsolete]` warnings across the 4.147.0/4.148.0 line are now `[Obsolete("…", error: true)]`, and obsolete enum values no longer appear in the reference assembly. Any consumer still using these members will fail to compile. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))
- **.NET Interactive (Polyglot Notebooks) support removed** — the deprecated `SkiaSharp.DotNet.Interactive` integration is gone. ([#4186](https://github.com/mono/SkiaSharp/pull/4186))

## Engine

- **Skia milestone 150** — SkiaSharp moved onto the upstream m150 branch. ([#4146](https://github.com/mono/SkiaSharp/pull/4146), [skia#253](https://github.com/mono/skia/pull/253))
- **chrome/m150 upstream sync** — pulled upstream chrome/m150 bug fixes into the Skia submodule. ([#4190](https://github.com/mono/SkiaSharp/pull/4190), [skia#259](https://github.com/mono/skia/pull/259))
- **harfbuzz 14.2.1** — HarfBuzz patch bump; see [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md). ([#4167](https://github.com/mono/SkiaSharp/pull/4167), [skia#257](https://github.com/mono/skia/pull/257))

## API Surface

- **`SKPaint.GetFastBounds()`** — computes a conservative bounding rectangle for a paint's effect on a source rect, letting callers cheaply reject work that lies outside the target region. ([#4271](https://github.com/mono/SkiaSharp/pull/4271), [skia#271](https://github.com/mono/skia/pull/271))
- **`SKColorFilter.CreateOverdraw()`** — a colour filter that maps sample counts to distinct colours, so drawing with it visualises overdraw hot-spots. ([#4227](https://github.com/mono/SkiaSharp/pull/4227), [skia#262](https://github.com/mono/skia/pull/262))
- **`SKImageFilter.CreateCrop()`** — a filter that clips its input to a rectangle without allocating an intermediate layer. ([#4229](https://github.com/mono/SkiaSharp/pull/4229), [skia#261](https://github.com/mono/skia/pull/261))
- **`SKImageFilter.CreateEmpty()`** — a filter that produces an empty result, useful as a null/no-op sentinel in filter graphs. ([#4228](https://github.com/mono/SkiaSharp/pull/4228), [skia#260](https://github.com/mono/skia/pull/260))

## GPU & Rendering

- **`SKSurface.Canvas` wrapper cached across calls** — the `Canvas` accessor previously reallocated the managed wrapper on each call; caching it removes an allocation and a P/Invoke round-trip from every draw call. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4247](https://github.com/mono/SkiaSharp/pull/4247))

## Bug Fixes

- **Small-sigma raster blur regression fixed** — small-radius raster blurs went through a slow path that also produced wrong results in some cases; defining `SK_AVOID_SLOW_RASTER_PIPELINE_BLURS` restores the correct fast path. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4269](https://github.com/mono/SkiaSharp/pull/4269))
