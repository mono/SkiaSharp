# Version 4.150.0

> **Preview release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/4.150.0-preview)

> **API changes** · [SkiaSharp API diff](4.150.0/index.md) · [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md)

## Highlights

SkiaSharp 4.150.0 updates the Skia engine to Chrome milestone m150 and ships alongside HarfBuzzSharp 14.2.1 (see the [HarfBuzzSharp 14.2.1](harfbuzzsharp/14.2.1.md) page for binding details). This release finalizes the v4 API transition — thanks to community contributor [@ramezgerges](https://github.com/ramezgerges), remaining pre-v4 obsolete APIs are now compile errors and outdated enums have been trimmed from the reference assembly. Deprecated .NET Interactive (Polyglot Notebooks) support has also been removed.

## ⚠️ Breaking Changes

- **Pre-v4 obsoletes promoted to compile errors** — APIs deprecated as part of the v3→v4 migration are now `[Obsolete(error: true)]`. Any code still calling these APIs will fail to compile; migrate to the v4 equivalents before upgrading. Obsolete enums have also been trimmed from the reference assembly. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4195](https://github.com/mono/SkiaSharp/pull/4195))
- **Polyglot Notebooks / .NET Interactive support removed** — The previously deprecated integration with .NET Interactive (Polyglot Notebooks) has been removed. Migrate to the standard `SkiaSharp.Views.Blazor` package for Blazor scenarios. ([#4186](https://github.com/mono/SkiaSharp/pull/4186))

## New Features

### Engine

- **Skia milestone m150** — Updates the underlying Skia graphics engine to Chrome milestone m150, bringing the latest upstream rendering improvements, bug fixes, and new capabilities. ([#4146](https://github.com/mono/SkiaSharp/pull/4146))

### Platform

- **Tizen Docker builds** — Tizen native dependencies are now built inside a Docker container, improving build reproducibility and isolating the Tizen toolchain from the host environment. ([#4154](https://github.com/mono/SkiaSharp/pull/4154))
