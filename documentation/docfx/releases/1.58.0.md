# Version 1.58.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.58.0)

> **API changes** · [SkiaSharp API diff](1.58.0/index.md)

## Highlights

Servicing release with no user-facing PR work. The binding refresh drops the batching helpers from `GRContextOptions`, the color-cube helpers from `SKColorFilter`, and the `UnderlineText`/`StrikeThruText` shortcut properties from `SKPaint`.

## Breaking Changes

- **SKPaint** — the shortcut properties `UnderlineText`, `StrikeThruText`, and `TextAlign`-based batch setters were removed. Set flags on the paint's `FakeBoldText` / decoration APIs or use `SKTextBlob` for styled runs.
- **SKColorFilter** — `CreateColorCube(SKData cubeData, int cubeDimension)` and its byte-array overload were removed. Use `SKColorFilter.CreateTable` or a shader-based lookup for the same effect.
- **GRContextOptions** — the batching-related properties (`ExplicitlyAllocateGPUResources`, `AllowMultipleGlyphCacheTextures`, `BufferMapThreshold`, `MinimumRasterTileSize`, and related fields) were removed. Their behavior is now controlled internally by Skia.
- **GRPixelConfig** — several enum values were removed and the remainder re-numbered. Only code that persists raw integer values is affected.
