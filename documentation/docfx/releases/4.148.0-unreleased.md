# Version 4.148.0

> **Upcoming release** · In development · Not yet available on NuGet

## Highlights

This release bumps the Skia engine to milestone m148 and continues the API surface cleanup. Legacy `SKPaint` text and font properties now produce compile-time errors, and singleton object lifetimes have been reworked for correctness and predictability. Community contributor [@ramezgerges](https://github.com/ramezgerges) drove both major user-facing changes in this release.

## Breaking Changes

- **`SKPaint` text/font legacy members promoted to compile errors** — Properties and methods previously marked `[Obsolete]` as warnings (text size, typeface, and related font properties) now produce compile-time errors. Migrate to `SKFont`-based APIs. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))

## New Features

### Engine

- **Skia milestone m148** — Updates the underlying Skia engine to milestone m148, bringing upstream rendering improvements, bug fixes, and performance work from Google. ([#4125](https://github.com/mono/SkiaSharp/pull/4125))

### API Surface

- **Singleton lifecycle rework** — The disposal and lifetime management of singleton instances has been reworked for correctness, preventing accidental premature disposal and improving deterministic cleanup. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
- **Remaining legacy `SKPaint` state-reading APIs obsoleted** — The last remaining state-reading APIs on `SKPaint` that were previously moved to `SKFont` are now marked `[Obsolete]`, completing the paint/font API separation. ([#4114](https://github.com/mono/SkiaSharp/pull/4114))
- **Native compatibility check moved to `SkiaApi` static constructor** — The native library version compatibility gate was moved from `[ModuleInitializer]` to the `SkiaApi` static constructor, improving reliability on platforms with limited module initializer support. ([#4133](https://github.com/mono/SkiaSharp/pull/4133))

## Bug Fixes

- **`SKPixmap.GetPixelSpan<T>(x,y)` offset calculation** — Fixed a bug where the pixel offset used `Height` instead of `Width` when computing the byte position, producing incorrect results for non-square pixmaps. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **WinUI Projection DLL not found on .NET 9** — Fixed a package reference issue that caused `System.Private.WinRT.dll` to be missing when consuming `SkiaSharp.Views.WinUI` in .NET 9 projects. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))

## Security

- **expat 2.8.1** — Updates the expat XML parsing library to 2.8.1, picking up upstream security and correctness fixes. ([#4079](https://github.com/mono/SkiaSharp/pull/4079))
- **HarfBuzz 14.2.0** — Updates the HarfBuzz text shaping library to 14.2.0. ([#4035](https://github.com/mono/SkiaSharp/pull/4035))
