## Highlights

**SkiaSharp 4.148.0 marks the debut of the 4.x era** — the first major version bump since 2.x — advancing the Skia engine from milestone m132 through m148 and delivering the most feature-rich release in the library's history. Variable font support arrives in both SkiaSharp and HarfBuzzSharp, animated WebP encoding is now supported, color font palettes are exposed, and zero-copy stream-to-data conversion lands via `SKStream.GetData()`. This release rolls up all preview work from 4.147.0, which shipped preview-only and was never released as stable; community contributors [@ramezgerges](https://github.com/ramezgerges), [@4Darmygeometry](https://github.com/4Darmygeometry), [@SimonvBez](https://github.com/SimonvBez), [@ebariche](https://github.com/ebariche), [@Copilot](https://github.com/Copilot), and [@sasakrsmanovic](https://github.com/sasakrsmanovic) drove many of the new features and fixes.

## ⚠️ Breaking Changes

- **Major version 4.x** — The package version has jumped from 3.x to 4.x, signaling intentional breaking changes. Review your existing code for removed or renamed APIs. ([#3640](https://github.com/mono/SkiaSharp/pull/3640))
- **`SKPaint` text/font deprecated members are now errors** — Members of `SKPaint` that were marked `[Obsolete]` for text and font operations now produce compile-time errors. Migrate to the `SKFont`-based API. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))
- **WASM pre-.NET 8 builds dropped** — Native WebAssembly builds for Emscripten versions targeting frameworks older than .NET 8 are no longer produced. ([#4022](https://github.com/mono/SkiaSharp/pull/4022))

## New Features

### Engine

- **Skia milestones m132 through m148** — Cumulative engine upgrade spanning Skia milestones m132, m133, m147, and m148, bringing years of upstream graphics improvements. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3560](https://github.com/mono/SkiaSharp/pull/3560), [#3660](https://github.com/mono/SkiaSharp/pull/3660), [#3702](https://github.com/mono/SkiaSharp/pull/3702), [#4125](https://github.com/mono/SkiaSharp/pull/4125))
- **Skia m147 upstream bug fix syncs** — Applied upstream Skia chrome/m147 bug fixes from Google's release branch. ([#4081](https://github.com/mono/SkiaSharp/pull/4081), [#4044](https://github.com/mono/SkiaSharp/pull/4044))

### Text & Fonts

- **Variable font support** — Full variable font support added to both SkiaSharp and HarfBuzzSharp, enabling font axis manipulation and rich typographic variation. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palettes** — Exposes Skia's color font palette API, enabling full control over color font rendering and palette selection. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct** — Adds a proper `HBColor` struct for correct `hb_color_t` representation in HarfBuzzSharp. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Default typeface resolution** — Moved default-typeface resolution from native to the managed layer, fixing font fallback behavior across platforms. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))
- **HarfBuzz 14.2.0** — Updated HarfBuzz text-shaping library. ([#4035](https://github.com/mono/SkiaSharp/pull/4035))

### API Surface

- **Animated WebP encoding** — New `SKWebpEncoder` class supports encoding animated WebP image sequences. ([#3771](https://github.com/mono/SkiaSharp/pull/3771))
- **`SKStream.GetData()`** — Zero-copy stream-to-data conversion without extra allocation. ([#3772](https://github.com/mono/SkiaSharp/pull/3772))
- **`SKSamplingOptions` on draw** — `SKSurface.Draw` and `SKCanvas.DrawSurface` now accept `SKSamplingOptions` for per-call filtering control. ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))
- **Missing API wrappers** — Added missing C API wrappers and corrected naming convention violations throughout the binding. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))
- **Legacy paint APIs obsoleted** — Remaining legacy `SKPaint` state-reading APIs are now marked `[Obsolete]` to guide migration. ([#4114](https://github.com/mono/SkiaSharp/pull/4114))

### Platform

- **Tizen x64 and ARM64 support** — Tizen native builds now support x64 and arm64 architectures. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **Linux Bionic native assets** — Added native asset support for Linux Bionic (bionic libc, as used by Android-on-Linux). ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **C# 13 on legacy TFMs** — Added PolySharp to enable C# 13 language features on downlevel target frameworks. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))
- **Apple platform TFM fix** — Apple library projects now use versioned TFMs (26.0); app projects use the unversioned form. ([#3798](https://github.com/mono/SkiaSharp/pull/3798))
- **Linux Docker build migration** — Linux native builds migrated to .NET official cross-compilation Docker images for consistency. ([#4062](https://github.com/mono/SkiaSharp/pull/4062))

## Bug Fixes

- **`SKPixmap`/`SKBitmap.GetPixelSpan` stride** — `GetPixelSpan` now correctly uses `RowBytes` for stride, fixing out-of-bounds reads on padded images. ([#4148](https://github.com/mono/SkiaSharp/pull/4148))
- **`SKPixmap.GetPixelSpan<T>(x, y)` offset** — Fixed pixel offset calculation that incorrectly used `Height` instead of `Width`. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **`SKPath` finalizer crash** — Fixed a crash when `SKPathBuilder` was garbage-collected before the `SKPath` it created. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3796](https://github.com/mono/SkiaSharp/pull/3796))
- **Singleton lifecycle** — Reworked lifecycle management of singleton instances to prevent premature disposal. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
- **Android MAUI `SKGLView` tab switch** — Fixed `SKGLView` not rendering after switching tabs in a MAUI `TabBar` on Android. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **WinUI Projection DLL for .NET 9** — Fixed "DLL not found" for WinUI consumers targeting .NET 9. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
- **Native-compat gate** — Moved the native compatibility check from `[ModuleInitializer]` to the `SkiaApi` static constructor for more reliable startup diagnostics. ([#4133](https://github.com/mono/SkiaSharp/pull/4133))
