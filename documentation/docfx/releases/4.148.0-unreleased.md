## Highlights

Version 4.148.0 is a landmark release — the first major version bump in SkiaSharp's history, stepping up from **3.x to 4.x**. Built on Skia milestone 148, it delivers animated WebP encoding, color font palette support, full variable font support (with HarfBuzz integration), and a sweeping API cleanup that promotes long-deprecated `SKPaint` text members to compile errors. This release supersedes the previewed **4.147.0** line, which was never shipped as stable, and rolls up all that work cumulatively. Community contributors [@ramezgerges](https://github.com/ramezgerges), [@4Darmygeometry](https://github.com/4Darmygeometry), [@SimonvBez](https://github.com/SimonvBez), [@ebariche](https://github.com/ebariche), and [@Copilot](https://github.com/Copilot) drove significant portions of this release.

## Breaking Changes

- **Major version bump (3.x → 4.x)** — SkiaSharp has been promoted to version 4. Package IDs are unchanged; update version constraints to accept `4.*`. ([#3640](https://github.com/mono/SkiaSharp/pull/3640))
- **`SKPaint` text/font obsolete members are now errors** — APIs previously marked `[Obsolete]` as warnings have been promoted to compile errors. Migrate to `SKFont` equivalents before upgrading. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))
- **Singleton lifecycle rework** — The lifecycle of globally shared singleton instances has been redesigned for correctness; code that relied on specific initialization order may need updates. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
- **Remaining legacy paint state-reading APIs obsoleted** — The last `SKPaint` APIs that read back text/font state are now `[Obsolete]` and will become errors in a future release. ([#4114](https://github.com/mono/SkiaSharp/pull/4114))
- **Native-compat check moved to `SkiaApi` static constructor** — The native library compatibility gate has moved from `[ModuleInitializer]` to the `SkiaApi` cctor. Unusual host scenarios that control module initialization order may be affected. ([#4133](https://github.com/mono/SkiaSharp/pull/4133))

## New Features

### Engine

- **Skia milestone 148** — Updated to Skia m148, bringing the latest upstream rendering improvements, bug fixes, and GPU backend changes. ([#4125](https://github.com/mono/SkiaSharp/pull/4125))

### Images & Encoding

- **Animated WebP encoding (`SKWebpEncoder`)** — New `SKWebpEncoder` API supports encoding multi-frame animated WebP files, complementing existing animated GIF support. ([#3771](https://github.com/mono/SkiaSharp/pull/3771))
- **`SKStream.GetData()` for zero-copy stream access** — Converts a stream to `SKData` without an extra memory copy, eliminating redundant allocations in common pipeline scenarios. ([#3772](https://github.com/mono/SkiaSharp/pull/3772))

### Text & Fonts

- **Variable font support** — Full support for OpenType variable fonts via `SKFont` and HarfBuzzSharp, enabling dynamic weight/width/style axes at runtime. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette support** — New APIs for querying and selecting color font palettes (e.g. emoji skin tones, light/dark variants). ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct for HarfBuzz** — Adds a proper `HBColor` struct to HarfBuzzSharp for correct `hb_color_t` representation. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Default typeface resolution moved to managed layer** — Fixes cross-platform inconsistencies in default typeface lookup. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))

### API Surface

- **`SKSamplingOptions` for `SKSurface.Draw` and `SKCanvas.DrawSurface`** — These methods now accept `SKSamplingOptions`, giving callers fine-grained control over resampling quality. ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))
- **Missing API wrappers and naming fixes** — Several previously unbound C API functions are now wrapped, and naming convention violations in the public surface have been corrected. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))

### Platform

- **Linux Bionic native assets** — New native library assets for Linux Bionic environments, enabling use in Android NDK-based Linux runtimes. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 native build support** — Tizen gains proper x64 and arm64 architecture builds. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **C# 13 support on legacy TFMs via PolySharp** — C# 13 language features are now available on .NET Framework and older TFMs through PolySharp polyfills. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))

## Bug Fixes

- **`SKPixmap.GetPixelSpan<T>(x, y)` wrong pixel offset** — Fixed a bug where the pixel offset calculation used `Height` instead of `Width`, returning incorrect data in non-square images. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **`SKPath` finalizer crash when `SKPathBuilder` is collected first** — Fixed a crash triggered when `SKPathBuilder` was garbage-collected before its derived `SKPath`, invalidating the path's native handle. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3796](https://github.com/mono/SkiaSharp/pull/3796))
- **Android `SKGLView` goes blank after MAUI tab switch** — Fixed `SKGLView` failing to re-render when navigating away and back in a MAUI `TabBar`. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **WinUI Projection DLL not found for .NET 9** — Fixed a missing `WinUI.Projection` DLL that caused runtime failures for .NET 9 WinUI consumers. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
- **Apple platform TFM alignment** — Libraries now use versioned Apple TFMs (`26.0`) while app targets use unversioned ones, fixing compatibility issues on the latest Apple OS versions. ([#3798](https://github.com/mono/SkiaSharp/pull/3798))

## Preview History

This release supersedes **4.147.0**, which shipped previews but was skipped before going stable. All 4.147.0 preview work is included in the 4.148.0 rollup above.

### Preview 3

Updated to Skia milestone m148, obsoleted remaining legacy paint state-reading APIs, reworked singleton lifecycle, and fixed the `SKPixmap` pixel offset bug.

### Preview 2

Added variable font support, color font palette APIs, `SKStream.GetData()`, animated WebP encoding, `SKSamplingOptions` for surface drawing, and Linux Bionic native assets.

### Preview 1

Major version upgrade from 3.x to 4.x, updated to Skia milestone 147, added Tizen x64/arm64 support, and delivered foundational CI and build system improvements.
