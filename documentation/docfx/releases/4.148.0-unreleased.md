This is a landmark release: SkiaSharp jumps to major version 4, ships the Skia m148 engine, and delivers major new capabilities including **variable font support**, **color font palettes**, and **animated WebP encoding**. It also includes several long-awaited bug fixes — including Android `SKGLView` rendering in MAUI TabBar, `SKPixmap.GetPixelSpan` stride correctness, and `SKPath` finalizer crash. All work from the 4.147.0 preview cycle (Skia m147, Linux Bionic support, Tizen x64/arm64, the Uno SkiaFiddle sample, and the new Blazor gallery) is rolled up cumulatively. Community contributors [@ramezgerges](https://github.com/ramezgerges), [@SimonvBez](https://github.com/SimonvBez), [@4Darmygeometry](https://github.com/4Darmygeometry), [@ebariche](https://github.com/ebariche), [@sasakrsmanovic](https://github.com/sasakrsmanovic), and [@Copilot](https://github.com/Copilot) each drove user-facing changes in this release.

## Breaking Changes

- **Major version bump to 4.x** — SkiaSharp has moved from 3.x to 4.x. Package references must be updated to `>= 4.0.0`. ([#3640](https://github.com/mono/SkiaSharp/pull/3640))
- **Obsoleted legacy paint text/font APIs promoted to error** — APIs deprecated in 3.x that read text/font state from `SKPaint` now produce compile errors. Migrate to `SKFont` and `SKTextBlob`. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))
- **WASM: drop pre-.NET 8 Emscripten native builds** — Native WASM binaries targeting Emscripten versions older than .NET 8 are no longer produced. ([#4022](https://github.com/mono/SkiaSharp/pull/4022))

## New Features

### Engine

- **Skia milestone m148** — Updates the underlying Skia engine to milestone 148 with the latest rendering improvements from Google. ([#4125](https://github.com/mono/SkiaSharp/pull/4125))
- **Skia milestone m147** *(rolled up from 4.147.0 preview)* — Skia m147 engine update. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3702](https://github.com/mono/SkiaSharp/pull/3702))
- **Skia milestone m133** — Skia m133 engine update. ([#3660](https://github.com/mono/SkiaSharp/pull/3660))
- **Skia milestone m132** *(rolled up from 4.147.0 preview)* — Skia m132 engine update. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3560](https://github.com/mono/SkiaSharp/pull/3560))

### Text & Fonts

- **Variable font support** — Full variable font support in both SkiaSharp and HarfBuzzSharp, enabling control of font axes (weight, width, slant, optical size, and custom axes). ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette support** — `SKFontManager` and `SKTypeface` now expose color font palette information, enabling rendering of emoji and color fonts with their intended color tables. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct** — New `HBColor` struct properly represents `hb_color_t` in the HarfBuzz bindings. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Default typeface resolution in managed layer** — Fixes default typeface handling so system fonts resolve correctly without requiring a native fallback. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))

### Images & Documents

- **Animated WebP encoding (`SKWebpEncoder`)** — SkiaSharp can now encode animated WebP files. ([#3771](https://github.com/mono/SkiaSharp/pull/3771))
- **`SKStream.GetData()`** — New zero-copy `SKStream.GetData()` method converts a stream to `SKData` without a copy where the underlying data is contiguous. ([#3772](https://github.com/mono/SkiaSharp/pull/3772))

### API Surface

- **`SKSamplingOptions` for `SKSurface.Draw` and `SKCanvas.DrawSurface`** — Adds `SKSamplingOptions` overloads to `SKSurface.Draw` and `SKCanvas.DrawSurface` for quality-controlled blitting. ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))
- **New API wrappers and naming fixes** — Several missing C API wrappers added and naming convention violations corrected. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))
- **Obsolete legacy paint state-reading APIs** — Remaining legacy APIs that read text and font state from `SKPaint` are now marked obsolete with migration guidance toward `SKFont`. ([#4114](https://github.com/mono/SkiaSharp/pull/4114))

### Platform

- **Linux Bionic native assets** — New `libSkiaSharp` native binaries for the Linux Bionic ABI (used by Android on Linux desktop). ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 native builds** — Tizen now has proper native builds for both x64 and arm64 architectures. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **Apple TFM fix** — Corrects Apple platform TFMs to use `26.0` for library packages and unversioned for app packages, fixing resolution on newer SDK versions. ([#3798](https://github.com/mono/SkiaSharp/pull/3798))
- **WinUI .NET 9 fix** — Fixes `WinUI Projection DLL not found` errors when consuming SkiaSharp in .NET 9 WinAppSDK projects. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))

### Gallery & Samples

- **SkiaFiddle Uno Platform WebAssembly sample** — New SkiaFiddle interactive code editor running on Uno Platform WASM. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3762](https://github.com/mono/SkiaSharp/pull/3762))
- **SkiaFiddle WASM size reduction (-60%)** — SkiaFiddle application bundle size reduced by approximately 60%. ❤️ [@ebariche](https://github.com/ebariche) ([#3849](https://github.com/mono/SkiaSharp/pull/3849))
- **Blazor gallery redesign** — Gallery overhauled with an API Atlas view featuring tag-cloud filtering, plus 5 new interactive samples. ([#3970](https://github.com/mono/SkiaSharp/pull/3970), [#3757](https://github.com/mono/SkiaSharp/pull/3757))

## Bug Fixes

- **`SKPixmap`/`SKBitmap` `GetPixelSpan` stride** — `GetPixelSpan` now correctly uses `RowBytes` for stride instead of pixel width, fixing off-by-stride reads in non-packed bitmaps. ([#4148](https://github.com/mono/SkiaSharp/pull/4148))
- **`SKPixmap.GetPixelSpan<T>(x,y)` offset** — Fixed an offset calculation that incorrectly used `Height` instead of `Width`, returning wrong pixel addresses. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **`SKPath` finalizer crash** — Fixed a crash in the `SKPath` finalizer when `SKPathBuilder` was garbage-collected before the path it built. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3796](https://github.com/mono/SkiaSharp/pull/3796))
- **Android `SKGLView` not rendering after MAUI TabBar switch** — Fixes `SKGLView` going blank when switching back to a tab containing a MAUI `TabBar`. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **Singleton lifecycle** — Reworked how singleton SkiaSharp objects are retained and released, fixing edge-case lifetime issues. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
