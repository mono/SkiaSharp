## Highlights

SkiaSharp 4.148.0 is the first stable release of the major 4.x series, cumulatively rolling up all preview work from the skipped 4.147.0 milestone. This release brings **variable font support** in SkiaSharp and HarfBuzzSharp, **animated WebP encoding**, a new **Linux Bionic** native target, and multiple critical bug fixes. The Skia engine advances through milestones m132 through m148. Community contributors [@ramezgerges](https://github.com/ramezgerges), [@4Darmygeometry](https://github.com/4Darmygeometry), [@SimonvBez](https://github.com/SimonvBez), [@ebariche](https://github.com/ebariche), and [@Copilot](https://github.com/Copilot) drove significant user-facing improvements in this release.

## Breaking Changes

- **Major version bump (3.x → 4.x)** — SkiaSharp has been upgraded from the 3.x series to 4.x, reflecting the Skia engine milestone progression and accumulated API improvements. ([#3640](https://github.com/mono/SkiaSharp/pull/3640))
- **SKPaint text/font obsolete members are now errors** — APIs marked `[Obsolete]` in previous versions for text and font state on `SKPaint` now produce compile errors. Migrate to `SKFont` and text blob APIs. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4068](https://github.com/mono/SkiaSharp/pull/4068))
- **WebAssembly: pre-.NET 8 Emscripten support dropped** — Native WASM builds for Emscripten versions targeting runtimes before .NET 8 are no longer produced. ([#4022](https://github.com/mono/SkiaSharp/pull/4022))

## New Features

### Engine

- **Skia milestone 148** — Updates the rendering engine to m148, incorporating upstream performance improvements, GPU fixes, and new rendering capabilities. ([#4125](https://github.com/mono/SkiaSharp/pull/4125))
- **Upstream Chrome/m147 bug fixes** — Cherry-picked upstream Skia fixes from the Chrome m147 branch. ([#4081](https://github.com/mono/SkiaSharp/pull/4081), [#4044](https://github.com/mono/SkiaSharp/pull/4044))

### Text & Fonts

- **Variable font support** — Full variable font support in both SkiaSharp and HarfBuzzSharp, enabling dynamic adjustment of font axes (weight, width, slant, etc.). ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette support** — Added support for color font palettes, enabling correct rendering of emoji and multi-color OpenType fonts. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct** — Introduces `HBColor` for proper `hb_color_t` representation in HarfBuzzSharp. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))

### API Surface

- **Animated WebP encoding (`SKWebpEncoder`)** — New `SKWebpEncoder` API for encoding animated WebP images from frame sequences. ([#3771](https://github.com/mono/SkiaSharp/pull/3771))
- **`SKStream.GetData()`** — Zero-copy stream-to-`SKData` conversion for efficient data access without unnecessary allocations. ([#3772](https://github.com/mono/SkiaSharp/pull/3772))
- **`SKSamplingOptions` for `SKSurface.Draw` and `SKCanvas.DrawSurface`** — Adds sampling quality control when drawing surfaces. ❤️ [@Copilot](https://github.com/Copilot) ([#3491](https://github.com/mono/SkiaSharp/pull/3491))
- **Additional API wrappers and naming fixes** — Several missing API wrappers added and naming convention violations corrected for improved consistency. ([#4001](https://github.com/mono/SkiaSharp/pull/4001))
- **Legacy paint state-reading APIs obsoleted** — Remaining legacy paint state-reading APIs are now marked obsolete in favour of the modern equivalents. ([#4114](https://github.com/mono/SkiaSharp/pull/4114))

### Platform

- **Linux Bionic native assets** — Adds native library support for Android Bionic-based Linux environments. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 native builds** — Native library support extended to Tizen x64 and arm64 architectures. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **C# 13 support on legacy TFMs via PolySharp** — Uses PolySharp to backport C# 13 language features to legacy target frameworks. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))
- **Improved singleton instance lifecycle** — Reworked the lifecycle management of singleton instances for more predictable behaviour. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
- **Safer native-compat gate** — Moved the native compatibility check from `[ModuleInitializer]` to the `SkiaApi` static constructor for more reliable startup. ([#4133](https://github.com/mono/SkiaSharp/pull/4133))
- **Fixed Apple platform TFMs** — Corrects versioned vs. unversioned TFM usage for Apple platform libraries and apps. ([#3798](https://github.com/mono/SkiaSharp/pull/3798))

## Bug Fixes

- **`SKPixmap`/`SKBitmap` `GetPixelSpan` stride** — Fixes `GetPixelSpan` to use `RowBytes` for stride instead of width, preventing incorrect pixel access on non-contiguous bitmaps. ([#4148](https://github.com/mono/SkiaSharp/pull/4148))
- **`SKPixmap.GetPixelSpan<T>(x, y)` offset** — Fixes the row offset calculation that was using `Height` instead of `Width`. ([#4128](https://github.com/mono/SkiaSharp/pull/4128))
- **`SKPath` finalizer crash** — Fixes a crash when `SKPath` is finalised after its associated `SKPathBuilder` has already been garbage collected. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3796](https://github.com/mono/SkiaSharp/pull/3796))
- **WinUI Projection DLL not found on .NET 9** — Fixes a `DllNotFoundException` for WinUI projection builds targeting .NET 9 consumers. ([#4084](https://github.com/mono/SkiaSharp/pull/4084))
- **Android `SKGLView` not rendering after tab switch** — Fixes `SKGLView` failing to render when switching back to its tab in a MAUI `TabBar`. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **Default typeface resolution** — Moves default-typeface resolution to the managed layer for consistent cross-platform font fallback behaviour. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3730](https://github.com/mono/SkiaSharp/pull/3730))
