# Version 2.88.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.0)

> **API changes** · [SkiaSharp API diff](2.88.0/index.md) · [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md)

## Highlights

The 2.88 line is the .NET 6 + .NET MAUI baseline — adds MAUI compat views and handlers, Mac Catalyst native builds, and Blazor WebAssembly component support. Ships a new "Native Asset" package split, HarfBuzz 2.8.2, Uno 4.0 support, and long-awaited fixes for `SKCanvasView.IgnorePixelScaling` and Blazor infinite-loop rendering. HarfBuzzSharp 2.8.2 ships alongside this release — see [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md) for its binding details.

## Breaking Changes

- **`GRContext` now inherits from `GRRecordingContext`** — Consumers that pattern-match `GRContext` against `SKObject` directly should switch to `GRRecordingContext` or a more general base. Callers that pass `GRContext` where `GRRecordingContext` was expected keep working.

## Engine

- **HarfBuzz update to 2.8.2** ([#1766](https://github.com/mono/SkiaSharp/pull/1766) by @mattleibow) — Refreshes the HarfBuzz native to 2.8.2, shipped as [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md).

## GPU & Rendering

- **`SKCanvasView.IgnorePixelScaling` works correctly** ([#1804](https://github.com/mono/SkiaSharp/pull/1804) by @mattleibow) — Fixes the property so pixel-scaling is respected across the affected views.

## API Surface

- **Add some of the new APIs** ([#1828](https://github.com/mono/SkiaSharp/pull/1828) by @mattleibow) — Layers in the m88-era additions surfaced by the Skia rebase.

## Bug Fixes

- **[Blazor] prevent infinite loop in `SKCanvasView`** ([#1889](https://github.com/mono/SkiaSharp/pull/1889) by @JensKrumsieck) — Breaks the recursive render loop the Blazor canvas could hit.
- **Protect NRE in .NET MAUI `SKCanvasView`** ([#1734](https://github.com/mono/SkiaSharp/pull/1734) by @jsuarezruiz) — Avoids a null reference before the MAUI handler is attached.
- **Replace `HandleDictionary` lock with a critical section under Windows** ([#1817](https://github.com/mono/SkiaSharp/pull/1817) by @toptensoftware) — Improves lock-contention behaviour on Windows.
- **Never dispose the native statics** ([#1863](https://github.com/mono/SkiaSharp/pull/1863) by @mattleibow) — Prevents rare tear-down crashes on process exit.
- **Work around enums not being blittable** ([#1857](https://github.com/mono/SkiaSharp/pull/1857) by @mattleibow) — Sidesteps a marshalling issue with enums on some runtimes.
- **Try p/invoking to `libdl.so.2` before `libdl.so`** ([#2010](https://github.com/mono/SkiaSharp/pull/2010) by @akoeplinger) — Fixes libdl loading on newer Linux distros.
- **iOS requires that the view have a background** ([#1730](https://github.com/mono/SkiaSharp/pull/1730) by @mattleibow) — Applies the default background so iOS renders content.
- **So many issues resolved** ([#1958](https://github.com/mono/SkiaSharp/pull/1958) by @mattleibow) — A grab-bag PR that closes out a batch of long-standing issues.

## Platform

- **.NET MAUI views (compat and handlers)** ([#1678](https://github.com/mono/SkiaSharp/pull/1678) by @mattleibow) — Adds `SKCanvasView` + `SKGLView` for MAUI in both handler and compat forms.
- **Mac Catalyst support** ([#1760](https://github.com/mono/SkiaSharp/pull/1760), [#1681](https://github.com/mono/SkiaSharp/pull/1681), [#1762](https://github.com/mono/SkiaSharp/pull/1762), [#1865](https://github.com/mono/SkiaSharp/pull/1865) by @mattleibow) — Adds Mac Catalyst as a target, wires it up via frameworks, and iteratively fixes the Catalyst build inputs/outputs.
