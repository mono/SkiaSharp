# Version 2.88.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.0)

> **API changes** · [SkiaSharp API diff](2.88.0/index.md) · [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md)

The `.NET MAUI` release. Sixty-three PRs across fourteen previews take SkiaSharp from XF-centric `2.80` all the way to the .NET MAUI RC-4 baseline, add native symbol splitting, and land ASP.NET Blazor / WebAssembly-Component support alongside a HarfBuzz `2.8.2` bump. HarfBuzz `2.8.2` ships alongside — see [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md).

## Highlights

- Add .NET MAUI views (compat and handlers) ([#1678](https://github.com/mono/SkiaSharp/pull/1678)) and iterate through MAUI Previews 7 → RC 4 across the release window (see the trailing preview list).
- Support ASP.NET Blazor WebAssembly components ([#1811](https://github.com/mono/SkiaSharp/pull/1811)); `[Blazor]` — prevent an infinite loop in `SKCanvasView` ([#1889](https://github.com/mono/SkiaSharp/pull/1889)).
- Add Mac Catalyst support ([#1681](https://github.com/mono/SkiaSharp/pull/1681)) and split `SkiaSharp` into smaller `NativeAsset` packages ([#1758](https://github.com/mono/SkiaSharp/pull/1758)) with native Windows symbols and `HarfBuzzSharp.NativeAssets.*` ([#1797](https://github.com/mono/SkiaSharp/pull/1797)).
- Add some of the new APIs on top of the modernised `2.88` surface ([#1828](https://github.com/mono/SkiaSharp/pull/1828)).
- Add main PDB files to the packages so downstream debuggers can step through ([#1781](https://github.com/mono/SkiaSharp/pull/1781)).

## Fixes

- Fix `IgnorePixelScaling`'s behaviour ([#1804](https://github.com/mono/SkiaSharp/pull/1804)).
- Protect against NRE in the .NET MAUI `SKCanvasView` ([#1734](https://github.com/mono/SkiaSharp/pull/1734)); iOS requires the view to have a background ([#1730](https://github.com/mono/SkiaSharp/pull/1730)).
- Never dispose the native statics ([#1863](https://github.com/mono/SkiaSharp/pull/1863)) and work around enums not being blittable ([#1857](https://github.com/mono/SkiaSharp/pull/1857)).
- Replace the `HandleDictionary` lock with a critical section under Windows ([#1817](https://github.com/mono/SkiaSharp/pull/1817)).
- Ensure VS is set up correctly ([#1736](https://github.com/mono/SkiaSharp/pull/1736)) and try to build faster on demand ([#1859](https://github.com/mono/SkiaSharp/pull/1859)).
- Fix the Mac Catalyst build task inputs/outputs ([#1865](https://github.com/mono/SkiaSharp/pull/1865)).
- Try p/invoking to `libdl.so.2` before `libdl.so` ([#2010](https://github.com/mono/SkiaSharp/pull/2010)) — the change that unblocked several newer Linux distros.

## Engine

- Update HarfBuzz to `2.8.2` ([#1766](https://github.com/mono/SkiaSharp/pull/1766)).
- Build without manually installing .NET Core ([#1956](https://github.com/mono/SkiaSharp/pull/1956)); build with the newer/better/faster 1ES agents ([#1946](https://github.com/mono/SkiaSharp/pull/1946)); build using .NET 6.0.300 public ([#2031](https://github.com/mono/SkiaSharp/pull/2031)); use Python 3.x and Clang 12.x ([#1959](https://github.com/mono/SkiaSharp/pull/1959)); build using the older gcc `6.2` and new llvm `10` ([#1806](https://github.com/mono/SkiaSharp/pull/1806)); force a single CPU on macOS/Linux ([#1944](https://github.com/mono/SkiaSharp/pull/1944)); reduce the size of build artifacts ([#1788](https://github.com/mono/SkiaSharp/pull/1788)); update to the WindowsAppSDK ([#1800](https://github.com/mono/SkiaSharp/pull/1800)); switch to the .NET-tool Cake ([#1793](https://github.com/mono/SkiaSharp/pull/1793)); use `Invoke` for actions ([#1754](https://github.com/mono/SkiaSharp/pull/1754)); add support for `emscripten 2.0.23` ([#1746](https://github.com/mono/SkiaSharp/pull/1746)).
