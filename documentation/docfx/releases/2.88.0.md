# Version 2.88.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.0)

> **API changes** · [SkiaSharp API diff](2.88.0/index.md) · [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md)

<!-- Generated: 2026-07-03T21:22:41Z by claude-opus-4.7 -->

The first release aligned with .NET MAUI, plus Mac Catalyst, ASP.NET Blazor WebAssembly components, and refreshed Uno Platform integration. This is also when the SkiaSharp package family expanded to include the smaller "Native Asset" split. [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md) ships alongside; refer to its page for shaping-side binding details.

## Highlights

- Added .NET MAUI views (both compat renderers and handlers) so MAUI apps can drop in `SKCanvasView` and `SKGLView` on day one ([#1678](https://github.com/mono/SkiaSharp/pull/1678)).
- Added Mac Catalyst support so iOS apps ported to macOS keep their SkiaSharp rendering ([#1760](https://github.com/mono/SkiaSharp/pull/1760)).
- Split the main SkiaSharp package into smaller `SkiaSharp.NativeAssets.*` runtime packages so consumers only pull the platform-specific bits they need ([#1758](https://github.com/mono/SkiaSharp/pull/1758)).
- Added ASP.NET Blazor WebAssembly component support ([#1811](https://github.com/mono/SkiaSharp/pull/1811)).
- Added `net6.0-*` targets for the binding projects so .NET 6 apps target the platform TFMs directly ([#1707](https://github.com/mono/SkiaSharp/pull/1707)).
- Updated to WindowsAppSDK, keeping the WinUI 3 story current with the platform ([#1800](https://github.com/mono/SkiaSharp/pull/1800)).
- Updated HarfBuzz to 2.8.2 alongside a set of `HarfBuzzSharp.NativeAssets.*` packages that mirror the SkiaSharp split ([#1766](https://github.com/mono/SkiaSharp/pull/1766), [#1797](https://github.com/mono/SkiaSharp/pull/1797)).

## New features

- **.NET MAUI views** — `SKCanvasView` and `SKGLView` with both compatibility renderers and MAUI handlers ([#1678](https://github.com/mono/SkiaSharp/pull/1678)).
- **Mac Catalyst target** — full Skia support in Catalyst apps ([#1760](https://github.com/mono/SkiaSharp/pull/1760)).
- **Blazor WebAssembly components** — first-class `SKCanvasView` for Blazor Wasm ([#1811](https://github.com/mono/SkiaSharp/pull/1811)).
- **Native asset packages** — `SkiaSharp.NativeAssets.Win32`, `Linux`, `macOS`, `WebAssembly`, etc. ([#1758](https://github.com/mono/SkiaSharp/pull/1758)).
- **HarfBuzzSharp native asset packages** — matching split for the shaping libraries ([#1797](https://github.com/mono/SkiaSharp/pull/1797)).
- **`IgnorePixelScaling`** — clarified behaviour so it works as documented on Blazor and other backends ([#1804](https://github.com/mono/SkiaSharp/pull/1804)).
- **Software Bill of Materials (SBOM) manifest** — generated at build time for supply-chain audits ([#1954](https://github.com/mono/SkiaSharp/pull/1954) ❤️).
- **Wasm Emscripten 2.0.23** — bumped the WebAssembly toolchain for compatibility with Uno's latest ([#1746](https://github.com/mono/SkiaSharp/pull/1746) ❤️).

## Bug fixes

- Fixed the Mac Catalyst build task's `Inputs`/`Outputs` so incremental builds no longer over- or under-execute ([#1865](https://github.com/mono/SkiaSharp/pull/1865)).
- Prevented an infinite loop in the Blazor `SKCanvasView` ([#1889](https://github.com/mono/SkiaSharp/pull/1889) ❤️).
- Never disposed the native static instances so long-running processes don't lose default fonts/color spaces ([#1863](https://github.com/mono/SkiaSharp/pull/1863)).
- Replaced the shared `HandleDictionary` lock with a Windows critical section for lower contention ([#1817](https://github.com/mono/SkiaSharp/pull/1817) ❤️).
- Worked around enums that are not blittable on some runtimes ([#1857](https://github.com/mono/SkiaSharp/pull/1857)).
- Protected against `NullReferenceException` in the MAUI `SKCanvasView` handler ([#1734](https://github.com/mono/SkiaSharp/pull/1734) ❤️).
- Ensured iOS views have a background so a transparent host doesn't render as black ([#1730](https://github.com/mono/SkiaSharp/pull/1730)).
- Fixed the `p/invoke` fallback on newer Linux distros where `libdl.so.2` exists but `libdl.so` does not ([#2010](https://github.com/mono/SkiaSharp/pull/2010) ❤️).
- Excluded `.so` files from Android class libraries so consumers don't ship duplicate natives ([#1895](https://github.com/mono/SkiaSharp/pull/1895) ❤️).
- Trimmed extra fields in `Resource.designer.cs` to reduce Android app size ([#1812](https://github.com/mono/SkiaSharp/pull/1812) ❤️).

## Platform support

- **.NET MAUI** — RC1–RC4 with both compat and handler-based views.
- **Mac Catalyst** — first-class rendering target.
- **Blazor WebAssembly** — component-based `SKCanvasView`.
- **WindowsAppSDK / WinUI 3** — modernised on top of the new SDK.
- **Uno Platform** — bumped to 4.0.
- **.NET 6** — first-class managed target across the binding projects.
