# Version 2.88.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.0)

> **API changes** · [SkiaSharp API diff](2.88.0/index.md) · [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md)

## Highlights

Modernizes SkiaSharp for the .NET 6 / MAUI era: adds Mac Catalyst, .NET 6 target frameworks, .NET MAUI views (compat + handlers), ASP.NET Blazor WebAssembly components, WindowsAppSDK/WinUI 3 updates, and Xamarin.Forms on `netcoreapp3.1`. Splits the SkiaSharp mega-package into per-runtime native asset packages, adds PDBs, and lands SBOM generation. Ships alongside **[HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md)** — see that page for its binding details.

Community contributions from [@jonathanpeppers](https://github.com/jonathanpeppers) (Android class-library asset hygiene + Resource.designer.cs trim), [@akoeplinger](https://github.com/akoeplinger) (`libdl.so.2` fallback), [@jeromelaban](https://github.com/jeromelaban) (Uno 4.0 bump + WASM emscripten 2.0.23), [@jsuarezruiz](https://github.com/jsuarezruiz) (`SKCanvasView` NRE guard for MAUI), [@toptensoftware](https://github.com/toptensoftware) (Windows critical-section `HandleDictionary`), [@JensKrumsieck](https://github.com/JensKrumsieck) (Blazor infinite-loop fix), and [@mjbond-msft](https://github.com/mjbond-msft) (SBOM manifest).

## Breaking Changes

- **iOS/tvOS view assemblies renamed/reshaped** — `SkiaSharp.Views.iOS` and `SkiaSharp.Views.tvOS` see signature changes tied to the .NET 6 iOS/tvOS bindings and Mac Catalyst enablement. Recompile after retargeting your `Views.iOS`/`Views.tvOS` references. See the [iOS](2.88.0/SkiaSharp.Views/SkiaSharp.Views.iOS.breaking.md) and [tvOS](2.88.0/SkiaSharp.Views/SkiaSharp.Views.tvOS.breaking.md) breaking-change lists.
- **`SkiaSharp` core signature shifts** — Assorted signature clean-ups accompany the .NET 6 upgrade. See the [SkiaSharp breaking-change list](2.88.0/SkiaSharp/SkiaSharp.breaking.md) for exact members.
- **Native asset splitting** — `SkiaSharp` no longer bundles every RID's native binary in one package; consumers pull the appropriate `SkiaSharp.NativeAssets.*` package for their target(s). Most consumers get this transparently through package references. ([#1758](https://github.com/mono/SkiaSharp/pull/1758))

See the [full API diff](2.88.0/index.md) for the complete list.
