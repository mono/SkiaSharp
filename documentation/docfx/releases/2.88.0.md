# Version 2.88.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.0)

> **API changes** · [SkiaSharp API diff](2.88.0/index.md) · [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md)

<!-- Generated: 2026-07-03T22:30:00Z by claude-opus-4.7 -->

## Highlights

The 2.88 line. Adds first-class .NET MAUI views (compat + handler), Mac Catalyst support, ASP.NET Blazor WebAssembly components, Native Asset package splits, and a `GRRecordingContext` base type for `GRContext`. Ships alongside [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md).

- 🎨 **Core** — new APIs added along the way; `IgnorePixelScaling` semantics fixed; native statics never disposed to avoid teardown crashes; enum blittability workarounds; HarfBuzz updated to 2.8.2.
- 🍎 **Apple** — Mac Catalyst support and frameworks-based layout via `gn`; iOS view now sets a background so it actually renders.
- 🪟 **Windows / Uno / WinUI** — switched to WindowsAppSDK; Uno bumped to 4.0 (❤️ @jeromelaban); `HandleDictionary` lock replaced with a Windows critical section for lower contention (❤️ @toptensoftware).
- 🌐 **WebAssembly / Blazor** — ASP.NET Blazor WebAssembly components supported; Emscripten 2.0.23 support (❤️ @jeromelaban); `SKCanvasView` infinite-loop fix (❤️ @JensKrumsieck).
- 🤖 **Android** — `.so` files no longer bundled into class libraries; large-scale `Resource.designer.cs` field reduction (❤️ @jonathanpeppers).
- 🏗️ **Build / CI** — Native Asset package split (`SkiaSharp.NativeAssets.*` / `HarfBuzzSharp.NativeAssets.*`), main PDBs shipped in the packages, SBOM manifest generated (❤️ @mjbond-msft), .NET 6 baseline, and cake moved to a .NET tool.
- 📦 **General** — `[android]` fields eliminated from generated `Resource.designer.cs`, MAUI RC 4 baseline.

## Breaking Changes

- **SkiaSharp**
  - **GRContext** — its base type changed from `SKObject` to `GRRecordingContext`. Code that unified against `SKObject` still compiles (since `GRRecordingContext : SKObject`), but code that referenced `GRContext`'s direct base explicitly must switch.
- **SkiaSharp.Views.iOS / .Views.tvOS** — minor internal shape changes on the extension entry points; see the per-assembly API diff for the specifics.
