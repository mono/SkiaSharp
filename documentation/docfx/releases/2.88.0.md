# Version 2.88.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/2.88.0)

> **API changes** · [SkiaSharp API diff](2.88.0/index.md) · [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md)

## Highlights

SkiaSharp 2.88.0 lights up .NET MAUI, Mac Catalyst, and Blazor WebAssembly as first-class targets, alongside a big pass on the Android and Uno stacks. HarfBuzzSharp ships in lock-step at 2.8.2 — see the linked API changes above for the exact surface, and the [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md) page for the binding details.

## Breaking Changes

*No source-breaking API changes in this release.* The new-platform TFMs (`net6.0-android`, `net6.0-ios`, `net6.0-maccatalyst`, `net6.0-tizen`) introduce fresh assemblies rather than modifying existing ones. Review the [SkiaSharp API diff](2.88.0/index.md) for the full surface.

## New Features

### Platforms

- **.NET MAUI & .NET 6** — SkiaSharp and SkiaSharp.Views ship official `net6.0-*` builds for MAUI targets (Android, iOS, Mac Catalyst, Tizen), including the MAUI-facing helpers and `SKGLView`/`SKCanvasView` controls tuned for the new lifecycle. ([#2001](https://github.com/mono/SkiaSharp/pull/2001), [#1976](https://github.com/mono/SkiaSharp/pull/1976), [#2043](https://github.com/mono/SkiaSharp/pull/2043))
- **Mac Catalyst** — Native Mac Catalyst binaries and Views packages ship for the first time. ([#1902](https://github.com/mono/SkiaSharp/pull/1902))
- **Blazor WebAssembly** — `SKCanvasView`/`SKGLView` Blazor components ship in `SkiaSharp.Views.Blazor` with a matching WASM native build. ([#1893](https://github.com/mono/SkiaSharp/pull/1893), [#2049](https://github.com/mono/SkiaSharp/pull/2049))
- **Tizen refresh** — Tizen native and managed packages are rebuilt against a modern SDK. ([#1912](https://github.com/mono/SkiaSharp/pull/1912))
- **Linux legacy loader** — Fall back to `libdl.so.2` on distros that dropped the legacy `libdl.so.1` symlink. ❤️ [@akoeplinger](https://github.com/akoeplinger) ([#1928](https://github.com/mono/SkiaSharp/pull/1928))

### HarfBuzzSharp

- **HarfBuzz 2.8.2 co-release** — HarfBuzzSharp jumps to 2.8.2 with matching MAUI/Mac Catalyst/Blazor targets. See [HarfBuzzSharp 2.8.2](harfbuzzsharp/2.8.2.md) for details.

### API Additions

- **HandleDictionary batching** — New batched lookups and eviction hooks reduce lock contention in high-throughput scenarios. ❤️ [@toptensoftware](https://github.com/toptensoftware) ([#1856](https://github.com/mono/SkiaSharp/pull/1856))

## Bug Fixes

- **Android** — Restore the shipped `.so` layout so Android apps continue to resolve native SkiaSharp under .NET 6. ❤️ [@jonathanpeppers](https://github.com/jonathanpeppers) ([#1957](https://github.com/mono/SkiaSharp/pull/1957))
- **Android** — Regenerate `Resource.designer.cs` deterministically to avoid noisy diffs. ❤️ [@jonathanpeppers](https://github.com/jonathanpeppers) ([#1958](https://github.com/mono/SkiaSharp/pull/1958))
- **Uno Platform** — Update to Uno 4.0 packaging conventions and drop stale UWP-only assets. ❤️ [@jeromelaban](https://github.com/jeromelaban) ([#1990](https://github.com/mono/SkiaSharp/pull/1990))
- **WASM** — Refresh the Emscripten toolchain to 2.0.23 for `net6.0-browser` builds. ❤️ [@jeromelaban](https://github.com/jeromelaban) ([#2039](https://github.com/mono/SkiaSharp/pull/2039))
- **Blazor** — Fix an infinite invalidation loop when `SKCanvasView` re-renders. ❤️ [@JensKrumsieck](https://github.com/JensKrumsieck) ([#2079](https://github.com/mono/SkiaSharp/pull/2079))
- **Xamarin.Forms** — Fix a `NullReferenceException` when the native handle is torn down mid-render. ❤️ [@jsuarezruiz](https://github.com/jsuarezruiz) ([#1874](https://github.com/mono/SkiaSharp/pull/1874))
