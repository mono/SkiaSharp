# Version 1.53.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.53.0)

## Highlights

Marks the 1.49 → 1.53 stable transition of SkiaSharp. Only one PR is recorded in the diff — the native-library rename ([#81](https://github.com/mono/SkiaSharp/pull/81)) that established the `libSkiaSharp` naming convention still in use today — but the assembly version jumped from 1.49.0.0 to 1.53.0.0 and the API surface saw a large early cleanup as part of stabilization.

## Breaking Changes

- **Native library renamed** — The shipped native binary was renamed as part of establishing the current naming convention. Consumers linking against the older name must update their references. ([#81](https://github.com/mono/SkiaSharp/pull/81))
- **`SKAlphaType` values renumbered** — `Opaque`, `Premul`, and `Unpremul` shifted from `0/1/2` to `1/2/3`. Recompile against the new assembly; do not persist numeric values across upgrade boundaries.
- **`SKBitmap.Decode` / `DecodeBounds` — `pref` overloads removed** — The overloads taking a `SKColorType pref` (over `SKStreamRewindable`, `byte[]`, and `string`) were removed. Use the current `Decode` overloads plus explicit `SKImageInfo` when you need a specific color type.
- **`SKCanvas`** — Several legacy method shapes were retired as part of the surface consolidation. See the [full API diff](1.53.0/index.md).

## Category
