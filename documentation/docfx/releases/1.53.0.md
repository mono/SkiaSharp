# Version 1.53.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.53.0)

> **API changes** · [SkiaSharp API diff](1.53.0/index.md)

<!-- Generated: 2026-07-03T22:30:00Z by claude-opus-4.7 -->

## Highlights

The first tagged 1.x release. Only one PR is credited to the branch, but the API diff is substantial: the codec subsystem was rewritten around `SKCodec`, `SKImageDecoder` and its supporting enums were removed, and several `SKColorType`/`SKAlphaType` values were re-numbered to match Skia's canonical layout.

- 🎨 **Core** — `SKCodec` is now the recommended decoder entry point; the old `SKImageDecoder` façade is gone.

## Breaking Changes

- **SKImageDecoder** — the class and its supporting types (`SKImageDecoderFormat`, `SKImageDecoderMode`, `SKImageDecoderResult`) were removed. Use `SKCodec` / `SKImage.FromEncodedData` instead.
- **SKBitmap.Decode(SKColorType)** — the overload keyed on color type was removed; decode via `SKBitmap.Decode(stream)` and convert with `SKBitmap.Copy(SKColorType)` if needed.
- **SKCanvas** — the `DrawText`/`DrawPositionedText` overloads that took raw byte arrays and text-encoding parameters were removed; use the string-based overloads or `SKTextBlob`.
- **SKColorType / SKAlphaType** — several enum values were re-numbered to match upstream Skia. Only code that persists raw integer values (network, disk) is affected.
