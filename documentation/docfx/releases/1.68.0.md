## Highlights

Establishes the 1.68 line as the CI-modernized baseline for SkiaSharp: the recorded diff is entirely infrastructure (statically-linked Windows binaries, PR-builder testing, Linux CI improvements, Groovy CI script), but the assembly bumped from 1.60.0.0 to 1.68.0.0 and the API surface saw significant cleanup — most visibly in the GPU/UWP layers. Ships alongside **[HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md)** — see that page for its binding details.

## Breaking Changes

- **`GRBackend` enum renumbered** — `OpenGL` changed from `0` to `1`, `Vulkan` from `1` to `2`. Recompile against the new assembly; do not persist numeric values across upgrade boundaries.
- **`GRContext.Create(GRBackend, ...)` overloads removed** — The two overloads accepting `GRBackend backend` (with `GRGlInterface backendContext` and with `IntPtr backendContext`) were removed. Use the backend-specific factories: `GRContext.CreateGl(...)` and `GRContext.CreateVulkan(...)`.
- **`GRGlInterface.Clone()` removed** — There is no supported clone path; hold a reference to the original interface instead.
- **`GRPixelConfig`** — `Rgba8888SInt` removed and several enum values renumbered to close the gap. Recompile after auditing any explicit references.
- **UWP `SkiaSharp.Views.UWP.AngleSwapChainPanel`** — The ANGLE-related properties (`BackingOption`, `Context`, `DepthFormat`, `Multisampling`, `StencilFormat`) and the supporting `Gles*` types (`GlesBackingOption`, `GlesContext`, `GlesDepthFormat`, `GlesMultisampling`, `GlesRenderTarget`, `GlesStencilFormat`) were removed. `ContentsScale` is now read-only. Migrate to the current SkiaSharp UWP view API.

See the [full API diff](1.68.0/index.md) for the complete list.
