# Version 1.68.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.0)

> **API changes** · [SkiaSharp API diff](1.68.0/index.md) · [HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md)

<!-- Generated: 2026-07-03T21:22:41Z by claude-opus-4.7 -->

A CI-focused milestone that solidified the build pipelines and moved Windows binaries to a fully-static link so consumers no longer need to worry about matching Visual C++ runtimes on the target machine. [HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md) ships alongside for text-shaping consumers pinning that transitive dependency; refer to its page for binding details.

## Highlights

- Windows binaries are now statically linked, removing the runtime dependency on external VC++ redistributables ([#662](https://github.com/mono/SkiaSharp/pull/662)).
- Reworked the Linux and PR-builder CI so the pipelines produce consistent packages across platforms ([#635](https://github.com/mono/SkiaSharp/pull/635), [#616](https://github.com/mono/SkiaSharp/pull/616), [#638](https://github.com/mono/SkiaSharp/pull/638)).

Plus several CI and documentation improvements.
