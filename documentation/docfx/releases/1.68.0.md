# Version 1.68.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.0)

> **API changes** · [SkiaSharp API diff](1.68.0/index.md) · [HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md)

## Highlights

Packaging and build-infrastructure refresh for the 1.x line. Windows binaries are now statically linked, and the CI pipeline received several rounds of cleanup that made the release process more repeatable. HarfBuzzSharp 1.4.6.2 ships alongside this release — see [HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md) for its binding details.

## Breaking Changes

None in this release.

## Platform

- **Statically Link Windows Binaries** ([#662](https://github.com/mono/SkiaSharp/pull/662) by @mattleibow) — Removes the runtime dependency on the Visual C++ runtime by statically linking the CRT into the shipped Windows native binaries, simplifying deployment.

## Lifecycle & Internals

- **CI refresh** ([#615](https://github.com/mono/SkiaSharp/pull/615), [#616](https://github.com/mono/SkiaSharp/pull/616), [#635](https://github.com/mono/SkiaSharp/pull/635), [#638](https://github.com/mono/SkiaSharp/pull/638) by @mattleibow) — Adds the initial Groovy CI script, introduces the PR builder, tightens the Linux CI, and lands a round of miscellaneous pipeline improvements.
