# Version 1.68.0

## Highlights

Statically-linked Windows binaries and a full overhaul of the CI infrastructure. This was the first release cut through the new Jenkins/Groovy pipeline, and lays the groundwork for the reliable multi-platform builds the rest of the 1.x line depends on. Ships alongside [HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md).

- 🪟 **Windows** — the native binaries are now statically linked, eliminating a class of missing-runtime deployment failures.
- 🐧 **Linux** — Linux CI reliability improvements land.
- 🏗️ **Build / CI** — the Groovy-based CI pipeline lands and a PR-builder is stood up.

## Breaking Changes

The 1.60→1.68 jump absorbed a major Skia rev; the API diff itemizes the drift:

- **SkiaSharp**
  - **GRBackend** — enum values were re-numbered to match upstream Skia; only code that persists raw integer values is affected.
  - **GRContext.Create** — the legacy static `Create()` factory was removed; use `GRContext.CreateGl(GRGlInterface)` (or the current per-backend factories).
  - **GRGlInterface.Clone** — the instance `Clone()` was removed. Callers can hold on to the original interface — it is already reference-counted.
  - **SKColorType** — several enum values were re-numbered to match upstream.
  - **SKCodecResult** — enum values were re-numbered.
  - **SKImageFilter** — `CreatePictureForLocalspace(SKPicture, SKRect, SKFilterQuality)` was removed. Use `CreatePicture(SKPicture, SKRect)` combined with a matrix or the current sampling APIs.
- **SkiaSharp.Views.UWP** — miscellaneous internal shape changes around the XAML host; see the assembly diff.
