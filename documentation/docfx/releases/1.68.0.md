# Version 1.68.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.0)

> **API changes** · [SkiaSharp API diff](1.68.0/index.md) · [HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md)

Rebases SkiaSharp on the upstream Skia `m68` milestone and modernises the CI. Five PRs shipped in the run-up to the final tag, all around build infrastructure and static-linking of the Windows binaries. HarfBuzz `1.4.6.2` ships alongside — see [HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md) for the standalone binding.

## Highlights

- Statically link the Windows native binaries so consumers no longer need to redistribute the runtime pieces separately ([#662](https://github.com/mono/SkiaSharp/pull/662)).
- Move CI improvements into place: PR builder, Linux CI hardening and a Groovy pipeline script ([#638](https://github.com/mono/SkiaSharp/pull/638), [#635](https://github.com/mono/SkiaSharp/pull/635), [#616](https://github.com/mono/SkiaSharp/pull/616), [#615](https://github.com/mono/SkiaSharp/pull/615)).

## Breaking Changes

Moving to Skia `m68` reshapes the managed surface across both `SkiaSharp` and `SkiaSharp.Views.UWP`. Walk the [SkiaSharp API diff](1.68.0/index.md) before upgrading from `1.6x` builds.
