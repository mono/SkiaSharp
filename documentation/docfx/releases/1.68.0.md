# Version 1.68.0

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.0)

The 1.68 line pivots the Windows build to static linking and modernizes the CI plumbing that had been growing under the 1.6x series. HarfBuzzSharp 1.4.6.2 co-ships — see the [HarfBuzzSharp 1.4.6.2](harfbuzzsharp/1.4.6.2.md) page for its binding details.

## Highlights

- Windows binaries are now statically linked ([#662](https://github.com/mono/SkiaSharp/pull/662)), eliminating the runtime dependency on a matching Visual C++ redistributable.
- CI overhaul across the Linux, PR-builder, and Groovy pipelines ([#615](https://github.com/mono/SkiaSharp/pull/615), [#616](https://github.com/mono/SkiaSharp/pull/616), [#635](https://github.com/mono/SkiaSharp/pull/635), [#638](https://github.com/mono/SkiaSharp/pull/638)) to make branch validation reliable ahead of the 2.x work that follows.
- HarfBuzzSharp 1.4.6.2 ships alongside — see the linked page for binding details.

## Breaking Changes

The signature diff picks up minor adjustments in `SkiaSharp` and `SkiaSharp.Views.UWP` — see the [SkiaSharp API diff](1.68.0/index.md) for the per-assembly detail.
