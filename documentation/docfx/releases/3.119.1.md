# Version 3.119.1

> **Community bug-fix servicing** · Released September 12, 2025 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.1) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.1)

> **API changes** · [SkiaSharp API diff](3.119.1/index.md) · [HarfBuzzSharp 8.3.1.2](harfbuzzsharp/8.3.1.2.md)

## Highlights

Community-driven servicing release for the 3.119 line. Fixes cluster around
Metal rendering on Apple, WinUI on WASDK, and small correctness bugs; one new
`SKTypeface.PostScriptName` property was added, and a native `zlib` bump
(v1.3.0.1+) reduces the CVE footprint of the shipped Skia. The HarfBuzz binding
is republished as [HarfBuzzSharp 8.3.1.2](harfbuzzsharp/8.3.1.2.md).

## Breaking Changes

None in this release. See the [3.119.1 API diff](3.119.1/index.md) to confirm
the only signature addition is `SKTypeface.PostScriptName`.

## API Surface

- **`SKTypeface.PostScriptName`** — expose the font's PostScript name for
  callers that need to round-trip through PDF or system font-lookup APIs. ❤️
  [@jonlipsky](https://github.com/jonlipsky)
  ([#3263](https://github.com/mono/SkiaSharp/pull/3263),
  [mono/skia#158](https://github.com/mono/skia/pull/158))

## Bug Fixes

- **Metal hotfix for iOS 15 and lower** — the Metal path regressed on older iOS
  versions and is restored. ❤️ [@taublast](https://github.com/taublast)
  ([#3293](https://github.com/mono/SkiaSharp/pull/3293))
- **`SKAutoCanvasRestore` no longer crashes when the canvas is already
  disposed** — a defensive check prevents the double-dispose crash. ❤️
  [@taublast](https://github.com/taublast)
  ([#3291](https://github.com/mono/SkiaSharp/pull/3291))
- **`SKXamlCanvas` is no longer opaque on Mac Catalyst** — the platform view
  now correctly honors transparency. ❤️ [@spouliot](https://github.com/spouliot)
  ([#3236](https://github.com/mono/SkiaSharp/pull/3236))
- **`SKMetalView` no longer initializes twice** — a redundant init call is
  removed. ❤️ [@jeremy-visionaid](https://github.com/jeremy-visionaid)
  ([#3256](https://github.com/mono/SkiaSharp/pull/3256))
- **`GRContext` disposal no longer crashes on Metal** — regression test added
  for the Metal backend teardown path. ❤️
  [@jeremy-visionaid](https://github.com/jeremy-visionaid)
  ([#3258](https://github.com/mono/SkiaSharp/pull/3258),
  [mono/skia#159](https://github.com/mono/skia/pull/159))
- **UTC `DateTime` conversion fixed** — the marshalling path now preserves
  UTC kind instead of shifting into local time. ❤️
  [@jeremy-visionaid](https://github.com/jeremy-visionaid)
  ([#3333](https://github.com/mono/SkiaSharp/pull/3333))

## Native Engine

- **zlib native updated to v1.3.0.1+** — the Skia bundled zlib is refreshed to
  reduce the CVE footprint of the shipped native binaries.
  ([#3058](https://github.com/mono/SkiaSharp/pull/3058),
  [mono/skia#160](https://github.com/mono/skia/pull/160))

## Platform

- **WinUI/WASDK native build flags straightened out** — the WinUI interop is
  built with `/MT`, and EGL/GLESv2 are built as win32 / WASDK targets
  respectively to match the SDK expectations.
  ([#3341](https://github.com/mono/SkiaSharp/pull/3341),
  [#3338](https://github.com/mono/SkiaSharp/pull/3338))
