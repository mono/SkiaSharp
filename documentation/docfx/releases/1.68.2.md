# Version 1.68.2

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.2)

> **API changes** · [SkiaSharp API diff](1.68.2/index.md) · [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md)

## Highlights

Adds `SKColorF` + `SKColorSpace` support for wide-gamut/HDR workflows, brings GTK views to Xamarin.Forms, adds improved-Perlin-noise and buffer-samples APIs, and rounds out `SKImage`, `SKRegion`, and `SKCanvas` with more members. Ships with a refreshed managed/native lifetime story: reflection-based object creation is retired, disposal is tightened, and a raft of view-lifetime and concurrency issues are resolved. Ships alongside **[HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md)** — see that page for its binding details.

Community contributions from [@Gillibald](https://github.com/Gillibald) (reflection removal), [@ziriax](https://github.com/ziriax) (`SKImageFilter.Merge` null args), [@zbyszekpy](https://github.com/zbyszekpy) (`SKGLViewRenderer` disposal), [@validvoid](https://github.com/validvoid) (UWP `SKSwapChainPanel` antialiasing), [@alexandrvslv](https://github.com/alexandrvslv) (WPF mouse-wheel touch), [@mscherotter](https://github.com/mscherotter) (device-type + pressure/eraser in touch events), [@daltonks](https://github.com/daltonks), [@bender2k14](https://github.com/bender2k14), and [@terrajobst](https://github.com/terrajobst).

## Breaking Changes

- **`SKColorF` overloads on the paint/canvas surface** — New `SKColorF` (float-channel) overloads are added and existing color-space-aware entry points may shift signatures. Recompile after upgrading. ([#1116](https://github.com/mono/SkiaSharp/pull/1116))
- **`SKImageFilter.Merge` accepts null arguments** — The merge-filter factory overloads now accept `null` where they previously threw; downstream code guarding against `null` inputs no longer needs to. ❤️ [@ziriax](https://github.com/ziriax) ([#1185](https://github.com/mono/SkiaSharp/pull/1185))
- **Method rename pass to match .NET conventions** — Several less-idiomatic method names were renamed to match .NET conventions. ([#1255](https://github.com/mono/SkiaSharp/pull/1255))
- **Removed the `[Preserve]` attribute** — Type-preservation for linkers is now handled without a custom attribute. ([#1229](https://github.com/mono/SkiaSharp/pull/1229))
- **Removed unused `lang` parameter** — The unused `lang` argument was trimmed off its call site. ([#1190](https://github.com/mono/SkiaSharp/pull/1190))

See the [full API diff](1.68.2/index.md) for the complete list.
