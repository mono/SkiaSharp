# Version 1.68.2

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.2)

> **API changes** · [SkiaSharp API diff](1.68.2/index.md) · [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md)

<!-- Generated: 2026-07-03T21:22:41Z by claude-opus-4.7 -->

A stability-focused point release for 1.68 that fixed disposal races across the view hierarchy, closed several concurrency bugs, and rounded out the `SKImage`, `SKRegion`, and `SKColorF` binding surfaces. [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md) ships alongside; refer to its page for shaping-side binding details.

## Highlights

- Reworked view disposal so `SKGLView`, `SKCanvasView` and friends no longer leak or double-dispose native resources ([#1180](https://github.com/mono/SkiaSharp/pull/1180), [#1213](https://github.com/mono/SkiaSharp/pull/1213), [#1256](https://github.com/mono/SkiaSharp/pull/1256)).
- Resolved concurrency issues in the shared object cache and removed reflection from `SKObject` creation to keep hot paths allocation-free ([#1200](https://github.com/mono/SkiaSharp/pull/1200), [#1209](https://github.com/mono/SkiaSharp/pull/1209) ❤️).
- Filled out the `SKImage`, `SKRegion`, `SKCanvas`, and `SKColorF` surfaces with the members that were still missing after 1.68.1 ([#1126](https://github.com/mono/SkiaSharp/pull/1126), [#1125](https://github.com/mono/SkiaSharp/pull/1125), [#1090](https://github.com/mono/SkiaSharp/pull/1090), [#1116](https://github.com/mono/SkiaSharp/pull/1116)).
- Modernised the build: VS2019 + .NET Core 3.0, Docker-based Linux builds, and updated `expat` to 2.2.9 ([#1030](https://github.com/mono/SkiaSharp/pull/1030), [#1036](https://github.com/mono/SkiaSharp/pull/1036), [#1214](https://github.com/mono/SkiaSharp/pull/1214)).

## New features

- **GTK views for Xamarin.Forms** — new backend for Linux desktop apps ([#1089](https://github.com/mono/SkiaSharp/pull/1089)).
- **`SKShader` improved Perlin noise** — binding for the higher-quality noise generator ([#1056](https://github.com/mono/SkiaSharp/pull/1056)).
- **`SKColorF` overloads with `SKColorSpace`** — first-class support for wide-gamut color throughout the drawing surface ([#1116](https://github.com/mono/SkiaSharp/pull/1116)).
- **Additional `SKImage`, `SKRegion`, `SKCanvas` members** — parity fill-ins across the binding ([#1126](https://github.com/mono/SkiaSharp/pull/1126), [#1125](https://github.com/mono/SkiaSharp/pull/1125), [#1090](https://github.com/mono/SkiaSharp/pull/1090)).
- **Writable generic pixel span** — expose the raw pixel buffer as `Span<T>` so callers can mutate it in-place ([#1242](https://github.com/mono/SkiaSharp/pull/1242)).
- **Touch device types + pressure/eraser handling** — WPF/UWP touch handler now surfaces pen, mouse, touch, pressure, and eraser data ([#1191](https://github.com/mono/SkiaSharp/pull/1191) ❤️, [#1212](https://github.com/mono/SkiaSharp/pull/1212) ❤️).
- **WPF mouse-wheel on Touch event** — mouse-wheel scroll surfaces through the touch pipeline ([#1079](https://github.com/mono/SkiaSharp/pull/1079) ❤️).
- **`SkiaSharp.Views.Windows.Forms`** — multi-targeted for netfx and .NET Core ([#1082](https://github.com/mono/SkiaSharp/pull/1082)).

## Bug fixes

- Removed the `[Preserve]` attribute so linked builds no longer keep unreferenced types around unnecessarily ([#1229](https://github.com/mono/SkiaSharp/pull/1229)).
- Stopped disposing system fonts, which could crash long-running processes that re-used the same typefaces ([#1224](https://github.com/mono/SkiaSharp/pull/1224)).
- Prevented the GC from collecting the `this` reference and pinned pixel buffers during native calls ([#1258](https://github.com/mono/SkiaSharp/pull/1258)).
- Fixed `SKTouchHandler` crash caused by passing incorrect parameters and added device-type handling ([#1212](https://github.com/mono/SkiaSharp/pull/1212) ❤️).
- Fixed the UWP `SKSwapChainPanel` antialiasing regression ([#1086](https://github.com/mono/SkiaSharp/pull/1086) ❤️).
- Fixed the macOS `SKGLView` display issues ([#1083](https://github.com/mono/SkiaSharp/pull/1083)).
- Queue the redraw for XF touch handling and prevented double-load races ([#1135](https://github.com/mono/SkiaSharp/pull/1135), [#1133](https://github.com/mono/SkiaSharp/pull/1133)).
- Corrected `SKGLViewRenderer.Dispose` ordering to avoid crashes on hot-reload ([#1095](https://github.com/mono/SkiaSharp/pull/1095) ❤️).
- Merge filter factory overloads now accept `null` arguments ([#1185](https://github.com/mono/SkiaSharp/pull/1185) ❤️).
- Fixed unit test crashes plus a WGL deadlock ([#1237](https://github.com/mono/SkiaSharp/pull/1237)).

## API polish

- Renamed several methods to follow more idiomatic .NET conventions ([#1255](https://github.com/mono/SkiaSharp/pull/1255)).
- Always reference `netstandard` so downstream projects don't need to add it manually ([#1250](https://github.com/mono/SkiaSharp/pull/1250)).
