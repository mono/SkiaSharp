# Version 1.68.2

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.2)

> **API changes** · [SkiaSharp API diff](1.68.2/index.md) · [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md)

The second `1.68` servicing wave — 71 PRs across seven previews, sanding down concurrency and disposal issues surfaced by the wider `1.68.1` audience and pulling more `SKCanvas`, `SKImage`, `SKRegion` and `SKColorF` bindings into the managed surface. HarfBuzz `2.6.1.2` ships alongside — see [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md).

## Highlights

- Expose a generic, writable `Span<T>` for pixels ([#1242](https://github.com/mono/SkiaSharp/pull/1242)) and rename several methods to match .NET conventions ([#1255](https://github.com/mono/SkiaSharp/pull/1255)).
- Fill in more `SKCanvas` members ([#1090](https://github.com/mono/SkiaSharp/pull/1090)) and the rest of `SKRegion` ([#1125](https://github.com/mono/SkiaSharp/pull/1125)); add more from `SKImage` ([#1126](https://github.com/mono/SkiaSharp/pull/1126)) and improve the generator and struct handling ([#1122](https://github.com/mono/SkiaSharp/pull/1122), [#1123](https://github.com/mono/SkiaSharp/pull/1123)).
- Add `SKColorF` (with `SKColorSpace`) overloads ([#1116](https://github.com/mono/SkiaSharp/pull/1116), (skia: [mono/skia#74](https://github.com/mono/skia/pull/74))).
- Land a Xamarin.Forms GTK view ([#1089](https://github.com/mono/SkiaSharp/pull/1089), (skia: [mono/skia#72](https://github.com/mono/skia/pull/72))) and multi-target `SkiaSharp.Views.Windows.Forms` ([#1082](https://github.com/mono/SkiaSharp/pull/1082)).
- Add an `SKShader` binding for the improved Perlin noise generator ([#1056](https://github.com/mono/SkiaSharp/pull/1056)).

## Fixes

- Resolve concurrency issues in the shared object bookkeeping ([#1200](https://github.com/mono/SkiaSharp/pull/1200)) and null out the field on disposal so double-dispose stops crashing (fixes [#960](https://github.com/mono/SkiaSharp/issues/960), [#1256](https://github.com/mono/SkiaSharp/pull/1256)).
- Prevent the GC from collecting `this` and `pixels` mid-flight ([#1258](https://github.com/mono/SkiaSharp/pull/1258)) and remove an unnecessary allocation on disposal ([#1257](https://github.com/mono/SkiaSharp/pull/1257)).
- Refresh the Android surface when recreated ([#1234](https://github.com/mono/SkiaSharp/pull/1234)) and re-work views to avoid incorrect disposal ([#1180](https://github.com/mono/SkiaSharp/pull/1180)).
- Fix WGL deadlock and misc unit-test crashes ([#1237](https://github.com/mono/SkiaSharp/pull/1237)); before drawing, `MakeCurrent` ([#1141](https://github.com/mono/SkiaSharp/pull/1141)); make sure the view is still in the hierarchy before drawing ([#1143](https://github.com/mono/SkiaSharp/pull/1143)).
- Change paint-encoding path instead of throwing ([#1152](https://github.com/mono/SkiaSharp/pull/1152), fixes [#1151](https://github.com/mono/SkiaSharp/issues/1151)); queue the redraw (fixes [#1074](https://github.com/mono/SkiaSharp/issues/1074), [#1135](https://github.com/mono/SkiaSharp/pull/1135)); prevent the double-load issue (fixes [#1118](https://github.com/mono/SkiaSharp/issues/1118), [#1133](https://github.com/mono/SkiaSharp/pull/1133)).
- Fix `#1187` ([#1206](https://github.com/mono/SkiaSharp/pull/1206)) and fix issues with buffer samples ([#1098](https://github.com/mono/SkiaSharp/pull/1098)).
- Do not dispose system fonts ([#1224](https://github.com/mono/SkiaSharp/pull/1224)) and release skia objects when the view is unloaded ([#1213](https://github.com/mono/SkiaSharp/pull/1213)).
- Correctly decode images ([#1055](https://github.com/mono/SkiaSharp/pull/1055)) and fix HarfBuzz string handling ([#1034](https://github.com/mono/SkiaSharp/pull/1034)); cherry-pick fixes for buffer-overflow bugs ([#1037](https://github.com/mono/SkiaSharp/pull/1037)).
- Correct `SkiaSharp.Views.Forms` behaviour on macOS `SKGLView` ([#1083](https://github.com/mono/SkiaSharp/pull/1083)) and fix broken antialiasing on UWP `SKSwapChainPanel` ([#1086](https://github.com/mono/SkiaSharp/pull/1086)).
- Merge filter-factory overloads now accept `null` arguments ([#1185](https://github.com/mono/SkiaSharp/pull/1185)).
- `SKGLViewRenderer` disposal corrections ([#1095](https://github.com/mono/SkiaSharp/pull/1095)); WPF mouse-wheel on touch events ([#1079](https://github.com/mono/SkiaSharp/pull/1079)); `SKTouchHandler` crash fix + pressure/eraser handling ([#1212](https://github.com/mono/SkiaSharp/pull/1212)); touch/mouse/pen device-type on touch events ([#1191](https://github.com/mono/SkiaSharp/pull/1191)).
- Remove the `[Preserve]` attribute now that the linker sees what it needs ([#1229](https://github.com/mono/SkiaSharp/pull/1229)).
- Remove reflection usage for `SKObject` creation ([#1209](https://github.com/mono/SkiaSharp/pull/1209)) and try to remove `WeakReference` instances safely ([#1039](https://github.com/mono/SkiaSharp/pull/1039)).

## Breaking Changes

Signatures moved across `SkiaSharp` and `SkiaSharp.Views.Forms` as part of the concurrency and disposal cleanup plus the `.NET`-style method renames ([#1255](https://github.com/mono/SkiaSharp/pull/1255)). Consult the [SkiaSharp API diff](1.68.2/index.md) for the full list.
