# Version 1.68.2

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.2)

> **API changes** · [SkiaSharp API diff](1.68.2/index.md) · [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md)

## Highlights

Second maintenance drop on the 1.68 line — 71 PRs focused on lifecycle correctness (view disposal, GC pinning, WGL deadlock, concurrency), broader `SKCanvas`/`SKImage`/`SKRegion` bindings, and platform-view stability improvements across UWP, macOS `SKGLView`, Windows Forms multi-targeting, and the new WPF-for-.NET-Core sample. Community contributors delivered UWP swap-chain antialiasing fixes, `SKGLViewRenderer` disposal corrections, pen-eraser touch handling, and a reflection-free `SKObject` creation path. HarfBuzzSharp 2.6.1.2 ships alongside this release — see [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md) for its binding details.

## Breaking Changes

- **`SKSize` / `SKSizeI` equality operator parameter rename** — `op_Equality` and `op_Inequality` on both size structs renamed the second parameter from `sz2` to `right` (with the first renamed to `left`). Named-argument callers need to update; positional callers are unaffected.

## Engine

- **`SKColorF` overloads with `SKColorSpace`** ([#1116](https://github.com/mono/SkiaSharp/pull/1116) by @mattleibow) — Fills in the wide-gamut colour surface (skia: mono/skia#74).
- **Improved Perlin noise `SKShader` binding** ([#1056](https://github.com/mono/SkiaSharp/pull/1056) by @mattleibow) — Exposes the improved-noise variant of the Perlin shader.
- **Bind more of `SKCanvas`, `SKImage`, and `SKRegion`** ([#1090](https://github.com/mono/SkiaSharp/pull/1090), [#1126](https://github.com/mono/SkiaSharp/pull/1126), [#1125](https://github.com/mono/SkiaSharp/pull/1125) by @mattleibow) — Adds several previously missing overloads to the drawing surface.

## API Surface

- **Writable generic span for pixels** ([#1242](https://github.com/mono/SkiaSharp/pull/1242) by @mattleibow) — Exposes a strongly-typed writable `Span<T>` for bitmap pixels, allowing allocation-free pixel manipulation.
- **Merge filter factory accepts nulls** ([#1185](https://github.com/mono/SkiaSharp/pull/1185) by @ziriax) — `SKImageFilter.CreateMerge` overloads now accept null arguments to make caller composition easier.
- **Rename some methods for .NET conventions** ([#1255](https://github.com/mono/SkiaSharp/pull/1255) by @mattleibow) — Aligns a handful of member names with .NET style guidelines.

## Bug Fixes

- **Prevent GC from collecting `this` and pixel arrays** ([#1258](https://github.com/mono/SkiaSharp/pull/1258) by @mattleibow) — Pins the correct instances during callback lifetimes to avoid rare use-after-free.
- **Null out fields on disposal** ([#1256](https://github.com/mono/SkiaSharp/pull/1256) by @mattleibow) — Fixes issue [#960](https://github.com/mono/SkiaSharp/issues/960) where a stale reference could resurrect a disposed object.
- **Concurrency fixes** ([#1200](https://github.com/mono/SkiaSharp/pull/1200) by @mattleibow) — Resolves several race conditions in the object cache.
- **Don't dispose system fonts** ([#1224](https://github.com/mono/SkiaSharp/pull/1224) by @mattleibow) — System fonts are shared refs and must not be finalised.
- **WGL deadlock and unit-test crashes** ([#1237](https://github.com/mono/SkiaSharp/pull/1237) by @mattleibow) — Ports [#1228](https://github.com/mono/SkiaSharp/pull/1228) to fix a Windows GL context deadlock.
- **`SKTouchHandler` crash from wrong parameter** ([#1212](https://github.com/mono/SkiaSharp/pull/1212) by @mscherotter) — Adds pressure and eraser handling while fixing a crash caused by an incorrect callback parameter.
- **Buffer-overflow cherry-picks** ([#1037](https://github.com/mono/SkiaSharp/pull/1037) by @mattleibow) — Ports a set of upstream buffer-overflow fixes into the 1.68 line.
- **Correctly decode images** ([#1055](https://github.com/mono/SkiaSharp/pull/1055) by @mattleibow) — Fixes an image-decoding regression.

## Platform

- **Re-work views to avoid incorrect disposal** ([#1180](https://github.com/mono/SkiaSharp/pull/1180) by @mattleibow) — Rewrites the shared view code so views survive detach/reattach correctly.
- **UWP `SKSwapChainPanel` antialiasing** ([#1086](https://github.com/mono/SkiaSharp/pull/1086) by @validvoid) — Restores broken antialiasing on the UWP swap-chain panel.
- **`SKGLViewRenderer` disposal corrections** ([#1095](https://github.com/mono/SkiaSharp/pull/1095) by @zbyszekpy) — Fixes over-eager disposal that broke re-created renderers.
- **macOS `SKGLView` fixes and multi-target Windows Forms** ([#1083](https://github.com/mono/SkiaSharp/pull/1083), [#1082](https://github.com/mono/SkiaSharp/pull/1082) by @mattleibow) — Addresses macOS `SKGLView` issues and multi-targets the Windows Forms package.
- **Android view refresh on recreation** ([#1234](https://github.com/mono/SkiaSharp/pull/1234) by @mattleibow) — Redraws the surface when the Android view is recreated.
- **GTK views for Xamarin.Forms** ([#1089](https://github.com/mono/SkiaSharp/pull/1089) by @mattleibow) — Adds the GTK backend to Forms (skia: mono/skia#72).
- **WPF Mouse Wheel on Touch Event** ([#1079](https://github.com/mono/SkiaSharp/pull/1079) by @alexandrvslv) — Routes mouse-wheel input through the WPF touch pipeline.

## Lifecycle & Internals

- **Remove `[Preserve]` attribute** ([#1229](https://github.com/mono/SkiaSharp/pull/1229) by @mattleibow) — Cleans up a linker attribute that is no longer required.
- **Remove reflection for `SKObject` creation** ([#1209](https://github.com/mono/SkiaSharp/pull/1209) by @Gillibald) — Replaces reflection with generated dispatchers, cutting startup cost.
- **`TryRemove` weak references** ([#1039](https://github.com/mono/SkiaSharp/pull/1039) by @daltonks) — Uses `TryRemove` on the weak-reference cache to avoid the previous throw-and-catch pattern.
- **VS 2019 / .NET Core 3.0** ([#1030](https://github.com/mono/SkiaSharp/pull/1030) by @mattleibow) — Upgrades the SDK stack for the build.
