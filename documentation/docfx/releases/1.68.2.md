# Version 1.68.2

## Highlights

Servicing rev on the 1.68 line that quietly landed a lot: GTK views for Xamarin.Forms, `SKColorF` / `SKColorSpace` overloads, `SKRegion`/`SKImage` API surface expansion, and a rewrite of view lifetime handling on Apple, Android, and UWP. Ships alongside [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md).

- 🎨 **Core** — `SKColorF`/`SKColorSpace` overloads added (see [mono/skia#74](https://github.com/mono/skia/pull/74)); `SKImage` and `SKRegion` gained several missing members; generic writable-`Span<T>` pixel access; the reflection-based `SKObject` factory was removed (❤️ @Gillibald); paint text-encoding now switches instead of throwing on mismatch.
- 🍎 **Apple** — `[Preserve]` no longer needed on the views; MacOS `SKGLView` fixes; ensure `MakeCurrent` before drawing.
- 🤖 **Android** — surface refresh on recreation; views release Skia objects on unload.
- 🪟 **Windows / WPF / UWP** — WGL deadlock fixed; UWP `SKSwapChainPanel` anti-aliasing regression fixed (❤️ @validvoid); WPF mouse-wheel-as-touch (❤️ @alexandrvslv); WinForms views multi-targeted.
- 🖼️ **Xamarin.Forms** — GTK backend added for Xamarin.Forms; `SKTouchHandler` fixes plus device-type support (touch/mouse/pen/eraser) (❤️ @mscherotter).
- 🏗️ **Build / CI** — ANGLE now built with vcpkg; expat native library updated to 2.2.9; cake updated to 0.37; cgmanifest completeness; new Tizen overload for later Xamarin.Forms.
- 📦 **General** — Code of Conduct linked (❤️ @terrajobst).

## Breaking Changes

- **SkiaSharp**
  - **SKSize / SKSizeI** — parameter names on `op_Equality` were normalized. Named-argument calls (`SKSize.op_Equality(a: …, b: …)`) may need to update; positional calls are unaffected.
- **SkiaSharp.Views.Forms** — the Tizen `SKImageSourceHandler.LoadImageAsync` overload targeting Xamarin.Forms was removed as part of the Tizen refactor; use the current handler surface.

## Contributors

Thanks to the community: ❤️ @Gillibald (reflection-free `SKObject` factory), ❤️ @ziriax (merge-filter `null` support), ❤️ @mscherotter (touch-handler pen/eraser + device-type), ❤️ @zbyszekpy (`SKGLViewRenderer` disposal), ❤️ @validvoid (UWP swap-chain AA), ❤️ @alexandrvslv (WPF mouse wheel), ❤️ @terrajobst (Code of Conduct link).
