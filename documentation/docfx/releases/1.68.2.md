# Version 1.68.2

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.2)

> **API changes** · [SkiaSharp API diff](1.68.2/index.md) · [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md)

<!-- Generated: 2026-07-06T23:24:50Z by claude-opus-4.7 -->

Second release on the 1.68 line. Removes the reflection-based `SKObject` creation path, ships new `SKShader` Perlin-noise and `SKColorF`+`SKColorSpace` overloads, expands `SKImage`/`SKRegion` bindings, and cleans up view lifecycles across UWP, WPF (mouse-wheel-as-touch), Xamarin.Forms/GTK, and macOS. HarfBuzzSharp 2.6.1.2 co-ships — see the [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md) page for its binding details.

## Highlights

- Reflection is out of the `SKObject` creation path, contributed by [@Gillibald](https://github.com/Gillibald) ([#1209](https://github.com/mono/SkiaSharp/pull/1209)) — trims linker output and improves cold-start allocation profile.
- `SKShader` Perlin-noise binding lands ([#1056](https://github.com/mono/SkiaSharp/pull/1056)); new `SKColorF`+`SKColorSpace` overloads expose the wide-color surface ([#1116](https://github.com/mono/SkiaSharp/pull/1116)); more `SKImage`, `SKCanvas`, and `SKRegion` members bound ([#1090](https://github.com/mono/SkiaSharp/pull/1090), [#1125](https://github.com/mono/SkiaSharp/pull/1125), [#1126](https://github.com/mono/SkiaSharp/pull/1126)); merge-filter-factory overloads accept nulls, contributed by [@ziriax](https://github.com/ziriax) ([#1185](https://github.com/mono/SkiaSharp/pull/1185)).
- Touch events now carry device type (touch/mouse/pen), pressure, and eraser data, contributed by [@mscherotter](https://github.com/mscherotter) ([#1191](https://github.com/mono/SkiaSharp/pull/1191), [#1212](https://github.com/mono/SkiaSharp/pull/1212)); WPF mouse-wheel now surfaces as a touch event, contributed by [@alexandrvslv](https://github.com/alexandrvslv) ([#1079](https://github.com/mono/SkiaSharp/pull/1079)).
- Antialiasing regression on UWP `SKSwapChainPanel` fixed, contributed by [@validvoid](https://github.com/validvoid) ([#1086](https://github.com/mono/SkiaSharp/pull/1086)); GTK views for Xamarin.Forms land ([#1089](https://github.com/mono/SkiaSharp/pull/1089)).
- Cherry-picked buffer-overflow fixes ([#1037](https://github.com/mono/SkiaSharp/pull/1037)); WPF-on-.NET Core sample ([#1140](https://github.com/mono/SkiaSharp/pull/1140)); Docker builds for Linux ([#1036](https://github.com/mono/SkiaSharp/pull/1036)).
- HarfBuzzSharp 2.6.1.2 ships alongside — see the linked page for binding details.

## API Surface

- `SKShader` binding for improved Perlin noise ([#1056](https://github.com/mono/SkiaSharp/pull/1056)); `SKColorF`+`SKColorSpace` overloads ([#1116](https://github.com/mono/SkiaSharp/pull/1116)).
- More `SKImage`, `SKCanvas`, and `SKRegion` members bound ([#1090](https://github.com/mono/SkiaSharp/pull/1090), [#1125](https://github.com/mono/SkiaSharp/pull/1125), [#1126](https://github.com/mono/SkiaSharp/pull/1126)); generator/struct improvements ([#1122](https://github.com/mono/SkiaSharp/pull/1122), [#1123](https://github.com/mono/SkiaSharp/pull/1123)); merge-filter-factory overloads accept nulls ([#1185](https://github.com/mono/SkiaSharp/pull/1185)); generic writable span exposed for pixel data ([#1242](https://github.com/mono/SkiaSharp/pull/1242)); some methods renamed to be more .NET-idiomatic ([#1255](https://github.com/mono/SkiaSharp/pull/1255)); a Tizen overload added for newer Forms ([#1117](https://github.com/mono/SkiaSharp/pull/1117)); unused `lang` parameter removed ([#1190](https://github.com/mono/SkiaSharp/pull/1190)).

## Engine

- expat native library bumped to 2.2.9 ([#1214](https://github.com/mono/SkiaSharp/pull/1214)); Cake bumped to 0.37 ([#1215](https://github.com/mono/SkiaSharp/pull/1215)); tooling refreshed ([#1175](https://github.com/mono/SkiaSharp/pull/1175)); ANGLE now built via vcpkg ([#1158](https://github.com/mono/SkiaSharp/pull/1158)).

## GPU & Rendering

- `MakeCurrent` before drawing to keep the GL context stable ([#1141](https://github.com/mono/SkiaSharp/pull/1141)); redraw is queued instead of run inline to fix #1074 ([#1135](https://github.com/mono/SkiaSharp/pull/1135)); prevent a WGL deadlock ([#1237](https://github.com/mono/SkiaSharp/pull/1237)); don't pick the maximum MSAA sample count blindly ([#1060](https://github.com/mono/SkiaSharp/pull/1060)); GTK views for Xamarin.Forms ([#1089](https://github.com/mono/SkiaSharp/pull/1089)); UWP `SKSwapChainPanel` antialiasing regression fixed ([#1086](https://github.com/mono/SkiaSharp/pull/1086)); macOS `SKGLView` issues cleaned up ([#1083](https://github.com/mono/SkiaSharp/pull/1083)).

## Platform

- WPF-on-.NET Core sample ([#1140](https://github.com/mono/SkiaSharp/pull/1140)); make sure the view is still in the hierarchy before rendering ([#1143](https://github.com/mono/SkiaSharp/pull/1143)); a couple more Xamarin.Forms Tizen fixes ([#1117](https://github.com/mono/SkiaSharp/pull/1117)); Xamarin.Forms bumped to 4.2 ([#1109](https://github.com/mono/SkiaSharp/pull/1109)); use the correct OpenJDK URL on Linux ([#1245](https://github.com/mono/SkiaSharp/pull/1245)).
