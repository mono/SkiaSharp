# Version 1.68.2

> [NuGet](https://www.nuget.org/packages/SkiaSharp/1.68.2)

> **API changes** · [SkiaSharp API diff](1.68.2/index.md) · [HarfBuzzSharp 2.6.1.2](harfbuzzsharp/2.6.1.2.md)

## Highlights

The 1.68.2 stable release folds in GTK views for Xamarin.Forms, Improved Perlin noise, an `SKColorF`/`SKColorSpace` overload sweep, a broad set of GC/disposal fixes, and generic writable-span pixel access. Community contributions cover reflection removal, touch device-type support, WPF mouse-wheel handling, UWP antialiasing, and merge-filter null overloads. HarfBuzzSharp 2.6.1.2 co-ships — see the HarfBuzz link on the API changes line above for its per-line page.

## Breaking Changes

Some `SkiaSharp` and `SkiaSharp.Views.Forms` signatures were renamed for .NET conventions. See the linked breaking-changes files for the exact deltas.

## New Features

- **`SKColorF` + `SKColorSpace` overloads** — Sweeping addition across the API surface. ([#1116](https://github.com/mono/SkiaSharp/pull/1116))
- **Improved Perlin noise shader** — `SKShader` factory for the improved variant. ([#1056](https://github.com/mono/SkiaSharp/pull/1056))
- **Generic writable span for pixels** — Cleaner zero-copy access. ([#1242](https://github.com/mono/SkiaSharp/pull/1242))
- **More `SKImage` and `SKRegion` members** ([#1125](https://github.com/mono/SkiaSharp/pull/1125), [#1126](https://github.com/mono/SkiaSharp/pull/1126))
- **`SKCanvas` completeness** — A few more members. ([#1090](https://github.com/mono/SkiaSharp/pull/1090))
- **GTK views for Xamarin.Forms** — First-class GTK renderers. ([#1089](https://github.com/mono/SkiaSharp/pull/1089))
- **Touch device type on `SKTouch`** — Distinguishes mouse, pen, and touch. ❤️ [@mscherotter](https://github.com/mscherotter) ([#1191](https://github.com/mono/SkiaSharp/pull/1191))
- **`Merge` filter accepts `null` arguments** ❤️ [@ziriax](https://github.com/ziriax) ([#1185](https://github.com/mono/SkiaSharp/pull/1185))
