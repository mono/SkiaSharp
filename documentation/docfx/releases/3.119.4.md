# Version 3.119.4

> **Platform expansion release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.4-preview.1.1) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.4-preview.1.1)

## Highlights

This release significantly expands platform support with new GTK 4 view bindings, Linux Bionic native assets, and Tizen x64/arm64 native builds. Windows users benefit from D3D12 delay-loading to prevent crashes on systems without DirectX 12, and Android users get a fix for `SKGLView` not rendering after tab switches. Community contributors ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry), [@SimonvBez](https://github.com/SimonvBez), and [@ramezgerges](https://github.com/ramezgerges) drove key improvements in this release.

## ⚠️ Breaking Changes

- **Minimum .NET Framework raised to net48** — The full framework target has been updated from net462 to net48. Projects targeting older .NET Framework versions will need to retarget. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3631](https://github.com/mono/SkiaSharp/pull/3631))

## New Features

### Platform

- **GTK 4 view bindings** — New `SkiaSharp.Views.Gtk4` package using GirCore.Gtk-4.0 for native GTK 4 support on Linux. ([#3527](https://github.com/mono/SkiaSharp/pull/3527))
- **Linux Bionic native assets** — Adds Android Bionic library build support for native Linux builds. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 native builds** — Expands Tizen platform support beyond the existing ARM build to include x64 and arm64 architectures. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **Update to .NET 10 SDK** — Moves the build infrastructure to .NET 10 with workload version set pinning. ([#3514](https://github.com/mono/SkiaSharp/pull/3514))

### API Surface

- **PolySharp for C# 13 on legacy TFMs** — Enables modern C# language features when targeting older frameworks via PolySharp polyfills. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))
- **SKManagedStream refactor** — Replaces the parent/child chain with an SKData snapshot for more reliable duplicate/fork behavior. ([#3589](https://github.com/mono/SkiaSharp/pull/3589))

## Bug Fixes

- **Fix SKGLView not rendering after tab switch on Android** — Resolves an issue where `SKGLView` was not re-rendered when switching tabs in a MAUI TabBar. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **Delay-load D3D12 DLLs on Windows** — Prevents crashes on Windows systems without DirectX 12 by delay-loading `d3d12.dll`. ([#3633](https://github.com/mono/SkiaSharp/pull/3633))
- **Fix tvOS device/simulator native builds** — Splits tvOS native builds into separate device and simulator variants to match the iOS pattern, fixing build and runtime issues. ([#3561](https://github.com/mono/SkiaSharp/pull/3561), [#3563](https://github.com/mono/SkiaSharp/pull/3563))
- **Fix Cairo assertion crash in GTK 4** — Resolves an assertion failure in the GTK 4 `SKDrawingArea`. ([#3562](https://github.com/mono/SkiaSharp/pull/3562))
- **Fix loongarch64 Linux builds** — Corrects the architecture mapping for LoongArch64 Linux builds. ([#3648](https://github.com/mono/SkiaSharp/pull/3648))
- **Enforce `--no-undefined` linker flag** — Ensures all symbols are resolved at link time for Linux and Android native builds, catching missing symbols earlier. ([#3629](https://github.com/mono/SkiaSharp/pull/3629))
- **Fix WindowsForms DesignMode detection** — Uses the `DesignMode` property directly for more reliable Visual Studio designer detection. ([#3614](https://github.com/mono/SkiaSharp/pull/3614))

Plus several CI, sample, and documentation improvements.
