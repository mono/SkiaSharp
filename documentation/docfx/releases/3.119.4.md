# Version 3.119.4

> **Platform reach, samples, docs** · Released May 22, 2026 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.4) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.4)

> **Supersedes [3.119.3](3.119.3.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](3.119.4/index.md) · [HarfBuzzSharp 8.3.1.5](harfbuzzsharp/8.3.1.5.md)

## Highlights

Focused stable that supersedes the never-released 3.119.3 preview and rolls
up its work cumulatively. Native reach grows with Linux Bionic and Tizen
x64/arm64 assets. The samples are rewritten across every platform, a new
`SkiaSharp.Views.Gtk4` package lands, and `SKManagedStream` is rebuilt on
`SKData` for safer AoT and multi-thread scenarios. The shipped native
`libpng`, `libwebp`, `brotli`, and `libexpat` are refreshed. Four community
contributors landed at least one change. Ships with
[HarfBuzzSharp 8.3.1.5](harfbuzzsharp/8.3.1.5.md).

## Breaking Changes

- **`SkiaSharp.Views.Blazor._Imports` no longer derives from
  `ComponentBase`** — the internal Blazor `_Imports` type is regenerated
  without a component base and no longer implements `IComponent`,
  `IHandleAfterRender`, or `IHandleEvent`. This type is generated by the
  Razor tooling and is not meant to be consumed directly; no app code should
  be affected. See the [3.119.4 API diff](3.119.4/index.md#breaking-changes)
  for the full signature list.

## Native Engine

- **`libpng` → 1.6.58, `libwebp` → 1.6.0, `brotli` → 1.2.0** — routine
  native-dependency refresh for the codec chain.
  ([#3723](https://github.com/mono/SkiaSharp/pull/3723),
  [#3478](https://github.com/mono/SkiaSharp/pull/3478),
  [#3469](https://github.com/mono/SkiaSharp/pull/3469),
  [#3452](https://github.com/mono/SkiaSharp/pull/3452),
  [mono/skia#192](https://github.com/mono/skia/pull/192),
  [mono/skia#167](https://github.com/mono/skia/pull/167),
  [mono/skia#162](https://github.com/mono/skia/pull/162))
- **Unused Skia deps removed** — the Skia dependency graph is pruned of
  externals we do not use, keeping the native binary smaller.
  ([#3463](https://github.com/mono/SkiaSharp/pull/3463),
  [mono/skia#164](https://github.com/mono/skia/pull/164))
- **`-Wl,--no-undefined` enforced on Linux / Android native builds** —
  unresolved symbols now fail the link instead of surfacing as runtime
  crashes.
  ([#3629](https://github.com/mono/SkiaSharp/pull/3629),
  [mono/skia#176](https://github.com/mono/skia/pull/176))

## API Surface

- **New: `SkiaSharp.Views.Gtk4`** — a Gtk 4 view package built on the
  GirCore Gtk-4.0 bindings, next to the existing Gtk 3 package.
  ([#3527](https://github.com/mono/SkiaSharp/pull/3527))
- **`SKManagedStream` rebuilt on `SKData` snapshots** — the managed stream
  wrapper no longer chains parent/child managed handles; it snapshots into
  an `SKData` up front, which is safer under AoT and under concurrent
  disposal. ([#3589](https://github.com/mono/SkiaSharp/pull/3589))

## Platform

## Bug Fixes

- **Android: `SKGLView` re-renders after a MAUI TabBar tab switch** — the
  GL view no longer goes blank when the user tabs away and back in a
  MAUI `TabBar`. ❤️ [@SimonvBez](https://github.com/SimonvBez)
  ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **AoT: no crash when `IlcDisableReflection=true`** — the AoT publish path
  no longer crashes when reflection is disabled at trim time.
  ([#3485](https://github.com/mono/SkiaSharp/pull/3485))
- **GTK 4: no more Cairo assertion crash in `SKDrawingArea`** — the GTK 4
  drawing area no longer trips a Cairo assertion at render time.
  ([#3562](https://github.com/mono/SkiaSharp/pull/3562))
- **tvOS device / simulator native frameworks split correctly** — the tvOS
  build now ships separate device and simulator frameworks, matching the
  iOS layout, so tvOS apps link the right slice.
  ([#3561](https://github.com/mono/SkiaSharp/pull/3561),
  [#3563](https://github.com/mono/SkiaSharp/pull/3563))
- **Windows: D3D12 DLLs delay-loaded** — SkiaSharp no longer fails to load
  on Windows systems without DirectX 12 present; the D3D12 DLLs are now
  delay-loaded on demand.
  ([#3633](https://github.com/mono/SkiaSharp/pull/3633))
- **Linux ARM64: fontconfig runtime library present in cross-sysroot** —
  fixes a Linux-ARM64 crash caused by the runtime `fontconfig` library
  missing from the cross-compile sysroot.
  ([#3494](https://github.com/mono/SkiaSharp/pull/3494))
- **WinForms: use `DesignMode` directly** — the WinForms controls use the
  built-in `DesignMode` property instead of a custom check, fixing false
  positives inside designers.
  ([#3614](https://github.com/mono/SkiaSharp/pull/3614))
- **`loongarch` Linux build mapping fixed** — the Linux CI matrix now maps
  `loongarch` correctly, so the LoongArch64 slice ships every build.
  ([#3648](https://github.com/mono/SkiaSharp/pull/3648))
