# Version 3.119.4

> **Platform expansion release** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.4-preview.1.1) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.4-preview.1.1)

## Highlights

This release dramatically expands platform reach with new GTK 4 views, Linux Bionic native support, Tizen multi-architecture builds, and .NET 10 SDK adoption. Three community contributors drove key improvements: ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) added Linux Bionic support, C# 13 language features on legacy TFMs, and .NET Framework 4.8 targeting, while ❤️ [@SimonvBez](https://github.com/SimonvBez) fixed an Android rendering regression in MAUI TabBar.

## Breaking Changes

*None in this release.*

## New Features

### Platform

- **GTK 4 views** — New `SkiaSharp.Views.Gtk4` package using GirCore.Gtk-4.0 bindings brings first-class GTK 4 support to SkiaSharp. ([#3527](https://github.com/mono/SkiaSharp/pull/3527))
- **Linux Bionic native support** — Adds native assets for Android's Bionic libc on Linux, enabling SkiaSharp in Bionic-based environments. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64** — Native build support for Tizen on both x64 and arm64 architectures. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **D3D12 delay-loading** — D3D12 DLLs are now delay-loaded on Windows to prevent crashes on systems without DirectX 12. ([#3633](https://github.com/mono/SkiaSharp/pull/3633))

### API Surface

- **SKManagedStream refactor** — Replaced the parent/child chain with an `SKData` snapshot for duplicate/fork operations, improving reliability and simplifying the stream lifecycle. ([#3589](https://github.com/mono/SkiaSharp/pull/3589))

### Build & Compatibility

- **.NET 10 SDK** — Updated to .NET 10 SDK with workload version set pinning. ([#3514](https://github.com/mono/SkiaSharp/pull/3514))
- **C# 13 on legacy TFMs** — Added PolySharp to enable C# 13 language features on older target frameworks. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))
- **.NET Framework 4.8** — Updated full framework target from net462 to net48. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3631](https://github.com/mono/SkiaSharp/pull/3631))
- **Stricter native linking** — Enforced `--no-undefined` linker flag for Linux and Android native builds to catch missing symbols at build time. ([#3629](https://github.com/mono/SkiaSharp/pull/3629))

## Bug Fixes

- **Fixed SKGLView not rendering after tab switch** — Resolved an issue where `SKGLView` on Android would stop rendering when switching tabs in a MAUI TabBar. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **Fixed tvOS native builds** — Split device and simulator native builds to match the iOS pattern, resolving build failures. ([#3561](https://github.com/mono/SkiaSharp/pull/3561), [#3563](https://github.com/mono/SkiaSharp/pull/3563))
- **Fixed GTK 4 Cairo assertion crash** — Resolved a Cairo assertion crash in `GTK4 SKDrawingArea`. ([#3562](https://github.com/mono/SkiaSharp/pull/3562))
- **Fixed loongarch64 Linux builds** — Corrected architecture mapping for LoongArch64 Linux builds. ([#3648](https://github.com/mono/SkiaSharp/pull/3648))
- **Fixed WinForms designer detection** — Uses `DesignMode` property directly in WindowsForms controls for more reliable designer detection. ([#3614](https://github.com/mono/SkiaSharp/pull/3614))

Plus several CI, documentation, sample, and tooling improvements.
