## Highlights

Big servicing rev on the 3.119 line. It **supersedes 3.119.3** (which stayed preview-only), so all of that work — Spectre-mitigated Linux/Windows natives, AoT-with-reflection-disabled fixes, cross-compile fontconfig fix — is included here cumulatively, along with .NET 10 SDK adoption, GTK 4 views, MSIX-packaged WinUI, delay-loaded D3D12, a fully redesigned Gallery, and native-dep bumps for libpng / libwebp / libexpat / brotli. Ships alongside [HarfBuzzSharp 8.3.1.5](harfbuzzsharp/8.3.1.5.md).

- 🎨 **Core** — `SKManagedStream` internals rebuilt on top of `SKData` snapshots (❤️ @Copilot); AoT crash fix when `IlcDisableReflection=true` (❤️ @Copilot); Skia-side dep bumps: [libpng 1.6.58](https://github.com/mono/skia/pull/192), [libpng 1.6.54](https://github.com/mono/skia/pull/162), [libwebp 1.6.0](https://github.com/mono/skia/pull/167), libexpat 2.7.3, brotli 1.2.0, unused-dep cleanup ([mono/skia#164](https://github.com/mono/skia/pull/164)); [`-Wl,--no-undefined`](https://github.com/mono/skia/pull/176) linker enforcement on Linux/Android; [Spectre mitigation](https://github.com/mono/SkiaSharp/pull/3502) flag on `libSkiaSharp.so` and `libSkiaSharp.dll` (❤️ @sshumakov).
- 🐧 **Linux** — Linux Bionic native assets support (❤️ @4Darmygeometry); loong64 build path fixed; libfontconfig-dev updated for loong64 (❤️ @Copilot); Linux ARM64 fontconfig sysroot cross-compile crash fixed.
- 🤖 **Android** — `SKGLView` no longer stops rendering after a MAUI `TabBar` tab-switch (❤️ @SimonvBez); Android CI disk-space and emulator startup improvements (❤️ automation).
- 🪟 **Windows** — D3D12 DLLs are now delay-loaded so SkiaSharp doesn't crash on machines without DX12; WinUI 3 sample now MSIX-packaged; `net48` full-framework target added for Windows validation (❤️ @4Darmygeometry).
- 🍏 **Apple** — tvOS `NativeReference` now splits device/simulator frameworks correctly, matching the iOS pattern.
- 🍽️ **Tizen** — Studio 6.1 support, TFM fixes, x64/arm64 native builds ([mono/skia#175](https://github.com/mono/skia/pull/175)), and split per-architecture native CI.
- 🌐 **WASM** — [WASM 2.0.6](https://github.com/mono/SkiaSharp/pull/4045); WASM natives built directly in the docs workflow; Docker WASM build scripts fixed for ARM hosts.
- 🖼️ **Gallery / Samples** — Gallery redesigned as a Blazor WASM showcase with 21 interactive demos (❤️ @Copilot); every platform sample refreshed with multi-page layouts and READMEs (Android Material 3, iOS split view, Catalyst/tvOS/macOS tab bars, WPF/WinForms/WinUI3/Blazor/MAUI/Console/GTK 3/GTK 4/Browser WASM/Web).
- 🖼️ **New views** — new [`SkiaSharp.Views.Gtk4`](https://github.com/mono/SkiaSharp/pull/3527) package using `GirCore.Gtk-4.0` bindings (❤️ @Copilot).
- 🏗️ **Build / CI** — updated to .NET 10 SDK (10.0.105 pinned), workload version-set pinning, `PolySharp` for C# 13 on legacy TFMs (❤️ @4Darmygeometry), NuGet mega-package split (<500 MB feed limit), WPF sample uses `DesignMode` directly, `generate.ps1` propagates errors, generator picks up macOS SDK system includes.
- 📦 **Docs / site** — homepage redesigned (hero + code showcase + gallery + resources); landing page and unified deployment moved to `main`; docs deploy race fixed via serialized "Go Live"; docs moved into `documentation/dev/`; daily API-docs generation now runs.

## Breaking Changes

- **SkiaSharp.Views.Blazor** — `_Imports` no longer derives from `Microsoft.AspNetCore.Components.ComponentBase` (it is now a plain `System.Object`) and no longer implements `IComponent`, `IHandleAfterRender`, or `IHandleEvent`. This is an auto-generated Razor imports helper — user code should not have inherited from it, but any project that referenced it as a component must switch to a proper Razor component instead.
