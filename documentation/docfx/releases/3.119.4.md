# Version 3.119.4

> **Platform expansion and stability** · Preview only · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.4-preview.1.1) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.4-preview.1.1)

## Highlights

SkiaSharp 3.119.4 expands platform support with a new GTK 4 view package, Tizen x64/arm64 native builds, and Android Bionic native library support. The build toolchain moves to .NET 10 SDK and the .NET Framework minimum target is updated to net48 with C# 13 language feature support via PolySharp. Several stability fixes address crashes on Windows (D3D12 delay-loading), GTK 4, tvOS, and Android. Community contributors [@4Darmygeometry](https://github.com/4Darmygeometry) and [@SimonvBez](https://github.com/SimonvBez) drove key platform and rendering improvements.

## Breaking Changes

*None in this release.*

## New Features

### Platform

- **SkiaSharp.Views.Gtk4** — New GTK 4 view package using GirCore.Gtk-4.0 bindings, bringing first-class Linux desktop support via the modern GTK toolkit. ([#3527](https://github.com/mono/SkiaSharp/pull/3527))
- **Android Bionic native library support** — Adds native assets for Linux Bionic targets used by Android. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 native builds** — Extends native build support to Tizen x64 and arm64 architectures. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **.NET 10 SDK** — Updates the build toolchain to .NET 10 SDK with workload version set pinning. ([#3514](https://github.com/mono/SkiaSharp/pull/3514))

### API Surface

- **C# 13 language features on legacy TFMs** — Adds PolySharp to enable C# 13 language features on .NET Framework 4.8 and other older target frameworks. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3642](https://github.com/mono/SkiaSharp/pull/3642))
- **.NET Framework 4.8 targeting** — Updates the full framework minimum target from net472 to net48. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3631](https://github.com/mono/SkiaSharp/pull/3631))

## Bug Fixes

- **Fixed SKGLView not rendering after tab switch on Android** — Resolved an issue where `SKGLView` stopped rendering when switching tabs in a MAUI TabBar. ❤️ [@SimonvBez](https://github.com/SimonvBez) ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- **Delay-load D3D12 DLLs on Windows** — Prevents crashes on systems without DirectX 12 by delay-loading D3D12 DLLs. ([#3633](https://github.com/mono/SkiaSharp/pull/3633))
- **Fixed Cairo assertion crash in GTK 4** — Resolved a crash in `SKDrawingArea` caused by a Cairo assertion in GTK 4. ([#3562](https://github.com/mono/SkiaSharp/pull/3562))
- **Fixed tvOS device/simulator builds** — Split tvOS native builds and NativeReference targets to properly separate device and simulator frameworks, matching the iOS pattern. ([#3561](https://github.com/mono/SkiaSharp/pull/3561), [#3563](https://github.com/mono/SkiaSharp/pull/3563))
- **Fixed loongarch64 Linux builds** — Corrected the architecture mapping for loongarch64 Linux native builds. ([#3648](https://github.com/mono/SkiaSharp/pull/3648))
- **Fixed WinForms designer mode** — Uses `DesignMode` property directly in Windows Forms controls to avoid runtime issues. ([#3614](https://github.com/mono/SkiaSharp/pull/3614))
- **Refactored SKManagedStream** — Replaced parent/child chain with SKData snapshot for duplicate/fork operations, improving stream reliability. ([#3589](https://github.com/mono/SkiaSharp/pull/3589))
- **Enforced `--no-undefined` linker flag** — Linux and Android native builds now catch missing symbols at link time rather than at runtime. ([#3629](https://github.com/mono/SkiaSharp/pull/3629))
