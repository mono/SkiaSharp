# Version 3.119.4

> **Major platform expansion and sample showcase overhaul** · Released April 23, 2026 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.4-preview.1.1)

## Highlights

This release dramatically expands platform reach with new GTK4 views, Linux Bionic support, and Tizen x64/arm64 native builds. A D3D12 delay-loading fix eliminates crashes on Windows systems without DirectX 12. On the developer experience side, PolySharp enables C# 13 language features on older target frameworks, the sample gallery has been completely redesigned as an interactive Blazor WASM showcase, and SKManagedStream received a significant internal refactor. Two community contributors ([@4Darmygeometry](https://github.com/4Darmygeometry) and [@SimonvBez](https://github.com/SimonvBez)) helped make this release possible.

## Breaking Changes

*None in this release.*

## New Features

### Platform

- **SkiaSharp.Views.Gtk4** — New GTK4 view integration using GirCore.Gtk-4.0 bindings, bringing first-class SkiaSharp support to modern GTK applications. ([#3527](https://github.com/mono/SkiaSharp/pull/3527))
- **Linux Bionic native assets** — Adds native library support for Linux Bionic (Android's C library on non-Android Linux). ❤️ @4Darmygeometry ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 builds** — Native build support for Tizen on both x64 and arm64 architectures, expanding IoT and embedded device coverage. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))

### GPU & Rendering

- **D3D12 delay-loading** — D3D12 DLLs are now delay-loaded to prevent crashes on Windows systems without DirectX 12 installed. ([#3633](https://github.com/mono/SkiaSharp/pull/3633))

### API Surface

- **PolySharp for C# 13** — Adds PolySharp so projects targeting older TFMs can use C# 13 language features with SkiaSharp. ❤️ @4Darmygeometry ([#3642](https://github.com/mono/SkiaSharp/pull/3642))
- **SKManagedStream refactor** — Replaced the parent/child chain in `SKManagedStream` with an `SKData` snapshot for duplicate/fork operations, improving reliability and simplifying internals. ([#3589](https://github.com/mono/SkiaSharp/pull/3589))

### Samples & Gallery

- **Interactive Gallery redesign** — The sample gallery has been rebuilt as an interactive Blazor WASM showcase with 21 demos, deployed to GitHub Pages. ([#3578](https://github.com/mono/SkiaSharp/pull/3578))
- **Platform sample overhaul** — Every platform sample (Android, iOS, macOS, Mac Catalyst, tvOS, WPF, WinForms, WinUI 3, GTK3, GTK4, Blazor, MAUI, Console, ASP.NET Core, Docker) has been modernized with multi-page layouts, improved drawing demos, and READMEs. ([#3590](https://github.com/mono/SkiaSharp/pull/3590)–[#3606](https://github.com/mono/SkiaSharp/pull/3606))

## Bug Fixes

- Fix SKGLView not rendering after tab switch in MAUI TabBar. ❤️ @SimonvBez ([#3076](https://github.com/mono/SkiaSharp/pull/3076))
- Fix tvOS native builds: split device/simulator builds to match iOS pattern. ([#3561](https://github.com/mono/SkiaSharp/pull/3561), [#3563](https://github.com/mono/SkiaSharp/pull/3563))
- Fix Cairo assertion crash in GTK4 `SKDrawingArea`. ([#3562](https://github.com/mono/SkiaSharp/pull/3562))
- Fix loongarch64 Linux build mapping. ([#3648](https://github.com/mono/SkiaSharp/pull/3648))
- Fix `DesignMode` property usage in WindowsForms controls. ([#3614](https://github.com/mono/SkiaSharp/pull/3614))
- Enforce `-Wl,--no-undefined` linker flag for Linux and Android native builds to catch missing symbols at link time. ([#3629](https://github.com/mono/SkiaSharp/pull/3629))
- Update `FullFrameworkTargetFrameworks` to net48. ❤️ @4Darmygeometry ([#3631](https://github.com/mono/SkiaSharp/pull/3631))
