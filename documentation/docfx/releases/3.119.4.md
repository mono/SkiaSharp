# Version 3.119.4

> [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.4)

> **Supersedes [3.119.3](3.119.3.md)** · Rolls up preview-only work that was never released as stable — those changes are included cumulatively below.

> **API changes** · [SkiaSharp API diff](3.119.4/index.md) · [HarfBuzzSharp 8.3.1.5](harfbuzzsharp/8.3.1.5.md)

## Highlights

3.119.4 is the fourth stable patch on the 3.119 line. It rolls up the 3.119.3 preview work (never released as stable) into a single cumulative release and adds a raft of infrastructure, packaging, and CI improvements on top. Tizen native binaries are now built for x64 and arm64, `PolySharp` powers C# 13 syntax on legacy TFMs, D3D12 DLLs are delay-loaded to keep SkiaSharp usable on machines without DirectX 12, a MAUI-only Android tab-switch rendering regression in `SKGLView` is fixed, and native crashes on AOT (with `IlcDisableReflection=true`), on cross-compiled Linux ARM64 (missing fontconfig), and in the GTK4 `SKDrawingArea` (Cairo assertion) are addressed. Native dependencies get another sweep — libpng 1.6.58, libwebp 1.6.0, libexpat 2.7.3, brotli 1.2.0, freetype 2.14.3, zlib 1.3.2.1-motley, plus Spectre mitigation flags on both `libSkiaSharp.so` and `libSkiaSharp.dll`. NuGet packages are split to stay under the 500 MB feed limit, and a large repository/documentation modernization ships alongside. HarfBuzzSharp 8.3.1.5 ships alongside this release — see [HarfBuzzSharp 8.3.1.5](harfbuzzsharp/8.3.1.5.md) for its details.

## Breaking Changes

None in this release.

The only public break flagged in the API diff is limited to `SkiaSharp.Views.Blazor` — an internal-facing signature adjustment on a small helper as part of the .NET 9 ASP.NET Blazor fixes. Callers using the documented public MAUI/Blazor view APIs should see no source or binary breaks.

See the [full API diff](3.119.4/index.md) for the exact signatures.
