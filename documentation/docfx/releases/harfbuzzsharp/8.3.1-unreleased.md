# HarfBuzzSharp 8.3.1

> **Upcoming release** · In development · Ships with the upcoming SkiaSharp 3.118.0 · Not yet available on NuGet

## Highlights

The HarfBuzzSharp binding that will ship alongside SkiaSharp 3.118.0 picks up the
WASM build reshuffle: the Blazor workaround was moved into the WASM target so
consumers no longer need to apply it in their apps. Nothing about the shaping
surface changes.

## Breaking Changes

None in this release.

## Platform

- **WASM Blazor workaround now built in** — the runtime fix that consumers used
  to apply themselves is folded into the WASM build, so Blazor apps consuming
  HarfBuzzSharp no longer need to carry the shim.
  ([#3082](https://github.com/mono/SkiaSharp/pull/3082),
  [#3088](https://github.com/mono/SkiaSharp/pull/3088))

Plus various CI, documentation, and internal tooling improvements.
