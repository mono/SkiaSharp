# HarfBuzzSharp 8.3.1.5

> **Bionic and Tizen native assets** · Ships with [SkiaSharp 3.119.4](../3.119.4.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.1.5) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.4)

> **API changes** · [HarfBuzzSharp API diff](8.3.1.5/index.md) · Ships with [SkiaSharp 3.119.4](../3.119.4.md)

## Highlights

Companion release for SkiaSharp 3.119.4. Extends HarfBuzz native asset coverage to Linux Bionic and to Tizen x64/arm64 (Tizen Studio 6.1), and fixes tvOS `NativeReference` targets to split device and simulator frameworks the same way iOS does. Also moves the build onto the .NET 10 SDK.

## Breaking Changes

None in this release — see the [HarfBuzzSharp API diff](8.3.1.5/index.md).

## Platform

- **Linux Bionic native assets** — HarfBuzz native binaries are now produced for Linux Bionic (Termux, Android userspace). ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 native builds** — HarfBuzz now builds and ships native binaries for Tizen x64 and arm64. ([#3620](https://github.com/mono/SkiaSharp/pull/3620), [skia#175](https://github.com/mono/skia/pull/175))
- **Tizen Studio 6.1 + TFM fixes + 3-tab sample** — updates the Tizen support to Studio 6.1 with corrected TFM handling. ([#3619](https://github.com/mono/SkiaSharp/pull/3619))
- **tvOS device/simulator framework split** — corrects `NativeReference` targets so device and simulator frameworks are packaged separately (matches iOS). ([#3561](https://github.com/mono/SkiaSharp/pull/3561), [#3563](https://github.com/mono/SkiaSharp/pull/3563))

## Lifecycle & Internals

- **.NET 10 SDK with workload version pinning** — the build now targets the .NET 10 SDK, with the workload version set pinned for reproducibility. ([#3514](https://github.com/mono/SkiaSharp/pull/3514))
