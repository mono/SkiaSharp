## Highlights

**HarfBuzzSharp 8.3.1.5** ships alongside [SkiaSharp 3.119.4](../3.119.4.md) and widens the native runtime matrix. Linux Bionic gets first-class native assets, Tizen adds x64 and arm64 builds with tooling refreshed to Tizen Studio 6.1, and the .NET 10 SDK becomes the pinned build baseline. tvOS packaging finally matches the iOS split-device/simulator layout so its native assets resolve correctly.

## Platform support

- **Linux Bionic** — Add native assets for Bionic-based Linux distributions so HarfBuzzSharp ships to that runtime (❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) in [#3217](https://github.com/mono/SkiaSharp/pull/3217)).
- **Tizen** — Add x64 and arm64 to the native build matrix ([#3620](https://github.com/mono/SkiaSharp/pull/3620), [mono/skia#175](https://github.com/mono/skia/pull/175)) and update the Tizen SDK to 6.1 with TFM fixes ([#3619](https://github.com/mono/SkiaSharp/pull/3619)).
- **tvOS** — Split device/simulator native builds and NativeReference targets so tvOS apps pick up the correct architecture-specific frameworks ([#3561](https://github.com/mono/SkiaSharp/pull/3561), [#3563](https://github.com/mono/SkiaSharp/pull/3563)).
- **.NET 10 SDK baseline** — Update SkiaSharp/HarfBuzzSharp to build against the .NET 10 SDK with workload version sets pinned ([#3514](https://github.com/mono/SkiaSharp/pull/3514)).
