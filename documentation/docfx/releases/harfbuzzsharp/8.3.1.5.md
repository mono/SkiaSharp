# HarfBuzzSharp 8.3.1.5

> Ships with [SkiaSharp 3.119.4](../3.119.4.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.1.5) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.4)

> **API changes** · [HarfBuzzSharp API diff](8.3.1.5/index.md) · Ships with [SkiaSharp 3.119.4](../3.119.4.md)

## Highlights

The HarfBuzzSharp binding that ships with SkiaSharp 3.119.4 adds Linux Bionic and
Tizen x64/arm64 to the native matrix, splits tvOS device/simulator native builds
to match the iOS layout, and moves the whole toolchain up to the .NET 10 SDK.

## Breaking Changes

None in this release.

## Platform

- **Linux Bionic native assets** — HarfBuzzSharp now ships pre-built native
  binaries for Linux Bionic (Android-style libc). ❤️
  [@4Darmygeometry](https://github.com/4Darmygeometry)
  ([#3217](https://github.com/mono/SkiaSharp/pull/3217))
- **Tizen x64 and arm64 native builds** — Tizen now has proper x64 and arm64
  native binaries and the Tizen support code was refreshed for Studio 6.1 with
  TFM fixes and an updated three-tab sample.
  ([#3620](https://github.com/mono/SkiaSharp/pull/3620),
  [#3619](https://github.com/mono/SkiaSharp/pull/3619),
  [mono/skia#175](https://github.com/mono/skia/pull/175))
- **tvOS: split device and simulator native builds** — tvOS now follows the same
  device/simulator split as iOS, and the `NativeReference` targets are wired up
  to select the right slice.
  ([#3561](https://github.com/mono/SkiaSharp/pull/3561),
  [#3563](https://github.com/mono/SkiaSharp/pull/3563))

## Lifecycle & Internals

- **.NET 10 SDK with pinned workload versions** — the build moves to the .NET 10
  SDK and pins its workload set for reproducible restores across CI and dev
  boxes. ([#3514](https://github.com/mono/SkiaSharp/pull/3514))
