# HarfBuzzSharp 2.6.1.8

> Ships with [SkiaSharp 2.80.3](../2.80.3.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/2.6.1.8) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v2.80.3)

> **API changes** · [HarfBuzzSharp API diff](2.6.1.8/index.md) · Ships with [SkiaSharp 2.80.3](../2.80.3.md)

## Highlights

This release improves the HarfBuzzSharp interop pipeline and extends native support across WebAssembly, musl-based Linux environments, and newer .NET runtimes.

## Breaking Changes

*None in this release.*

## New Features

- **Generated interop** — Moves HarfBuzz interop generation into the build, making the bindings easier to keep in sync. ([#1447](https://github.com/mono/SkiaSharp/pull/1447))

## Platform Support

- **WebAssembly coverage** — Ships all Emscripten variants needed for the WASM distributions. ([#1590](https://github.com/mono/SkiaSharp/pull/1590))
- **Musl detection** — Properly detects musl-based systems so the correct native assets can be loaded. ([#1657](https://github.com/mono/SkiaSharp/pull/1657))
- **Runtime and toolchain updates** — Adds .NET 5 support and keeps compatibility with older Visual Studio 2013 toolchains where possible. ([#1697](https://github.com/mono/SkiaSharp/pull/1697), [#1536](https://github.com/mono/SkiaSharp/pull/1536))
