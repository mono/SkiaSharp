The HarfBuzz binding that ships in the SkiaSharp 2.80.3 package family. This is primarily a packaging and platform-parity release: the interop layer was regenerated to match the SkiaSharp generator, and the native asset now targets .NET 5, VS 2013, and the full Emscripten matrix for WASM. See the API diff above for the exposed HarfBuzz surface at this milestone.

## Highlights

- Regenerated the HarfBuzz P/Invoke layer using the same generator that produces the SkiaSharp bindings, keeping signatures and naming in lockstep ([#1447](https://github.com/mono/SkiaSharp/pull/1447)).
- Built and included all supported Emscripten versions for the WASM native asset so Blazor and Uno consumers can pick the version their toolchain expects ([#1590](https://github.com/mono/SkiaSharp/pull/1590)).
- Adopted .NET 5 as the primary managed target for the binding ([#1697](https://github.com/mono/SkiaSharp/pull/1697)).
- Detects and loads the musl-flavoured native asset on Alpine and other musl-libc systems ([#1657](https://github.com/mono/SkiaSharp/pull/1657)).
- Restored VS 2013 compatibility for the native asset build ([#1536](https://github.com/mono/SkiaSharp/pull/1536)).
