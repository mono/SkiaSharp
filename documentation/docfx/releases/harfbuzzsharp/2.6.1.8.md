# HarfBuzzSharp 2.6.1.8

> Ships with [SkiaSharp 2.80.3](../2.80.3.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/2.6.1.8) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v2.80.3)

> **API changes** · [HarfBuzzSharp API diff](2.6.1.8/index.md) · Ships with [SkiaSharp 2.80.3](../2.80.3.md)

## Highlights

Ships as part of SkiaSharp 2.80.3 and consolidates the HarfBuzz binding around the generated interop. Adds musl-based Linux detection at the HarfBuzz native layer and rounds out the emscripten variants for WebAssembly.

## Breaking Changes

- **`OpenTypeMetrics` is now a reference type** — It was previously a struct (`ValueType`), and its `OpenTypeMetrics(IntPtr font)` constructor has been removed. Callers should get an instance from the owning `Font` rather than constructing one directly and update any `default(OpenTypeMetrics)` / by-value passing patterns.

## API Surface

- **Generate the interop with harfbuzz** ([#1447](https://github.com/mono/SkiaSharp/pull/1447) by @mattleibow) — Regenerates the HarfBuzz P/Invoke layer using the same generator SkiaSharp adopted on the 1.68 line.

## Platform

- **Build and include all the emscripten versions for WASM** ([#1590](https://github.com/mono/SkiaSharp/pull/1590) by @mattleibow) — Ships every supported emscripten variant of the HarfBuzz native.
- **Properly detect and handle musl systems** ([#1657](https://github.com/mono/SkiaSharp/pull/1657) by @mattleibow) — Picks the correct native at load time on Alpine and other musl distros.

## Lifecycle & Internals

- **Update to .NET 5.0** ([#1697](https://github.com/mono/SkiaSharp/pull/1697) by @mattleibow) — Targets .NET 5 alongside the rest of the SkiaSharp 2.80.3 stack.
