## Highlights

HarfBuzzSharp companion for SkiaSharp 2.80.3, folding in the HarfBuzz-touching work from that release line: a WASM/Emscripten rebuild pass, .NET 5 build support, Alpine/musl detection, and the switch to the generated HarfBuzz interop layer.

## Breaking Changes

- **`OpenTypeMetrics` is now a class** — The type's base changed from `System.ValueType` to `System.Object`, and the `OpenTypeMetrics(IntPtr font)` constructor was removed. Use `Font.OpenTypeMetrics` instead of constructing the struct directly.

## Build & Interop

- **Generate the HarfBuzz interop** — Interop bindings are now produced by the shared code generator so future updates stay in lockstep with SkiaSharp. ([#1447](https://github.com/mono/SkiaSharp/pull/1447))
- **All-Emscripten WASM builds** — Ship every supported Emscripten flavor for WebAssembly. ([#1590](https://github.com/mono/SkiaSharp/pull/1590))
- **Musl/Alpine detection** — Correctly identify Alpine Linux (musl libc) so the right native asset is loaded. ([#1657](https://github.com/mono/SkiaSharp/pull/1657))
- **.NET 5 support** — Adds .NET 5.0 target framework alongside the existing ones. ([#1697](https://github.com/mono/SkiaSharp/pull/1697))
