# HarfBuzzSharp 2.6.1.8

> Ships with [SkiaSharp 2.80.3](../2.80.3.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/2.6.1.8) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v2.80.3)

> **API changes** · [HarfBuzzSharp API diff](2.6.1.8/index.md) · Ships with [SkiaSharp 2.80.3](../2.80.3.md)

## Highlights

HarfBuzzSharp 2.6.1.8 ships alongside SkiaSharp 2.80.3. The line picks up the interop generator work started in `SkiaSharp 2.80.3-preview.18`, the WASM emscripten build support, and the .NET 5 modernization, plus proper musl detection on Linux.

## Breaking Changes

*None in this release.*

## Highlights of the shipped work

- **Generated interop with HarfBuzz** — Reworks the bindings around the auto-generated interop layer used across the 2.80.x line. ([#1447](https://github.com/mono/SkiaSharp/pull/1447))
- **All Emscripten flavors packaged for WASM** — Ensures every WASM configuration ships in the native artifacts. ([#1590](https://github.com/mono/SkiaSharp/pull/1590))
- **.NET 5.0 support** — Lines HarfBuzzSharp up with the SkiaSharp 2.80.3 .NET 5 update. ([#1697](https://github.com/mono/SkiaSharp/pull/1697))
- **musl detection on Linux** — Correctly resolves the right native at load time on musl systems. ([#1657](https://github.com/mono/SkiaSharp/pull/1657))
- **VS 2013 build support** — Keeps the older toolchain building for downstream consumers that still need it. ([#1536](https://github.com/mono/SkiaSharp/pull/1536))
