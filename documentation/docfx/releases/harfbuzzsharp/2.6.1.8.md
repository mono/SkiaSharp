# HarfBuzzSharp 2.6.1.8

> Ships with [SkiaSharp 2.80.3](../2.80.3.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/2.6.1.8) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v2.80.3)

> **API changes** · [HarfBuzzSharp API diff](2.6.1.8/index.md) · Ships with [SkiaSharp 2.80.3](../2.80.3.md)

The HarfBuzz binding that ships alongside SkiaSharp `2.80.3`. The signal item this cycle is the switch to a **generated** HarfBuzz interop — the same source-generator that produces the SkiaSharp bindings — plus platform coverage for the WASM and .NET 5 side of that release.

## Highlights

- Generate the interop with HarfBuzz ([#1447](https://github.com/mono/SkiaSharp/pull/1447)) — moves the HarfBuzz P/Invoke surface onto the generator, sharing the same tooling as SkiaSharp itself.
- Properly detect and handle **musl** systems ([#1657](https://github.com/mono/SkiaSharp/pull/1657)) so `HarfBuzzSharp.NativeAssets.Linux` lights up correctly on Alpine and other musl-libc distros.
- Update to **.NET 5.0** ([#1697](https://github.com/mono/SkiaSharp/pull/1697)) alongside the SkiaSharp bump.
- Build and include all the Emscripten variants for WASM ([#1590](https://github.com/mono/SkiaSharp/pull/1590)).
- Support VS 2013 because it is easy to do so ([#1536](https://github.com/mono/SkiaSharp/pull/1536)).
