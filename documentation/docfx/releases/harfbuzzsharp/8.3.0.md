# HarfBuzzSharp 8.3.0

> Ships with [SkiaSharp 3.0.0](../3.0.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.0.0)

> **API changes** · [HarfBuzzSharp API diff](8.3.0/index.md) · Ships with [SkiaSharp 3.0.0](../3.0.0.md)

## Highlights

HarfBuzzSharp 8.3.0 ships alongside SkiaSharp 3.0.0. This is the first HarfBuzz release on the 3.x line and picks up the .NET 8 modernization, the SDK/workload refresh, and the function-pointer + `LibraryImport` interop revamp from [@maxkatz6](https://github.com/maxkatz6).

## Breaking Changes

See the linked HarfBuzzSharp API diff for the signature changes rolled up from the removal of `[Obsolete]` members and the interop rewrite.

## Highlights of the shipped work

- **Function pointers and `LibraryImport`** — Replaces reflection-heavy delegate P/Invoke with the modern function-pointer / source-generated `LibraryImport` interop. ❤️ [@maxkatz6](https://github.com/maxkatz6) ([#2917](https://github.com/mono/SkiaSharp/pull/2917))
- **HarfBuzz bumped to 7.3.0** — Rolls the native HarfBuzz version forward as part of the 3.0 preview. ([#2582](https://github.com/mono/SkiaSharp/pull/2582))
- **.NET 8 SDK and workloads** — Aligns HarfBuzzSharp with the SkiaSharp 3.0 .NET 8 modernization. ([#2927](https://github.com/mono/SkiaSharp/pull/2927), [#2971](https://github.com/mono/SkiaSharp/pull/2971))
- **Removed all `[Obsolete]` members and types** — Cleans up API surface as part of the 3.0 preview. ([#2539](https://github.com/mono/SkiaSharp/pull/2539))
- **Repository modernization for SkiaSharp 3.0** — Foundational build/tooling refresh that HarfBuzzSharp 8.3.0 also builds on. ([#2505](https://github.com/mono/SkiaSharp/pull/2505))
- **APIScan runs on the built packages** — Post-packaging APIScan for compliance. ([#2730](https://github.com/mono/SkiaSharp/pull/2730))
