# HarfBuzzSharp 8.3.0

> Ships with [SkiaSharp 3.0.0](../3.0.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.0.0)

HarfBuzzSharp release that ships with SkiaSharp 3.0.0. This build brings a fresh HarfBuzz engine drop and rebuilds the P/Invoke surface on the modern .NET runtime with function pointers and `LibraryImport` in place of the old marshaller.

## Engine

- HarfBuzz native engine bumped to 7.3.0 ([#2582](https://github.com/mono/SkiaSharp/pull/2582)) — later re-anchored on 8.3.0 alongside the SkiaSharp 3.0 preview 5 native updates.

## API Surface

- Repository modernization for SkiaSharp 3.0 also refreshed HarfBuzzSharp's project structure and target-framework list ([#2505](https://github.com/mono/SkiaSharp/pull/2505)).
- All `[Obsolete]` members and types accumulated in the 2.x line are removed from the 3.0 baseline ([#2539](https://github.com/mono/SkiaSharp/pull/2539)).

## Platform

- P/Invoke migrated to function pointers and `LibraryImport` on modern TFMs by [@maxkatz6](https://github.com/maxkatz6) ([#2917](https://github.com/mono/SkiaSharp/pull/2917)), letting the trimmer keep only the interop the app actually uses.
- .NET target frameworks and workloads refreshed to .NET 8 ([#2927](https://github.com/mono/SkiaSharp/pull/2927), [#2971](https://github.com/mono/SkiaSharp/pull/2971)).

## Lifecycle & Internals

- APIScan now runs after packaging so scans are executed against the actual shipping HarfBuzzSharp NuGet ([#2730](https://github.com/mono/SkiaSharp/pull/2730)).

## Breaking Changes

The 3.0 modernization pass removes obsolete members that had been marked for deletion on the 2.x line ([#2539](https://github.com/mono/SkiaSharp/pull/2539)). Consult the [HarfBuzzSharp API diff](8.3.0/index.md) for the exact removals.
