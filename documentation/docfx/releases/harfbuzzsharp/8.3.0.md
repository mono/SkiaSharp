# HarfBuzzSharp 8.3.0

> Ships with [SkiaSharp 3.0.0](../3.0.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.0.0)

> **API changes** · [HarfBuzzSharp API diff](8.3.0/index.md) · Ships with [SkiaSharp 3.0.0](../3.0.0.md)

## Highlights

Ships as part of SkiaSharp 3.0.0 and bumps the HarfBuzz native from 7.3.0 to 8.3.0 — HarfBuzz version cadence now tracks the SkiaSharp major/minor for readable side-by-side compatibility. Modernises the interop with function pointers and `LibraryImport`, drops all `[Obsolete]` members from the previous line, and updates the toolchain to .NET 8.

## Breaking Changes

- **`Blob.AsSpan()` removed** — The method was previously public on `Blob`. Callers should switch to the pointer/length accessors (`Data` + `Length`) and build the `ReadOnlySpan<byte>` themselves.
- **All `[Obsolete]` members and types removed** — Everything previously marked `[Obsolete]` on the 2.x HarfBuzz surface is gone. Callers should follow the migration guidance from the individual obsolete annotations in the 2.88.x line.

## Engine

- **Update HarfBuzz to 7.3.0 → 8.3.0** ([#2582](https://github.com/mono/SkiaSharp/pull/2582) by @mattleibow) — Refreshes the HarfBuzz native from the 2.8.2 SkiaSharp shipped to the modern 7.3.0 line, then again to 8.3.0 as the release finalises (skia: mono/skia#104).

## API Surface
