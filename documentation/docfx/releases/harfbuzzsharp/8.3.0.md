# HarfBuzzSharp 8.3.0

> Ships with [SkiaSharp 3.0.0](../3.0.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.0.0)

The HarfBuzz binding that ships in the SkiaSharp 3.0.0 package family. This is a major version bump aligned with the SkiaSharp 3.0 modernisation: HarfBuzz itself rolled to 7.3.0, the binding sheds every `[Obsolete]` member accumulated during 1.x/2.x, and the interop uses function pointers via `LibraryImport` on .NET 8. See the API diff above for the exposed surface at this milestone.

## Highlights

- Upgraded the underlying HarfBuzz native library to 7.3.0 (via mono/skia [#104](https://github.com/mono/skia/pull/104)) — see [#2582](https://github.com/mono/SkiaSharp/pull/2582).
- Removed every `[Obsolete]` member and type inherited from the 1.x/2.x series, so the 8.x surface is what's supported going forward ([#2539](https://github.com/mono/SkiaSharp/pull/2539)).
- Modernised the repository layout in preparation for SkiaSharp 3.0, aligning the HarfBuzz binding's build and packaging with the new pipelines ([#2505](https://github.com/mono/SkiaSharp/pull/2505)).
- Adopted function pointers and `LibraryImport` on .NET 8, eliminating the runtime marshalling shims where possible ([#2917](https://github.com/mono/SkiaSharp/pull/2917) ❤️).
- Updated the managed target to .NET 8 across the binding and its samples ([#2927](https://github.com/mono/SkiaSharp/pull/2927), [#2971](https://github.com/mono/SkiaSharp/pull/2971)).
