# HarfBuzzSharp 8.3.0

> Ships with [SkiaSharp 3.0.0](../3.0.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.0.0)

> **API changes** · [HarfBuzzSharp API diff](8.3.0/index.md) · Ships with [SkiaSharp 3.0.0](../3.0.0.md)

## Highlights

The HarfBuzzSharp binding that ships with SkiaSharp 3.0.0 was modernized alongside
the SkiaSharp 3.0 push: the native HarfBuzz was bumped to the 8.3 series, the
interop layer moved to function pointers / `LibraryImport`, `[Obsolete]` members
were culled, and the whole project graph was refreshed for .NET 8.

## Breaking Changes

Legacy `[Obsolete]` members and types were removed as part of the 3.0 cleanup. See
the [HarfBuzzSharp API diff](8.3.0/index.md) for the complete signature deltas.

## Native Engine

- **HarfBuzz native updated to the 8.3 series** — brings the shaper up to a
  current upstream cut, ahead of the SkiaSharp 3.0 release.
  ([#2582](https://github.com/mono/SkiaSharp/pull/2582))

## API Surface

- **Interop modernized with function pointers and `LibraryImport`** — the shim
  layer moves off the legacy DllImport model onto the source-generated
  `LibraryImport` for lower P/Invoke overhead and full AOT compatibility.
  ❤️ [@maxkatz6](https://github.com/maxkatz6)
  ([#2917](https://github.com/mono/SkiaSharp/pull/2917))
- **Legacy `[Obsolete]` surface removed** — the 3.0 cleanup drops types and
  members that had been marked obsolete on the 2.x line.
  ([#2539](https://github.com/mono/SkiaSharp/pull/2539))

## Lifecycle & Internals

- **Repository, SDK, and workload refresh for .NET 8** — the project graph and
  toolchain were bumped in tandem with the SkiaSharp 3.0 modernization.
  ([#2505](https://github.com/mono/SkiaSharp/pull/2505),
  [#2927](https://github.com/mono/SkiaSharp/pull/2927),
  [#2971](https://github.com/mono/SkiaSharp/pull/2971))
