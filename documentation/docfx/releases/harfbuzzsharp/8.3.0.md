# HarfBuzzSharp 8.3.0

> **First HarfBuzz 8 release** · Ships with [SkiaSharp 3.0.0](../3.0.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.0.0)

> **API changes** · [HarfBuzzSharp API diff](8.3.0/index.md) · Ships with [SkiaSharp 3.0.0](../3.0.0.md)

## Highlights

The 8.x line paired with the SkiaSharp 3.0.0 modernization. Native HarfBuzz jumps to 7.3.0, the P/Invoke layer switches to `LibraryImport` and function-pointer callbacks for .NET 8, and the binding drops every previously-obsoleted member. The single breaking signature is the removal of `Blob.AsSpan()`, which had been marked obsolete.

## Breaking Changes

- **`HarfBuzzSharp.Blob.AsSpan()` removed** — the method was previously marked obsolete; consumers should use the alternative accessors documented on `Blob`. See the [HarfBuzzSharp API diff](8.3.0/index.md) for the full signature summary.

## Engine

- **Native HarfBuzz updated to 7.3.0** — brings all upstream HarfBuzz shaping and OpenType fixes forward. ([#2582](https://github.com/mono/SkiaSharp/pull/2582), [skia#104](https://github.com/mono/skia/pull/104))

## API Surface

- **Removed obsoleted members and types** — long-deprecated APIs across the binding are now gone; migrate to the replacements called out in prior deprecation warnings. ([#2539](https://github.com/mono/SkiaSharp/pull/2539))

## Lifecycle & Internals

- **Function pointers + `LibraryImport`** — the P/Invoke layer now uses the modern function-pointer callback ABI and source-generated `LibraryImport` marshalling. ❤️ [@maxkatz6](https://github.com/maxkatz6) ([#2917](https://github.com/mono/SkiaSharp/pull/2917))
- **.NET 8 SDK and workloads** — build and target moved to the .NET 8 SDK and workloads. ([#2927](https://github.com/mono/SkiaSharp/pull/2927), [#2971](https://github.com/mono/SkiaSharp/pull/2971))
