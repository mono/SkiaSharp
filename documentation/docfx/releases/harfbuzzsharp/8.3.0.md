# HarfBuzzSharp 8.3.0

> Ships with [SkiaSharp 3.0.0](../3.0.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.0.0)

> **API changes** · [HarfBuzzSharp API diff](8.3.0/index.md) · Ships with [SkiaSharp 3.0.0](../3.0.0.md)

The HarfBuzz binding that ships alongside SkiaSharp `3.0.0`. Version numbers realign with upstream HarfBuzz (`7.3.0`, then jumping to `8.3.0` to match), the P/Invoke surface moves to **function pointers and `LibraryImport`**, and every `[Obsolete]` member from the previous major line is removed.

## Highlights

- Bump `harfbuzz` to `7.3.0` ([#2582](https://github.com/mono/SkiaSharp/pull/2582), skia: [mono/skia#104](https://github.com/mono/skia/pull/104)) — the native upgrade that anchors this release; the `8.x` version stamp is applied later in the cycle to bring HarfBuzzSharp in line with upstream HarfBuzz.
- Function pointers and `LibraryImport` ([#2917](https://github.com/mono/SkiaSharp/pull/2917)) replace the legacy `[DllImport]` interop, matching what SkiaSharp `3.0.0` does on `.NET 8+`.
- Remove all `[Obsolete]` members and types ([#2539](https://github.com/mono/SkiaSharp/pull/2539)) — the same wholesale cleanup as the SkiaSharp `3.0.0` binding.
- Modernize the repository and prepare SkiaSharp 3.0 ([#2505](https://github.com/mono/SkiaSharp/pull/2505)).
- Update `.NET` to `8.0` ([#2927](https://github.com/mono/SkiaSharp/pull/2927)) and refresh SDK/workloads/projects ([#2971](https://github.com/mono/SkiaSharp/pull/2971)).

## Infrastructure

- Run APIScan after packaging on the packages ([#2730](https://github.com/mono/SkiaSharp/pull/2730)).

## Breaking Changes

The entire pre-3.0 `[Obsolete]` HarfBuzzSharp surface has been removed ([#2539](https://github.com/mono/SkiaSharp/pull/2539)). Anything you were still calling with a deprecation warning under `2.88.x` no longer compiles. See the [HarfBuzzSharp API diff](8.3.0/index.md) for the exact list — the P/Invoke retooling in [#2917](https://github.com/mono/SkiaSharp/pull/2917) also changes signatures around `LibraryImport` targets, so users who call HarfBuzzSharp interop directly should re-check their bindings against the diff.
