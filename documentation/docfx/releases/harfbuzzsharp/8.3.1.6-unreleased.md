# HarfBuzzSharp 8.3.1.6

> **Upcoming release** · In development · Ships with the upcoming SkiaSharp 4.147.0 · Not yet available on NuGet

<!-- Generated: 2026-07-08T08:15:00Z by claude-opus-4.7 -->

## Highlights

**HarfBuzzSharp 8.3.1.6** is in preparation alongside SkiaSharp **4.147.0**. The headline addition is the new `HBColor` struct that gives callers a proper managed representation of `hb_color_t` for color-font work. Color-font support in SkiaSharp is complemented by a wider variable-font path that spans both bindings.

## New APIs

- **`HBColor`** — A new value type that maps `hb_color_t` correctly for palette and color-font APIs, replacing ad-hoc `uint` handling at call sites ([#4000](https://github.com/mono/SkiaSharp/pull/4000)).
- **Variable font support** — Extends HarfBuzzSharp to expose variable-font instances and axes so shaping can consume the same fonts as SkiaSharp's rendering path (❤️ [@ramezgerges](https://github.com/ramezgerges) in [#3703](https://github.com/mono/SkiaSharp/pull/3703), [mono/skia#185](https://github.com/mono/skia/pull/185)).

## Fixes

- **Warnings-as-errors cleanup** — Tighten the binding assemblies so `TreatWarningsAsErrors` stays on for future work, including in HarfBuzzSharp ([#3752](https://github.com/mono/SkiaSharp/pull/3752)).
- **WASM native builds** — Drop the pre-.NET 8 Emscripten flavours so HarfBuzzSharp's WebAssembly artefacts stay in step with the shared toolchain ([#4022](https://github.com/mono/SkiaSharp/pull/4022)).

## Platform support

- **Linux Bionic** — Add native assets for Linux Bionic runtimes so HarfBuzzSharp ships correctly to that platform (❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) in [#3217](https://github.com/mono/SkiaSharp/pull/3217)).
- **Tizen x64 / arm64** — Extend the native build matrix so HarfBuzzSharp's Tizen native library covers all supported architectures ([#3620](https://github.com/mono/SkiaSharp/pull/3620), [mono/skia#175](https://github.com/mono/skia/pull/175)).
