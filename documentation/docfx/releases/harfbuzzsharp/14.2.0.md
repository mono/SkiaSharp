# HarfBuzzSharp 14.2.0

> Ships with [SkiaSharp 4.148.0](../4.148.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/14.2.0) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v4.148.0)

> **API changes** · [HarfBuzzSharp API diff](14.2.0/index.md) · Ships with [SkiaSharp 4.148.0](../4.148.0.md)

## Highlights

First stable HarfBuzzSharp on the v4 line and the biggest HB release in a long
time: native HarfBuzz jumps to the milestone-14.2 build alongside Skia m147,
color font palettes and variable fonts are wired in, an `HBColor` value type
replaces the ad-hoc `hb_color_t` marshalling, and the singleton lifecycle was
reworked so the shared registries clean up predictably.

## Breaking Changes

The singleton lifecycle rework changes how the shared registries are torn down —
see the [HarfBuzzSharp API diff](14.2.0/index.md) for the exact signature and
lifetime deltas.

## API Surface

- **`HBColor` struct for `hb_color_t`** — a real value type replaces the ad-hoc
  packed-`uint` marshalling used before, giving palette callers named channels
  and a stable layout. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))
- **Variable font support** — variable-axis coordinates now flow through
  HarfBuzzSharp for shaping, keeping HB in step with the SkiaSharp side. ❤️
  [@ramezgerges](https://github.com/ramezgerges)
  ([#3703](https://github.com/mono/SkiaSharp/pull/3703),
  [mono/skia#185](https://github.com/mono/skia/pull/185))
- **Color font palettes and variable-font robustness** — palette selection lands
  and the variable font path gets its rough edges filed off.
  ([#3742](https://github.com/mono/SkiaSharp/pull/3742),
  [mono/skia#197](https://github.com/mono/skia/pull/197))

## Native Engine

- **Native HarfBuzz refreshed alongside Skia m147** — the HB natives are rebuilt
  from the mono/skia milestone-147 branch so shaping tracks the same upstream
  cut as Skia. ❤️ [@ramezgerges](https://github.com/ramezgerges)
  ([#3702](https://github.com/mono/SkiaSharp/pull/3702),
  [mono/skia#184](https://github.com/mono/skia/pull/184))

## Lifecycle & Internals

- **Singleton instance lifecycle reworked** — the process-wide shared registries
  (blob, face, and shaper singletons) now clean up predictably, unblocking hot-
  reload and long-running host scenarios. ❤️
  [@ramezgerges](https://github.com/ramezgerges)
  ([#4080](https://github.com/mono/SkiaSharp/pull/4080))
- **`TreatWarningsAsErrors` in binding/** — the binding project graph now fails
  on any C# warning, so nothing quietly slips through.
  ([#3752](https://github.com/mono/SkiaSharp/pull/3752))
- **Skottie/Uno sample downgrade fix (NU1605)** — the Uno gallery Skottie sample
  restores cleanly again after a downstream package version was pinned. ❤️
  [@ramezgerges](https://github.com/ramezgerges)
  ([#3785](https://github.com/mono/SkiaSharp/pull/3785))

## Platform

- **WASM native builds drop pre-.NET 8 Emscripten** — the WASM natives now
  target only .NET 8+ Emscripten, matching the SkiaSharp WASM support policy.
  ([#4022](https://github.com/mono/SkiaSharp/pull/4022))
- **Linux Bionic and Tizen x64/arm64 native builds** — the extra native slices
  from the 3.119.4 side ship on the v4 line too. ❤️
  [@4Darmygeometry](https://github.com/4Darmygeometry)
  ([#3217](https://github.com/mono/SkiaSharp/pull/3217),
  [#3620](https://github.com/mono/SkiaSharp/pull/3620),
  [mono/skia#175](https://github.com/mono/skia/pull/175))
