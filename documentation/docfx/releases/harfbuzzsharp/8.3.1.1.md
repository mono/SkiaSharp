# HarfBuzzSharp 8.3.1.1

> Ships with [SkiaSharp 3.119.0](../3.119.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.1.1) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.0)

> **API changes** · [HarfBuzzSharp API diff](8.3.1.1/index.md) · Ships with [SkiaSharp 3.119.0](../3.119.0.md)

## Highlights

The HarfBuzzSharp binding shipped with SkiaSharp 3.119.0 rolls up the WASM/Blazor
fixes from the skipped 8.3.1 preview and adds native builds for the new
architectures the SkiaSharp 3.119 release added: LoongArch64, RISC-V 64, and
Windows on ARM.

## Breaking Changes

None in this release.

## Platform

- **New CPU architectures — LoongArch64 and RISC-V 64** — native HarfBuzz binaries
  are now produced for both architectures, matching the SkiaSharp 3.119.0
  platform expansion. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry),
  [@kasperk81](https://github.com/kasperk81)
  ([#3198](https://github.com/mono/SkiaSharp/pull/3198),
  [#3192](https://github.com/mono/SkiaSharp/pull/3192),
  [#3194](https://github.com/mono/SkiaSharp/pull/3194))
- **ARM target fixes and WASM/Blazor workaround baked in** — build targets for
  ARM machines were straightened out, and the WASM Blazor shim consumers used
  to apply themselves is now part of the WASM build.
  ([#3190](https://github.com/mono/SkiaSharp/pull/3190),
  [#3082](https://github.com/mono/SkiaSharp/pull/3082),
  [#3088](https://github.com/mono/SkiaSharp/pull/3088))
