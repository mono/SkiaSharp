## Highlights

**HarfBuzzSharp 8.3.1.1** ships alongside [SkiaSharp 3.119.0](../3.119.0.md) and broadens the Linux native build matrix. New architectures — LoongArch64 and RISC-V 64 — join the supported list, and ARM Linux targets are corrected so the shipped runtime assets load reliably on those devices.

## Platform support

- **LoongArch64** — Add native build support so HarfBuzzSharp works on Loongson-based Linux distributions (❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) in [#3198](https://github.com/mono/SkiaSharp/pull/3198)).
- **RISC-V 64** — Add native build support for the `riscv64` Linux architecture (❤️ [@kasperk81](https://github.com/kasperk81) in [#3192](https://github.com/mono/SkiaSharp/pull/3192)).
- **ARM Linux** — Fix the target definitions so Linux ARM builds pick up the correct runtime identifier and package the right assets ([#3190](https://github.com/mono/SkiaSharp/pull/3190)).

## Fixes

- **Linux test infrastructure** — Fix Linux CI test wiring so HarfBuzzSharp's Linux native validation stays green (❤️ [@kasperk81](https://github.com/kasperk81) in [#3194](https://github.com/mono/SkiaSharp/pull/3194)).
- **Package layout** — Set the correct NuGet `PackagePath` so runtime assets are placed in the expected folders ([#3100](https://github.com/mono/SkiaSharp/pull/3100)).

## Other changes

Plus various CI, documentation, and internal tooling improvements — Android NDK bumped to r27c with 16 KB page-size alignment.
