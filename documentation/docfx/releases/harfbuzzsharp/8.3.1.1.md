# HarfBuzzSharp 8.3.1.1

> **Broader Linux architectures** · Ships with [SkiaSharp 3.119.0](../3.119.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.1.1) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.0)

> **API changes** · [HarfBuzzSharp API diff](8.3.1.1/index.md) · Ships with [SkiaSharp 3.119.0](../3.119.0.md)

## Highlights

Companion release for SkiaSharp 3.119.0. HarfBuzz now ships native binaries for LoongArch64 and riscv64, plus fixed ARM targets and Linux tests. Also picks up the WASM/Blazor workaround relocation and the Android NDK r27c refresh from the 3.118 preview line.

## Breaking Changes

None in this release — see the [HarfBuzzSharp API diff](8.3.1.1/index.md).

## Platform

- **LoongArch64 build support** — HarfBuzz native assets are now produced for `linux-loongarch64`. ❤️ [@4Darmygeometry](https://github.com/4Darmygeometry) ([#3198](https://github.com/mono/SkiaSharp/pull/3198))
- **riscv64 build support** — HarfBuzz native assets are now produced for `linux-riscv64`. ❤️ [@kasperk81](https://github.com/kasperk81) ([#3192](https://github.com/mono/SkiaSharp/pull/3192))
- **ARM target fixes** — corrects the target triplets used for ARM machines so the right binary is picked up at build time. ([#3190](https://github.com/mono/SkiaSharp/pull/3190))
- **Blazor/WASM initialization moved into the WASM package** — the workaround the app used to wire up is now part of the WASM native package itself. ([#3082](https://github.com/mono/SkiaSharp/pull/3082), [#3088](https://github.com/mono/SkiaSharp/pull/3088))
- **Android NDK r27c with 16 KB page alignment** — rebuilt against NDK r27c so the native library aligns to the 16 KB page size Android 15 devices are moving to. ([#3096](https://github.com/mono/SkiaSharp/pull/3096))

## Bug Fixes

- **Linux tests fixed** — the Linux test job runs green again on the new architectures. ❤️ [@kasperk81](https://github.com/kasperk81) ([#3194](https://github.com/mono/SkiaSharp/pull/3194))

## Lifecycle & Internals

- Plus various CI, documentation, and internal tooling improvements — including a NuGet `PackagePath` attribute correction ([#3100](https://github.com/mono/SkiaSharp/pull/3100)).
