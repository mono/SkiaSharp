# HarfBuzzSharp 8.3.1.6

> Ships with [SkiaSharp 4.147.0](../4.147.0.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.1.6) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v4.147.0)

> **API changes** · [HarfBuzzSharp API diff](8.3.1.6/index.md) · Ships with [SkiaSharp 4.147.0](../4.147.0.md)

## Highlights

HarfBuzzSharp 8.3.1.6 ships with SkiaSharp 4.147.0 and is the first HarfBuzz release under the 4.x line. It adds variable font support, the new `HBColor` type for CPAL/COLR color fonts, extended Tizen and WASM platform coverage, and a binding-layer code quality pass. Community contributor [@ramezgerges](https://github.com/ramezgerges) led the variable font work and Skia engine integration.

## Breaking Changes

*None in this release.*

## New Features

### Text & Fonts

- **Variable font support** — Adds full variable font support to HarfBuzzSharp alongside SkiaSharp, enabling programmatic access to variable font axes (weight, width, slant, and custom axes). ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3703](https://github.com/mono/SkiaSharp/pull/3703))
- **Color font palette support** — Improves color font (CPAL/COLR) support and variable font robustness, supporting palette access and manipulation alongside the color glyph APIs. ([#3742](https://github.com/mono/SkiaSharp/pull/3742))
- **`HBColor` struct** — Introduces `HBColor` as the correct managed representation of `hb_color_t`, replacing the previous ad-hoc usage and improving color glyph API accuracy. ([#4000](https://github.com/mono/SkiaSharp/pull/4000))

### Engine

- **Skia milestone 147** — Updates the underlying Skia engine to milestone 147, pulling in upstream HarfBuzz improvements and text-shaping fixes. ❤️ [@ramezgerges](https://github.com/ramezgerges) ([#3702](https://github.com/mono/SkiaSharp/pull/3702))

## Platform

- **Tizen x64 and arm64** — Adds x64 and arm64 native build targets for Tizen, extending HarfBuzzSharp coverage to those architectures. ([#3620](https://github.com/mono/SkiaSharp/pull/3620))
- **WebAssembly — dropped pre-.NET 8 builds** — Removes native WebAssembly builds targeting pre-.NET 8 Emscripten toolchains; .NET 8+ is now the minimum for WASM. ([#4022](https://github.com/mono/SkiaSharp/pull/4022))

## Code Quality

- **`TreatWarningsAsErrors` enabled** — Enables `TreatWarningsAsErrors` across the binding project and fixes all resulting warnings, improving long-term code quality. ([#3752](https://github.com/mono/SkiaSharp/pull/3752))
