## Highlights

**HarfBuzzSharp 8.3.1.3** ships alongside [SkiaSharp 3.119.2](../3.119.2.md) as a security-hardening and Android-symbols refresh. No HarfBuzz binding API changes land here — the native library adopts additional Windows security mitigations (Control Flow Guard and buffer-security checks) and Android now preserves debug symbols in the shipped packages.

## Breaking Changes

None in this release.

## Platform support

- **Windows security mitigations** — Enable Control Flow Guard and `BufferSecurityCheck` for all Windows native DLLs to satisfy BinSkim BA2007 and improve exploit resistance (❤️ [@Aguilex](https://github.com/Aguilex) in [#3397](https://github.com/mono/SkiaSharp/pull/3397) and [#3404](https://github.com/mono/SkiaSharp/pull/3404)).
- **Android symbols** — Preserve and pack Android debug symbols so profilers and crash reporters can resolve HarfBuzzSharp frames ([#3374](https://github.com/mono/SkiaSharp/pull/3374), [mono/skia#161](https://github.com/mono/skia/pull/161)).
