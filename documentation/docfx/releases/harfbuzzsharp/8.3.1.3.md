# HarfBuzzSharp 8.3.1.3

> **Windows security hardening** · Ships with [SkiaSharp 3.119.2](../3.119.2.md) · [NuGet](https://www.nuget.org/packages/HarfBuzzSharp/8.3.1.3) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.2)

> **API changes** · [HarfBuzzSharp API diff](8.3.1.3/index.md) · Ships with [SkiaSharp 3.119.2](../3.119.2.md)

## Highlights

Companion release for SkiaSharp 3.119.2. Extends Windows security hardening to the HarfBuzz native DLL (Control Flow Guard, BufferSecurityCheck for BinSkim BA2007) and picks up the Android symbol packaging change so crash reports resolve HarfBuzz frames correctly.

## Breaking Changes

None in this release — see the [HarfBuzzSharp API diff](8.3.1.3/index.md).

## Platform

- **Android symbols preserved and packed** — HarfBuzz Android AARs now include symbol files, so crash reports and profiler traces resolve function names correctly. ([#3374](https://github.com/mono/SkiaSharp/pull/3374), [skia#161](https://github.com/mono/skia/pull/161))

## Security

- **Control Flow Guard on the HarfBuzz Windows DLL** — HarfBuzz native DLL is now built with `/guard:cf`. ❤️ [@Aguilex](https://github.com/Aguilex) ([#3397](https://github.com/mono/SkiaSharp/pull/3397))
- **BufferSecurityCheck enabled — BinSkim BA2007 clean** — the `/GS` flag is now applied, resolving the BinSkim buffer-security-check warning. ❤️ [@Aguilex](https://github.com/Aguilex) ([#3404](https://github.com/mono/SkiaSharp/pull/3404))
