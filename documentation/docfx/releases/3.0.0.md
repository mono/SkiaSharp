# Version 3.0.0

> **Preview only** · Superseded by [3.116.0](3.116.0.md) · Never released as stable — these changes rolled up into 3.116.0 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.0.0-preview)

> **API changes** · [SkiaSharp API diff](3.0.0/index.md) · [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md)

## Highlights

The 3.0 preview line was the modernization pass on top of the 2.88.x codebase: it removed every `[Obsolete]` API, moved P/Invoke to function pointers with `LibraryImport`, targeted .NET 8 (and later .NET 9), bumped Skia to milestone 116, refreshed harfbuzz (8.3.0), webp, and libjpeg-turbo, added Metal on Mac Catalyst, and introduced first-class WinUI/Uno.WinUI, WPF `SKGLElement`, and Blazor DPI support. **This preview line was never released as stable; everything below rolled up into [3.116.0](3.116.0.md).** HarfBuzzSharp 8.3.0 shipped alongside — see its [linked page](harfbuzzsharp/8.3.0.md) for the binding changes.

## Breaking Changes

Every `[Obsolete]` member and type was removed across every assembly ([#2539](https://github.com/mono/SkiaSharp/pull/2539)) — the [SkiaSharp API diff](3.0.0/index.md) lists the full set of removals per assembly (`SkiaSharp`, `SkiaSharp.Views` on every platform, `SkiaSharp.Views.Desktop.Common`, `SkiaSharp.Views.WindowsForms`, `SkiaSharp.Views.Maui.Controls`, `SkiaSharp.Views.WinUI`, `SkiaSharp.Views.Uno.WinUI`). Migrate to the replacements each `[Obsolete]` warning pointed at in the 2.88.x line. Package versions can now be no longer than 64 characters ([#2707](https://github.com/mono/SkiaSharp/pull/2707)), and `GRMtlTextureInfo` fields are readonly ([#2833](https://github.com/mono/SkiaSharp/pull/2833)).

## Engine

- **Skia milestone 116** — moved SkiaSharp onto the upstream m116 branch. ([#2829](https://github.com/mono/SkiaSharp/pull/2829), [skia#122](https://github.com/mono/skia/pull/122))
- **Skia refresh late in the cycle** — final Skia bump from Redth before rolling to 3.116.0. ❤️ [@Redth](https://github.com/Redth) ([#3026](https://github.com/mono/SkiaSharp/pull/3026), [skia#137](https://github.com/mono/skia/pull/137))
- **libjpeg-turbo 3.0.0 → reverted to 2.1.5.1** — jumped to 3.0.0 for CVE coverage, then rolled back after ABI issues; the release ships with 2.1.5.1. ([#2583](https://github.com/mono/SkiaSharp/pull/2583), [#2699](https://github.com/mono/SkiaSharp/pull/2699), [skia#115](https://github.com/mono/skia/pull/115))
- **libwebp 1.3.2** — WebP encoder/decoder refresh. ([#2622](https://github.com/mono/SkiaSharp/pull/2622), [skia#112](https://github.com/mono/skia/pull/112))
- **harfbuzz 7.3.0 → 8.3.0** — pair of native HarfBuzz refreshes across preview 1 and preview 5. ([#2582](https://github.com/mono/SkiaSharp/pull/2582), [#2624](https://github.com/mono/SkiaSharp/pull/2624), [skia#104](https://github.com/mono/skia/pull/104), [skia#118](https://github.com/mono/skia/pull/118))

## GPU & Rendering

- **Metal on Mac Catalyst** — Mac Catalyst now uses Metal as its rendering backend. ([#2747](https://github.com/mono/SkiaSharp/pull/2747))
- **Metal APIs on a common .NET TFM** — the Metal-specific type surfaces are now available on every Apple platform under a single TFM. ❤️ [@maxkatz6](https://github.com/maxkatz6) ([#2788](https://github.com/mono/SkiaSharp/pull/2788))
- **`SKGLView` implemented** — the OpenGL-backed view control lands for platforms that were still missing it. ([#2598](https://github.com/mono/SkiaSharp/pull/2598))
- **WinUI accelerated views** — first-class support for WinUI's DirectX-accelerated `SwapChainPanel`-backed surface. ([#2733](https://github.com/mono/SkiaSharp/pull/2733))
- **`SKGLElement` for WPF** — a new `SKGLElement` control lets WPF apps render into an OpenGL-backed surface. ❤️ [@gmurray81](https://github.com/gmurray81) ([#2317](https://github.com/mono/SkiaSharp/pull/2317))

## API Surface

- **`SKImage.ToRawShader`** — new API to obtain a raw shader for the image. ([#2748](https://github.com/mono/SkiaSharp/pull/2748), [skia#119](https://github.com/mono/skia/pull/119))

- **`SKBlender` exposed** — new type binding for the Skia blender abstraction. ([#2830](https://github.com/mono/SkiaSharp/pull/2830), [skia#123](https://github.com/mono/skia/pull/123))

- **Memory leaks in MAUI views** — fixes a set of leaks where MAUI-hosted SkiaSharp views weren't releasing native resources. ([#2955](https://github.com/mono/SkiaSharp/pull/2955))
- **`SKPaint` clone disposal** — no longer throws an `AccessViolationException` when the cloned paint is disposed. ❤️ [@jeremy-visionaid](https://github.com/jeremy-visionaid) ([#2904](https://github.com/mono/SkiaSharp/pull/2904))
- **Metal snapshot returns real colour on Apple** — fixes a Metal path that was returning a magenta snapshot on Apple GPUs. ❤️ [@taublast](https://github.com/taublast) ([#2804](https://github.com/mono/SkiaSharp/pull/2804))
- **`SKXamlCanvas` on Uno Skia uses `Bgra8888`** — fixes the pixel format so `SKXamlCanvas` renders correctly under Uno Skia. ❤️ [@Youssef1313](https://github.com/Youssef1313) ([#2918](https://github.com/mono/SkiaSharp/pull/2918))
- **`XamlRoot` may be null when `SKXamlCanvas` is unloaded** — no longer throws on teardown. ❤️ [@jeromelaban](https://github.com/jeromelaban) ([#2854](https://github.com/mono/SkiaSharp/pull/2854))
- **`SKXamlCanvas` async-void avoided on Uno** — avoids a fire-and-forget async void that could crash on Uno. ❤️ [@lindexi](https://github.com/lindexi) ([#2720](https://github.com/mono/SkiaSharp/pull/2720))
- **`SKXamlCanvas` WASM: no throw when canvas can't be found** — fails soft on WASM host removal. ❤️ [@jeromelaban](https://github.com/jeromelaban) ([#2563](https://github.com/mono/SkiaSharp/pull/2563))
- **Uno: ensure canvas context is active when rendering** — fixes a rendering race in the Uno path. ❤️ [@jeromelaban](https://github.com/jeromelaban) ([#2559](https://github.com/mono/SkiaSharp/pull/2559))
- **`GetKerningPairAdjustments` API fix** — corrects the underlying `SKTypeface` kerning-pair adjustment path. ❤️ [@pdjonov](https://github.com/pdjonov) ([#2858](https://github.com/mono/SkiaSharp/pull/2858))
- **`SKFilterQuality` respected in obsolete methods** — obsolete drawing methods now honour the paint's `FilterQuality` when converting to `SKSamplingOptions`. ([#2924](https://github.com/mono/SkiaSharp/pull/2924))
- **AOT `IBufferByteAccess`** — fixes an AOT scenario where the `IBufferByteAccess` interop broke. ([#2920](https://github.com/mono/SkiaSharp/pull/2920))
- **`SKRectI` conversion floors outwards** — corrects the rounding so integer rects always contain the float rect. ([#2568](https://github.com/mono/SkiaSharp/pull/2568))
- **Do not stretch the background** — WPF/WinForms host controls no longer stretch the background layer. ([#2922](https://github.com/mono/SkiaSharp/pull/2922))
