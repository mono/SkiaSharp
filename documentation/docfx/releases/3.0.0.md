# Version 3.0.0

> **Preview only** · Superseded by [3.116.0](3.116.0.md) · Never released as stable — these changes rolled up into 3.116.0 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.0.0-preview)

> **API changes** · [SkiaSharp API diff](3.0.0/index.md) · [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md)

## Highlights

The five-preview journey that took SkiaSharp from 2.88.x to a fresh v3 line. Preview 1 modernized the repo and cut back to .NET 6+/8, Preview 2 introduced Metal on Mac Catalyst and Skottie's Animation Builder, Preview 3 brought WPF `SKGLElement`, function pointers and MSVC builds, Preview 4 landed the big Skia m116 bump, `SKBlender`, R-tree recording and Span-based color filters, and Preview 5 delivered .NET 9 support, `LibraryImport` interop, and the SDK/workload refresh. This line never shipped as stable — everything rolled up into [3.116.0](3.116.0.md). See the linked API diff for the full v2→v3 surface change.

## Breaking Changes

3.0.0 is a major-version bump — the surface change is large. Highlights (see the [API diff](3.0.0/index.md) for the complete list):

- **All previously `[Obsolete]` members were removed.** ([#2539](https://github.com/mono/SkiaSharp/pull/2539))
- **`SKPngEncoderOptions` behaviour on `SKBitmap.Encode` was reverted** to match 2.88.8 semantics. ([#3014](https://github.com/mono/SkiaSharp/pull/3014))
- **`SKFilterQuality` compatibility overloads** were added to ease migration to `SKSamplingOptions`. ([#2963](https://github.com/mono/SkiaSharp/pull/2963))
- **Views assemblies (Android/Mac/iOS/tvOS/Tizen/Blazor/Uno.WinUI/WinUI/WPF/WinForms/MAUI/Desktop.Common)** all carry breaking signature changes documented in their per-assembly diffs under [3.0.0/](3.0.0/index.md).
