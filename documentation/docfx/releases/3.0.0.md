## Highlights

**Preview only** — this preview stream was never released as stable. All 122 PRs listed below were rolled into **[3.116.0](3.116.0.md)** as the first stable 3.x release. Refer to the [3.116.0 release notes](3.116.0.md) for the polished narrative, breaking-change guidance, and the full API-diff walk-through; the trailing preview sections below are kept for the historical record.

The 3.0 preview stream modernized SkiaSharp end-to-end: text-based blobs came back, all `[Obsolete]` members from the 2.x line were removed, function pointers and `LibraryImportAttribute` replaced older P/Invoke plumbing (contributed by [@maxkatz6](https://github.com/maxkatz6)), Metal became the Mac Catalyst backend, WinUI accelerated views landed, and Skia was walked forward from milestone 104 all the way to m116. HarfBuzz was bumped from 7.3.0 to **8.3.0** (see the [HarfBuzzSharp 8.3.0](harfbuzzsharp/8.3.0.md) page for the binding-level detail), libwebp went to 1.3.2, and libjpeg-turbo to 3.0.0. A `SKGLElement` for WPF from [@gmurray81](https://github.com/gmurray81), `SKImage.ToRawShader`, `SKBlender`, Skottie's Animation Builder, `SKFilterQuality` compatibility overloads, and a MAUI-views memory-leak fix rounded out the stream.

## Breaking Changes

- **All `[Obsolete]` members and types from the 2.x line are removed** ([#2539](https://github.com/mono/SkiaSharp/pull/2539)) — Any call site that had been living on a deprecation warning will now fail to compile. See the [SkiaSharp breaking-change list](3.0.0/SkiaSharp/SkiaSharp.breaking.md) for exact members.
- **`SKFilterQuality` restored via compatibility overloads** ❤️ [@Youssef1313](https://github.com/Youssef1313) ([#2963](https://github.com/mono/SkiaSharp/pull/2963)) — Adds ABI-compatible entry points for callers still passing `SKFilterQuality`.
- **Text-based `SKTextBlob` constructors are back** ([#2545](https://github.com/mono/SkiaSharp/pull/2545)) — Restores the ergonomic text-first blob API the 2.x line had lost.
- **View-assembly signature shifts** — Assorted per-platform view signature changes accompany the WinUI accelerated view, `SKGLView`, and MAUI leak fixes. See the per-assembly `*.breaking.md` files linked at the top for the exact members.

See the [full API diff](3.0.0/index.md) for the complete list.
