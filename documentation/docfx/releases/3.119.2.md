# Version 3.119.2

> **Windows hardening + tvOS/Android fixes** · Released February 6, 2026 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.2) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.2)

> **API changes** · [SkiaSharp API diff](3.119.2/index.md) · [HarfBuzzSharp 8.3.1.3](harfbuzzsharp/8.3.1.3.md)

## Highlights

Servicing release for the 3.119 line focused on Windows security hardening —
Control Flow Guard and Buffer Security Check are now enabled on every Windows
native DLL — plus missing tvOS `SKSurface.Create` overloads and packed Android
debug symbols for crash diagnostics. Ships with
[HarfBuzzSharp 8.3.1.3](harfbuzzsharp/8.3.1.3.md).

## Breaking Changes

None in this release. See the [3.119.2 API diff](3.119.2/index.md).

## API Surface

- **Missing tvOS `SKSurface.Create` overloads added** — the tvOS surface now
  has the same set of `Create` overloads as iOS. ❤️
  [@MartinZikmund](https://github.com/MartinZikmund)
  ([#3342](https://github.com/mono/SkiaSharp/pull/3342))

## Security

- **Control Flow Guard and Buffer Security Check on all Windows native DLLs** —
  the Windows binaries now build with CFG and `/GS`-strong buffer checks so
  they satisfy BinSkim BA2007. ❤️ [@Aguilex](https://github.com/Aguilex)
  ([#3397](https://github.com/mono/SkiaSharp/pull/3397),
  [#3404](https://github.com/mono/SkiaSharp/pull/3404))
- **Spectre mitigation flag on native Windows and Linux binaries** — the
  Windows and Linux native builds enable the Spectre-mitigation code-gen flag.
  ([#3497](https://github.com/mono/SkiaSharp/pull/3497),
  [#3503](https://github.com/mono/SkiaSharp/pull/3503))

## Platform

- **Android debug symbols preserved and packed** — the Android `.so` binaries
  are now built with symbols preserved and packed into the NuGet, so crash
  stacks resolve properly.
  ([#3374](https://github.com/mono/SkiaSharp/pull/3374),
  [mono/skia#161](https://github.com/mono/skia/pull/161))

## Lifecycle & Internals

- **Binding generator improvements** — small correctness fixes in the
  generator make the generated files a bit tidier and preserve doc comments on
  generated enum members.
  ([#3360](https://github.com/mono/SkiaSharp/pull/3360),
  [#3361](https://github.com/mono/SkiaSharp/pull/3361))
