# Version 3.119.2

> **Security hardening release** · Released February 7, 2026 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.2) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.2)

## Highlights

This release focuses on hardening the native libraries against common exploit techniques on Windows and Linux. Spectre mitigations, Control Flow Guard, and BufferSecurityCheck are now enabled across all Windows and Linux native builds — addressing BinSkim compliance and reducing attack surface. It also adds missing tvOS `SKSurface.Create` overloads. Three community contributors drove every user-facing change in this release.

## Breaking Changes

*None in this release.*

## New Features

### Platform

- **tvOS `SKSurface.Create` overloads** — Adds the missing `SKSurface.Create` overloads for tvOS, bringing parity with other Apple platforms. ❤️ @MartinZikmund ([#3342](https://github.com/mono/SkiaSharp/pull/3342))

## Security

- **Spectre mitigation for Windows** — Enables the Spectre mitigation compiler flag (`/Qspectre`) for `libSkiaSharp.dll` on Windows. ❤️ @sshumakov ([#3496](https://github.com/mono/SkiaSharp/pull/3496), [#3497](https://github.com/mono/SkiaSharp/pull/3497))
- **Spectre mitigation for Linux** — Enables the Spectre mitigation flag (`-mretpoline`) for `libSkiaSharp.so` on Linux. ❤️ @sshumakov ([#3502](https://github.com/mono/SkiaSharp/pull/3502), [#3503](https://github.com/mono/SkiaSharp/pull/3503))
- **Control Flow Guard (CFG)** — Enables Control Flow Guard for all Windows native DLLs, preventing control-flow hijacking attacks. ❤️ @Aguilex ([#3397](https://github.com/mono/SkiaSharp/pull/3397))
- **BufferSecurityCheck (`/GS`)** — Enables BufferSecurityCheck for Windows native DLLs, resolving BinSkim rule BA2007. ❤️ @Aguilex ([#3404](https://github.com/mono/SkiaSharp/pull/3404))
