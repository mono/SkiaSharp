> **Windows hardening + tvOS Surface** · Released February 6, 2026 · [NuGet](https://www.nuget.org/packages/SkiaSharp/3.119.2) · [GitHub Release](https://github.com/mono/SkiaSharp/releases/tag/v3.119.2)

## Highlights

**3.119.2** hardens the Windows native binaries with Control Flow Guard and buffer-security checks, preserves Android debug symbols so profilers can resolve SkiaSharp frames, and adds the missing tvOS `SKSurface.Create` overloads. HarfBuzz 8.3.1.3 ships alongside — see [its release notes](harfbuzzsharp/8.3.1.3.md) for the binding details.

## New APIs

- **tvOS `SKSurface.Create` overloads** — Fill in the tvOS `SKSurface.Create` overloads that were missing so tvOS apps can create GPU-backed surfaces from a Metal drawable (❤️ [@MartinZikmund](https://github.com/MartinZikmund) in [#3342](https://github.com/mono/SkiaSharp/pull/3342)).

## Platform support

- **Windows security mitigations** — Enable Control Flow Guard for all Windows native DLLs and `BufferSecurityCheck` to resolve BinSkim BA2007 (❤️ [@Aguilex](https://github.com/Aguilex) in [#3397](https://github.com/mono/SkiaSharp/pull/3397) and [#3404](https://github.com/mono/SkiaSharp/pull/3404)).
- **Android debug symbols** — Preserve and pack the Android debug symbols so profilers and crash reporters can resolve SkiaSharp frames ([#3374](https://github.com/mono/SkiaSharp/pull/3374), [mono/skia#161](https://github.com/mono/skia/pull/161)).

## Other changes

Plus various CI, documentation, and internal tooling improvements — the first tranche of AI-assisted docs, release-tagging and CI skills.
