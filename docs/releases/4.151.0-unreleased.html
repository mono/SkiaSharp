Version 4.151.0 (Unreleased)

New platforms and performance

Highlights

SkiaSharp 4.151.0 adds Vulkan/Silk.NET support, FreeType on Nano Server, and several performance fixes. This release expands platform reach with a new Vulkan binding package and FreeType-backed font support on Windows Nano Server. It also includes a memory-leak fix in raster image creation and two performance improvements in text encoding and runtime-effect uniforms.

Breaking Changes

None in this preview line.

API Surface

  • New SkiaSharp.Vulkan.Silk.NET package — A dedicated NuGet package integrates SkiaSharp with Vulkan via the Silk.NET bindings, enabling GPU-accelerated rendering on Linux and Android Vulkan back-ends. (#4527)
  • FreeType font support on Windows Nano Server — Nano Server containers lack the GDI font stack; SkiaSharp now falls back to FreeType so font rendering works in that environment. — ❤️ @jeffska (#4280)

Bug Fixes

  • Memory leak in SKImage.Create raster failure path fixed — A CoTaskMem pixel buffer was not freed when the underlying Skia raster-image creation failed, causing a native memory leak. The buffer is now released on the failure path. (#4455)

Lifecycle & Internals

  • Avoid intermediate string allocation in GetEncodedText null-terminated path — The hot path that reads null-terminated text no longer allocates an intermediate string, reducing GC pressure on text-heavy workloads. (#4498)
  • Pre-size uniform lookup map in SKRuntimeEffectUniforms — The uniform-name lookup dictionary is now sized to the known count at construction time, eliminating rehash allocations for typical runtime-effect usage. (#4512)
  • Solutions converted to .slnx format — All .sln solution files have been migrated to the lighter .slnx format for improved tooling compatibility. (#4528)

Platform

  • Containerized test legs added: Azure Linux, Alpine/musl, Nano Server — CI now runs a full test matrix on Azure Linux, Alpine (musl libc), and Windows Nano Server containers, catching platform-specific regressions earlier. (#4515)
