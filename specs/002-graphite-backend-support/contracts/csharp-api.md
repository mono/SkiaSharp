# Contract: SkiaSharp C# Public API

**Date**: 2026-04-30 · **Last amended**: 2026-05-01 · **Branch**: `002-graphite-backend-support`

This contract defines the public managed surface added by this feature. It is the durable interface shipped to SkiaSharp's downstream consumers.

> **Status legend**:
> ✅ Shipped in MVP (US1, smoke-gate green) · 🔶 Deferred to US2 (Metal/Dawn) · 🔷 Deferred to US3 (BackendTexture wrap, Image-Wrap) · ⚙️ Auto-generated by `pwsh ./utils/generate.ps1` from the C headers.

---

## Discoveries during implementation (2026-05-01)

Three things shaped the contract beyond what was originally specified:

1. **`SKGraphiteVkBackendContext` is a class, not a struct (Variant A ownership transfer).** Originally specified as a `public struct` with public fields including a `GetProc` delegate. A struct cannot safely own a `GCHandle`: copy semantics, no finalizer, no enforceable single-owner. So it's a `public sealed class : IDisposable` mirroring the existing `GRVkBackendContext` shape. Furthermore, `SKGraphiteContext.CreateVulkan` *takes ownership* of both the GCHandle pinning the GetProc delegate and the native backend-context wrapper at create time — callers may safely `Dispose` the `SKGraphiteVkBackendContext` immediately afterwards. (This differs from the existing `GRVkBackendContext` lifetime rule, which requires the backend-context to outlive the GR context. We're greenfield for Graphite so we did it right.)
2. **`SKGraphiteContext.ReadPixels` is a managed-side composition, not a C-shim invention.** `SKSurface.ReadPixels` is `GPU_TEST_UTILS`-gated in upstream Skia and returns false in production. The C API stays honest to upstream by exposing only the async primitives (`asyncRescaleAndReadPixelsSurface`, `checkAsyncWorkCompletion`, `AsyncReadResult` accessors); the sync wrapper is composed in C# by spinning on completion.
3. **Many auto-generated value types ship without hand-written wrappers.** The SkiaSharp generator produces `SKGraphiteContextOptions`, `SKGraphiteSubmitInfo`, `SKGraphiteInsertRecordingInfo`, `SKGraphiteBackend`, `SKGraphiteInsertStatus`, `SKGraphiteSyncToCpu`, `SKGraphiteMarkFrameBoundary`, `SKGraphiteRescaleGamma`, `SKGraphiteRescaleMode` directly from the C headers + a few JSON mappings in `binding/libSkiaSharp.json`. The contract below describes the shipped *shape* (which the generator produces); it intentionally does not duplicate the hand-rolled struct definitions originally drafted here.

## Namespace and assembly

- All new public types live in the existing `SkiaSharp` namespace, in the existing `SkiaSharp` assembly.
- New types use the `SKGraphite*` prefix to keep them visually distinct from Ganesh's `GR*` types and from generic `SK*` types.
- New methods that extend existing types (`SKSurface`, `SKImage`) are added as **overloads only**. No existing signatures change (FR-013).

```csharp
    public int               MaxTextureSize { get; }
    public bool              SupportsProtectedContent { get; }
    public long              CurrentBudgetedBytes { get; }
    public long              MaxBudgetedBytes { get; set; }

    // Recording — ✅
    public SKGraphiteRecorder CreateRecorder(long recorderBudgetBytes = -1);

    // Submission — ✅; non-Success status values are NOT exceptions, callers branch.
    public SKGraphiteInsertStatus InsertRecording(SKGraphiteRecording recording);
    public SKGraphiteInsertStatus InsertRecording(SKGraphiteInsertRecordingInfo info);
    public bool                   Submit();
    public bool                   Submit(SKGraphiteSubmitInfo submitInfo);

    // Resource management — ✅
    public void FreeGpuResources();
    public void PerformDeferredCleanup(TimeSpan duration);
    // 🔷 US3:
    // public void DeleteBackendTexture(SKGraphiteBackendTexture texture);

    // Async work completion — ✅
    /// <summary>Drives any pending async readback or finished-callback work.
    /// Callbacks fire on the calling thread.</summary>
    public void CheckAsyncWorkCompletion();

    // Synchronous CPU readback from a Graphite-backed surface — ✅
    /// <summary>
    /// Composed in managed code from the async primitives (the C/C++ side does
    /// NOT ship a sync wrapper). Blocks the calling thread by spinning on
    /// CheckAsyncWorkCompletion until the async callback fires; cost is roughly
    /// equivalent to glFinish.
    ///
    /// Required because SKSurface.ReadPixels is gated on GPU_TEST_UTILS in
    /// upstream Skia and returns false on Graphite-backed surfaces in
    /// production builds.
    /// </summary>
    public bool ReadPixels(SKSurface surface, SKImageInfo dstInfo,
                           IntPtr dstPixels, int dstRowBytes,
                           int srcX, int srcY);
}
```

**Behavior**:

- Constructors are not provided — backend-specific factories are the only public entry. Each returns `null` if the requested backend is not available, the backend context is invalid, or the underlying GPU rejected the create call.
- All properties are pure queries; none allocate.
- `InsertRecording` returns the status from the C API verbatim. Use of a recording vended from another context returns `InvalidRecording`. **Does not throw.**
- `Submit` returns `false` if submission failed; does not throw.
- `Dispose` releases the underlying `Context*` AND frees the backend-delegate GCHandle (acquired at `CreateVulkan` time via Variant A ownership transfer). `Dispose` is idempotent. **Disposal order matters**: dependents (recorders, recordings, surfaces) MUST be disposed before the context. Disposing the context first leaves dependents holding dangling native pointers; using them after that point is undefined behavior (likely native-side crash) — same convention as everywhere else in SkiaSharp.
- `ReadPixels` returns `false` on any failure (rect out of bounds, async callback signaled error, lost device). Does not throw.
- `CheckAsyncWorkCompletion` is single-thread-affine — must be called from the same thread that owns the context.

### `SKGraphiteVkBackendContext` — ✅ shipped (Variant A: ctx takes ownership)

```csharp
/// <summary>
/// Caller-supplied Vulkan handles + procedure-address callback used to bring up a
/// Graphite Vulkan context.
///
/// LIFETIME (Variant A): on a successful SKGraphiteContext.CreateVulkan call,
/// ownership of the GetProc delegate's pinned GCHandle AND the underlying native
/// backend-context wrapper transfer to the SKGraphiteContext. The caller may safely
/// Dispose this object immediately afterwards; double-disposal is a no-op. The
/// Vulkan handles themselves (VkInstance/VkDevice/VkQueue) are never freed by
/// SkiaSharp — they remain owned by the caller for the lifetime of the
/// SKGraphiteContext.
/// </summary>
public sealed class SKGraphiteVkBackendContext : IDisposable
{
    public IntPtr  VkInstance         { get; set; }
    public IntPtr  VkPhysicalDevice   { get; set; }
    public IntPtr  VkDevice           { get; set; }
    public IntPtr  VkQueue            { get; set; }
    public uint    GraphicsQueueIndex { get; set; }
    public uint    MaxApiVersion      { get; set; }
    public bool    ProtectedContext   { get; set; }
    public SKGraphiteVkGetProcedureAddressDelegate GetProcedureAddress { get; set; }

    public void Dispose();
}

/// <summary>
/// Vulkan procedure-address resolver: routes Skia's vk* function-pointer lookups
/// through the caller's Vulkan loader. Called from inside Skia's pipeline-creation
/// and dispatch threads — the Vulkan loader is thread-safe by spec, so the body
/// can simply forward to vkGetDeviceProcAddr / vkGetInstanceProcAddr.
/// </summary>
public delegate IntPtr SKGraphiteVkGetProcedureAddressDelegate(string name, IntPtr instance, IntPtr device);
```

Internal pieces (not part of the public surface, included for completeness):

- `SKGraphiteVkBackendContextNative` — the marshalled struct produced by the SkiaSharp generator from `sk_graphite_vk_backend_context_init_t`. Internal because it carries a managed delegate field.
- `SKGraphiteVkBackendContext.TransferGetProcHandle()` / `ReleaseNativeHandle()` — `internal` methods invoked by `SKGraphiteContext.CreateVulkan` to perform the Variant A ownership transfer.

## ABI rules

- Once a type or member ships, its signature is frozen. Future feature additions append overloads.
- Adding fields to `SKGraphiteVkBackendContext` is allowed because it's a class with property setters (additive); adding fields to auto-generated value-type structs (`SKGraphiteContextOptions`, `SKGraphiteSubmitInfo`, etc.) is **forbidden** in patch releases. Field additions to value structs go through versioned types (e.g., `SKGraphiteContextOptionsV2`).
- Returning `null` from a factory is a non-ABI-breaking change of behavior; throwing `OutOfMemoryException` from a factory that previously returned null IS ABI-breaking (caller branches on null) and must wait for a major version.
- **Variant A note**: relaxing `SKGraphiteVkBackendContext`'s lifetime rule (caller can now dispose it earlier) is a *non-breaking* contract change. It would only be ABI-breaking if a future revision re-tightened the rule.
- **Per-backend context lifetimes (Metal/Dawn, US2)** will follow the same Variant A pattern — backend-context disposable immediately after the corresponding `Create*` factory call.
