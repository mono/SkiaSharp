## üöÄ Getting Started

### I want to...

#### Use SkiaSharp in my app
- Check the main [README](../README.md) for installation

#### Report a bug or request a feature
- Read [../CONTRIBUTING.md](../CONTRIBUTING.md)
- Search existing [issues](https://github.com/mono/SkiaSharp/issues)

#### Fix a bug or add a feature
1. Read [../CONTRIBUTING.md](../CONTRIBUTING.md)
2. Study [../design/QUICKSTART.md](../design/QUICKSTART.md)
3. Follow [../design/adding-new-apis.md](../design/adding-new-apis.md) if adding APIs
4. Build using [Building-SkiaSharp.md](Building-SkiaSharp.md)

#### Build SkiaSharp from source
- **Managed only:** [Building-SkiaSharp.md](Building-SkiaSharp.md#managed-only-building)
- **Full build:** [Building-SkiaSharp.md](Building-SkiaSharp.md#native-building)
- **Linux:** [Building-on-Linux.md](Building-on-Linux.md)

#### Understand SkiaSharp's architecture
- Start with [../design/architecture-overview.md](../design/architecture-overview.md)
- Study pointer types in [../design/memory-management.md](../design/memory-management.md)
- Review error handling in [../design/error-handling.md](../design/error-handling.md)

#### Become a maintainer
- Review [Being-a-Maintainer.md](Being-a-Maintainer.md)
- Understand [Release-Checklist.md](Release-Checklist.md)
- Learn [Branching.md](Branching.md) and [Versioning.md](Versioning.md)

## ü§ù Contributing to Documentation

Documentation improvements are welcome! To contribute:

1. Fork the repository
2. Make changes to files in `wiki/` or `design/`
3. Submit a pull request
4. Maintainers will review and merge

See [../CONTRIBUTING.md](../CONTRIBUTING.md) for details.
